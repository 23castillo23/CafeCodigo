Introducción — ¿Por qué importa el editor?
El instrumento de trabajo más importante de un programador · 0:00 – 1:22

Todo en el mundo digital —páginas web, apps móviles, inteligencia artificial— empieza como texto escrito en un archivo. Ese texto es el código. Y el lugar donde escribes ese código es el editor.

🎻 Analogía: el instrumento del músico

Un guitarrista puede tocar con cualquier guitarra, hasta con una de juguete. Pero un guitarrista profesional cuida su instrumento, lo afina, lo personaliza. El editor de código es exactamente eso: tu instrumento. Puedes escribir código en el Bloc de Notas, pero es como tocar con una guitarra de plástico. VS Code es la guitarra profesional: misma música, mil veces más fácil y poderosa.

💡 Lo que aprenderás en esta guía

Esta guía cubre el curso completo de VS Code de Soy Dalto: desde cero hasta dominar la herramienta con la que escribirás código todos los días. No hace falta saber programar para empezar — el editor se aprende primero, el código viene después.

Entendiendo VS Code — ¿Qué es exactamente?
Editor vs IDE, y por qué VS Code es la mejor opción · 1:22 – 8:09

Visual Studio Code (también llamado VSCode o simplemente "VS Code") es un editor de código creado por Microsoft. Es gratuito, open source, y actualmente el más usado en el mundo por programadores de todos los niveles.

Editor de código vs IDE — ¿cuál es la diferencia?

📝
Editor de código
Un programa que te ayuda a escribir texto (código) con autocompletado, colores y detección de errores. Ligero y rápido. Ej: VS Code, Sublime Text, Notepad++.
🏭
IDE (Entorno de Desarrollo Integrado)
Todo lo que tiene un editor MÁS herramientas integradas: emuladores, compiladores, depuradores avanzados, gestión de bases de datos. Más pesado. Ej: Visual Studio, IntelliJ, Android Studio.
🦄
VS Code: el híbrido
Nace como editor ligero pero con extensiones se convierte en un IDE completo. Lo mejor de ambos mundos: arranca rápido y crece según lo que necesites.
🧩
IntelliSense
El motor de autocompletado de VS Code. Mientras escribes, sugiere cómo terminar el código, qué parámetros acepta una función, y detecta errores en tiempo real.
🌿
Control de versiones
Sistema para guardar el historial de cambios del código. VS Code tiene Git integrado — puedes subir archivos a GitHub sin salir del editor.
🐛
Debugger (Depurador)
Herramienta para encontrar errores ejecutando el código línea por línea y viendo qué valor tiene cada variable en cada momento.

🏎 Analogía: el auto de carreras vs el auto familiar

Un IDE es como un auto de Fórmula 1: potentísimo, lleno de botones, especializado para un circuito específico. Un editor de código es como un auto deportivo: rápido, ágil, y con el tiempo le puedes agregar los accesorios que necesitas. VS Code es ese auto deportivo al que puedes agregarle turbo, GPS, sistema de audio de carreras — pero solo cuando lo necesitas.

🟢 ¿Por qué no usar el Bloc de Notas?

  • Sin colores — todo se ve igual, los errores no se destacan
  • Sin autocompletado — tienes que escribir cada carácter manualmente
  • Sin detección de errores — sabrás que algo falló solo cuando ejecutes el código
  • Sin atajos útiles — cada tarea que en VS Code son 2 teclas, en el bloc son 30 clics

Descarga e instalación de VS Code
Desde cero, paso a paso · 8:09 – 14:12

Si ya tienes VS Code instalado y quieres empezar desde cero (sin configuraciones viejas), primero hay que resetear todo. Si es tu primera instalación, salta al paso 4.

Cómo resetear VS Code (instalación limpia)

🔄 Reset completo — borra toda configuración anterior Opcional
  1. Abre VS Code → presiona Ctrl + Shift + P para abrir la Paleta de Comandos.
  2. Escribe Open User Settings (JSON) y presiona Enter. Borra todo el contenido del archivo y guarda.
  3. Cierra VS Code.
  4. Abre el explorador de archivos y ve a la carpeta de usuario: C:\Users\TuNombre\AppData\Roaming\Code (en Windows). Borra la carpeta Code completa.
  5. Desinstala VS Code desde Panel de Control → Programas → Desinstalar.

Ahora sí, instalación limpia.

Instalación paso a paso

Ve a code.visualstudio.com en tu navegador. Es la página oficial de Microsoft.

Haz clic en "Download for Free". El botón detecta automáticamente tu sistema operativo (Windows, Mac o Linux).

Ejecuta el instalador descargado. Acepta el acuerdo de licencia.

Durante la instalación, marca la opción "Agregar al PATH" y "Crear acceso directo en el escritorio". Esto te permitirá abrir VS Code desde la terminal escribiendo code .

Haz clic en Instalar y espera que termine.

Abre VS Code. Al primer arranque te preguntará qué tema de color prefieres. El curso recomienda Dark Modern: buen contraste, no cansa la vista.

🏠 Analogía: mudarte a una casa nueva

Instalar VS Code es como mudarte a una casa vacía. La casa ya tiene todo lo estructural (paredes, puertas, ventanas), pero aún está sin muebles. En los siguientes pasos vas a decorarla: agregarás extensiones (muebles), configurarás temas (pintura), y crearás tus propios atajos (costumbres del hogar). Al final será tu casa, no la de nadie más.

🟢 Sobre el idioma — no lo cambies a español

Dalto lo dice claro en el curso: no cambies VS Code al español. El 99% de la documentación, tutoriales, errores y soluciones en internet están en inglés. Si usas VS Code en inglés desde el principio, vas a entender el lenguaje técnico antes y vas a poder buscar soluciones mucho más fácil cuando algo falle.

Interfaz de VS Code — Conocer el taller
Cada parte de la pantalla y para qué sirve · 14:12 – 21:29

VS Code tiene una interfaz pensada para que todo esté a uno o dos clics de distancia. Entender qué es cada parte te ahorra horas de confusión.

① Activity Bar — Barra vertical izquierda. Los íconos principales del editor.
② Side Bar — Panel que se abre al hacer clic en un ícono de la Activity Bar.

④ Panel — Parte inferior: Terminal, Problemas, Output, Debug Console.
⑤ Status Bar — Barra del fondo: idioma del archivo, línea/columna, Git, errores.

💡 La Paleta de Comandos — el control central

Presiona Ctrl + Shift + P en cualquier momento. Se abre una caja de búsqueda donde puedes ejecutar cualquier comando de VS Code escribiendo su nombre en inglés. Si alguna vez no sabes dónde está algo, ábrela y escribe lo que buscas. Es el GPS del editor.

✈️ Analogía: la cabina del avión

Un avión tiene docenas de botones, palancas e indicadores. Al principio parece imposible. Pero los pilotos aprenden que hay instrumentos que usan todo el tiempo (altímetro, velocímetro) y otros que usan solo en situaciones específicas. VS Code es igual: la mayoría del tiempo solo usas el Explorer y el Editor. El resto lo aprendes cuando lo necesitas, y la Paleta de Comandos es la cabina de control desde donde puedes acceder a todo.

La Status Bar — el panel de información

La barra de abajo del todo te dice en todo momento:

  • Rama de Git activa — ej. main o feature/login
  • Errores y advertencias — número en rojo/amarillo, clic para verlos
  • Línea y columna — dónde está el cursor exactamente
  • Codificación — normalmente UTF-8 (acepta tildes, ñ, emojis)
  • Saltos de línea — LF (Linux/Mac) o CRLF (Windows)
  • Lenguaje del archivo — clic para cambiar el resaltado de sintaxis

⚠️ LF vs CRLF — el problema invisible

Windows usa dos caracteres para los saltos de línea (CRLF), mientras que Linux/Mac usan uno solo (LF). Si subes un archivo creado en Windows a un servidor Linux, puede dar errores extraños. Lo más seguro: configura VS Code para que siempre use LF. En Settings busca files.eol y ponlo en \n.

El Explorer es tu vista de archivos y carpetas. Siempre deberías trabajar con una carpeta abierta (no archivos sueltos), porque así VS Code puede ayudarte con rutas, importaciones y Git.

💡 Indentación — la presentación del código

Por defecto VS Code usa 4 espacios por nivel de indentación. Para web (HTML/CSS/JS), la convención más extendida son 2 espacios. Puedes cambiarlo en Settings → editor.tabSize. La indentación no afecta cómo funciona el código (excepto en Python), pero sí afecta muchísimo cómo se lee.

💡 UTF-8 — por qué tu ñ no aparece rota

UTF-8 es el estándar de codificación que permite usar tildes (á, é, í, ó, ú), la ñ, emojis y caracteres de cualquier idioma. VS Code lo usa por defecto. Si alguna vez abres un archivo y ves símbolos raros como Ã± en vez de ñ, el archivo fue guardado con otra codificación. En la Status Bar haz clic en UTF-8 y elige "Reopen with encoding".

🟢 El punto en el nombre del archivo

Cuando abres VS Code y un archivo tiene cambios sin guardar, aparece un punto • blanco en la pestaña. Es pequeño pero importante. Acostúmbrate a mirar ese punto y a guardar seguido con Ctrl+S. O activa el guardado automático (lo veremos en Settings).

El panel de Search no solo busca texto — puede buscar en todos los archivos del proyecto a la vez, distinguir mayúsculas, buscar palabras completas, usar expresiones regulares, y reemplazar texto en masa con un solo clic.

📚 Analogía: el buscador de libros

Con Search Global + Reemplazar todo, en 10 segundos cambia en los 50 archivos simultáneamente. Es como tener un asistente que lee todos los libros de la biblioteca y tach tacha la palabra en cada página.

🟢 Word Wrap — ver líneas largas sin scroll horizontal

Si tienes líneas muy largas que se van de la pantalla, presiona Ctrl+Shift+P → escribe word wrap → Toggle Word Wrap. El texto se partirá visualmente para ajustarse a la ventana (sin afectar el archivo real). También puedes activarlo permanentemente en Settings → editor.wordWrap: "on".

Source Control — Git integrado en VS Code
Control de versiones sin salir del editor · 43:02 – 51:50

Git es el sistema de control de versiones más usado en el mundo. VS Code lo tiene integrado de forma nativa — puedes hacer commits, crear ramas, subir a GitHub y sincronizar cambios sin abrir ninguna terminal.

📸 Analogía: el álbum de fotos de tu código

Imagina que cada vez que avanzas en tu proyecto tomas una foto del estado actual. Si algo sale mal, puedes volver a cualquier foto anterior. Eso es exactamente lo que hace Git: cada commit es una foto del código en ese momento. Y GitHub es el álbum en la nube donde guardas todas esas fotos para que no se pierdan aunque se rompa tu computadora.

⚠️ Aprende Git antes de continuar

Dalto lo dice explícitamente en el curso: si no sabes Git, aprende Git primero. Esta sección de VS Code asume que ya conoces los conceptos básicos (init, add, commit, push, pull, branches). Si no, busca el curso de Git de Soy Dalto en YouTube antes de seguir aquí.

Flujo básico con Git desde VS Code

Inicializar repositorio: En el panel Source Control, haz clic en "Initialize Repository". Equivale a git init en la terminal.

Ver cambios: Cada archivo modificado aparece en la lista. Una M significa "Modified", una U significa "Untracked" (nuevo).

Hacer staging: Haz clic en el + junto al archivo para agregarlo al staging. Equivale a git add archivo.html.

Escribir el commit: En el campo de texto de arriba, escribe un mensaje descriptivo. Ej: "Agrego formulario de contacto".

Hacer commit: Haz clic en el botón azul "Commit" o presiona Ctrl+Enter. Equivale a git commit -m "mensaje".

Subir a GitHub: Haz clic en "Publish Branch" la primera vez (te pedirá autenticarte con GitHub). Después usa "Sync Changes" para hacer push/pull.

Operaciones avanzadas desde VS Code

  • Crear/cambiar ramas: Clic en el nombre de la rama en la Status Bar (abajo a la izquierda)
  • Ver historial de commits: Explorer → Timeline (parte inferior del panel)
  • Clonar repositorio: Ctrl+Shift+P → "Git: Clone" → pega la URL del repositorio
  • Git fetch automático: Settings → git.autofetch: true — mantiene actualizado el repositorio local

🟢 Git Bash como terminal por defecto

Si instalaste Git para Windows, puedes configurar VS Code para usar Git Bash como terminal integrada. Ctrl+Shift+P → "Terminal: Select Default Profile" → elige "Git Bash". Así puedes escribir comandos de Linux/Unix directamente en VS Code sin instalar nada extra.

Run & Debug — Encontrar errores como un detective
Ejecutar y depurar código paso a paso · 51:50 – 1:00:20

El Debugger (depurador) es una de las herramientas más poderosas de VS Code. Te permite ejecutar el código línea por línea, pausarlo en cualquier punto, y ver exactamente qué valor tiene cada variable en ese preciso instante.

🔬 Analogía: el microscopio del código

Cuando el código falla y no sabes por qué, tienes dos opciones: poner console.log en todos lados (como buscar bacterias a simple vista) o usar el debugger (como usar un microscopio). El microscopio te muestra exactamente dónde está el problema, con precisión total, sin contaminar el código con logs que después tienes que borrar.

Conceptos clave del Debugger

🔴
Breakpoint (punto de interrupción)
Una marca en una línea de código donde la ejecución se pausará. Haz clic en el número de línea (a la izquierda) para activarlo — aparece un punto rojo.
⏭
Step Over
Ejecuta la línea actual y pasa a la siguiente, sin entrar en funciones. El atajo es F10.
⬇️
Step Into
Entra dentro de la función que se llama en la línea actual para ver qué hace por dentro. El atajo es F11.
▶️
Continue
Reanuda la ejecución hasta el siguiente breakpoint (o hasta el final si no hay más). El atajo es F5.
🔁
Restart
Reinicia la depuración desde el principio sin cerrar la sesión. Útil cuando corriges un error y quieres volver a probar.
📊
Variables Panel
Panel izquierdo que muestra el valor actual de todas las variables en el momento en que está pausada la ejecución.

Ejemplo práctico — depurar JavaScript

JavaScript// Ejemplo con un error intencional function sumar(a, b) { return a + b; // ← pon un breakpoint aquí (clic en el número de línea) } let resultado = sumar(3, 4); console.log(resultado);
✏️ Ejercicio — Tu primera sesión de debugging Principiante
  1. Crea un archivo index.html con la estructura básica y enlaza un main.js.
  2. En main.js, escribe el código del ejemplo de arriba.
  3. Haz clic en el número de la línea return a + b para poner un breakpoint (aparece un punto rojo).
  4. Ve al panel Run & Debug (Ctrl+Shift+D) y haz clic en "Run and Debug".
  5. Elige "Chrome" como entorno de ejecución.
  6. La ejecución se pausará en el breakpoint. En el panel izquierdo verás que a = 3 y b = 4.
  7. Presiona F10 para avanzar línea por línea y ver cómo cambian las variables.

Resultado esperado: Ves en tiempo real cómo resultado pasa de undefined a 7 cuando se ejecuta la función.

Extensions — Superpoderes para tu editor
Instalar, configurar y las extensiones esenciales · 1:00:20 – 1:11:48

Las extensiones son mini-programas que se agregan a VS Code para darle nuevas funcionalidades. Hay más de 40,000 extensiones gratuitas en el Marketplace. Con ellas, VS Code se convierte en el editor exactamente que tú necesitas.

🛠 Analogía: las apps del celular

Tu celular nuevo viene con apps básicas: teléfono, cámara, mensajes. Pero tú le instalas WhatsApp, Instagram, Spotify — lo personalizas para tu uso. VS Code es igual: viene con lo esencial, y tú le instalas las "apps" (extensiones) que necesitas según qué tipo de proyectos haces. Un desarrollador web instala unas, un científico de datos instala otras.

Cómo instalar una extensión

Abre el panel de Extensions con Ctrl+Shift+X.

En la barra de búsqueda escribe el nombre de la extensión.

Haz clic en la extensión que aparece (verifica el autor para evitar fakes).

Haz clic en el botón azul "Install".

Algunas extensiones requieren reiniciar VS Code — si aparece un botón "Reload Required", haz clic en él.

Extensiones esenciales recomendadas por Dalto

Prettier
by Prettier

Formatea tu código automáticamente al guardar: indentación, espacios, comillas — todo queda ordenado y consistente sin que lo hagas manualmente.

🧹 Como tener un conserje que ordena tu escritorio cada vez que sales de la oficina. Tú escribes rápido y sucio, Prettier lo limpia.
Live Server
by Ritwick Dey

Lanza un servidor local y recarga el navegador automáticamente cada vez que guardas un archivo HTML/CSS/JS. Imprescindible para desarrollo web.

🪞 Como tener un espejo en tiempo real: cada cambio que haces se refleja al instante en el navegador sin tener que recargar manualmente.
GitLens
by GitKraken

Superpoderes para Git: muestra quién escribió cada línea de código, cuándo, y en qué commit. Visualiza el historial de ramas en forma de gráfico.

🕵️ Como el historial de ediciones de Google Docs pero para código: ves exactamente quién cambió qué y cuándo.
IntelliCode
by Microsoft

Autocompletado inteligente potenciado por IA. Aprende de millones de líneas de código para sugerir qué vas a escribir a continuación con más precisión que el IntelliSense básico.

🔮 Como un teclado predictivo que ha leído millones de programas y ya sabe qué función usas después de array.
Indent Rainbow
by oderwat

Colorea los diferentes niveles de indentación con colores distintos. Hace que la estructura anidada del código sea visualmente obvia de un vistazo.

🌈 Como tener niveles de un edificio pintados de diferentes colores: el sótano es rojo, planta baja azul, primer piso verde — nunca te pierdes.
Temas de color
varios autores

Cambian los colores del editor. Los más populares: One Dark Pro, Andromeda, Catppuccin, Dracula. El tema es personal — prueba varios y quédate con el que no te canse la vista.

🎨 Como pintar las paredes de tu oficina. Mismo trabajo, diferente ambiente. Elige el que te ponga de mejor humor para programar.

💡 Linked Editing — edita etiquetas HTML en pareja

Esta funcionalidad viene integrada en VS Code, sin necesidad de extensión. Actívala en Settings → busca Linked Editing → activa el toggle. Cuando edites una etiqueta de apertura (ej: cambias <div> por <section>), la etiqueta de cierre se actualiza automáticamente. Un ahorro enorme en HTML.

User Settings — Personaliza tu entorno
Configurar VS Code a tu medida · 1:11:48 – 1:26:34

Las Settings (configuraciones) de VS Code son un archivo JSON donde defines cómo debe comportarse el editor. Piénsalas como el CSS de tu editor: cambias propiedades para adaptar la apariencia y el comportamiento a lo que tú necesitas.

Cómo abrir Settings

  • Interfaz visual: Ctrl+, — abre la pantalla de configuración con campos y toggles
  • JSON directo: Ctrl+Shift+P → "Open User Settings (JSON)" — editas el archivo de configuración directamente

Configuraciones recomendadas por Dalto

JSON — settings.json{ // ── EDITOR ──────────────────────────────── "editor.fontSize": 16, // Tamaño de fuente (defecto: 14) "editor.tabSize": 2, // 2 espacios por tabulación (web estándar) "editor.wordWrap": "on", // Líneas largas se parten visualmente "editor.cursorBlinking": "smooth", // Cursor con animación suave "editor.smoothScrolling": true, // Scroll con animación "editor.linkedEditing": true, // Editar etiquetas HTML en pareja "editor.renderLineHighlight": "line", // Resaltar línea actual // ── GUARDADO AUTOMÁTICO ─────────────────── "files.autoSave": "afterDelay", // Guarda automáticamente "files.autoSaveDelay": 3000, // ... cada 3 segundos "files.eol": "\n", // Saltos de línea estilo Linux (LF) // ── FORMATEO ────────────────────────────── "editor.formatOnSave": true, // Prettier formatea al guardar "editor.defaultFormatter": "esbenp.prettier-vscode", // ── TERMINAL ────────────────────────────── "terminal.integrated.fontSize": 14, "terminal.integrated.defaultProfile.windows": "Git Bash", // ── SNIPPETS ────────────────────────────── "editor.snippetSuggestions": "top" // Tus snippets aparecen primero }

💡 Breadcrumbs — las migas de pan del código

Activa "breadcrumbs.enabled": true en Settings. Aparece una barra encima del editor que muestra la ruta completa de donde estás: carpeta → archivo → función → clase. Muy útil cuando trabajas con archivos grandes o proyectos con muchos archivos anidados.

🟢 Editor Rules — las guías de ancho

Puedes poner líneas verticales en el editor para recordarte cuándo una línea es demasiado larga. Agrega en settings: "editor.rulers": [80, 120]. La convención más usada es que las líneas de código no superen los 80 o 120 caracteres de ancho — más que eso se vuelve difícil de leer.

Atajos de teclado — Programar a velocidad de pensamiento
Los shortcuts más importantes del curso · 1:26:34 – 1:39:01

Los atajos de teclado son la diferencia entre un programador lento y uno rápido. No los memorices todos de golpe — aprende 3 por semana y en dos meses los tendrás todos incorporados.

🎹 Analogía: el músico y los acordes

Un pianista principiante busca cada tecla con los ojos. Un pianista experimentado toca sin mirar porque sus dedos ya memorizaron los movimientos. Los atajos de teclado funcionan igual: al principio tienes que pensar en ellos, con la práctica se vuelven reflejos automáticos y tu velocidad de trabajo se multiplica.

Abre la Paleta de Comandos → "Open Keyboard Shortcuts". Puedes buscar cualquier acción y asignarle la combinación de teclas que quieras. Si alguna combinación ya está ocupada, VS Code te avisa del conflicto.

🟢 Consejo de Dalto sobre los atajos

No te obsesiones con aprenderlos todos. Los atajos más valiosos son los que usas todos los días: Ctrl+S, Ctrl+Z, Ctrl+/, Alt+↑↓ y Ctrl+D. Con esos cinco ya eres notablemente más rápido. Los demás los incorporarás naturalmente con el tiempo.

Snippets — Plantillas de código a medida
Crea tus propios atajos de código reutilizable · 1:39:01 – 1:49:07

Un snippet es un fragmento de código que puedes insertar escribiendo una palabra clave y presionando Tab. Son como sellos de goma para código: en vez de escribir 20 líneas repetidas, escribes 3 letras y Tab.

🖊 Analogía: las respuestas predefinidas del chat

Los agentes de servicio al cliente tienen respuestas predefinidas para preguntas frecuentes: en vez de escribir "Estimado cliente, lamentamos el inconveniente..." cada vez, hacen clic en un botón y sale la respuesta completa. Los snippets son exactamente eso para el código: escribes el "atajo" y sale el bloque completo que usas una y otra vez.

Crear un snippet personalizado

Abre la Paleta de Comandos (Ctrl+Shift+P) → escribe "Configure User Snippets".

Elige el lenguaje para el snippet: html, javascript, python, etc. O elige "New Global Snippets File" para que funcione en todos los lenguajes.

Se abre un archivo JSON. Escribe tu snippet siguiendo la estructura de abajo.

Guarda el archivo. Ya puedes usar el snippet en cualquier archivo del lenguaje que elegiste.

Estructura de un snippet

JSON{ "Estructura HTML básica": { "prefix": "htmlbase", // La palabra que escribes para activarlo "body": [ // El código que se inserta (array de líneas) "<!DOCTYPE html>", "<html lang=\"es\">", "<head>", " <meta charset=\"UTF-8\">", " <title>$1</title>", // $1 = primer campo donde va el cursor "</head>", "<body>", " $2", // $2 = segundo cursor (Tab salta aquí) "</body>", "</html>" ], "description": "Estructura HTML completa desde cero" } }

✏️ Ejercicio — Crea tu primer snippet Principiante
  1. Ctrl+Shift+P → "Configure User Snippets" → elige html.
  2. Borra el comentario de ejemplo y escribe este snippet:
JSON{ "Enlace con clase": { "prefix": "enlace", "body": ["<a href=\"${1:#}\" class=\"${2:btn}\">${3:texto}</a>"], "description": "Enlace HTML con href y class" } }
  1. Guarda el archivo (Ctrl+S).
  2. Abre cualquier archivo HTML, escribe enlace y presiona Tab.
  3. Aparece el enlace completo. El cursor está en el href. Escribe la URL, presiona Tab para ir a la clase, Tab de nuevo para el texto.

💡 Snippets vs Emmet — ¿cuándo usar cada uno?

Emmet (la siguiente sección) es mejor para estructuras HTML dinámicas y variadas. Snippets son mejores para bloques que siempre repites igual: un comentario de sección, un console.log de debugging, una configuración que usas en todos tus proyectos. Ambas herramientas se complementan perfectamente.

Emmet — HTML y CSS a velocidad de rayo
El plugin que ya viene en VS Code y nadie usa al máximo · 1:49:07 – 2:07:37

Emmet es un plugin que viene integrado en VS Code (sin necesidad de instalarlo). Convierte abreviaciones cortas en bloques completos de HTML o CSS. Es una de las herramientas más poderosas para desarrollo web.

⚡ Analogía: el taquígrafo del código

Los taquígrafos escriben a velocidades imposibles usando símbolos abreviados que después se convierten en texto completo. Emmet es el taquígrafo del HTML: escribes ul>li*5 (15 caracteres) y Tab — aparecen 30 líneas de código completas. No es magia, es eficiencia.

Pair Programming & GitHub Copilot — Tu copiloto IA
Programar en pareja con inteligencia artificial · 2:07:37 – 2:17:20

El pair programming (programación en pares) es una técnica donde dos programadores trabajan juntos: uno escribe el código (el Driver) y el otro lo revisa, sugiere y detecta errores (el Copiloto). Hoy, la IA puede ocupar el rol de copiloto.

✈️ Analogía: el piloto y el copiloto

En un avión, el piloto conduce pero el copiloto revisa los instrumentos, lee los procedimientos y alerta sobre problemas. Tú eres el piloto: tomas las decisiones, escribes el código, defines el rumbo. GitHub Copilot es el copiloto: observa lo que escribes, anticipa lo que necesitas y te alerta cuando algo podría estar mal. Ninguno reemplaza al otro — juntos vuelan más seguro.

¿Qué es GitHub Copilot?

GitHub Copilot es un asistente de IA desarrollado por GitHub y OpenAI. Analiza el código que estás escribiendo y el contexto de tu proyecto para sugerir completados de línea, funciones enteras e incluso archivos completos.

GitHub Copilot
by GitHub — Pago (prueba gratis 30 días)

El más potente. Sugiere código basándose en el contexto completo de tu proyecto. Puede escribir funciones enteras, explicar código seleccionado, y corregir errores automáticamente.

🏆 La opción premium. Más inteligente y contextual que las alternativas gratuitas.

by Codium — Gratuito

Alternativa gratuita a Copilot. Genera tests automáticamente, sugiere código y analiza el comportamiento de las funciones. Buena opción para aprender sin pagar.

Code GPT
by Daniel Sanchez — Gratuito

Conecta VS Code con modelos de lenguaje (GPT-4, Claude, Gemini). Puedes hacer preguntas sobre tu código directamente desde el editor.

Cómo instalar GitHub Copilot

Abre Extensions (Ctrl+Shift+X) y busca "GitHub Copilot". Instala la extensión oficial de GitHub.

VS Code te pedirá que inicies sesión con tu cuenta de GitHub. Haz clic en "Sign in to GitHub".

Se abrirá el navegador para autorizar la conexión. Acepta los permisos.

De vuelta en VS Code, verás el ícono de Copilot en la Status Bar. Si tiene un tilde verde, está activo.

Escribe un comentario describiendo lo que quieres hacer (ej: // función que recibe un array y devuelve el promedio) y presiona Enter. Copilot sugerirá el código completo — presiona Tab para aceptar.

⚠️ Importante — el copiloto no es el piloto

Dalto lo enfatiza: no copies el código de Copilot sin entenderlo. Si aceptas sugerencias que no comprendes, construyes sobre bases que no controlas y cuando algo falla no sabes por qué. Usa Copilot para acelerar código que ya sabes hacer, no para saltarte el aprendizaje.

💡 El futuro de la programación con IA

Los asistentes de IA van a ser cada vez más comunes en el desarrollo. La habilidad que vale no es "escribir código rápido" — esa la tiene la IA. La habilidad que vale es saber qué pedirle, entender lo que genera, detectar sus errores, y decidir cuándo no usarla. El programador del futuro es el que sabe dirigir la IA, no el que la teme.
