La filosofía de los proyectos finales
Por qué construir proyectos reales es lo más importante del aprendizaje

Las partes A–I te dieron todas las herramientas. Esta parte es diferente: aquí no aprendes herramientas nuevas, aprendes a tomar decisiones. En un proyecto real no hay una respuesta correcta única. Tienes que elegir cuándo usar Grid vs Flexbox, cuándo una variable CSS vale la pena, cómo organizar el código para que sea mantenible.

Antes de empezar cualquier proyecto: el workflow

  1. Define el objetivo y el usuario ¿Qué hace esta página? ¿A quién va dirigida? ¿Qué acción quiero que haga el usuario?
  2. Diseña la estructura HTML primero Sin CSS. Escribe el HTML semántico completo: header, main, sections, article, aside, footer. Si el HTML tiene sentido sin CSS, vas bien.
  3. Define el sistema de diseño en :root Variables de colores, tipografía, espaciado, sombras, radios. Todo antes de escribir un solo selector.
  4. Construye mobile-first Los estilos base son para móvil. Usa min-width para expandir a tablet y desktop.
  5. Layout con Grid y Flexbox Grid para la estructura de página, Flexbox para los componentes internos.
  6. Añade interactividad sutil Transitions en hover, animaciones de entrada, skeleton screens. Siempre con prefers-reduced-motion.
  7. Verifica accesibilidad Alt en imágenes, labels en formularios, contraste suficiente, navegación con teclado.
  8. Optimiza y publica Imágenes en WebP con srcset, fuentes con preconnect, un archivo HTML bien organizado con comentarios.

🟢 Lo más importante de una landing page

  • El CTA (Call to Action) debe verse sin hacer scroll. Botón claro, texto de acción ("Empezar gratis", "Probar 14 días", no "Click aquí")
  • Jerarquía visual: h1 para el beneficio principal, subtítulo que explica cómo, botón de acción
  • Social proof: logos de clientes, testimonios con foto real, número de usuarios
  • La página debe funcionar perfectamente en móvil. El 60%+ del tráfico es móvil.

🟢 Consejos para un portafolio que funciona

  • Máximo 3–5 proyectos, bien presentados, mejor que 15 mediocres
  • Cada proyecto debe tener: captura de pantalla, descripción breve, tecnologías usadas, link al demo y al código
  • El tiempo de carga es parte del portafolio — optimiza las imágenes
  • Incluye un CV descargable en PDF con un link download
  • El formulario de contacto es importante — usa Formspree o EmailJS para que funcione sin backend

⚠️ Clon con fines educativos únicamente

El proyecto de clon de app (Spotify, Netflix, etc.) es solo para practicar CSS y HTML. Nunca publiques un clon de una app comercial en internet ni uses logos, imágenes o marcas registradas reales. El objetivo es aprender a replicar layouts complejos, no crear una copia de un producto.

🟢 Regla de los componentes independientes

Cada componente (navbar, card, modal, etc.) debe funcionar correctamente sin importar dónde lo pongas en la página. Un componente bien escrito no depende de su padre para verse bien. Esto es lo que hace que el código sea reutilizable y mantenible a largo plazo.

¿Y ahora qué? — El camino después del curso
Dónde ir desde aquí para seguir creciendo como desarrollador

Completar este curso significa que dominas los fundamentos de la web. HTML y CSS ya no son un obstáculo para ti — son herramientas que puedes usar. Pero el desarrollo web es un campo enorme. Aquí están los pasos más naturales:

⚡ JavaScript

El siguiente paso natural. Con JS puedes hacer tus páginas interactivas: formularios que validan, menús hamburguesa, dark mode con toggle, APIs para datos reales. Soy Dalto también tiene un curso completo de JS.

⚛️ React (o Vue)

Después de JS, un framework de componentes. React es el más demandado en el mercado laboral. Todo lo que aprendiste de CSS y HTML se usa directamente en React — los componentes son HTML y CSS con superpoderes.

🎨 Diseño UI/UX

Figma es la herramienta estándar de diseño. Aprender a leer y crear diseños en Figma te hace mucho más eficiente desarrollando y te permite colaborar con diseñadores profesionales.

🚀 Publicar proyectos

Vercel y Netlify permiten publicar páginas HTML/CSS gratis en segundos. Cada proyecto que publiques y pongas en tu portafolio vale más que cualquier certificado. El código público en GitHub habla por ti.

💼 Freelancing

Con lo que sabes de HTML y CSS ya puedes hacer landings y portafolios para clientes. Empieza con proyectos pequeños para amigos o negocios locales. La experiencia real vale más que cualquier práctica.

📚 Recursos de referencia

MDN Web Docs es tu biblia (developer.mozilla.org). CSS Tricks para técnicas. web.dev para mejores prácticas de Google. caniuse.com para ver qué navegadores soportan qué. Úsalos continuamente.
