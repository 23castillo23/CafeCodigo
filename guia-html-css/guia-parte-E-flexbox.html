Flexbox — El sistema de layout de una dimensión
Contenedor y elementos hijos: los dos mundos de Flexbox

Flexbox (Flexible Box Layout) es un sistema de CSS que organiza elementos en una sola dimensión: una fila o una columna. Lo activas con display: flex en el elemento padre y, a partir de ese momento, todos sus hijos directos se convierten en flex items que puedes controlar con precisión.

🚂 Analogía: el tren con vagones

El contenedor flex es la locomotora: decide la dirección del tren (¿va hacia adelante o hacia atrás?), cómo se distribuyen los vagones (¿pegados, separados, centrados?) y qué altura tienen. Los flex items son los vagones: pueden crecer para llenar espacio vacío, encogerse si hay poco espacio, o mantener un tamaño fijo. Y cada vagón puede pedir estar en un lugar diferente del tren.

Los dos mundos de Flexbox

/* Propiedades del CONTENEDOR (el padre) */
.padre {
  display: flex;Activa Flexbox
  flex-direction: row;Dirección del eje principal
  flex-wrap: nowrap;¿Se rompe en varias líneas?
  justify-content: flex-start;Distribución en eje principal
  align-items: stretch;Alineación en eje cruzado
  align-content: normal;Alineación de líneas (con wrap)
  gap: 0;Espacio entre hijos
}

/* Propiedades de los HIJOS (flex items) */
.hijo {
  flex-grow: 0;¿Puede crecer para llenar espacio?
  flex-shrink: 1;¿Puede encogerse si no hay espacio?
  flex-basis: auto;Tamaño base antes de grow/shrink
  flex: 0 1 auto;Shorthand de los tres anteriores
  align-self: auto;Sobreescribe align-items solo para este hijo
  order: 0;Orden visual (sin cambiar el HTML)
}

Los dos ejes de Flexbox

Flexbox tiene dos ejes que cambian según la dirección:

Eje principal (Main Axis)

La dirección en la que fluyen los flex items. Si flex-direction: row, el eje principal va de izquierda a derecha. Si es column, de arriba a abajo.

Lo controla: justify-content

Eje cruzado (Cross Axis)

El eje perpendicular al principal. Si el eje principal es horizontal, el cruzado es vertical (y viceversa).

Lo controla: align-items / align-self

⚠️ El error más común de Flexbox

Confundir justify-content con align-items. Recuerda: justify siempre controla el eje principal (donde fluyen los elementos), y align siempre controla el eje cruzado (perpendicular). Si cambias flex-direction, los ejes se invierten y por lo tanto también lo que hacen justify y align.

flex-direction — La dirección del tren
¿Los elementos van en fila o en columna? ¿Normal o al revés?

flex-direction define el eje principal: la dirección en la que los flex items se colocan dentro del contenedor. Tiene cuatro valores posibles.

🟢 ¿Cuándo usar column?

Usa flex-direction: column cuando necesitas apilar elementos verticalmente con control fino sobre su alineación y espaciado: menús verticales, tarjetas que deben tener el botón siempre al fondo, formularios, sidebars. Para layouts de página completa (columnas lado a lado), usa Grid.

justify-content — Distribuir en el eje principal
Cómo se distribuye el espacio libre entre los flex items

justify-content controla cómo se distribuyen los flex items a lo largo del eje principal. Solo tiene efecto cuando hay espacio libre en ese eje (es decir, cuando los items no llenan todo el contenedor).

CSS.contenedor { display: flex; justify-content: flex-start; /* Items al inicio (por defecto) */ justify-content: flex-end; /* Items al final */ justify-content: center; /* Items centrados */ justify-content: space-between; /* Espacio entre items, sin espacio en bordes */ justify-content: space-around; /* Espacio alrededor de items (bordes = mitad) */ justify-content: space-evenly; /* Espacio igual entre items Y en bordes */ }

💡 space-between vs space-around vs space-evenly

between: el espacio va entre los items. Los extremos tocan los bordes del contenedor. Ideal para navbars.
around: cada item tiene el mismo espacio a sus lados, pero los espacios de dos items adyacentes se suman, haciendo el espacio interno el doble del externo.
evenly: todos los espacios (incluidos bordes) son iguales. El más "matemáticamente perfecto".

align-items y align-content — El eje cruzado
Alinear verticalmente (o perpendicular al eje principal)

align-items controla cómo se alinean los flex items en el eje cruzado (perpendicular al principal). Si la dirección es row, el eje cruzado es vertical. Si es column, el eje cruzado es horizontal.

align-content — cuando hay varias filas (con wrap)

align-content solo funciona cuando hay flex-wrap: wrap y los items se distribuyen en múltiples líneas. Controla cómo se distribuyen esas líneas en el eje cruzado, igual que justify-content pero perpendicular.

CSS.contenedor { display: flex; flex-wrap: wrap; /* Necesario para que align-content funcione */ align-content: flex-start; /* Líneas pegadas arriba */ align-content: center; /* Líneas centradas */ align-content: space-between;/* Espacio entre líneas */ }
🔀
flex-wrap — ¿Se rompe en varias líneas?
Controlar qué pasa cuando los items no caben en una sola línea

Por defecto, Flexbox intenta meter todos los items en una sola línea, encogiendo los items si es necesario. Con flex-wrap: wrap le dices que si no caben, que pase los sobrantes a la línea siguiente.

🥚 Analogía: la caja de huevos

Una caja de huevos normal tiene una fila de 6 slots. Si tienes 10 huevos, no los aplastas para que quepan: usas una segunda fila. Eso es flex-wrap: wrap: en vez de comprimir los items para que quepan en una línea, los pasa a la siguiente.

CSS.contenedor { display: flex; flex-wrap: nowrap; /* Una línea, items se encogen si no caben (por defecto) */ flex-wrap: wrap; /* Pasa a siguiente línea si no caben */ flex-wrap: wrap-reverse; /* Wrap pero las líneas van en orden inverso */ } /* Shorthand: flex-flow = flex-direction + flex-wrap */ .contenedor { flex-flow: row wrap; } /* Lo más común en layouts de cards */ .contenedor { flex-flow: column nowrap; } /* Para listas verticales */

flex-grow y flex-shrink — Crecer y encogerse
Cómo se distribuye el espacio sobrante (o faltante) entre los items

Estas dos propiedades controlan cómo reaccionan los flex items cuando hay espacio sobrante o cuando el espacio es insuficiente.

🎈 Analogía: globos en una cuerda

Imagina tres globos atados a una cuerda. El flex-grow es como la capacidad de inflarse: si un globo tiene flex-grow: 2 y los otros dos tienen flex-grow: 1, el primero se infla el doble que los demás con el aire disponible. El flex-shrink es lo contrario: si la cuerda es muy corta, ¿cuánto puede desinflarse cada globo?

flex-grow — repartir el espacio sobrante

CSS/* flex-grow: 0 = no crece (por defecto) */ /* flex-grow: 1 = crece para ocupar el espacio sobrante */ /* El número es una proporción — no píxeles ni porcentaje */ .item-a { flex-grow: 1; } /* Recibe 1 parte del espacio sobrante */ .item-b { flex-grow: 2; } /* Recibe 2 partes — el doble que item-a */ .item-c { flex-grow: 1; } /* Recibe 1 parte — igual que item-a */ /* Total: 4 partes. item-b ocupa el 50%, a y c el 25% cada uno */
flex-grow: 1 | 2 | 1 — el del medio recibe el doble de espacio libre

flex-shrink — encogerse cuando no hay espacio

CSS/* flex-shrink: 1 = puede encogerse (por defecto) */ /* flex-shrink: 0 = NO puede encogerse — mantiene su tamaño */ /* flex-shrink: 2 = se encoge el doble que un item con shrink:1 */ .logo { flex-shrink: 0; } /* El logo nunca se encoge */ .nav { flex-shrink: 1; } /* La nav puede encogerse si es necesario */ .sidebar { flex-shrink: 0; } /* El sidebar mantiene su ancho fijo */ .content { flex-grow: 1; } /* El contenido ocupa el espacio restante */

🟢 El patrón más útil: flex-shrink: 0 en elementos fijos

Cuando tienes una navbar con logo + links + botón, el logo y el botón normalmente tienen tamaño fijo (flex-shrink: 0) y los links en el centro pueden comprimirse. Esto evita que el logo se deforme en pantallas pequeñas.

📏
flex-basis y el shorthand flex
El tamaño inicial antes de que entren grow y shrink

flex-basis define el tamaño base del flex item antes de que flex-grow y flex-shrink ajusten el espacio. Es como el punto de partida desde el cual el item puede crecer o encogerse.

CSS.item { flex-basis: auto; } /* Usa width/height del elemento (por defecto) */ .item { flex-basis: 200px; } /* Empieza en 200px, luego crece/encoge */ .item { flex-basis: 30%; } /* Empieza en 30% del contenedor */ .item { flex-basis: 0; } /* Empieza en 0, todo su tamaño viene de flex-grow */ /* ─────────────────────────────────────────── */ /* El shorthand flex = grow shrink basis */ /* ─────────────────────────────────────────── */ .item { flex: 1; } /* = flex: 1 1 0 (crece y encoge, base 0) */ .item { flex: auto; } /* = flex: 1 1 auto */ .item { flex: none; } /* = flex: 0 0 auto (tamaño fijo, no crece ni encoge) */ .item { flex: 2 1 200px; } /* Crece el doble, puede encoger, base 200px */ /* Patrón sidebar + contenido principal */ .sidebar { flex: 0 0 260px; } /* Fijo en 260px, no crece ni encoge */ .contenido{ flex: 1; } /* Ocupa todo el espacio restante */

💡 ¿Cuándo usar flex: 1?

flex: 1 es el valor más usado en el día a día. Significa: "crece para llenar el espacio disponible". Si pones flex: 1 en todos los items, se reparten el espacio en partes iguales. Si lo pones solo en uno, ese uno ocupa todo el espacio que los demás no usan.

🎯
align-self — La excepción individual
Sobreescribir align-items solo para un hijo específico

El contenedor define con align-items cómo se alinean todos sus hijos en el eje cruzado. Pero con align-self en un hijo específico puedes decir "tú te alineas diferente al resto". Es como una excepción individual.

🏋️ Analogía: el equipo de formación y el jugador estrella

El entrenador (contenedor) le dice a todo el equipo: "todos se colocan en la línea del medio". Pero el portero (un hijo con align-self) dice "yo me quedo al fondo". align-self es ese jugador que tiene su propia posición independiente de la instrucción general del entrenador.

CSS.contenedor { display: flex; align-items: flex-start; /* Regla general: todos arriba */ } .hijo-especial { align-self: flex-end; /* Este va abajo, ignorando la regla general */ } .hijo-centrado { align-self: center; /* Este va al centro */ } .hijo-estirado { align-self: stretch; /* Este se estira al alto total */ }

order — Reordenar visualmente sin tocar el HTML
Cambiar el orden de aparición de los flex items solo con CSS

Por defecto, los flex items aparecen en el mismo orden que en el HTML. Con order puedes cambiar ese orden visualmente sin modificar el HTML. El valor por defecto es 0; items con menor número aparecen primero.

🎬 Analogía: el orden de créditos en una película

En una película, los actores están en el set en cierto orden, pero en los créditos finales puedes ordenarlos como quieras: "primero el protagonista, luego el director, luego el reparto". El HTML es el set de filmación; el CSS con order es el editor que decide en qué orden aparecen en pantalla.

CSS/* HTML original: A, B, C */ /* Con order, aparecen: B (order:1), C (order:2), A (order:3) */ .item-a { order: 3; } /* Aparece de último */ .item-b { order: 1; } /* Aparece de primero */ .item-c { order: 2; } /* Aparece de segundo */
En el HTML están en orden A→B→C, pero order los reordena visualmente:
A
order:3
B
order:1
C
order:2

Resultado visual: B · C · A — aunque en el HTML el orden sigue siendo A, B, C

⚠️ order y la accesibilidad

order cambia el orden visual pero no el orden del DOM. Los lectores de pantalla y la navegación con Tab siguen el orden del HTML, no el visual. Úsalo para ajustes visuales menores (como en responsive), pero no para cambios de orden que afecten la comprensión del contenido.

🟢 Uso real de order: responsive mobile-first

En desktop, un artículo tiene: [imagen | texto]. En móvil quieres [texto | imagen] (el texto primero). Con order puedes hacerlo sin cambiar el HTML: la imagen tiene order: 2 en móvil y vuelve a su orden natural en desktop. Perfecto combinado con media queries.

📦
gap — El espacio entre flex items
La forma moderna y limpia de separar elementos

gap define el espacio entre los flex items. Es mucho más limpio que usar margin en los hijos porque no agrega espacio extra en los bordes externos del contenedor.

CSS.contenedor { display: flex; gap: 1rem; /* Mismo espacio horizontal y vertical */ gap: 1rem 2rem; /* vertical horizontal (filas columnas) */ row-gap: 1rem; /* Solo espacio entre filas */ column-gap: 1.5rem; /* Solo espacio entre columnas */ } /* ❌ La forma antigua con margin — problemática */ .item { margin-right: 1rem; } .item:last-child { margin-right: 0; } /* Tenías que quitar el margen del último */ /* ✅ La forma moderna con gap — sin excepciones */ .contenedor { gap: 1rem; }
gap: 1rem — espacio limpio entre todos los items

Ejercicio integrador — App de resultados deportivos
Construye una interfaz completa usando solo Flexbox
✏️ Ejercicio — Pantalla de resultados deportivos Avanzado

Crea un archivo resultados.html con CSS en el mismo archivo. La pantalla debe estar construida exclusivamente con Flexbox (sin Grid por ahora).

  1. Navbar con justify-content: space-between: logo a la izquierda, links en el centro (display:flex; gap:1.5rem), botón a la derecha. Usa flex-shrink: 0 en logo y botón.
  2. Hero horizontal: Dos equipos enfrentados. Usa Flexbox con align-items: center; justify-content: center; gap: 2rem. En el centro el marcador, a los lados el logo y nombre de cada equipo.
  3. Barra de stats (posesión, tiros, tarjetas): Tres columnas con flex: 1 cada una. Cada stat tiene un número grande arriba, una barra de progreso y la etiqueta abajo. Haz la barra de progreso con dos divs flex.
  4. Grid de partidos recientes: Mínimo 6 tarjetas de partidos usando flex-wrap: wrap y flex: 1 1 240px en cada tarjeta. Cada tarjeta debe tener el botón al fondo usando flex-direction: column y margin-top: auto.
  5. Layout principal: Sidebar izquierdo (flex: 0 0 200px) con la lista de torneos, y el contenido principal a la derecha (flex: 1) que contiene el hero y las tarjetas.
  6. Footer: Tres columnas con flex: 1 cada una y justify-content: space-between.

Bonus: En el hero, usa order para que en móvil (con una media query) el marcador aparezca primero y los equipos debajo, usando flex-direction: column.
