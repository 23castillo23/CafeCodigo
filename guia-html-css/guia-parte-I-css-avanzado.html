filter — Efectos visuales sobre elementos
blur, brightness, contrast, grayscale, sepia, hue-rotate y más

La propiedad filter aplica efectos visuales a un elemento y todo su contenido: imágenes, texto, fondos — todo. Es como los filtros de Instagram pero directamente en CSS, sin necesitar Photoshop ni JavaScript.

📸 Analogía: los filtros de una cámara fotográfica

En fotografía, un filtro es un cristal que se pone delante del objetivo y modifica la luz antes de que llegue al sensor. El filtro de polarización oscurece, el de densidad neutra reduce la luz, el de color añade tonos. CSS filter es exactamente eso: un "cristal" que se pone delante del elemento y modifica cómo se ve, sin cambiar el elemento en sí.

Todos los filtros disponibles

brightness(n)Controla el brillo0=negro, 1=normal, 1.5=más brillante

Combinar filtros y animarlos

CSS/* Combinar varios filtros en una sola declaración */ .imagen { filter: grayscale(1) brightness(.8); } .imagen:hover { filter: grayscale(0) brightness(1.1); } /* Transición suave de filtros */ .foto { transition: filter .4s ease; } /* Patrón clásico para galerías: fotos en gris, color al hover */ .galeria img { filter: grayscale(1) opacity(.85); transition: filter .3s; } .galeria img:hover { filter: grayscale(0) opacity(1); } /* drop-shadow vs box-shadow — la diferencia crucial */ /* box-shadow: sombra del rectángulo del elemento */ .con-box-shadow { box-shadow: 4px 4px 12px rgba(0,0,0,.3); } /* drop-shadow: sombra que sigue los píxeles visibles — ideal para PNGs y SVGs */ .con-drop-shadow { filter: drop-shadow(4px 4px 12px rgba(0,0,0,.3)); }

backdrop-filter — El efecto vidrio esmerilado (glassmorphism)
Filtrar lo que está detrás del elemento, no el elemento mismo

Mientras que filter afecta al elemento y su contenido, backdrop-filter aplica el filtro al fondo que está detrás del elemento. Para que funcione, el elemento debe tener algo de transparencia (background con alpha o rgba).

🪟 Analogía: el vidrio esmerilado

Una puerta de vidrio esmerilado no cambia lo que está del otro lado, pero sí cómo lo ves: todo se ve borroso y difuminado a través del vidrio. backdrop-filter: blur() hace exactamente eso: lo que hay detrás del elemento se ve borroso, creando el efecto "glassmorphism" que es muy popular en diseño moderno.

CSS/* El truco del glassmorphism */ .glass-card { background: rgba(255, 255, 255, 0.15); /* Fondo semitransparente — obligatorio */ backdrop-filter: blur(12px) saturate(1.5); /* Desenfoca lo que hay detrás */ -webkit-backdrop-filter: blur(12px) saturate(1.5); /* Safari */ border: 1px solid rgba(255, 255, 255, 0.3); /* Borde sutil de luz */ border-radius: 16px; } /* Navbar con efecto vidrio al hacer scroll */ .navbar { background: rgba(255, 255, 255, 0.8); backdrop-filter: blur(20px); -webkit-backdrop-filter: blur(20px); border-bottom: 1px solid rgba(0, 0, 0, 0.1); } /* Fallback para navegadores sin soporte */ @supports not (backdrop-filter: blur(1px)) { .glass-card { background: rgba(255, 255, 255, 0.9); } }

transform — Mover, rotar, escalar y distorsionar
Todas las funciones de transform y cómo combinarlas

transform modifica la apariencia visual de un elemento sin afectar el layout. Los demás elementos no "ven" el transform: el espacio del elemento sigue siendo el mismo aunque visualmente se mueva o crezca. Por eso es ideal para animaciones.

CSS/* ── TRANSLATE — mover ── */ .elemento { transform: translate(20px, -10px); } /* x e y */ .elemento { transform: translateX(20px); } /* solo horizontal */ .elemento { transform: translateY(-10px); } /* solo vertical */ .elemento { transform: translateZ(50px); } /* hacia/lejos del espectador */ /* ── ROTATE — rotar ── */ .elemento { transform: rotate(45deg); } /* 2D: en el plano */ .elemento { transform: rotateX(30deg); } /* 3D: eje X (arriba/abajo) */ .elemento { transform: rotateY(45deg); } /* 3D: eje Y (izquierda/derecha) */ .elemento { transform: rotateZ(90deg); } /* = rotate() */ /* ── SCALE — escalar ── */ .elemento { transform: scale(1.2); } /* Uniforme: 20% más grande */ .elemento { transform: scaleX(1.5); } /* Solo ancho */ .elemento { transform: scaleY(.8); } /* Solo alto */ .elemento { transform: scale(-1); } /* Espejo (flip) */ /* ── SKEW — inclinar ── */ .elemento { transform: skewX(15deg); } /* Inclina horizontalmente */ .elemento { transform: skewY(5deg); } /* Inclina verticalmente */ /* ── COMBINAR — el orden importa ── */ .elemento { transform: translateY(-4px) scale(1.05) rotate(2deg); } /* Las transformaciones se aplican de derecha a izquierda */ /* Primero rota, luego escala, luego traslada */ /* transform-origin — el punto desde donde se aplica la transformación */ .elemento { transform-origin: center; } /* Por defecto: el centro */ .elemento { transform-origin: top left; } /* Desde la esquina superior izquierda */ .elemento { transform-origin: 0% 100%; } /* Desde la esquina inferior izquierda */

Transform 3D — Añadir profundidad a los elementos
perspective, rotateX/Y, translateZ y preserve-3d

CSS puede renderizar elementos en un espacio tridimensional. La clave es la propiedad perspective en el contenedor padre, que define desde qué distancia "ve" el espectador el elemento 3D. Menor perspective = efecto más dramático.

📷 Analogía: la distancia al cuadro

Si te acercas mucho a un cuadro en un museo, la perspectiva parece exagerada: los ángulos se distorsionan. Si te alejas, la perspectiva es más sutil. perspective: 300px es como estar muy cerca (efecto dramático), y perspective: 1200px es como estar lejos (efecto sutil).

CSS/* perspective va en el CONTENEDOR PADRE, no en el elemento */ .escena { perspective: 600px; /* La "distancia del espectador" */ perspective-origin: 50% 50%; /* Desde dónde mira (centro por defecto) */ } .carta { transform-style: preserve-3d; /* Los hijos mantienen posición 3D */ transition: transform .6s ease; } .carta:hover { transform: rotateY(180deg); /* Voltea como una tarjeta de presentación */ } /* Tarjeta con flip completo (técnica card flip) */ .carta-frente, .carta-reverso { position: absolute; inset: 0; backface-visibility: hidden; /* Oculta el reverso cuando da la vuelta */ } .carta-reverso { transform: rotateY(180deg); /* Empieza volteada, se muestra al flipear */ } /* translateZ: acercar/alejar del espectador */ .boton-3d:hover { transform: translateZ(20px); /* "Sale" hacia el espectador */ box-shadow: 0 20px 40px rgba(0,0,0,.3); }

Variables CSS — Custom Properties en profundidad
El sistema de diseño que hace tu CSS escalable y mantenible

Las custom properties (variables CSS) son valores reutilizables que defines una vez y usas en todo el CSS. A diferencia de las variables de Sass/Less, son nativas, viven en el DOM, se heredan, y pueden cambiarse en tiempo real con JavaScript.

🎨 Analogía: la paleta del pintor

Un pintor mezcla sus colores en una paleta antes de pintar. Si decide cambiar el azul por un azul más claro, cambia el color en la paleta y todos los trazos que hizo con ese color en el cuadro… bueno, en la vida real no cambian, pero en CSS sí: si cambias --primary: #c44a2a a --primary: #2a6cc4, todos los lugares donde usaste var(--primary) cambian automáticamente.

Sintaxis completa

/* Definir variables en :root (disponibles en todo el documento) */
:root {
  --nombre-variable: valor;siempre empieza con --
  --primary: #c44a2a;
  --font-size-base: 16px;
  --spacing-md: 1rem;
}

/* Usar variables con var() */
.boton {
  background: var(--primary);usar la variable
  padding: var(--spacing-md);
  color: var(--text-on-primary, white);valor de respaldo si no existe
}
CSS/* ── SISTEMA DE DISEÑO COMPLETO CON VARIABLES ── */ :root { /* Colores */ --color-primary: #c44a2a; --color-secondary: #1a7a5a; --color-accent: #b07010; --color-bg: #f7f4ef; --color-text: #1a1814; --color-muted: #8a8480; /* Tipografía */ --font-body: 'DM Sans', sans-serif; --font-mono: 'DM Mono', monospace; --text-sm: .875rem; --text-base: 1rem; --text-lg: 1.125rem; --text-xl: 1.25rem; --text-2xl: 1.5rem; /* Espaciado */ --space-1: .25rem; --space-2: .5rem; --space-4: 1rem; --space-8: 2rem; /* Bordes */ --radius-sm: 6px; --radius-md: 10px; --radius-lg: 16px; --radius-full: 9999px; /* Sombras */ --shadow-sm: 0 1px 3px rgba(0,0,0,.08); --shadow-md: 0 4px 16px rgba(0,0,0,.12); --shadow-lg: 0 12px 40px rgba(0,0,0,.18); /* Transiciones */ --transition-fast: .15s ease; --transition-base: .25s ease; --transition-slow: .4s ease; } /* Modo oscuro: solo sobreescribes las variables */ @media (prefers-color-scheme: dark) { :root { --color-bg: #0f0f0f; --color-text: #f0ece6; --color-muted: #6b6b6b; } } /* Variables locales — solo para un componente */ .tarjeta { --tarjeta-padding: 1.5rem; --tarjeta-radio: 12px; padding: var(--tarjeta-padding); border-radius: var(--tarjeta-radio); } /* Modificar variable con JavaScript */ // document.documentElement.style.setProperty('--color-primary', '#2a6cc4'); // const valor = getComputedStyle(root).getPropertyValue('--color-primary');

Variables en cálculos — el verdadero poder

CSS/* Variables + calc() = sistema de espaciado matemático */ :root { --base: 8px; } .elemento { margin: calc(var(--base) * 2); /* 16px */ padding: calc(var(--base) * 3); /* 24px */ gap: calc(var(--base) * 1.5); /* 12px */ } /* Variables para animaciones parametrizables */ :root { --velocidad: 1s; } .animado { animation-duration: var(--velocidad); } /* Con JS: element.style.setProperty('--velocidad', '0.3s'); */

calc(), min(), max() y clamp() — Matemáticas en CSS
Cálculos dinámicos que mezclan unidades y resuelven problemas reales

CSS tiene funciones matemáticas que permiten mezclar unidades distintas (px + %, rem + vw) y crear valores dinámicos. Son una de las herramientas más poderosas del CSS moderno.

calc() — Mezclar unidades en una sola expresión

🔧 Analogía: el carpintero que mide sobre la marcha

Un carpintero que quiere que una estantería ocupe todo el ancho de la pared menos 2cm de cada lado no usa un número fijo: mide el ancho de la pared (100%) y resta 4cm en total. calc(100% - 40px) hace exactamente eso: "todo el ancho del contenedor, menos 40px".

CSS/* calc() acepta +, -, *, / con espacios alrededor de + y - */ .sidebar { width: 260px; } .contenido{ width: calc(100% - 260px - 2rem); } /* Ancho - sidebar - gap */ /* Centrar un elemento con ancho conocido */ .centrado { margin-left: calc(50% - 150px); } /* Mitad menos la mitad del ancho */ /* Espaciado dinámico */ .seccion { padding: calc(2rem + 5vw); } /* Base + extra según el ancho */ /* Mezclar variables y calc */ :root { --nav-height: 60px; } .main { min-height: calc(100vh - var(--nav-height)); } /* Cuadrícula manual con calc */ .col-3 { width: calc((100% - 2rem) / 3); } /* 3 columnas con 1rem de gap entre ellas */
calc(100% - 40px) — barra que nunca toca los bordes

La barra roja tiene width: calc(100% - 40px). Se adapta al ancho del contenedor siempre dejando 20px de cada lado.

min() y max() — El valor más pequeño o más grande

CSS/* min(a, b): elige el valor MÁS PEQUEÑO de los dos */ .caja { width: min(600px, 90%); } /* En desktop (>666px): usa 600px */ /* En móvil (<666px): usa 90% del contenedor */ /* Equivale a: max-width: 600px + width: 90% */ /* max(a, b): elige el valor MÁS GRANDE de los dos */ .texto { font-size: max(1rem, 2vw); } /* Mínimo 1rem siempre, escala con la pantalla si 2vw > 1rem */ /* Casos de uso reales */ .padding-seguro { padding: max(1rem, 4vw); } /* Al menos 1rem */ .ancho-util { width: min(920px, 100% - 2rem); } /* Reemplaza max-width + padding */
📐
clamp() — El valor perfecto entre mínimo y máximo
Tipografía y espaciado que escala fluidamente sin media queries

clamp(mínimo, preferido, máximo) elige el valor preferido, pero nunca sale del rango entre mínimo y máximo. Es como min() y max() combinados. Su uso más potente es en tipografía responsive: texto que crece fluidamente con la pantalla sin necesitar media queries.

🌡️ Analogía: el termostato con límites

Un termostato inteligente tiene una temperatura deseada (21°C) pero con límites: nunca menos de 18°C (mínimo) y nunca más de 26°C (máximo). clamp(18°, 21°, 26°). El CSS hace lo mismo: quiere cierto valor en base al viewport, pero tiene pisos y techos.

CSS/* clamp(mínimo, preferido, máximo) */ /* Tipografía fluida — el uso más importante */ h1 { font-size: clamp(1.5rem, 5vw, 3rem); } /* En móvil: 1.5rem como mínimo */ /* En desktop: crece hasta 5vw del ancho de pantalla */ /* En pantallas muy grandes: no supera 3rem */ h2 { font-size: clamp(1.25rem, 3.5vw, 2rem); } h3 { font-size: clamp(1rem, 2.5vw, 1.5rem); } body { font-size: clamp(.9rem, 1.2vw, 1rem); } /* Espaciado fluido — secciones más espaciadas en desktop */ .seccion { padding-block: clamp(2rem, 8vw, 6rem); } /* Ancho de contenedor con máximo */ .contenedor { width: clamp(300px, 90%, 1200px); margin: 0 auto; } /* Gap que crece con la pantalla */ .grid { gap: clamp(.5rem, 2vw, 2rem); }

Unidades modernas de viewport y otras
svh, dvh, lvh, svw, cqi, ch, ex y cuándo usar cada una

Las unidades de viewport clásicas (vw, vh) tenían un problema en móviles: el 100vh no consideraba la barra de URL del navegador. CSS moderno introdujo nuevas unidades que resuelven esto y más.

svh / svw
Small Viewport: el viewport más pequeño posible, asumiendo que todas las barras del navegador están visibles.
Ideal para elementos que deben caber incluso con la barra de URL visible. height: 100svh nunca se corta.
lvh / lvw
Large Viewport: el viewport más grande posible, asumiendo que las barras están ocultas (al hacer scroll).
Equivale al antiguo 100vh en desktop. En móvil puede causar el problema de la barra URL.
dvh / dvw
Dynamic Viewport: se actualiza dinámicamente cuando las barras del navegador aparecen o desaparecen al hacer scroll.
La más precisa pero puede causar reflows. Úsala cuando la precisión sea crítica.
cqi / cqb
Container Query inline/block size. El % del tamaño del contenedor, para usar con container queries.
font-size: 5cqi = 5% del ancho del contenedor. Perfecto para tipografía dentro de containers.
ch
El ancho del carácter "0" en la fuente actual. Ideal para definir el ancho máximo de columnas de texto.
max-width: 65ch es la longitud de línea óptima para lectura (entre 60–75 caracteres).
lh / rlh
La altura de línea del elemento (lh) o del elemento raíz (rlh). Útil para espaciado vertical consistente con la tipografía.
margin-top: 2lh = dos alturas de línea. Mantiene el ritmo vertical tipográfico.
CSS/* La solución moderna para el hero que ocupa toda la pantalla en móvil */ /* ❌ Antes — 100vh se cortaba con la barra de URL en iOS Safari */ .hero { height: 100vh; } /* ✅ Ahora — svh es más seguro */ .hero { height: 100svh; } /* Con fallback para navegadores viejos */ .hero { height: 100vh; /* Fallback */ height: 100svh; /* Sobreescribe en navegadores modernos */ } /* Texto legible con ch */ .articulo p { max-width: 65ch; } /* Columna de texto de lectura óptima */ /* Contenedor centrado moderno con min() */ .wrapper { width: min(920px, 100% - 2rem); margin-inline: auto; /* margin-inline = margin-left + margin-right */ }

Scroll Snap y Scroll Behavior
Carruseles nativos, scroll suave y control del desplazamiento

Scroll Snap hace que el scroll se "enganche" en posiciones específicas al terminar, como un carrusel. Scroll Behavior: smooth hace que los saltos de ancla (#seccion) sean suaves en vez de instantáneos.

📸 Analogía: el proyector de diapositivas

Cuando avanzas diapositivas en un proyector, no te quedas a medias entre dos slides: siempre terminas en una diapositiva completa. Eso es scroll-snap: el scroll siempre "cae" en las posiciones definidas, nunca a medias entre dos elementos.

CSS/* ── SCROLL SNAP ── */ /* En el contenedor: activa el snap */ .carrusel { display: flex; overflow-x: scroll; scroll-snap-type: x mandatory; /* x=horizontal, mandatory=siempre snappea */ scroll-snap-type: y proximity; /* y=vertical, proximity=snappea si está cerca */ scroll-behavior: smooth; /* Animación de scroll */ -webkit-overflow-scrolling: touch; /* Inercia en iOS */ } /* En cada item: define el punto de snap */ .item { scroll-snap-align: start; /* El borde izquierdo/superior snappea al inicio */ scroll-snap-align: center; /* El centro del item snappea al centro del contenedor */ scroll-snap-align: end; /* El borde derecho/inferior snappea al final */ flex-shrink: 0; /* Los items no se encogen */ } /* ── SCROLL BEHAVIOR GLOBAL ── */ html { scroll-behavior: smooth; } /* Ahora todos los links de ancla (#seccion) scrollean suavemente */ /* ── SCROLL MARGIN — espacio entre el elemento y el top al hacer ancla ── */ .seccion { scroll-margin-top: 80px; } /* Evita que el header fijo tape el inicio de la sección */ /* ── SCROLL PADDING — en el contenedor ── */ html { scroll-padding-top: 80px; } /* Alternativa global: reserva espacio superior para el header fijo en todos los anclas */

aspect-ratio — Proporciones perfectas siempre
Mantener la relación ancho/alto sin trucos de padding

aspect-ratio mantiene automáticamente la proporción entre el ancho y el alto de un elemento. Si defines el ancho, el alto se calcula solo. Si defines el alto, el ancho se calcula solo. Acabó el truco del "padding-bottom hack" para mantener proporciones.

CSS/* aspect-ratio: ancho / alto */ .video-container { aspect-ratio: 16 / 9; } /* Proporciones de pantalla widescreen */ .cuadrado { aspect-ratio: 1 / 1; } /* Cuadrado perfecto */ .foto-clasica { aspect-ratio: 4 / 3; } /* Proporciones de foto clásica */ .portrait { aspect-ratio: 9 / 16; } /* Vertical (historias, reels) */ .panoramica { aspect-ratio: 21 / 9; } /* Ultrawide */ /* Combinado con object-fit para imágenes */ .tarjeta-img { width: 100%; aspect-ratio: 16 / 9; object-fit: cover; /* La imagen llena la proporción sin deformarse */ object-position: center; } /* Antes del aspect-ratio (el padding hack) — para referencia histórica */ /* .video-container::before { content:''; padding-top:56.25%; display:block; } */ /* Con aspect-ratio ya NO necesitas ese truco */

Ejercicio integrador — Perfil de jugador premium
filter, transform, variables, clamp, scroll-snap y glassmorphism
✏️ Ejercicio — Página de perfil de jugador de fútbol premium Avanzado

Crea perfil-jugador.html que use todas las técnicas de esta parte para construir un perfil visual de tu jugador favorito.

  1. Variables CSS del sistema de diseño: Define en :root al menos 12 variables: colores del equipo (primario, secundario), tipografía, espaciado (--space-1 al --space-8), bordes y sombras. Todo el CSS del proyecto usa solo variables, nunca valores hardcodeados.
  2. Hero con glassmorphism: Un hero con fondo de imagen/gradiente de colores del equipo. Encima, una tarjeta con backdrop-filter: blur(12px) y background: rgba(255,255,255,.15) que muestra el nombre y posición del jugador.
  3. Foto del jugador con filter: La foto de perfil en escala de grises en reposo (filter: grayscale(1)), y a todo color con brightness(1.1) al hacer hover. Con transition: filter .4s y aspect-ratio: 1/1 para que sea siempre cuadrada.
  4. Tarjetas 3D de estadísticas: Mínimo 4 tarjetas de stats (goles, asistencias, partidos, rating). Al hacer hover cada tarjeta aplica transform: rotateY(15deg) translateY(-4px) con el contenedor teniendo perspective: 800px.
  5. Tipografía fluida con clamp: El nombre del jugador usa font-size: clamp(2rem, 6vw, 4rem). Los subtítulos usan clamp(1rem, 3vw, 1.5rem). El cuerpo de texto tiene max-width: 65ch.
  6. Carrusel de partidos con Scroll Snap: Una sección horizontal de tarjetas de partidos recientes. Cada tarjeta tiene aspect-ratio: 4/3 y usa scroll-snap-align: start.
  7. Galería de acciones: Grid de fotos donde en reposo tienen filter: grayscale(.7) brightness(.9) y al hover filter: none; transform: scale(1.03). Cada foto tiene aspect-ratio: 16/9; object-fit: cover.

Bonus: Agrega modo oscuro con @media (prefers-color-scheme: dark) que sobreescriba las variables de color. El glassmorphism cambia a rgba(0,0,0,.2) en modo oscuro y el fondo de las tarjetas a colores más oscuros.

  • transform 2D: translate, rotate, scale, skew y sus variantes X/Y
  • transform-origin: el punto desde donde se aplica la transformación
  • Transform 3D: perspective en el padre, rotateX/Y, translateZ, transform-style: preserve-3d, backface-visibility
  • Variables CSS: --nombre, var(--nombre, fallback), :root global, locales por componente, dinámicas con JS
  • calc(): mezcla unidades distintas, funciona con variables
  • min() / max(): el valor más pequeño / más grande de dos opciones
  • clamp(min, preferido, max): tipografía y espaciado fluido sin media queries
  • svh/dvh/lvh: viewport heights modernas sin el bug de iOS
  • ch: ancho del "0" — ideal para max-width de columnas de texto (65ch)
  • scroll-snap-type + scroll-snap-align: carruseles nativos sin JavaScript
  • scroll-behavior: smooth: anclas con scroll suave
  • scroll-margin-top / scroll-padding-top: espacio para headers fijos
  • aspect-ratio: proporciones perfectas sin el padding hack
