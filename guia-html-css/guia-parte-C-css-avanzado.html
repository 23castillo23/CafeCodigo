Selectores avanzados — Apunta exactamente donde quieres
Más allá de clases e ids: seleccionar con precisión quirúrgica

Ya conoces los selectores básicos: por etiqueta (p), por clase (.mi-clase) y por id (#mi-id). Los selectores avanzados te permiten apuntar a elementos según su relación con otros, sus atributos, o su posición en la página, sin necesitar agregar clases extra al HTML.

🔦 Analogía: las diferentes linternas

Un selector básico es como una linterna que ilumina toda una habitación. Los selectores avanzados son como linternas con zoom, filtros y detectores de movimiento: puedes iluminar exactamente el tercer elemento de la segunda fila, o solo los elementos que tienen cierto atributo, o solo el primero que sigue a otro específico. Mismo resultado visible, pero con control total.

Repaso rápido: los selectores básicos

CSSp { color: red; } /* Etiqueta: todos los <p> */ .tarjeta { color: blue; } /* Clase: todos los elementos con class="tarjeta" */ #logo { color: green; } /* ID: el elemento con id="logo" (único en la página) */ * { margin: 0; } /* Universal: todos los elementos */

Selector múltiple — aplicar el mismo estilo a varios

CSS/* Sin selector múltiple — repites el mismo código */ h1 { font-family: sans-serif; } h2 { font-family: sans-serif; } h3 { font-family: sans-serif; } /* Con selector múltiple — separas con coma */ h1, h2, h3 { font-family: sans-serif; } /* Puedes mezclar cualquier tipo de selector */ h1, .titulo, #encabezado-principal { font-weight: 800; }

Selector descendiente vs hijo directo

CSS/* Descendiente (espacio): selecciona TODOS los <a> dentro de nav, sin importar profundidad */ nav a { color: white; } /* Hijo directo (>): solo los <li> que son hijos DIRECTOS de ul, no nietos */ ul > li { list-style: none; }
¿Cuál es la diferencia?
nav a → selecciona el link aunque esté dentro de un <ul> <li> dentro del nav. Cualquier nivel de profundidad.
ul > li → selecciona solo los <li> que son hijos inmediatos del <ul>. Un <li> dentro de otro <li> (lista anidada) no quedaría seleccionado.

Combinadores — Seleccionar por relación entre elementos
Hermano adyacente, hermano general, hijo y descendiente

Los combinadores son los símbolos que van entre selectores para indicar qué relación debe existir entre ellos. Son cuatro: espacio, >, + y ~.

👨‍👩‍👧‍👦 Analogía: relaciones familiares

Imagina un árbol genealógico. El combinador de espacio es "cualquier descendiente" (hijos, nietos, bisnietos). El > es "solo mis hijos directos, no mis nietos". El + es "mi hermano que está justo a mi lado". El ~ es "todos mis hermanos menores". Misma familia, distintos grados de parentesco.

A B
Descendiente (espacio)
Selecciona B si está dentro de A, a cualquier nivel de profundidad.
/* Todos los <p> dentro de .articulo */ .articulo p { color: #555; }
A > B
Hijo directo
Selecciona B solo si es hijo inmediato de A. No afecta nietos.
/* Solo <li> directos del <ul>, no los <li> de listas anidadas */ ul > li { margin-bottom: .5rem; }
A + B
Hermano adyacente
Selecciona B si está justo después de A y son hermanos (mismo padre).
/* El <p> que viene JUSTO después de un <h2> (el primer párrafo) */ h2 + p { font-size: 1.1rem; font-weight: 500; }
A ~ B
Hermanos generales
Selecciona todos los B que siguen a A y son hermanos, no solo el primero.
/* TODOS los <p> que vienen después de un <h2> */ h2 ~ p { color: #666; }

Ejemplo práctico: estilizar formularios con combinadores

CSS/* El label que viene JUSTO después de un input inválido */ input:invalid + label { color: red; } /* El mensaje de error que sigue a un input inválido */ input:invalid ~ .error-msg { display: block; } /* Quitar margen al primer elemento dentro de una tarjeta */ .tarjeta > *:first-child { margin-top: 0; } /* Separadores entre items de una lista sin margen al primero */ .lista-item + .lista-item { border-top: 1px solid #eee; }

🟢 El truco del + más usado en el mundo real

El selector * + * (llamado "lobotomized owl" o búho lobotomizado) es uno de los más elegantes: selecciona cualquier elemento que tenga un hermano antes que él. Úsalo para agregar espacio entre elementos de forma automática sin tocar el primero:

CSS/* Agrega margen-top a todo elemento que tenga un hermano anterior */ /* El primero NO tiene margen porque no tiene hermano antes */ .contenedor > * + * { margin-top: 1rem; }

Selector de atributo — Seleccionar por características del HTML
Apuntar a elementos según sus atributos y valores

El selector de atributo te permite seleccionar elementos basándote en sus atributos HTML: si tiene un atributo, si tiene cierto valor, si el valor empieza o termina con algo. Se escribe entre corchetes [].

🏷️ Analogía: filtrar productos en una tienda online

En una tienda online puedes filtrar por "color: rojo", "talla: M", "marca: empieza con Nike". Los selectores de atributo son esos filtros aplicados a los elementos HTML: "dame todos los inputs que tengan type='email'", "dame todos los links que empiecen con https".

SelectorSeleccionaEjemplo en CSS
[atributo] Elementos que tienen ese atributo (sin importar el valor) [disabled] → todos los elementos disabled
[atributo="valor"] Elementos cuyo atributo es exactamente ese valor [type="email"] → inputs de tipo email
[atributo^="valor"] Elementos cuyo atributo empieza con ese valor (^ = inicio) [href^="https"] → links seguros
[atributo$="valor"] Elementos cuyo atributo termina con ese valor ($ = final) [href$=".pdf"] → links a PDFs
[atributo*="valor"] Elementos cuyo atributo contiene ese valor en cualquier posición (* = contiene) [class*="btn"] → cualquier clase que incluya "btn"

Ejemplos reales — los más útiles

CSS/* Estilizar inputs según su tipo — sin agregar clases al HTML */ input[type="text"] { border: 1px solid #ccc; } input[type="email"] { border: 1px solid blue; } input[type="password"] { letter-spacing: .2em; } /* Íconos automáticos en links externos vs internos */ a[href^="https"]::after { content: ' ↗'; font-size: .75em; } a[href$=".pdf"]::after { content: ' 📄'; } a[href$=".zip"]::after { content: ' 📦'; } /* Campos deshabilitados — sin necesidad de clase .disabled */ input[disabled] { background: #f5f5f5; cursor: not-allowed; opacity: .6; } /* Selector de atributo con insensibilidad a mayúsculas (i al final) */ a[href$=".PDF" i] { color: red; } /* Funciona con .pdf, .PDF, .Pdf */

Pseudoclases — Estilos según el estado o situación del elemento
:hover, :focus, :checked, :not(), :is(), :where() y muchas más

Una pseudoclase selecciona un elemento según su estado o su posición en el documento, no por su nombre o clase. Siempre empiezan con dos puntos :. Son uno de los conceptos más poderosos de CSS porque permiten cambiar estilos según lo que el usuario está haciendo.

🚦 Analogía: el semáforo y sus estados

Un semáforo siempre es el mismo objeto, pero tiene tres estados: rojo, amarillo, verde. Cada estado tiene un comportamiento diferente. Las pseudoclases son exactamente eso: el mismo elemento HTML, pero con estilos diferentes según su estado actual (el usuario está pasando el mouse, el campo está activo, el checkbox está marcado, etc.).

Pseudoclases de interacción del usuario

:hover

El mouse está encima del elemento. El más usado de todos.

.boton:hover { background: #9c3920; transform: translateY(-2px); }
:focus

El elemento está enfocado (el cursor está dentro de un input, o se llegó con Tab). Importantísimo para accesibilidad.

input:focus { border-color: var(--primary); box-shadow: 0 0 0 3px rgba(196,74,42,.2); outline: none; }
:focus-visible

Como :focus, pero solo muestra el estilo cuando el usuario navega con teclado, no con el mouse. Más moderno que :focus.

button:focus-visible { outline: 3px solid var(--primary); outline-offset: 2px; }
:active

El elemento está siendo presionado (el instante en que el usuario hace clic y no ha soltado).

.boton:active { transform: translateY(1px); box-shadow: none; }
:visited

Un link que el usuario ya visitó. Por privacidad, solo puedes cambiar color, background y font-style.

a:visited { color: purple; }
:checked

Un checkbox o radio que está marcado. Permite crear interactividad sin JavaScript.

input[type="checkbox"]:checked + label { color: var(--secondary); font-weight: 700; }
:disabled / :enabled

Elementos de formulario desactivados o activados.

button:disabled { opacity: .5; cursor: not-allowed; }
:valid / :invalid

Inputs según si su valor pasa o no la validación HTML (required, type, pattern).

input:valid { border-color: green; } input:invalid { border-color: red; }
:placeholder-shown

El input que aún muestra su placeholder (está vacío). Muy útil combinado con :not.

/* Input vacío */ input:placeholder-shown { border-style: dashed; } /* Input con contenido */ input:not(:placeholder-shown) { border-style: solid; }
:empty

El elemento no tiene hijos ni texto. Útil para ocultar contenedores vacíos.

.notificacion:empty { display: none; }
Prueba estos estados interactivos:

:hover y :active en botón

:focus en input + :valid/:invalid

Verde = válido · Rojo = inválido

:not() — excluir elementos

:not() selecciona todos los elementos que no coinciden con el selector del paréntesis. Muy útil para aplicar estilos a "todos excepto uno".

CSS/* Todos los botones excepto los disabled */ button:not([disabled]) { cursor: pointer; } /* Todos los párrafos excepto el que tiene clase .intro */ p:not(.intro) { color: #555; } /* Todos los li excepto el último (quita el borde del último) */ li:not(:last-child) { border-bottom: 1px solid #eee; } /* En CSS moderno, :not() acepta varios selectores separados por coma */ :not(h1, h2, h3) { font-size: 1rem; }

:is() y :where() — agrupar selectores de forma elegante

Estas dos pseudoclases modernas permiten agrupar selectores para no repetir código. La diferencia es la especificidad:

CSS/* Sin :is() — código repetitivo */ header h1, header h2, header h3 { color: white; } footer h1, footer h2, footer h3 { color: #ccc; } /* Con :is() — mucho más limpio */ header :is(h1, h2, h3) { color: white; } footer :is(h1, h2, h3) { color: #ccc; } /* :where() hace lo mismo pero con especificidad 0 (más fácil de sobreescribir) */ :where(h1, h2, h3, h4) { font-family: 'Fraunces', serif; } /* Útil en librerías y resets donde no quieres que interfiera con estilos del usuario */

💡 :is() vs :where() — la diferencia de especificidad

:is() toma la especificidad del selector más específico que tiene dentro. :where() siempre tiene especificidad 0, lo que lo hace ideal para estilos base que el usuario puede sobreescribir fácilmente. Para la mayoría de casos en proyectos propios, usa :is().

Pseudoclases de posición — nth-child y familia
Seleccionar elementos según su número de orden

Las pseudoclases de posición seleccionan elementos según su lugar dentro del padre: el primero, el último, el tercero, los pares, los impares, o cualquier patrón matemático que quieras.

🏟️ Analogía: los asientos numerados de un estadio

En un estadio, puedes decir "pinta de rojo los asientos 1, 3, 5, 7..." (impares), o "da descuento a los asientos múltiplos de 5", o "reserva el primer y último asiento de cada fila". Las pseudoclases de posición son exactamente eso: reglas que aplican según el número del elemento.

PseudoclaseSeleccionaEjemplo real
:first-childEl primer hijo dentro de su padreQuitar margen-top al primer párrafo de un artículo
:last-childEl último hijo dentro de su padreQuitar border-bottom al último item de una lista
:nth-child(n)El hijo número n exacto:nth-child(3) → el tercero
:nth-child(even)Todos los hijos en posición par (2, 4, 6…)Filas alternas de una tabla
:nth-child(odd)Todos los hijos en posición impar (1, 3, 5…)Filas alternas de una tabla
:nth-child(3n)Cada tercer hijo (3, 6, 9, 12…)Resaltar cada tercera tarjeta en un grid
:nth-child(3n+1)El primero de cada grupo de 3 (1, 4, 7, 10…)Grid de 3 columnas: el primero de cada fila
:only-childEl elemento que es hijo único dentro de su padreCentrar imagen cuando es la única en su contenedor
:nth-last-child(n)Como nth-child pero contando desde el final:nth-last-child(2) → el penúltimo

La fórmula de nth-child: An+B

CSS/* La fórmula es: An+B donde A=paso, B=inicio, n empieza en 0 */ li:nth-child(2n) /* even: 2, 4, 6, 8... → A=2, B=0 */ li:nth-child(2n+1) /* odd: 1, 3, 5, 7... → A=2, B=1 */ li:nth-child(3n) /* cada 3: 3, 6, 9, 12... → A=3, B=0 */ li:nth-child(3n+1) /* 1, 4, 7, 10... → A=3, B=1 */ li:nth-child(n+4) /* desde el 4 en adelante: 4, 5, 6... */ li:nth-child(-n+3) /* solo los 3 primeros: 1, 2, 3 */

:nth-of-type — por tipo de etiqueta

Pseudoelementos — Crear o apuntar partes que no existen en el HTML
::before, ::after, ::placeholder, ::selection, ::first-line y más

Los pseudoelementos se escriben con doble dos puntos :: (para distinguirlos de las pseudoclases que usan uno). Permiten dos cosas: apuntar a una parte específica de un elemento (la primera letra, la primera línea, el placeholder) o crear elementos virtuales antes y después del contenido sin tocar el HTML.

🎭 Analogía: el maquillaje y los accesorios

Un actor es el elemento HTML. El maquillaje que le ponen antes de salir a escena es ::before. El accesorio que le ponen detrás es ::after. El color de los labios pintados es ::selection (lo que se resalta al seleccionar). Ninguno de estos es parte del actor en sí, son capas que se agregan por encima sin modificar al actor.

::before y ::after — Los más poderosos

::before inserta contenido virtual antes del contenido real del elemento. ::after lo inserta después. Siempre necesitan la propiedad content (aunque sea vacía).

.elemento::before {
  content: "★";OBLIGATORIO — puede ser texto, emoji, o vacío ""
  display: block;por defecto es inline
  color: gold;
}

Casos de uso reales de ::before y ::after

1 — Comillas decorativas en citas

CSS.cita { position: relative; padding: 1rem 1.5rem; } .cita::before { content: '\201C'; /* " en código unicode */ font-size: 3rem; color: var(--primary); position: absolute; top: -.5rem; left: .4rem; } .cita::after { content: '\201D'; /* " en código unicode */ font-size: 3rem; color: var(--primary); position: absolute; bottom: -1.4rem; right: .4rem; }

La práctica hace al maestro. Escribe código todos los días, aunque sea un poquito.

2 — Tooltips sin JavaScript

CSS.tooltip { position: relative; display: inline-block; } .tooltip::before { /* content: attr(data-tip) lee el atributo data-tip del HTML */ content: attr(data-tip); position: absolute; bottom: calc(100% + 8px); left: 50%; transform: translateX(-50%); background: #1a1814; color: white; padding: 5px 10px; border-radius: 6px; font-size: .8rem; white-space: nowrap; opacity: 0; transition: opacity .2s; } .tooltip:hover::before { opacity: 1; }

Otros pseudoelementos importantes

::placeholder

El texto de placeholder de los inputs. Permite cambiar su color y estilo.

input::placeholder { color: #aaa; font-style: italic; font-size: .9rem; }
::selection

El texto que el usuario selecciona con el mouse. Un detalle visual que impresiona.

::selection { background: var(--primary); color: white; }
::first-line

La primera línea de un párrafo (la que el navegador calcula como primera, cambia con el ancho).

p::first-line { font-weight: 700; font-size: 1.05em; }
::first-letter

La primera letra de un párrafo. Clásico efecto de letra capital (drop cap).

p::first-letter { font-size: 3em; float: left; line-height: .8; margin-right: .1em; color: var(--primary); }
::marker

El marcador de las listas (el punto, el número). Puedes cambiar su color y tamaño.

li::marker { color: var(--primary); font-size: 1.2em; }
::backdrop

El fondo oscuro detrás de un elemento en modo pantalla completa o de un dialog nativo.

dialog::backdrop { background: rgba(0,0,0,.6); backdrop-filter: blur(4px); }

BEM — Block, Element, Modifier
La metodología para nombrar clases CSS como un profesional

BEM (Block, Element, Modifier) es una convención para nombrar clases CSS que hace que tu código sea predecible, fácil de leer, y evite conflictos entre estilos. No es una librería ni un framework: es simplemente una forma de pensar y nombrar clases.

🚗 Analogía: el sistema de identificación de un auto

Un auto (Block) tiene partes: puerta, ventana, rueda (Elements). Y esas partes pueden tener variantes: puerta-delantera, puerta-trasera (Modifiers). Si alguien te dice "reemplaza la puerta-trasera-izquierda del Sedán Rojo", sabes exactamente de qué habla. BEM hace lo mismo con las clases CSS: sabes exactamente qué es cada cosa solo leyendo el nombre.

La sintaxis de BEM

.bloqueEl componente principal
.bloque__elementoParte del bloque (doble guión bajo)
.bloque--modificadorVariante del bloque (doble guión)
.bloque__elemento--modificadorVariante de un elemento

BEM en la práctica — Tarjeta de proyecto

❌ Sin BEM — clases genéricas y confusas
<div class="card"> <img class="image"> <div class="body"> <h3 class="title"></h3> <p class="text"></p> <a class="btn featured"></a> </div> </div>

¿"title" es el título de la tarjeta o de la página? ¿"body" es el body del HTML? ¿"btn featured" — qué significa "featured" aquí?

✅ Con BEM — clases descriptivas y sin ambigüedad
<div class="tarjeta"> <img class="tarjeta__imagen"> <div class="tarjeta__cuerpo"> <h3 class="tarjeta__titulo"></h3> <p class="tarjeta__texto"></p> <a class="tarjeta__boton tarjeta__boton--destacado"></a> </div> </div>

Todo se entiende solo: "tarjeta__titulo" es el título de la tarjeta. "tarjeta__boton--destacado" es la variante destacada del botón de la tarjeta.

El CSS de BEM — sin anidamiento excesivo

❌ CSS sin BEM — muy anidado, difícil de sobreescribir
.card .body .title { ... } .card .body .btn { ... } .card .body .btn.featured { ... } /* Alta especificidad, difícil sobreescribir */
✅ CSS con BEM — plano, fácil de mantener
.tarjeta { ... } .tarjeta__titulo { ... } .tarjeta__boton { ... } .tarjeta__boton--destacado{ ... } /* Especificidad uniforme, fácil de entender */

Ejemplo completo — Componente de botón con BEM

HTML + CSS<!-- HTML --> <button class="boton">Normal</button> <button class="boton boton--primario">Primario</button> <button class="boton boton--peligro">Peligro</button> <button class="boton boton--grande">Grande</button> <button class="boton boton--primario boton--grande">Primario Grande</button> /* CSS */ /* BLOQUE: estilos base del botón */ .boton { padding: .5rem 1.25rem; border: 2px solid #ccc; border-radius: 8px; background: white; cursor: pointer; font-family: inherit; font-size: 1rem; } /* MODIFICADORES: variantes del botón */ .boton--primario { background: #c44a2a; color: white; border-color: #c44a2a; } .boton--peligro { background: #dc2626; color: white; border-color: #dc2626; } .boton--grande { padding: .75rem 2rem; font-size: 1.1rem; }

🟢 Reglas de oro de BEM

  • El Block es un componente que tiene sentido por sí solo: .menu, .tarjeta, .formulario
  • El Element no tiene sentido sin su bloque: .menu__item, .tarjeta__titulo
  • El Modifier es una variante: .menu--horizontal, .tarjeta--destacada
  • Los elementos BEM se escriben con un solo nivel: nunca .bloque__elemento__subelemento. Si llegas ahí, crea un nuevo bloque.
  • En CSS, los selectores BEM son casi todos de una sola clase: especificidad baja y uniforme.

⚠️ BEM no es obligatorio, pero sí muy recomendado

BEM es una convención, no una regla del lenguaje. No hay nada que te obligue a usarlo. Pero cuando trabajas en proyectos medianos o grandes, o en equipo, BEM hace que todos entiendan el código sin tener que preguntar. Es una de las habilidades que más diferencian a un desarrollador junior de uno senior.

Ejercicio integrador — Tarjeta de jugador con CSS avanzado
Selectores, pseudoclases, pseudoelementos y BEM en un solo proyecto
✏️ Ejercicio — Componente de tarjeta de jugador Avanzado

Crea un archivo tarjeta-jugador.html con CSS en el mismo archivo. Vas a construir un componente de tarjeta de jugador de fútbol aplicando todo lo de esta parte.

  1. HTML con BEM: Estructura la tarjeta con clases BEM. El bloque es .jugador-card. Necesita estos elementos: __imagen, __nombre, __posicion, __stats, __stat, __boton. Crea al menos 3 tarjetas: una normal, una con modificador --destacado y una con --lesionado.
  2. Selectores avanzados:
    • Usa .jugador-card + .jugador-card para agregar separación entre tarjetas
    • Usa .jugador-card > .jugador-card__imagen para estilizar solo la imagen directa
    • Usa input[type="search"] para estilizar un campo de búsqueda sin clase
  3. Pseudoclases:
    • :hover en la tarjeta: que suba con transform: translateY(-4px)
    • :focus en el input de búsqueda: borde de color y sombra
    • :nth-child(odd) en los stats para alternar colores de fondo
    • :not(.jugador-card--lesionado) para estilizar solo los jugadores disponibles
  4. Pseudoelementos:
    • ::before en .jugador-card--destacado para agregar una estrella ⭐ decorativa
    • ::after en .jugador-card--lesionado para mostrar el texto "Lesionado 🩹"
    • ::selection personalizado en toda la página
    • ::placeholder con estilo en el input de búsqueda
  5. Contador CSS: Usando counter-reset y counter-increment, agrega un número automático a cada tarjeta con ::before.

Bonus desafiante: Usa input[type="checkbox"]:checked ~ .jugador-card--lesionado para ocultar/mostrar los jugadores lesionados cuando el usuario marca un checkbox que diga "Mostrar solo disponibles" — todo sin JavaScript.
