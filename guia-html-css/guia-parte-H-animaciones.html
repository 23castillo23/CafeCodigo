Transition vs Animation — cuándo usar cuál
Dos herramientas distintas para dos tipos de movimiento distintos

Ya conoces las transitions de la Parte D. Las animaciones (@keyframes + animation) son más poderosas pero también más complejas. La diferencia clave es:

✅ Usa transition cuando…

  • El cambio es de un estado A a un estado B
  • Se dispara por una interacción del usuario (:hover, :focus, clase JS)
  • La animación solo tiene punto de inicio y punto de fin
  • Ejemplos: hover en botón, apertura de menú, cambio de color

✅ Usa animation (@keyframes) cuando…

  • La animación corre sola, sin que el usuario haga nada
  • Necesita repetirse en loop infinito
  • Tiene más de dos estados intermedios (0%, 30%, 70%, 100%)
  • Ejemplos: loaders, pulsaciones, efectos de entrada al cargar

🎭 Analogía: el actor y el bailarín

Una transition es un actor que reacciona a lo que hace el público: si alguien lo aplaude (hover), él responde con un gesto. Una animation es un bailarín con una coreografía propia: baila con o sin público, sigue su partitura desde el principio hasta el final, puede repetirla varias veces, y tiene pasos intermedios planificados.

🎬
@keyframes — La coreografía de la animación
Definir qué ocurre en cada punto del tiempo de la animación

@keyframes es donde defines los "pasos" de la animación. Le das un nombre y describes cómo debe verse el elemento en distintos puntos del tiempo: al 0% (inicio), al 50% (mitad), al 100% (fin), o cualquier porcentaje intermedio.

🎥 Analogía: el storyboard de una película

Antes de filmar una escena de acción, el director dibuja un storyboard: una secuencia de ilustraciones que muestran qué pasa en cada momento clave. El keyframe 0% es el primer dibujo, el 50% es el del medio, el 100% el final. El navegador "interpola" (dibuja los frames intermedios) entre cada keyframe, igual que los animadores rellenan los frames entre los dibujos clave.

Anatomía de @keyframes

@keyframes nombre-animacion {
  0% o from {
    transform: translateX(-50px);
    opacity: 0;
  }
  50% {
    transform: translateX(10px);
    opacity: .8;
  }
  100% o to {
    transform: translateX(0);
    opacity: 1;
  }
}
CSS/* Formas equivalentes de definir inicio y fin */ @keyframes mi-animacion { from { opacity: 0; } /* from = 0% */ to { opacity: 1; } /* to = 100% */ } /* Con porcentajes — más control */ @keyframes rebotar { 0% { transform: translateY(0); } 40% { transform: translateY(-30px); } 60% { transform: translateY(-15px); } 80% { transform: translateY(-5px); } 100% { transform: translateY(0); } } /* Varios porcentajes con el mismo estilo */ @keyframes parpadear { 0%, 100% { opacity: 1; } 50% { opacity: 0; } } /* Animación de múltiples propiedades */ @keyframes aparecerYSubir { from { opacity: 0; transform: translateY(30px) scale(.95); } to { opacity: 1; transform: translateY(0) scale(1); } }

La propiedad animation — Conectar keyframes con elementos
El shorthand y todas sus sub-propiedades explicadas

Una vez que defines los @keyframes, necesitas conectarlos con un elemento usando la propiedad animation. Puedes usar el shorthand o cada sub-propiedad por separado.

animation: nombre duración timing delay iteraciones dirección fill-mode play-state;

@keyframes ¿cuánto dura? aceleración espera ¿cuántas veces? ¿normal/reverse? estado al terminar pausa/corre

/* Ejemplo completo */
animation: aparecerYSubir .6s ease-out .2s 1 normal forwards running;
CSS/* Shorthand — el orden importa: nombre duración timing delay iteraciones dirección fill play-state */ .elemento { animation: girar 1s linear 0s infinite normal none running; } /* Sub-propiedades individuales (más legible para animaciones complejas) */ .elemento { animation-name: girar; animation-duration: 1s; animation-timing-function: linear; animation-delay: 0s; animation-iteration-count: infinite; animation-direction: normal; animation-fill-mode: none; animation-play-state: running; } /* Múltiples animaciones en el mismo elemento — separadas por coma */ .elemento { animation: girar 2s linear infinite, pulsar 1.5s ease infinite, deslizar .5s ease-out 1 forwards; }
🔧
Parámetros de animation — Cada uno explicado
duration, timing-function, delay, iteration-count, direction

animation-duration — ¿Cuánto dura?

CSSanimation-duration: .3s; /* Muy rápida — microinteracción */ animation-duration: 1s; /* Normal — la más usada */ animation-duration: 2.5s; /* Lenta — efectos dramáticos */ /* Consejo: la mayoría de animaciones de UI van entre .2s y .6s */

animation-timing-function — ¿Cómo acelera?

Los mismos valores que en transition: ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier(). Y además uno especial para animaciones:

CSS/* steps() — animación por "pasos" discretos, sin interpolación */ animation-timing-function: steps(4); /* 4 pasos iguales */ animation-timing-function: steps(1, end); /* 1 paso al final — flip instantáneo */ animation-timing-function: step-start; /* Salta al estado final inmediatamente */ animation-timing-function: step-end; /* Permanece en inicio, salta al final */ /* steps() es ideal para: sprites, relojes digitales, efecto typewriter */

animation-delay — ¿Cuánto espera antes de empezar?

CSSanimation-delay: 0s; /* Empieza inmediatamente (por defecto) */ animation-delay: .5s; /* Espera medio segundo */ animation-delay: -1s; /* ¡Delay negativo! Empieza como si ya hubiera pasado 1s */ /* El delay negativo es útil para animar elementos encadenados sin esperas al inicio */ /* Encadenado con nth-child — cada elemento aparece 0.1s después del anterior */ .item:nth-child(1) { animation-delay: 0s; } .item:nth-child(2) { animation-delay: .1s; } .item:nth-child(3) { animation-delay: .2s; }

animation-iteration-count — ¿Cuántas veces?

CSSanimation-iteration-count: 1; /* Una vez (por defecto) */ animation-iteration-count: 3; /* Exactamente 3 veces */ animation-iteration-count: infinite; /* Para siempre — loaders, pulsaciones */ animation-iteration-count: 2.5; /* 2 y media veces (puede ser decimal) */

animation-direction — ¿En qué dirección?

CSSanimation-direction: normal; /* 0% → 100% siempre (por defecto) */ animation-direction: reverse; /* 100% → 0% siempre */ animation-direction: alternate; /* 0%→100%, luego 100%→0%, alternando */ animation-direction: alternate-reverse; /* 100%→0%, luego 0%→100%, alternando */ /* alternate es muy útil para rebotes suaves: va y vuelve sin corte brusco */

animation-play-state — Pausar con CSS o JS

CSS.loader { animation-play-state: running; } /* Corriendo (por defecto) */ .loader { animation-play-state: paused; } /* Pausada */ /* Patrón útil: pausar animación al hacer hover */ .carrusel { animation: desplazar 10s linear infinite; } .carrusel:hover { animation-play-state: paused; } /* Pausar con JavaScript */ // elemento.style.animationPlayState = 'paused'; // elemento.style.animationPlayState = 'running';

animation-fill-mode — El estado antes y después
El parámetro más confuso y más importante para animaciones de entrada

animation-fill-mode controla qué estilos aplica el elemento antes de que empiece la animación (durante el delay) y después de que termine. Es el parámetro más importante para animaciones de entrada.

🎭 Analogía: el actor en el escenario

Antes de que empiece la obra (durante el delay), ¿el actor aparece en el escenario o espera entre bastidores? Cuando termina su actuación, ¿se va corriendo o se queda en la última pose? fill-mode: backwards lo esconde antes, forwards lo deja en la pose final, both hace ambas cosas.

ValorDurante el delay (antes)Al terminar (después)Cuándo usarlo
noneEstilos normales del elementoVuelve a los estilos normalesAnimaciones en loop que no tienen estado "final"
forwardsEstilos normalesSe queda en el keyframe finalAnimaciones de entrada que deben quedarse visibles
backwardsAplica el keyframe inicial (0%)Vuelve a estilos normalesCuando el elemento debe ser invisible durante el delay
bothAplica el keyframe inicial (0%)Se queda en el keyframe finalLa opción más completa para animaciones de entrada — usa esta casi siempre
CSS/* El patrón más común: animación de entrada con fill-mode: both */ @keyframes entrar { from { opacity: 0; transform: translateY(20px); } to { opacity: 1; transform: translateY(0); } } /* Sin fill-mode: both — el elemento aparece visible durante el delay, luego desaparece al terminar */ .malo { animation: entrar .5s ease .3s 1; } /* Con fill-mode: both — invisible durante el delay, se queda visible al terminar ✅ */ .bueno { animation: entrar .5s ease .3s 1 both; } /* El "both" al final del shorthand es animation-fill-mode */

Loaders — Indicadores de carga con CSS puro
Spinner, puntos, barras, ripple y barra de progreso

Los loaders son el caso de uso más clásico de las animaciones CSS. Son componentes pequeños que indican al usuario que algo está cargando. Con CSS puro puedes crear loaders profesionales sin imágenes ni JavaScript.

Cinco tipos de loader en CSS puro:
Spinner clásico
Spinner doble
Puntos pulsantes
Barras de audio
Onda ripple
Barra de progreso animada

Código de los loaders más usados

CSS/* ── SPINNER CLÁSICO ── */ @keyframes girar { to { transform: rotate(360deg); } } .spinner { width: 36px; height: 36px; border-radius: 50%; border: 3px solid #eee; /* Borde gris de fondo */ border-top-color: var(--primary); /* Solo el borde de arriba colorido */ animation: girar .8s linear infinite; } /* ── PUNTOS PULSANTES ── */ @keyframes pulsar { 0%, 100% { transform: scale(1); opacity: 1; } 50% { transform: scale(1.4); opacity: .6; } } .dots { display: flex; gap: 5px; } .dot { width: 8px; height: 8px; border-radius: 50%; background: var(--primary); animation: pulsar .8s ease-in-out infinite; } .dot:nth-child(2) { animation-delay: .15s; } .dot:nth-child(3) { animation-delay: .3s; } /* ── RIPPLE ── */ @keyframes onda { 0% { transform: scale(0); opacity: .8; } 100% { transform: scale(2.5); opacity: 0; } } .ripple-wrap { position: relative; width: 36px; height: 36px; } .ripple { position: absolute; inset: 0; border-radius: 50%; border: 2px solid var(--primary); animation: onda 1.5s ease-out infinite; } .ripple:nth-child(2) { animation-delay: .5s; }

Skeleton Screens — La forma moderna de mostrar carga
Un placeholder animado con la forma del contenido real

Un skeleton screen es un placeholder con la forma aproximada del contenido que está cargando, con un efecto shimmer animado. Es mucho mejor que un spinner porque el usuario sabe qué tipo de contenido viene y la página se siente más rápida.

🏗️ Analogía: el andamiaje de un edificio

Cuando construyen un edificio, ponen andamiaje metálico que tiene la forma aproximada del edificio final. Nadie confunde el andamiaje con el edificio, pero te da una idea de la escala y forma de lo que viene. El skeleton screen es ese andamiaje: tiene la forma del contenido final pero está hecho de cajas grises animadas.

Skeleton screen de una tarjeta de artículo deportivo:

El efecto shimmer se logra con un background-image: linear-gradient animado que se mueve horizontalmente.

Mientras el contenido real carga (con fetch/API), muestras este skeleton. Cuando llegan los datos, lo reemplazas con el contenido real.

CSS/* El efecto shimmer — gradiente que se desliza */ @keyframes shimmer { from { background-position: -600px 0; } to { background-position: 600px 0; } } /* La clase base de todos los elementos skeleton */ .sk { border-radius: 6px; background: linear-gradient( 90deg, #f0ece6 25%, /* Color base */ #e4e0da 50%, /* Color brillante (el shimmer) */ #f0ece6 75% /* Vuelve al base */ ); background-size: 600px 100%; /* Más ancho que el elemento para que el shimmer se vea */ animation: shimmer 1.5s ease-in-out infinite; } /* Elementos específicos */ .sk-imagen { height: 200px; width: 100%; } .sk-titulo { height: 20px; width: 80%; margin-bottom: .5rem; } .sk-linea { height: 14px; width: 100%; margin-bottom: .4rem; } .sk-corta { width: 65%; } .sk-avatar { width: 36px; height: 36px; border-radius: 50%; }

Animaciones de entrada — Dar vida al cargar la página
Fade, slide, scale y efectos encadenados con delay

Las animaciones de entrada son las que ocurren cuando un elemento aparece por primera vez en pantalla. Son sutiles, cortas (0.3s–0.6s) y hacen que la página se sienta premium y bien cuidada.

CSS/* Los @keyframes de entrada más usados */ @keyframes fadeIn { from { opacity: 0; } to { opacity: 1; } } @keyframes slideFromLeft { from { opacity: 0; transform: translateX(-40px); } to { opacity: 1; transform: translateX(0); } } @keyframes slideFromTop { from { opacity: 0; transform: translateY(-20px); } to { opacity: 1; transform: translateY(0); } } @keyframes scaleIn { from { opacity: 0; transform: scale(.8); } to { opacity: 1; transform: scale(1); } } /* Aplicarlos con fill-mode: both para que funcionen correctamente */ .hero-titulo { animation: slideFromTop .6s ease both; } .hero-subtitulo{ animation: slideFromLeft .6s ease .1s both; } .hero-boton { animation: scaleIn .5s ease .2s both; } /* Encadenado con nth-child — tarjetas aparecen una por una */ .card { animation: scaleIn .4s ease both; } .card:nth-child(1) { animation-delay: 0s; } .card:nth-child(2) { animation-delay: .08s; } .card:nth-child(3) { animation-delay: .16s; } .card:nth-child(4) { animation-delay: .24s; }

Rendimiento y accesibilidad — Animar bien
GPU vs CPU, propiedades baratas y prefers-reduced-motion

Propiedades baratas vs caras para animar

No todas las propiedades CSS cuestan lo mismo en rendimiento. El navegador tiene dos formas de renderizar cambios: usando la CPU (costoso, puede causar janking) o la GPU (gratis, súper fluido). Siempre que puedas, anima solo propiedades que use la GPU:

✅ Propiedades GPU (baratas)

  • transform — mover, escalar, rotar
  • opacity — transparencia
  • filter — blur, brightness
  • Siempre fluido, incluso con muchos elementos

❌ Propiedades CPU (costosas)

  • width, height — fuerza un recálculo
  • top, left — usa transform en su lugar
  • margin, padding — afecta el layout
  • background-color — mejor usar opacity
CSS/* ❌ Animar width — causa recálculo de layout en cada frame */ @keyframes malo { from { width: 0; } to { width: 200px; } } /* ✅ Animar transform — la GPU lo maneja sin tocar el layout */ @keyframes bueno { from { transform: scaleX(0); } to { transform: scaleX(1); } } /* will-change: adelanta a la GPU que este elemento va a animarse */ /* Solo úsalo en elementos que REALMENTE lo necesitan */ .loader { will-change: transform; }

prefers-reduced-motion — Respetar a los usuarios

Algunas personas tienen condiciones vestibulares o epilepsia que las hace sensibles al movimiento. Siempre respeta la preferencia del sistema operativo con esta media query:

CSS/* La regla más importante de accesibilidad en animaciones */ @media (prefers-reduced-motion: reduce) { /* Opción nuclear: deshabilitar TODAS las animaciones */ *, *::before, *::after { animation-duration: .01ms !important; animation-iteration-count: 1 !important; transition-duration: .01ms !important; } } /* Opción selectiva: desactivar solo las problemáticas */ @media (prefers-reduced-motion: reduce) { .carrusel { animation: none; } .paralax { transform: none !important; } /* Mantener animaciones sutiles de UI que son útiles */ .spinner { animation-duration: 2s; } /* Más lento, no eliminado */ } /* La forma recomendada: diseñar primero SIN animaciones, luego agregar */ .card { transition: none; } @media (prefers-reduced-motion: no-preference) { .card { transition: transform .3s ease, box-shadow .3s ease; } }

🟢 Reglas de oro para animar bien

  • Usa solo transform y opacity para la mayoría de animaciones
  • Duración corta para UI (0.2s – 0.5s), más larga solo para efectos dramáticos
  • Menos es más: 1-2 animaciones por página, no 10
  • Las animaciones deben tener propósito: guiar la atención, confirmar una acción, mostrar estado
  • Siempre incluye @media (prefers-reduced-motion: reduce)
  • animation-fill-mode: both en casi todas las animaciones de entrada

Ejercicio integrador — Pantalla de carga y resultados en vivo
Loaders, skeleton, animaciones de entrada y efectos avanzados
✏️ Ejercicio — App de resultados deportivos animada Avanzado

Crea resultados-animados.html. El objetivo es que la página tenga una experiencia de carga animada completa, como una app real.

  1. Pantalla de carga inicial: Al cargar la página, muestra durante 2 segundos un overlay con position:fixed que tiene:
    • Un spinner grande centrado con animation: girar
    • El texto "Cargando resultados..." con efecto typewriter
    • Con JS, después de 2s agrega la clase .oculto que hace opacity:0 y pointer-events:none con una transition
  2. Skeleton screen: Mientras "carga" (los primeros 2s), muestra 4 skeleton cards con efecto shimmer. Después reemplázalas con las tarjetas reales usando JS (setTimeout).
  3. Animaciones de entrada encadenadas: Cuando las tarjetas reales aparecen, cada una debe animarse con scaleIn y un animation-delay de 0.1s entre cada una.
  4. Marcador en vivo animado: Un marcador tipo "América 3 – 1 Chivas" donde:
    • Los números del marcador tienen animación pulsar
    • Hay un indicador "EN VIVO" con parpadear
    • Un punto rojo pulsante con animación de ripple
  5. Barras de estadísticas: Al hacer clic en "Ver estadísticas" (un botón que con JS agrega la clase .visible), las barras de posesión, tiros y precisión crecen con animation: crecer y delays encadenados.
  6. Gradiente animado en el hero/header con background-size: 400% 400%.
  7. prefers-reduced-motion: Agrega la media query que desactiva todas las animaciones para usuarios que la necesiten.

Bonus: Cuando el usuario hace hover en una tarjeta de resultado, muestra con CSS un tooltip con ::before (Parte C) que diga "Click para ver detalle". La aparición del tooltip debe tener una transition de opacity de 0.2s.

🟢 Resumen de lo que aprendiste en la Parte H

  • Transition vs Animation: transition = reacción a interacción; animation = coreografía propia
  • @keyframes: define los pasos con from/to o 0%–100%; los porcentajes pueden combinarse (0%, 100%)
  • animation shorthand: nombre duración timing delay iteraciones dirección fill-mode play-state
  • timing-function: ease, linear, ease-in/out, cubic-bezier(), steps() para animaciones discretas
  • animation-delay: puede ser negativo para saltar parte de la animación
  • animation-direction: alternate para rebotes suaves sin corte
  • animation-fill-mode: both — para casi todas las animaciones de entrada
  • animation-play-state: pausar/reanudar con CSS o JS
  • Múltiples animaciones: separadas por coma en el shorthand
  • Loaders: spinner, dots, barras, ripple, barra de progreso
  • Skeleton screen: shimmer con gradiente animado de position
  • Entrada: fadeIn, slideFrom*, scaleIn con fill-mode:both
  • Encadenado: animation-delay + nth-child para aparición secuencial
  • Avanzado: typewriter (steps), gradiente animado, barras de stats, sacudir error
  • Rendimiento: solo transform y opacity; will-change con moderación
  • Accesibilidad: @media (prefers-reduced-motion: reduce) siempre
