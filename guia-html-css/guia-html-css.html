Glosario — El vocabulario del desarrollador
Define cada término con exactitud antes de escribir código

Cuando aprendes a cocinar, primero necesitas saber qué es un "sofrito", una "reducción" o un "baño maría". Con HTML y CSS pasa igual: debes dominar el vocabulario antes de escribir código, o todo te sonará a idioma extranjero.

🏷️
Etiqueta (Tag)
Una instrucción escrita entre los signos < y >. Le dice al navegador qué tipo de contenido viene a continuación. Como una señal de tránsito que dice "viene una curva".
📦
Elemento
La etiqueta de apertura + el contenido + la etiqueta de cierre. El "paquete completo". Si la etiqueta es la caja, el elemento es la caja con todo lo que tiene adentro.
⚙️
Atributo
Información adicional que va dentro de la etiqueta de apertura para configurar cómo funciona el elemento. Como los botones y perillas de un aparato electrodoméstico.
💎
Valor
Lo que se le asigna a un atributo. Siempre va entre comillas. En color="red", el valor es red. Es la configuración específica del botón.
🎭
Clase (class)
Un apodo que tú inventas y que puedes ponerle a varios elementos para darles el mismo estilo. Como un uniforme de equipo: muchos jugadores, mismo uniforme.
🆔
ID
Un identificador único para un solo elemento. No puede repetirse en la misma página. Como el número de cédula o pasaporte: solo una persona en el mundo lo tiene.
🎨
CSS
El lenguaje que controla la apariencia visual. Si HTML es la estructura de un edificio (paredes, puertas), CSS es la decoración (pintura, muebles, iluminación).
🎯
Selector
La parte del CSS que indica a qué elemento(s) se les aplica el estilo. Como apuntar con una linterna en un cuarto oscuro: ilumina exactamente lo que tú quieres.
📋
Propiedad CSS
El aspecto visual que quieres cambiar: color, font-size, background. Es la pregunta "¿qué quieres cambiar?"
🔢
Valor CSS
La respuesta a "¿cómo quieres que quede?". Ejemplos: red, 16px, bold. Cada propiedad acepta ciertos valores específicos.
🌳
DOM
Document Object Model. La representación en árbol de todos los elementos HTML. El navegador lee tu HTML y lo convierte en este árbol para poder manipularlo.
🧱
Contenedor / Wrapper
Un elemento que solo existe para agrupar y organizar otros elementos dentro. Como una caja de cartón que usa de transporte: no importa por sí sola, importa lo que lleva.
🧬
Anidamiento
Poner un elemento dentro de otro. Como muñecas rusas (matrioshkas): una dentro de la otra. El que va dentro se llama "hijo" y el que lo contiene se llama "padre".
🌊
Cascada (en CSS)
El mecanismo por el cual, cuando hay reglas CSS contradictorias, se decide cuál gana. Funciona por orden, especificidad y herencia. El nombre CSS viene de aquí.
🧬
Herencia
Muchas propiedades CSS de un padre se transmiten automáticamente a sus hijos. Si el cuerpo tiene color: gray, todos los textos dentro serán grises, a menos que los sobreescribas.
📄
Elemento de bloque
Ocupa todo el ancho disponible y empuja el siguiente elemento hacia abajo. Como un párrafo en un libro: siempre empieza en nueva línea. Ej: <div>, <p>, <h1>.
↔️
Elemento en línea
Solo ocupa el espacio que necesita su contenido y convive en la misma línea con otros elementos. Como palabras en una oración. Ej: <span>, <a>, <strong>.
🔤
Etiqueta vacía
Una etiqueta que no tiene etiqueta de cierre porque no puede contener contenido dentro. Como una foto enmarcada: la foto en sí no contiene nada. Ej: <img>, <br>, <hr>.

🍕 Analogía maestra: HTML + CSS como construir una pizzería

HTML es la estructura del local: mesas, sillas, mostrador, horno, letreros. Sin HTML no hay nada que ver. CSS es la decoración y el diseño: los colores de las paredes, el tipo de letra del menú, si las sillas son altas o bajas. Sin CSS el local existe, pero es un cuarto vacío con muebles grises. JavaScript (que aprenderás después) es la interactividad: que el menú digital se actualice, que puedas pedir desde la mesa, que aparezca una notificación.

¿Cómo funciona un navegador?
Entender el proceso completo te hace mejor desarrollador

Antes de escribir código, necesitas entender qué hace el navegador con él. Este proceso pasa en milisegundos cada vez que abres una página web.

① Recibe el HTML — Tu archivo .html llega al navegador (ya sea desde internet o desde tu computadora).

② Lee el HTML de arriba hacia abajo — El navegador lee el código línea por línea, como tú lees un libro. Si encuentra un error, intenta corregirlo solo (esto puede causar comportamientos inesperados).

③ Construye el DOM — Convierte el HTML en un árbol de objetos en memoria. Cada elemento se convierte en un "nodo" del árbol.

④ Lee el CSS — Aplica los estilos a cada nodo del árbol. Calcula colores, tamaños, posiciones.

⑤ Renderiza (dibuja) — Pinta los píxeles en la pantalla. Lo que ves es el resultado de todos estos pasos.

🎬 Analogía: el director de teatro

El HTML es el guión de la obra (qué personajes hay, qué dicen, en qué orden salen). El CSS es el vestuario y la escenografía (cómo se ven los personajes, qué colores tiene el escenario). El navegador es el director que lee el guión y el vestuario, y luego organiza todo para que el público (tú) vea la obra terminada.

💡 Por qué importa esto

Si el CSS está en el <head>, el navegador lo procesa antes de mostrar contenido. Si pusiste el CSS al final del <body>, la página puede aparecer por un instante sin estilos (fea) y luego aplicar los estilos. Por eso los estilos siempre van en el <head>.

Editor de código — Tu taller de trabajo
La herramienta con la que escribirás todo tu código

Antes de escribir tu primera línea de HTML necesitas un lugar donde escribirlo. Ese lugar es el editor de código: un programa especializado que hace que escribir código sea más rápido, más cómodo y con muchos menos errores.

🔨 Analogía: el carpintero y su taller

Técnicamente podrías construir una silla con una piedra afilada, pero un carpintero profesional usa un taller con herramientas especializadas: sierra, lijadora, martillo, taladro. Un editor de código es ese taller para un desarrollador. Podrías escribir HTML en el Bloc de Notas, pero VS Code te da herramientas que multiplican tu productividad: autocompletado, colores para identificar el código, detección de errores, y mucho más.

VS Code — El editor más popular del mundo

Autocompletado (IntelliSense)

Cuando empiezas a escribir una etiqueta, VS Code sugiere cómo completarla. Escribe <h y aparecen todas las opciones. Presiona Tab o Enter para aceptar la sugerencia. Ahorra muchísimo tiempo y evita errores de escritura.

🔮 Como tener un asistente que predice lo que vas a escribir antes de que termines.

Resaltado de sintaxis

Cada parte del código tiene un color diferente: las etiquetas son de un color, los atributos de otro, los valores de otro más. Esto hace que el código sea mucho más fácil de leer y que los errores salten a la vista inmediatamente.

🌈 Como leer un libro donde los sustantivos, verbos y adjetivos están en colores diferentes — identificas la estructura de un vistazo.

Emmet — HTML en segundos

VS Code incluye Emmet, una herramienta que genera estructuras HTML completas con atajos. Escribe ! y presiona Tab → genera toda la estructura base HTML. Escribe ul>li*5 y Tab → genera una lista con 5 elementos.

⚡ Como tener sellos de caucho preformados: en vez de dibujar la misma figura repetidamente, estampas con un solo movimiento.

Live Server (extensión)

Esta extensión lanza un servidor local y abre tu página en el navegador. Cada vez que guardas un cambio en VS Code, el navegador se actualiza automáticamente — sin tener que recargar manualmente.

🪞 Como tener un espejo en tiempo real mientras te arreglas: ves el resultado de cada cambio al instante.

🟢 El truco del archivo index.html

Cuando un servidor web recibe una petición a una carpeta (ej: www.misitio.com/), automáticamente busca un archivo llamado index.html para mostrar. Es la convención universal de la web: si la página principal de tu proyecto se llama index.html, todo funciona automáticamente. Nunca cambies ese nombre para la página principal.

Estructura de un documento HTML
El esqueleto obligatorio de toda página web

Todo archivo HTML válido comienza con la misma estructura base. Piénsala como el esqueleto humano: sin él, el cuerpo no se sostiene. Todos los seres humanos tienen el mismo esqueleto básico aunque tengan apariencias distintas.

HTML <!-- ① Le dice al navegador: "esto es HTML5, la versión moderna" --> <!DOCTYPE html> <!-- ② El elemento raíz. TODO el documento va dentro de él --> <!-- lang="es" le dice a Google y lectores de pantalla: está en español --> <html lang="es"> <!-- ③ La "cabeza" del documento. El navegador lee esto pero NO lo muestra --> <!-- Aquí va toda la configuración e información "detrás de cámaras" --> <head> <!-- Permite letras especiales: tildes, ñ, emojis, etc. SIEMPRE la primera línea --> <meta charset="UTF-8"> <!-- Hace que la página se vea bien en celulares (responsive) --> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- El texto que aparece en la pestaña del navegador y en Google --> <title>Mi primera página web</title> <!-- Enlaza el archivo CSS externo --> <link rel="stylesheet" href="estilos.css"> </head> <!-- ④ El "cuerpo". TODO lo que el usuario VE va aquí --> <body> <h1>¡Hola Mundo!</h1> <p>Mi primera página web.</p> </body> </html>

🏠 Analogía: la casa

<html> es el terreno completo donde está la casa. <head> son los planos, los documentos de propiedad, las instrucciones del arquitecto — nadie los ve al vivir ahí, pero son esenciales. <body> es la casa que realmente habitas: las habitaciones, los muebles, todo lo que ves y tocas.

💡 La regla del anidamiento

Los elementos se abren y cierran como paréntesis: el que abre al último, cierra primero.

✅ Correcto: <div><p>texto</p></div> — el <p> abre y cierra dentro del <div>

❌ Incorrecto: <div><p>texto</div></p> — las cajas se cruzan

Etiquetas de texto
Para estructurar y dar significado al contenido escrito
<h1> — <h6>Headings / Títulos

Son los títulos de la página, del más importante (h1) al menos importante (h6). Tienen diferente tamaño por defecto, pero lo más importante es su significado jerárquico.

📰 Como en un periódico: el titular principal es h1 (grande, una sola vez por página), los subtítulos de sección son h2, los subtítulos dentro de esas secciones son h3, y así sucesivamente.

<h1>Título Principal</h1>
<h2>Subtítulo de sección</h2>
<h3>Subtítulo menor</h3>
<p>Paragraph / Párrafo

Define un párrafo de texto. El navegador agrega espacio arriba y abajo automáticamente para separar párrafos. Es un elemento de bloque.

📄 Exactamente igual que los párrafos en un libro: texto agrupado que habla de una misma idea, con espacio de separación antes y después.

<p>Este es mi primer párrafo.</p>
<p>Este es el segundo párrafo.</p>
<strong>Strong / Énfasis fuerte

Marca texto como muy importante. Por defecto se ve en negritas. Tiene significado semántico: "esto es importante", no solo "ponlo en negrita".

🔔 Como resaltar con marcador en un libro de texto las palabras clave que sí o sí tienes que recordar para el examen.

<p>Recuerda <strong>guardar tu trabajo</strong> seguido.</p>
<em>Emphasis / Énfasis

Marca texto con énfasis. Por defecto se ve en cursiva/itálica. Indica que esa palabra cambia el significado de la oración si se dice con otro tono.

🗣 Como cambiar el tono de voz al hablar. "Yo nunca dije eso" vs "Yo nunca dije eso" — el énfasis cambia el significado.

<p>Fue <em>increíblemente</em> difícil.</p>
<span>Span / Contenedor en línea

Un contenedor invisible en línea que no agrega significado por sí solo. Se usa para aplicar estilos CSS a una parte del texto dentro de un párrafo.

🖊 Como rodear con un círculo una sola palabra en un papel para poder señalar luego "colorea ese círculo de rojo". Sin el círculo, no podrías identificar exactamente qué colorear.

<p>El precio es <span class="precio">$299</span> pesos.</p>
<br>Break / Salto de línea

Inserta un salto de línea dentro del mismo elemento. Es una etiqueta vacía (no tiene cierre). Úsala solo cuando el salto de línea es parte del contenido.

⏎ Como la tecla Enter al escribir en Word: empuja el texto a la siguiente línea sin crear un nuevo párrafo con su propio espacio.

<p>Calle Álamo 42<br>Col. Centro<br>Veracruz, México</p>
<hr>Horizontal Rule / Línea

Dibuja una línea horizontal que separa secciones visualmente. También es etiqueta vacía. Originalmente era una "regla" tipográfica de separación.

━━━ Como la línea que dibuja el mesero en la comanda para separar las bebidas de los platillos.

<p>Sección 1</p>
<hr>
<p>Sección 2</p>
<blockquote>Cita en bloque

Marca una cita larga tomada de otra fuente. El navegador la indenta automáticamente. Para la fuente de la cita se usa el atributo cite.

📜 Como cuando en un ensayo pones una cita con sangría de otro autor para distinguirla de tu propio texto.

<blockquote>
<p>"La imaginación es más
importante que el conocimiento."</p>
</blockquote>
<code>Código en línea

Marca texto como código de computadora. Por defecto se muestra en fuente monoespaciada. Úsalo para mencionar nombres de variables, funciones o etiquetas dentro de un párrafo.

💻 Como cuando en un manual de instrucciones destacan los comandos que debes escribir con una tipografía especial para que no los confundas con el texto normal.

<p>Usa la función <code>alert()</code>.</p>
<pre>Texto preformateado

Muestra el texto exactamente como está escrito, respetando espacios y saltos de línea. Por defecto usa fuente monoespaciada. Se usa junto con <code> para bloques de código.

📠 Como un fax: lo que envías llega exactamente igual al otro lado, sin que nadie reorganice los espacios.

<pre><code>
function suma(a, b) {
return a + b;
}
</code></pre>
<small>Texto pequeño

Representa texto en letra pequeña: avisos legales, derechos de autor, notas al pie. No es solo estilo, tiene significado semántico de "texto de menor importancia".

📋 La letra chica de un contrato. Existe, es parte del documento, pero visualmente está subordinada al texto principal.

<small>© 2025 Mi Empresa. Todos los derechos reservados.</small>
<mark>Texto resaltado

Resalta texto como relevante en el contexto actual. Por defecto aparece con fondo amarillo (como un marcatextos). Útil para resultados de búsqueda.

🖍 Exactamente como un marcatextos físico: sigues leyendo el mismo documento pero algunas partes están visualmente destacadas.

<p>Busca la palabra <mark>HTML</mark> en el texto.</p>

⚠️ <b> y <i> vs <strong> y <em>

<b> y <i> son solo visuales: ponen negrita o cursiva sin significado. <strong> y <em> tienen significado semántico. Los motores de búsqueda y lectores de pantalla los interpretan diferente. Siempre prefiere <strong> y <em>.

Listas

<ul>Unordered List / Lista sin orden

Una lista donde el orden no importa. Los elementos aparecen con un punto (bullet) al inicio por defecto.

🛒 La lista del supermercado: da lo mismo si primero compras las manzanas o la leche, el orden no cambia nada.

<ul>
<li>HTML</li>
<li>CSS</li>
<li>JavaScript</li>
</ul>
<ol>Ordered List / Lista ordenada

Una lista donde el orden sí importa. Los elementos se numeran automáticamente. Puedes cambiar el tipo de numeración con el atributo type.

📋 La receta de cocina: primero bates los huevos, luego añades la harina. Si cambias el orden, el resultado es diferente.

<ol>
<li>Abre el editor</li>
<li>Escribe el código</li>
<li>Guarda el archivo</li>
</ol>
<li>List Item / Elemento de lista

Define cada elemento dentro de <ul> u <ol>. Nunca se usa solo: siempre debe ir dentro de una lista. Puede contener cualquier otro elemento HTML.

🧾 Cada renglón del ticket de caja. El ticket es la lista (<ul>/<ol>), cada producto es un <li>.

<li>Texto simple</li>
<li><strong>Con formato</strong></li>
<li><a href="#">Con enlace</a></li>
<dl> <dt> <dd>Definition List

Lista de definiciones. <dl> es la lista, <dt> es el término y <dd> es la definición. Perfecta para glosarios o preguntas frecuentes.

📚 Un diccionario: la palabra en negrita (<dt>) seguida de su significado indentado (<dd>).

<dl>
<dt>HTML</dt>
<dd>Lenguaje de marcado</dd>
<dt>CSS</dt>
<dd>Hojas de estilo</dd>
</dl>

El enlace: la etiqueta más poderosa de internet

<a>Anchor / Ancla — el hipervínculo

La etiqueta <a> (de "anchor", ancla en inglés) es la que hace posible internet tal como lo conocemos: la capacidad de saltar de una página a otra con un clic. Fue inventada por Tim Berners-Lee y es el fundamento de la "Web".

🔗 Como un portal mágico. Estás leyendo un libro y de repente hay una frase subrayada que, si la tocas, te teletransporta a otro libro relacionado. Eso es un hipervínculo.

🟢 ¿Por qué rel="noopener"?

Cuando usas target="_blank", la nueva pestaña puede acceder a tu página original por razones de seguridad. rel="noopener" bloquea eso. Siempre que uses target="_blank", añade rel="noopener noreferrer". Es una buena práctica de seguridad.

Darle significado a la estructura, no solo apariencia

El HTML semántico usa etiquetas que describen el significado del contenido, no solo su apariencia. Puedes usar <div> para todo — técnicamente funciona — pero es como escribir un libro donde todos los personajes se llaman "Personaje": funciona, pero no le dice nada al lector sobre quién es quién.

Especificidad y la Cascada
Por qué a veces tus estilos no se aplican — y cómo solucionarlo

La especificidad es el sistema que usa CSS para decidir cuál regla "gana" cuando dos reglas quieren cambiar la misma propiedad del mismo elemento. Es la fuente del 80% de las confusiones de los principiantes.

🏆 Analogía: el torneo de jerarquías

Imagina un torneo donde los jugadores tienen rangos. Un general (id) siempre le gana a un capitán (clase), y un capitán siempre le gana a un soldado (etiqueta). No importa cuántos soldados pongas: un solo general los derrota a todos. Pero hay alguien que le gana al general: el rey (inline style con style="" directamente en el elemento).

Unidades de medida en CSS
px, em, rem, %, vw, vh — cuándo usar cada una
px
Píxeles — absoluto

Una medida fija. 16px siempre es 16px, sin importar nada. Fácil de entender pero no se adapta bien a pantallas de diferente tamaño o preferencias del usuario.

%
Porcentaje — relativo al padre

Porcentaje del elemento padre. Si el padre tiene width: 500px, un hijo con width: 50% tendrá 250px. Perfecto para layouts fluidos.

em
Em — relativo al padre

Relativo al font-size del elemento padre. Si el padre tiene 16px, 2em = 32px. El problema: se acumula en elementos anidados (1.5em dentro de 1.5em = 2.25em).

rem
Root Em — relativo a la raíz

Relativo al font-size del elemento <html> (raíz). Por defecto son 16px. 1rem = 16px siempre, sin importar la anidación. El favorito de los profesionales.

vw
Viewport Width

1% del ancho de la ventana del navegador. 100vw = ancho completo de la pantalla. Ideal para elementos que deben cubrir toda la pantalla.

vh
Viewport Height

1% del alto de la ventana del navegador. 100vh = alto completo de la pantalla. Úsalo para crear secciones "hero" que cubren toda la pantalla.

📐 Analogía: unidades de construcción

px es como medir en centímetros: exacto y fijo. % es como decir "la mitad del cuarto" — depende del tamaño del cuarto. rem es como decir "X veces el ladrillo estándar" — el ladrillo siempre es el mismo tamaño, así que la proporción es predecible. vw/vh es "X% del terreno disponible" — se adapta a cualquier pantalla.

🟢 Guía rápida de cuándo usar cada una

  • px → bordes, sombras, cosas que no deben escalar: border: 1px solid
  • rem → tamaños de fuente, padding, margin — todo que debe escalar con las preferencias del usuario
  • % → anchos de columnas y elementos que deben ser proporcionales al padre
  • vw/vh → secciones hero, modales de pantalla completa, fondos
  • em → espaciado interno de componentes que debe ser relativo al tamaño de su propio texto

El Modelo de Caja (Box Model)
El concepto más importante de CSS — todo es una caja

En CSS, absolutamente todo elemento es una caja rectangular. Aunque visualmente parezca un círculo, un texto o una imagen, internamente el navegador lo trata como una caja. Esta caja tiene 4 capas concéntricas.

🟡 MARGIN — Espacio exterior. Transparente, separa este elemento del siguiente. No tiene color de fondo.
🔴 BORDER — El borde del elemento. Puede tener grosor, estilo y color.
🟢 PADDING — Espacio interior. Distancia entre el borde y el contenido. Toma el color de fondo del elemento.
CONTENT
Aquí va el texto, la imagen, o los elementos hijos.
CSS .tarjeta { /* CONTENT */ width: 320px; height: 200px; /* PADDING — espacio entre el borde y el contenido */ padding: 20px; /* los 4 lados iguales */ padding: 10px 20px; /* arriba/abajo izquierda/derecha */ padding: 10px 20px 15px 5px; /* arriba derecha abajo izquierda (reloj) */ padding-top: 10px; /* lado específico */ padding-right: 20px; padding-bottom: 15px; padding-left: 5px; /* BORDER */ border: 2px solid #4F46E5; /* grosor tipo color */ border-radius: 12px; /* esquinas redondeadas */ border-radius: 50%; /* círculo perfecto (si width=height) */ /* MARGIN — espacio fuera del elemento */ margin: 20px; /* los 4 lados */ margin: 0 auto; /* truco para centrar: arriba/abajo=0, izq/der=automático */ /* FONDO */ background-color: white; }

⚠️ El problema del box-sizing

Por defecto, cuando dices width: 300px y luego añades padding: 20px, el elemento queda con 340px de ancho total (300 + 20 izquierda + 20 derecha). Esto rompe layouts. La solución es box-sizing: border-box: el padding se incluye dentro del width definido, así que el elemento sigue midiendo 300px.

CSS — Reset recomendado (ponlo siempre al inicio) /* Esta regla hace que el box-sizing sea intuitivo en TODOS los elementos */ *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }

💡 Truco: margin: 0 auto para centrar

Para centrar horizontalmente un elemento de bloque con ancho fijo, usa margin: 0 auto. El auto a la izquierda y derecha distribuye el espacio disponible de manera equitativa, centrando el elemento. Esto solo funciona si el elemento tiene un width definido.

La propiedad display
Controlar cómo se comporta y ocupa espacio un elemento

La propiedad display es una de las más importantes de CSS. Controla fundamentalmente cómo se comporta un elemento en el flujo del documento y cómo organiza a sus hijos.

ValorComportamientoCuándo usarlo
block Ocupa todo el ancho disponible. Empieza en nueva línea. Respeta width, height, padding, margin. Es el valor por defecto de <div>, <p>, <h1>, etc.
inline Solo ocupa el ancho de su contenido. No rompe la línea. No respeta width/height verticales. Es el valor por defecto de <span>, <a>, <strong>.
inline-block Convive en línea como inline, PERO sí respeta width, height, padding y margin completamente. Botones, chips de categoría, badges — elementos que deben estar en línea pero con control de tamaño.
none El elemento desaparece completamente del layout. No ocupa espacio. Diferente a visibility: hidden que lo oculta pero sigue ocupando espacio. Menús móviles cerrados, modales cerrados, contenido oculto.
flex Activa Flexbox para organizar los hijos en una fila o columna flexible. Barras de navegación, tarjetas en fila, centrar contenido. El más usado hoy en día.
grid Activa CSS Grid para organizar los hijos en una cuadrícula de filas y columnas. Layouts de página completa, galerías de imágenes, diseños complejos.

🧱 Analogía: los bloques de construcción

block son como ladrillos horizontales: cada uno ocupa todo el ancho de la pared y el siguiente va en la fila de abajo. inline son como palabras en un texto: se ponen una al lado de la otra hasta que no caben y saltan a la siguiente línea. inline-block son como palabras pero que puedes cambiar de tamaño (¡como si las palabras tuvieran altura y ancho configurables!).

Posicionamiento CSS
Sacar elementos del flujo normal para posicionarlos con precisión
ValorComportamientoUsos típicos
static El valor por defecto. El elemento sigue el flujo normal del documento. Las propiedades top/left/right/bottom no tienen efecto. La mayoría de los elementos — no necesitas escribirlo.
relative Se desplaza relativo a su posición original. SÍ responde a top/left/right/bottom. El espacio original se mantiene (no lo "ceden" otros elementos). Pequeños ajustes de posición. También se usa como "contexto" para elementos hijos con position: absolute.
absolute Sale completamente del flujo normal. Se posiciona relativo al ancestro más cercano con position ≠ static. Si no hay ninguno, se posiciona relativo al viewport. Tooltips, badges de notificación, dropdowns, iconos sobre imágenes.
fixed Sale del flujo y se fija relativo a la ventana del navegador. No se mueve al hacer scroll. Barras de navegación que quedan fijas al hacer scroll, botones flotantes de chat o "volver arriba".
sticky Se comporta como relative hasta que el usuario hace scroll y llega a cierta posición, luego se vuelve fixed. Cabeceras de tabla que se quedan visibles al hacer scroll, menús de navegación que se fijan al bajar.

💡 z-index: capas como en Photoshop

Cuando los elementos se superponen visualmente, z-index controla cuál aparece encima. Es como el orden de capas en Photoshop: un número mayor = capa más arriba = aparece encima. Solo funciona en elementos con position diferente a static.

Flexbox — Layout flexible
El sistema moderno para organizar elementos en una dimensión

🚂 Analogía: el tren flexible

El contenedor flex es como un tren. Los elementos hijos son los vagones. Con Flexbox decides: ¿los vagones van en fila horizontal o en columna vertical? ¿cómo se distribuye el espacio entre ellos? ¿cómo se alinean verticalmente? ¿pueden doblar a una nueva línea si no caben? Todo esto lo controlas con propiedades del contenedor (el tren), y a veces también con propiedades de los hijos (los vagones).

CSS — Propiedades del CONTENEDOR (padre) .contenedor { display: flex; /* ACTIVA flexbox — esto es lo único obligatorio */ /* Dirección del eje principal */ flex-direction: row; /* → izquierda a derecha (default) */ flex-direction: row-reverse; /* ← derecha a izquierda */ flex-direction: column; /* ↓ arriba hacia abajo */ flex-direction: column-reverse; /* ↑ abajo hacia arriba */ /* Distribución en el eje principal (horizontal si row) */ justify-content: flex-start; /* al inicio */ justify-content: flex-end; /* al final */ justify-content: center; /* centrado */ justify-content: space-between; /* espacio entre elementos, sin bordes */ justify-content: space-around; /* espacio alrededor de cada elemento */ justify-content: space-evenly; /* espacio perfectamente igual entre todos */ /* Alineación en el eje secundario (vertical si row) */ align-items: stretch; /* estira al máximo alto (default) */ align-items: flex-start; /* arriba */ align-items: flex-end; /* abajo */ align-items: center; /* centrado vertical */ align-items: baseline; /* alineados por la línea base del texto */ /* Espacio entre los hijos */ gap: 20px; /* igual entre filas y columnas */ gap: 10px 20px; /* entre-filas entre-columnas */ row-gap: 10px; column-gap: 20px; /* ¿Los hijos saltan a nueva línea si no caben? */ flex-wrap: nowrap; /* no (default) — se comprimen */ flex-wrap: wrap; /* sí — saltan a la siguiente fila */ }
CSS — Propiedades de los HIJOS .hijo { /* flex: grow shrink basis */ flex: 1; /* shorthand de flex: 1 1 0 — crece para llenar espacio */ flex: 0 0 200px; /* no crece, no se encoge, siempre 200px */ /* Sobreescribir la alineación para este hijo específico */ align-self: flex-start; /* Cambiar el orden visual (sin cambiar el HTML) */ order: -1; /* va primero */ order: 2; /* va de tercero (el default es 0) */ }

CSS Grid — Layout en dos dimensiones
Para layouts de página completa y cuadrículas complejas

🏙 Analogía: el plano urbanístico

CSS Grid es como el plano de una ciudad: defines las manzanas (filas y columnas) y luego ubicas los edificios (elementos) en esas manzanas. Un edificio puede ocupar una manzana, o puede ser tan grande que ocupe 3 manzanas de ancho y 2 de alto. Flexbox es bueno para una calle (una dimensión). Grid es para planear toda la ciudad (dos dimensiones).

CSS .contenedor { display: grid; /* Define columnas — puedes mezclar unidades */ grid-template-columns: 200px 1fr 1fr; /* 200px fija + 2 iguales del resto */ grid-template-columns: repeat(3, 1fr); /* 3 columnas iguales */ grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* responsive automático */ /* Define filas */ grid-template-rows: 80px 1fr 60px; /* header, contenido, footer */ /* Espacio entre celdas */ gap: 20px; } /* Un hijo puede ocupar múltiples columnas o filas */ .destacado { grid-column: 1 / 3; /* ocupa desde columna 1 hasta columna 3 */ grid-row: 1 / 2; /* ocupa la primera fila */ } /* Forma abreviada: span = "ocupa N columnas/filas" */ .ancho-total { grid-column: 1 / -1; /* ocupa TODAS las columnas */ }

Diseño Responsive
Que tu sitio se vea bien en celular, tablet y escritorio

El diseño responsive (responsivo) significa que tu sitio web se adapta automáticamente a diferentes tamaños de pantalla. Hoy más del 60% del tráfico web viene de celulares.

💧 Analogía: el agua

Un diseño responsive es como el agua: toma la forma del recipiente que la contiene. En un vaso pequeño (celular) ocupa poco espacio en columna. En un recipiente grande (escritorio) se expande y usa todo el espacio disponible. La estructura del contenido es la misma, solo cambia cómo se organiza visualmente.

Media Queries

Los media queries son condiciones CSS: "aplica estos estilos SOLO si la pantalla tiene cierto tamaño".

CSS — Enfoque "Mobile First" (recomendado) /* Estilos base: para celular (pantalla pequeña) */ .galeria { display: flex; flex-direction: column; /* en móvil: columna (uno debajo del otro) */ gap: 1rem; } /* Para pantallas medianas (tablets) */ @media (min-width: 768px) { .galeria { flex-direction: row; /* en tablet: fila */ flex-wrap: wrap; } .tarjeta { width: calc(50% - 0.5rem); /* 2 columnas */ } } /* Para pantallas grandes (escritorio) */ @media (min-width: 1024px) { .tarjeta { width: calc(33.333% - 0.67rem); /* 3 columnas */ } nav { padding: 1rem 4rem; /* más padding lateral en escritorio */ } } /* Para pantallas muy grandes */ @media (min-width: 1280px) { .contenedor { max-width: 1200px; margin: 0 auto; /* centrar el contenido en pantallas enormes */ } }
