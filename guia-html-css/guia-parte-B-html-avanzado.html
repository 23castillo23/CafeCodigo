Links avanzados — Mucho más que href
target, download, tel, mailto, anclas y todo lo que <a> puede hacer

Ya conoces el link básico con <a href="url">. Pero la etiqueta <a> tiene muchas más capacidades que la mayoría de principiantes no conoce. Puede abrir en nueva pestaña, descargar archivos, llamar por teléfono, abrir el correo, o saltar a una sección de la misma página.

🚪 Analogía: los tipos de puertas de un edificio

Un link básico es como una puerta que lleva a otra habitación del mismo edificio. Pero también hay puertas que llevan a otro edificio (nueva pestaña), puertas que entregan un paquete (download), puertas que conectan un teléfono (tel:), y puertas que abren un buzón de correo (mailto:). Todas son puertas — todas son <a> — pero llevan a lugares diferentes.

El atributo target — ¿dónde se abre el link?

HTML<!-- _self: en la misma pestaña (comportamiento por defecto) --> <a href="pagina.html" target="_self">Ir a Pagina</a> <!-- _blank: en una pestaña NUEVA — muy útil para links externos --> <a href="https://google.com" target="_blank">Abrir Google</a> <!-- _parent: en el frame padre (para páginas con iframes) --> <a href="pagina.html" target="_parent">Ir al padre</a> <!-- _top: en la ventana completa, saliendo de todos los iframes --> <a href="pagina.html" target="_top">Ir al tope</a>

⚠️ Seguridad con target="_blank"

Cuando usas target="_blank", siempre agrega rel="noopener noreferrer". Sin esto, la página que abres puede acceder a tu página original mediante JavaScript (vulnerabilidad llamada "tabnabbing"). Con el rel queda seguro:

HTML<!-- ✅ Forma correcta y segura para abrir links externos --> <a href="https://github.com" target="_blank" rel="noopener noreferrer"> Ver mi GitHub </a>

Los tipos de links que puedes crear

🌐 Link a página web
<a href="https://mipagina.com"> Ir a mi página </a>
El link más común. Usa siempre https:// para URLs externas.
📂 Link a archivo interno
<a href="sobre-mi.html"> Sobre mí </a> <a href="./proyectos/web.html"> Proyectos </a>
Ruta relativa al archivo HTML en tu propio proyecto.
⬇️ Link de descarga
<a href="cv-hugo.pdf" download> Descargar mi CV </a> <a href="foto.png" download="mi-foto.png"> Descargar foto </a>
El atributo download fuerza la descarga en vez de abrir el archivo. Puedes darle un nombre personalizado al archivo descargado.
📧 Link de correo electrónico
<!-- Abre el cliente de correo --> <a href="mailto:hugo@email.com"> Envíame un correo </a> <!-- Con asunto y cuerpo predefinidos --> <a href="mailto:hugo@email.com ?subject=Hola Hugo &body=Te contacto desde..."> Escribir con asunto </a>
Abre la app de correo del usuario con el destinatario ya puesto. Muy usado en páginas de contacto.
📞 Link de teléfono
<!-- En móvil, llama directamente --> <a href="tel:+521234567890"> Llamar ahora </a> <!-- WhatsApp directo --> <a href="https://wa.me/521234567890" target="_blank"> WhatsApp </a>
En dispositivos móviles, tel: abre el marcador de llamadas directamente. Imprescindible en sitios de negocios.
⚓ Ancla — salto en la misma página
<!-- El link apunta al id --> <a href="#contacto"> Ir a Contacto </a> <!-- El destino tiene ese id --> <section id="contacto"> <h2>Contacto</h2> </section> <!-- Volver al inicio de la página --> <a href="#">↑ Subir arriba</a>
El #id hace que la página se desplace hasta el elemento con ese id. Muy usado en menús de navegación de una sola página (one-page).

🟢 Puedes poner cualquier cosa dentro de un <a>

Un link no tiene que ser solo texto. Puedes envolver imágenes, botones, tarjetas completas, íconos, o cualquier elemento HTML para que sean clicables:

HTML<!-- Link con imagen — la imagen entera es clicable --> <a href="proyecto.html"> <img src="captura.png" alt="Ver proyecto"> </a> <!-- Link con tarjeta completa --> <a href="proyecto.html" class="tarjeta"> <h3>Mi Proyecto</h3> <p>Descripción del proyecto...</p> </a>

Tablas HTML — Datos estructurados en filas y columnas
Para mostrar información tabular: precios, horarios, estadísticas, comparaciones

Las tablas son para mostrar datos que tienen una relación entre filas y columnas: una tabla de precios, los resultados de un torneo, un horario de clases, estadísticas de jugadores. Lo que NO se debe hacer es usar tablas para maquetar el diseño de una página (eso se hacía en los años 90, hoy se usa Flexbox y Grid).

📋 Analogía: la tabla de posiciones de la Liga MX

Cuando ves la tabla de posiciones del fútbol, cada fila es un equipo y cada columna es un dato: Pos, Equipo, PJ, G, E, P, GF, GC, Pts. Los datos tienen sentido porque están en la intersección correcta de fila y columna. Eso es una tabla HTML: cada celda está en el lugar exacto que le corresponde.

Anatomía completa de una tabla

<table>Contenedor de toda la tabla
  <caption>Título descriptivo</caption>Opcional: título arriba
  <thead>Cabecera — fila de títulos
    <tr>Table Row — una fila
      <th>Equipo</th>Table Header — celda de encabezado
    </tr>
  </thead>
  <tbody>Cuerpo — filas de datos
    <tr>
      <td>América</td>Table Data — celda de dato
    </tr>
  </tbody>
  <tfoot>Pie — totales o notas
  </tfoot>
</table>

Los elementos de una tabla y qué hace cada uno

ElementoNombrePara qué sirve
<table>TableContenedor principal de toda la tabla. Sin él, nada funciona.
<caption>CaptionTítulo descriptivo de la tabla. Aparece arriba. Importante para accesibilidad.
<thead>Table HeadAgrupa la fila o filas de encabezado. El navegador puede repetirlas al imprimir.
<tbody>Table BodyAgrupa las filas de datos. Si hay mucho contenido, puede tener scroll independiente.
<tfoot>Table FootFila o filas de pie: totales, promedios, notas. El navegador también puede repetirlas al imprimir.
<tr>Table RowUna fila completa de la tabla. Contiene los <th> o <td>.
<th>Table HeaderCelda de encabezado. El navegador la muestra en negrita y centrada por defecto. Semánticamente indica que es un título de columna o fila.
<td>Table DataCelda de dato normal. Aquí va la información de cada intersección fila-columna.

colspan y rowspan — Fusionar celdas

A veces necesitas que una celda ocupe varias columnas o varias filas, como en Excel cuando fusionas celdas. Para eso existen colspan y rowspan:

HTML<table> <thead> <tr> <!-- Esta celda ocupa 3 columnas horizontalmente --> <th colspan="3">Estadísticas del Torneo 2025</th> </tr> <tr> <th>Equipo</th> <th>Goles</th> <th>Puntos</th> </tr> </thead> <tbody> <tr> <!-- Esta celda ocupa 2 filas verticalmente --> <td rowspan="2">América</td> <td>42</td> <td>56</td> </tr> </tbody> </table>

Ejemplo completo — Tabla de posiciones de Liga MX

🟢 Estilizar tablas con CSS — lo básico

  • border-collapse: collapse en la table elimina el doble borde entre celdas
  • tr:nth-child(even) colorea las filas alternas (efecto "cebra") para facilitar la lectura
  • overflow-x: auto en un contenedor div hace que la tabla tenga scroll horizontal en móvil

Audio — Reproducción de sonido nativa
Sin plugins, sin Flash: audio directo con HTML5

HTML5 introdujo la etiqueta <audio> que permite reproducir audio directamente en el navegador sin necesidad de Flash ni plugins externos. El navegador muestra sus controles nativos: play, pausa, volumen y barra de progreso.

📻 Analogía: el radio integrado de un auto

En los autos viejos tenías que instalar un radio especial con adaptadores. Los autos modernos ya traen el radio integrado en el tablero: solo enchufas el USB y funciona. Eso es <audio>: el "radio" que el navegador moderno trae integrado, sin necesitar nada externo.

Anatomía de <audio>

<audio controlsMuestra los controles del navegador autoplayEmpieza a reproducir solo loopSe repite al terminar mutedEmpieza en silencio>
  <source src="cancion.mp3" type="audio/mpeg">
  <source src="cancion.ogg" type="audio/ogg">
  Tu navegador no soporta audio HTML5.
</audio>

Los atributos de <audio>

AtributoQué haceNota importante
controlsMuestra los controles nativos del navegador (play, volumen, barra)Sin este atributo, el audio es invisible e inutilizable para el usuario
autoplayEmpieza a reproducirse automáticamente al cargar la página⚠️ La mayoría de navegadores lo bloquean a menos que también tenga muted
loopCuando termina, vuelve a empezarÚtil para música de fondo o efectos de sonido
mutedEmpieza en silencio (el usuario puede subir el volumen)Combinado con autoplay sí funciona en todos los navegadores
preloadIndica cuánto debe precargar el navegadorauto, metadata (solo duración), o none (no precarga)

¿Por qué usar múltiples <source>?

No todos los navegadores soportan todos los formatos de audio. Por eso pones varias fuentes y el navegador elige la primera que puede reproducir:

HTML<audio controls> <!-- El navegador prueba de arriba para abajo y usa el primero que soporta --> <source src="audio/cancion.mp3" type="audio/mpeg"> <!-- MP3: soportado por todos --> <source src="audio/cancion.ogg" type="audio/ogg"> <!-- OGG: mejor calidad, Firefox --> <source src="audio/cancion.webm" type="audio/webm"> <!-- WEBM: Chrome/Edge --> <!-- Mensaje de respaldo si el navegador no soporta <audio> --> Tu navegador no soporta la reproducción de audio. <a href="audio/cancion.mp3">Descargar el audio</a> </audio>

🟢 Formato recomendado en 2025

Usa MP3 como formato principal: es soportado por absolutamente todos los navegadores modernos y tiene buena compresión. Si quieres ofrecer mejor calidad sin sacrificar mucho peso, agrega también OGG como alternativa.

Video — Reproducción de video nativa
Videos propios, YouTube embebido y todo lo que necesitas saber

La etiqueta <video> funciona exactamente igual que <audio> pero para video. Tiene los mismos atributos (controls, autoplay, loop, muted) más algunos exclusivos como poster y width/height>.

📺 Analogía: la televisión integrada vs el proyector externo

Antes de <video>, mostrar video en una web era como usar un proyector externo: necesitabas instalar Flash, configurarlo, rezar para que funcionara. Con <video>, es como una smart TV integrada en la pared: enchufa el cable (pon la ruta del video) y listo.

Anatomía de <video> con todos sus atributos

HTML<video controls <!-- Muestra los controles --> width="800" <!-- Ancho en píxeles --> height="450" <!-- Alto en píxeles --> poster="miniatura.jpg" <!-- Imagen que se muestra antes de darle play --> autoplay <!-- Reproduce solo (bloqueado sin muted) --> muted <!-- Necesario para que autoplay funcione --> loop <!-- Se repite al terminar --> preload="metadata" <!-- Solo carga la duración y miniatura --> playsinline <!-- En iPhone: reproduce dentro de la página, no en pantalla completa --> > <source src="video/presentacion.mp4" type="video/mp4"> <source src="video/presentacion.webm" type="video/webm"> Tu navegador no soporta video HTML5. </video>

El atributo poster — imagen de previsualización

El poster es la imagen que aparece en el reproductor de video antes de que el usuario le dé play, como la miniatura de un video de YouTube. Si no lo pones, el navegador muestra el primer fotograma del video (que a veces es negro).

HTML<!-- El video muestra "portada.jpg" hasta que el usuario presione play --> <video controls poster="portada.jpg" width="100%"> <source src="mi-video.mp4" type="video/mp4"> </video>

Embeber videos de YouTube / Vimeo con <iframe>

Para videos de YouTube no usas <video>: YouTube te da un código <iframe> para copiar y pegar. El <iframe> incrusta una página web dentro de otra.

HTML<!-- Código que YouTube te da al hacer clic en "Compartir → Insertar" --> <iframe width="560" height="315" src="https://www.youtube.com/embed/ID_DEL_VIDEO" title="Nombre descriptivo del video" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope" allowfullscreen ></iframe> <!-- Para que sea responsive, envuélvelo en un div con CSS --> <div style="position:relative; padding-bottom:56.25%; height:0; overflow:hidden;"> <iframe style="position:absolute; top:0; left:0; width:100%; height:100%;" src="https://www.youtube.com/embed/ID_DEL_VIDEO" allowfullscreen ></iframe> </div>

💡 ¿Cuándo usar <video> y cuándo <iframe>?

Usa <video> para videos propios que tienes en tu servidor (tu portafolio, demos, presentaciones). Usa <iframe> para videos de plataformas externas como YouTube o Vimeo: son más livianos (no guardas el video en tu servidor) y ya incluyen el reproductor completo con las funciones de la plataforma.

🟢 Video de fondo (hero) — técnica profesional

Para usar un video como fondo de una sección hero, se combina autoplay muted loop playsinline y se posiciona con CSS. El muted es obligatorio para que el autoplay funcione en todos los navegadores.

HTML + CSS<!-- HTML --> <section class="hero"> <video autoplay muted loop playsinline class="hero-video"> <source src="fondo.mp4" type="video/mp4"> </video> <div class="hero-content"> <h1>Bienvenido a mi Portafolio</h1> </div> </section> /* CSS */ .hero { position: relative; height: 100vh; overflow: hidden; } .hero-video { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); min-width: 100%; min-height: 100%; object-fit: cover; } .hero-content { position: relative; z-index: 1; color: white; }

Lazy Loading — Carga diferida de imágenes y videos
Haz tu página más rápida sin escribir JavaScript

Por defecto, cuando el navegador carga una página descarga todas las imágenes al mismo tiempo, aunque el usuario esté viendo solo la parte de arriba. Con lazy loading, las imágenes que están fuera de la pantalla no se descargan hasta que el usuario hace scroll y están a punto de verse.

🛒 Analogía: el supermercado que pide mercancía bajo demanda

Un supermercado normal tiene el almacén lleno desde el primer día. Un supermercado inteligente solo pide mercancía cuando el anaquel está a punto de vaciarse: ahorra espacio, dinero y esfuerzo. Lazy loading es eso: solo descarga las imágenes cuando el usuario está a punto de verlas.

Cómo activarlo — un solo atributo

HTML<!-- Sin lazy loading: se descarga aunque el usuario no la vea --> <img src="foto-grande.jpg" alt="Descripción"> <!-- Con lazy loading: solo se descarga cuando el usuario hace scroll hasta aquí --> <img src="foto-grande.jpg" alt="Descripción" loading="lazy"> <!-- También funciona en iframes (videos de YouTube) --> <iframe src="https://www.youtube.com/embed/ID" loading="lazy" allowfullscreen ></iframe>
Valor de loadingQué haceCuándo usar
lazySe descarga solo cuando está cerca del viewport (la parte visible)Todas las imágenes que estén debajo del "fold" (lo que no se ve al cargar)
eagerSe descarga inmediatamente (comportamiento por defecto)Imágenes críticas que deben aparecer al instante: el logo, la imagen hero
autoEl navegador decide (equivale a no poner el atributo)No recomendado: mejor ser explícito

⚠️ Nunca pongas lazy en la imagen hero o el logo

Las imágenes que aparecen en la parte visible inmediatamente al cargar la página (arriba del "fold") nunca deben tener loading="lazy". Si lo haces, el navegador las retrasará y verás un parpadeo feo al cargar. El lazy loading es solo para imágenes que están abajo en la página.

🟢 Siempre define width y height en las imágenes con lazy

Cuando usas lazy loading, el navegador necesita saber el tamaño de la imagen para reservar el espacio en el layout antes de descargarla. Sin width y height, la página "saltará" visualmente cuando la imagen cargue.

HTML<!-- ✅ Correcto: tiene dimensiones para que el navegador reserve el espacio --> <img src="proyecto-1.jpg" alt="Captura del proyecto de portafolio web" width="800" height="450" loading="lazy" >

HTML Deprecated — Etiquetas que ya no debes usar
Lo que HTML5 eliminó y por qué — con su alternativa moderna

A lo largo de la historia de HTML, muchas etiquetas fueron deprecadas (marcadas como obsoletas). Esto significa que todavía pueden funcionar en algunos navegadores, pero no debes usarlas en código nuevo porque pueden dejar de funcionar en cualquier momento y van en contra de las buenas prácticas modernas.

📼 Analogía: el videocasete VHS

Un VHS todavía "funciona" si tienes una reproductora vieja. Pero nadie hace películas nuevas en VHS porque hay mejores formatos: Blu-ray, streaming. Las etiquetas deprecated son como el VHS: técnicamente podrían funcionar en algún lugar, pero son tecnología del pasado que no debes usar para cosas nuevas.

Las etiquetas deprecated más comunes — y su reemplazo

❌ Deprecada
<center>
Centraba texto o elementos horizontalmente
✅ Alternativa moderna
CSS text-align / margin: auto
text-align: center para texto, margin: 0 auto para bloques, o Flexbox con justify-content: center
❌ Deprecada
<font>
Cambiaba la fuente, tamaño y color del texto con atributos HTML
✅ Alternativa moderna
CSS font-family, color, font-size
Todo el estilo de tipografía se maneja con CSS. HTML solo estructura, CSS decora.
❌ Deprecada
<marquee>
Creaba texto desplazándose horizontalmente (el famoso "ticker")
✅ Alternativa moderna
CSS animations / JS
Animaciones CSS con @keyframes o JavaScript para efectos similares con mucho más control.
❌ Deprecada
<blink>
Hacía parpadear el texto (era horrible). Solo funcionaba en Netscape.
✅ Alternativa moderna
CSS animation opacity
animation: blink 1s step-end infinite con @keyframes que alterna opacity entre 0 y 1.
❌ Deprecada
<b> y <i> (semánticamente)
Originalmente solo eran estilos visuales: negrita y cursiva sin significado semántico
✅ Alternativa moderna
<strong> y <em>
<strong> = importante (negrita + significado). <em> = énfasis (cursiva + significado). Los lectores de pantalla los pronuncian diferente.
❌ Deprecada
<frame> / <frameset>
Dividía la ventana del navegador en secciones independientes con URLs propias
✅ Alternativa moderna
<iframe> + CSS Grid/Flex
<iframe> para incrustar páginas externas. CSS Grid/Flex para dividir visualmente la página.

💡 La regla de oro de HTML moderno

HTML estructura, CSS decora. Casi todas las etiquetas deprecated existían porque en los años 90 HTML hacía también el trabajo visual. Con CSS moderno, toda la presentación (colores, fuentes, posición, animaciones) se maneja en CSS. HTML solo debe describir el significado del contenido, nunca su apariencia.

Accesibilidad Web — Diseñar para todos
alt, ARIA, roles, tabindex y las buenas prácticas que marcan la diferencia

La accesibilidad web (a11y) es diseñar y programar páginas que puedan ser usadas por todas las personas, incluyendo aquellas con discapacidades visuales, auditivas, motoras o cognitivas. No es solo ético: en muchos países es un requisito legal, y además mejora el SEO y la experiencia de todos los usuarios.

🏛️ Analogía: las rampas del edificio

Cuando un edificio tiene escaleras, una persona en silla de ruedas no puede entrar. La rampa es accesibilidad: permite que todos entren por la misma puerta. Y resulta que la rampa también la usan personas con carriolas, repartidores con carros, personas mayores. La accesibilidad web es exactamente así: hace el sitio mejor para todos, no solo para quienes tienen discapacidades.

El atributo alt — descripción de imágenes

El atributo alt es el más básico y más ignorado de la accesibilidad. Un lector de pantalla (software que usan personas con discapacidad visual) lee el alt en voz alta cuando encuentra una imagen.

HTML<!-- ❌ MAL: alt vacío en imagen importante --> <img src="equipo.jpg" alt=""> <!-- El lector de pantalla la ignora --> <img src="equipo.jpg"> <!-- Sin alt: lee la ruta del archivo (horrible) --> <img src="equipo.jpg" alt="imagen"> <!-- ❌ MAL: no describe nada --> <!-- ✅ BIEN: describe lo que hay en la imagen, con contexto --> <img src="equipo.jpg" alt="Equipo de América celebrando el campeonato 2025, con el trofeo al centro"> <!-- ✅ Imagen decorativa: alt vacío adrede (el lector de pantalla la omite) --> <img src="separador.png" alt="" role="presentation">

ARIA — Accessible Rich Internet Applications

ARIA es un conjunto de atributos que agregan información semántica a elementos HTML para que los lectores de pantalla entiendan mejor la interfaz. Se usa cuando el HTML semántico por sí solo no es suficiente.

aria-label

Da un nombre descriptivo a un elemento cuando no tiene texto visible.

<button aria-label="Cerrar menú"> ✕ </button> <!-- Sin aria-label, el lector diría solo "✕" -->
aria-hidden

Oculta un elemento de los lectores de pantalla (para íconos decorativos).

<i class="fa-star" aria-hidden="true"></i> <span>Favorito</span> <!-- Solo "Favorito" se anuncia, el ícono no -->
aria-expanded

Indica si un menú, acordeón o dropdown está abierto o cerrado.

<button aria-expanded="false" aria-controls="menu"> Menú </button> <ul id="menu" hidden>...</ul>
aria-describedby

Apunta a otro elemento que describe el elemento actual en detalle.

<input type="password" aria-describedby="ayuda-pass"> <p id="ayuda-pass"> Mínimo 8 caracteres </p>
aria-live

Anuncia cambios dinámicos del contenido (notificaciones, errores en vivo).

<div aria-live="polite"> <!-- El lector anuncia cuando este contenido cambia --> Cargando resultados... </div>
role

Define el rol semántico de un elemento cuando no es el correcto por defecto.

<div role="button" tabindex="0"> Hacer clic </div> <!-- Aunque es un div, se comporta como un botón -->

tabindex — Navegación con teclado

Muchos usuarios navegan con el teclado (Tab para moverse, Enter para activar). Por defecto, solo los elementos interactivos nativos (links, inputs, botones) son alcanzables con Tab. Con tabindex puedes controlarlo:

HTML<!-- tabindex="0": entra en el flujo natural del teclado --> <div role="button" tabindex="0">Elemento clickeable con teclado</div> <!-- tabindex="-1": NO entra en flujo del teclado, pero JS puede enfocarlo --> <div tabindex="-1" id="modal">Modal que recibe foco por JS</div> <!-- tabindex="1", "2", etc.: define el ORDEN del Tab (evitar — dificulta accesibilidad) --> <input tabindex="2"> <!-- ❌ Evitar: rompe el orden natural -->

✏️ Ejercicio — Página de jugador favorito Intermedio

Crea una página llamada jugador.html sobre tu jugador de fútbol favorito que use todos los elementos que aprendiste en esta parte.

  1. Header y navegación: Un <header> con el nombre del jugador y una <nav> con anclas (#estadisticas, #galeria, #videos) que salten a cada sección.
  2. Links de contacto/redes: En el header o footer, incluye al menos:
    • Un link al Instagram del jugador con target="_blank" rel="noopener noreferrer"
    • Un link de mailto: para "contactar al equipo"
    • Un link de download para "descargar estadísticas" (puede ser cualquier archivo)
  3. Tabla de estadísticas (id="estadisticas"): Crea una tabla con <thead>, <tbody> y <tfoot> mostrando temporadas: Año, Equipo, Partidos, Goles, Asistencias. En el <tfoot> pon los totales de carrera con colspan.
  4. Galería con lazy loading (id="galeria"): Al menos 4 imágenes del jugador con:
    • loading="lazy" en todas
    • alt descriptivo en cada una
    • width y height definidos
  5. Video (id="videos"): Embebe un video de YouTube del jugador usando <iframe> con loading="lazy" y el title correcto.
  6. Accesibilidad: Revisa que todos tus inputs tengan <label>, todas las imágenes tengan alt, el <html> tenga lang="es", y cualquier botón de ícono tenga aria-label.

Bonus: Agrega un elemento <audio> con el himno del equipo del jugador. Si no tienes el archivo, puedes usar cualquier MP3 de ejemplo.

🟢 Resumen de lo que aprendiste en la Parte B

  • Links avanzados: target, rel, download, mailto, tel, anclas con #id
  • Tablas: table, caption, thead, tbody, tfoot, tr, th, td, colspan, rowspan
  • Audio: <audio> con <source>, controls, autoplay, loop, muted, preload
  • Video: <video> con poster, playsinline; <iframe> para YouTube; video de fondo hero
  • Lazy Loading: atributo loading="lazy" en img e iframe
  • HTML Deprecated: center, font, marquee, blink, frameset y sus reemplazos
  • Accesibilidad: alt descriptivo, ARIA (aria-label, aria-hidden, aria-expanded, aria-live, role), tabindex, HTML semántico
