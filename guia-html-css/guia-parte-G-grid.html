CSS Grid — El sistema de layout bidimensional
Filas Y columnas al mismo tiempo: el poder que Flexbox no tiene

CSS Grid es el sistema de layout más poderoso de CSS. A diferencia de Flexbox (que trabaja en una dimensión: fila o columna), Grid trabaja en dos dimensiones simultáneamente: filas y columnas al mismo tiempo. Es ideal para layouts de página completa, galerías, dashboards y cualquier diseño con estructura bidimensional.

📰 Analogía: el tablero de ajedrez vs el tren

Flexbox es como un tren: los vagones van en una fila (o una columna). Grid es como un tablero de ajedrez: tienes filas Y columnas, y puedes colocar cada pieza en la intersección exacta que quieras, e incluso hacer que una pieza ocupe varias casillas a la vez. Cada cuadrícula del tablero tiene una dirección tanto horizontal como vertical.

Grid vs Flexbox — ¿cuándo usar cuál?

Usa Flexbox cuando…

  • El layout es en una dirección (fila o columna)
  • Necesitas distribuir espacio entre elementos
  • El tamaño del contenido determina el layout
  • Ejemplos: navbar, grupo de botones, lista de tags, input group

Usa Grid cuando…

  • El layout es en dos dimensiones (filas Y columnas)
  • El layout determina el tamaño del contenido
  • Necesitas alinear elementos en ambos ejes
  • Ejemplos: layouts de página, galerías, dashboards, formularios complejos

Activar Grid y los dos mundos

/* Propiedades del CONTENEDOR GRID */
.padre {
  display: grid;Activa Grid
  grid-template-columns: 1fr 1fr 1fr;Define las columnas
  grid-template-rows: auto 200px auto;Define las filas
  grid-template-areas: "header header";Nombra las áreas
  gap: 1rem;Espacio entre celdas
  justify-items: stretch;Alineación horizontal de celdas
  align-items: stretch;Alineación vertical de celdas
}

/* Propiedades de los HIJOS (grid items) */
.hijo {
  grid-column: 1 / 3;De línea 1 a línea 3
  grid-row: 2 / 4;De línea 2 a línea 4
  grid-area: header;Ocupa el área nombrada
  justify-self: center;Alineación horizontal de esta celda
  align-self: end;Alineación vertical de esta celda
}

grid-template-columns — Definir las columnas
La propiedad más importante de Grid: cuántas columnas y cuánto miden

grid-template-columns define cuántas columnas tiene el grid y cuánto mide cada una. Cada valor que escribas crea una columna. Puedes mezclar cualquier unidad: píxeles, porcentajes, fr, auto, minmax…

CSS.grid { display: grid; } /* 3 columnas iguales de 200px cada una */ .grid { grid-template-columns: 200px 200px 200px; } /* 3 columnas de ancho flexible igual */ .grid { grid-template-columns: 1fr 1fr 1fr; } /* Sidebar fijo + contenido flexible */ .grid { grid-template-columns: 260px 1fr; } /* 3 columnas mixtas */ .grid { grid-template-columns: 200px 1fr 2fr; } /* auto: tan ancho como su contenido */ .grid { grid-template-columns: auto 1fr auto; }

La unidad fr — Fracción del espacio disponible
La unidad estrella de CSS Grid que no existe en ningún otro contexto

La unidad fr (fracción) es exclusiva de CSS Grid. Representa una fracción del espacio disponible en el contenedor, después de restar los elementos con tamaño fijo (px, %) y los gaps. Es como el flex-grow de Flexbox pero aplicado a columnas y filas.

🍰 Analogía: repartir un pastel

Tienes un pastel (el espacio del grid). Si dices 1fr 1fr, cortas el pastel en 2 partes iguales. Si dices 1fr 2fr, una persona recibe 1/3 y la otra 2/3. Si dices 200px 1fr, primero apartas 200px del pastel (para el sidebar), y lo que sobra se lo llevas completo a la columna de 1fr. Los fr se reparten lo que queda.

CSS/* Contenedor de 900px con gap de 30px total */ /* grid-template-columns: 1fr 2fr 1fr */ /* Espacio disponible: 900 - 30(gaps) = 870px */ /* Total fr: 1+2+1 = 4 */ /* 1fr = 870/4 = 217.5px */ /* 2fr = 435px, 1fr = 217.5px */ /* Con elemento fijo: 200px 1fr 1fr */ /* Espacio disponible: 900 - 200 - gaps = 670px */ /* Cada 1fr = 670/2 = 335px */ /* El fr NUNCA puede ser menor que min-content */ /* Si el contenido es más ancho que el fr calculado, */ /* el fr crece para acomodarlo */ .grid { grid-template-columns: 1fr 1fr 1fr; } /* Igual a minmax(0, 1fr) */

🟢 fr vs porcentaje — ¿cuál usar?

Los porcentajes no descuentan el gap: si tienes 3 columnas de 33.33% con gap de 1rem, el total supera el 100% y se desborda. Los fr sí descuentan el gap automáticamente, por eso son superiores. Siempre prefiere fr sobre porcentajes en Grid.

🔁
repeat() — No repitas código
La función que genera columnas o filas automáticamente

Escribir 1fr 1fr 1fr 1fr 1fr 1fr 1fr 1fr 1fr 1fr 1fr 1fr para doce columnas iguales es horrible. repeat() lo hace por ti en una sola expresión limpia.

CSS/* Sin repeat — repetitivo y difícil de mantener */ .grid { grid-template-columns: 1fr 1fr 1fr 1fr; } /* Con repeat — limpio y fácil de cambiar */ .grid { grid-template-columns: repeat(4, 1fr); } /* repeat(cantidad, patrón) */ .grid { grid-template-columns: repeat(3, 200px); } .grid { grid-template-columns: repeat(4, 1fr); } .grid { grid-template-columns: repeat(2, 1fr 2fr); } /* = 1fr 2fr 1fr 2fr */ /* Mezclar repeat con otros valores */ .grid { grid-template-columns: 200px repeat(3, 1fr); } /* sidebar + 3 columnas */ .grid { grid-template-columns: repeat(3, 1fr) 100px; } /* 3 columnas + aside */

minmax() — Columnas con límites flexible
Define un rango: "no menos de X, no más de Y"

minmax(mínimo, máximo) define un rango de tamaño para una columna o fila. La columna puede crecer hasta el máximo y encogerse hasta el mínimo, pero nunca salir de ese rango. Es la clave para grids verdaderamente responsive.

👚 Analogía: la talla S-M de una camiseta

Una camiseta de talla S-M sirve a personas entre 60kg y 80kg. No es rígida para un tamaño exacto, sino que tiene un rango flexible. minmax(150px, 1fr) hace lo mismo: la columna no puede ser menor de 150px (se rompe el texto si es más chica) ni mayor de 1fr del espacio disponible (no necesita crecer más).

CSS/* Columna que mide mínimo 100px, máximo 300px */ .grid { grid-template-columns: minmax(100px, 300px); } /* El uso más importante: mínimo fijo + máximo flexible */ .grid { grid-template-columns: repeat(3, minmax(150px, 1fr)); } /* Con auto como mínimo: mínimo = ancho del contenido */ .grid { grid-template-columns: minmax(auto, 1fr); } /* Con auto como máximo: crece pero no más de su contenido */ .grid { grid-template-columns: minmax(200px, auto); } /* fit-content(valor): como max-content pero limitado al valor */ .grid { grid-template-columns: fit-content(300px) 1fr; }

auto-fill y auto-fit — El grid responsive sin media queries
El patrón más poderoso de Grid: columnas que se adaptan solas

Combinando repeat(), auto-fill o auto-fit con minmax(), obtienes un grid que automáticamente decide cuántas columnas caben según el ancho disponible. En pantalla grande caben 4 columnas; en móvil, 1. Sin escribir una sola media query.

🚗 Analogía: el estacionamiento inteligente

Un estacionamiento inteligente calcula automáticamente cuántos autos caben según el ancho disponible: en un terreno grande caben 6 autos en fila; en uno pequeño, 2. No necesita que alguien le diga cuántos caben: lo calcula solo. Así funciona auto-fill y auto-fit con minmax().

CSS/* La receta mágica — responsive sin media queries */ .grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); gap: 1rem; } /* En 900px: caben 4 columnas de ~200px */ /* En 600px: caben 2 columnas de ~280px */ /* En 300px: cabe 1 columna de 300px */

auto-fill vs auto-fit — la diferencia sutil

CSS/* auto-fill: llena el grid con columnas aunque estén vacías */ .grid-fill { grid-template-columns: repeat(auto-fill, minmax(100px, 1fr)); } /* Con 3 items en un espacio para 5: los 2 últimos slots quedan vacíos pero existen */ /* Resultado: los 3 items NO se estiran para llenar el espacio */ /* auto-fit: solo crea columnas para los items que existen */ .grid-fit { grid-template-columns: repeat(auto-fit, minmax(100px, 1fr)); } /* Con 3 items: los 3 items SE ESTIRAN para llenar todo el ancho */ /* Resultado: 3 columnas que juntas llenan el 100% del contenedor */

🟢 El one-liner más útil de CSS Grid

Esta sola línea crea un grid completamente responsive sin media queries. Copia y pega en todos tus proyectos:

CSS/* Cambia 250px por el ancho mínimo que necesites para tus cards */ .cards { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); gap: 1.5rem; }

grid-template-rows y grid-auto-rows
Controlar el alto de las filas del grid

Igual que defines columnas con grid-template-columns, puedes definir filas con grid-template-rows. Pero en la mayoría de casos las filas se crean automáticamente y solo necesitas controlar su alto con grid-auto-rows.

CSS/* Filas explícitas con alturas específicas */ .grid { grid-template-rows: 80px 1fr 60px; /* header + main + footer */ } /* grid-auto-rows: alto por defecto para filas creadas automáticamente */ .grid { grid-auto-rows: 200px; /* Cada fila generada mide 200px */ grid-auto-rows: minmax(100px, auto); /* Mínimo 100px, crece con el contenido */ } /* grid-auto-columns: para columnas implícitas (generadas automáticamente) */ .grid { grid-auto-columns: 1fr; } /* grid-auto-flow: dirección en que fluyen los items no colocados */ .grid { grid-auto-flow: row; } /* Por defecto: llena filas primero */ .grid { grid-auto-flow: column; } /* Llena columnas primero */ .grid { grid-auto-flow: dense; } /* Llena huecos automáticamente (galería masonry-like) */

span — Hacer que un elemento ocupe varias celdas
El equivalente de colspan/rowspan de las tablas, pero más poderoso

Con grid-column: span N o grid-row: span N haces que un grid item ocupe N columnas o filas. Es como el colspan y rowspan de las tablas HTML, pero aplicado a cualquier elemento con mucho más control.

CSS/* span en columnas */ .elemento { grid-column: span 2; } /* Ocupa 2 columnas */ .elemento { grid-column: span 3; } /* Ocupa 3 columnas (el ancho total si hay 3) */ /* span en filas */ .elemento { grid-row: span 2; } /* Ocupa 2 filas de alto */ /* span en ambas direcciones — elemento cuadrado grande */ .featured { grid-column: span 2; grid-row: span 2; }

Colocación explícita — grid-column y grid-row con líneas
Colocar cada elemento exactamente en la celda que quieres

Además de span, puedes colocar elementos usando los números de línea de grid. CSS Grid numera automáticamente todas las líneas de columnas y filas empezando desde 1. Con grid-column: inicio / fin dices "empieza en la línea X y termina en la línea Y".

🗺️ Analogía: las coordenadas de un mapa

En un mapa de ciudad, cada intersección tiene coordenadas. Si dices "empieza en la Calle 1 y termina en la Calle 3, entre la Avenida 2 y la Avenida 4", describes perfectamente un terreno. Eso hace grid-column: 1 / 3; grid-row: 2 / 4: describe exactamente qué espacio del grid ocupa ese elemento.

CSS/* Grid de 4 columnas — las líneas van de 1 a 5 */ /* 1 2 3 4 5 (líneas de columna) */ /* | | | | | */ /* [col1][col2][col3][col4] */ /* Ocupa de la línea 1 a la 3 (columnas 1 y 2) */ .elemento { grid-column: 1 / 3; } /* Shorthand: start / end */ .elemento { grid-column-start: 1; grid-column-end: 3; } /* = 1/3 */ /* -1 es la última línea (útil para "hasta el final") */ .full-width { grid-column: 1 / -1; } /* Ocupa todas las columnas */ /* Combinar filas y columnas */ .hero-img { grid-column: 1 / 3; /* Columnas 1 y 2 */ grid-row: 1 / 3; /* Filas 1 y 2 */ }

grid-template-areas — El layout visual más legible de CSS
Dibuja el layout con texto, como si fuera un mapa

grid-template-areas te permite definir el layout de tu página dibujándolo textualmente. En vez de coordenadas numéricas, describes visualmente cómo se distribuyen las áreas. Es el sistema más legible y mantenible para layouts de página completa.

🏠 Analogía: el plano de un arquitecto

Un arquitecto dibuja el plano de una casa: "aquí va la cocina, aquí el salón, aquí dos habitaciones". grid-template-areas hace exactamente eso con CSS: dibujas el plano del layout con texto y luego cada elemento HTML dice "yo soy la cocina" (grid-area: cocina). El resultado es un CSS que cualquiera puede leer y entender en segundos.

CSS.layout { display: grid; grid-template-columns: 200px 1fr; grid-template-rows: 60px 1fr 50px; grid-template-areas: "header header" /* Fila 1: header ocupa las 2 columnas */ "sidebar main" /* Fila 2: sidebar izq + main der */ "footer footer"; /* Fila 3: footer ocupa las 2 columnas */ gap: .5rem; min-height: 400px; } /* Cada elemento HTML usa grid-area para "ocupar" su área */ header { grid-area: header; } aside { grid-area: sidebar; } main { grid-area: main; } footer { grid-area: footer; } /* Para dejar una celda vacía, usa un punto */ .layout2 { grid-template-areas: "header header header" "sidebar main ." /* El punto = celda vacía */ "footer footer footer"; } /* Responsive: cambiar áreas con media query */ @media (max-width: 768px) { .layout { grid-template-columns: 1fr; grid-template-areas: "header" "main" "sidebar" "footer"; } }

Alineación en Grid — justify e align
Controlar la posición del contenido dentro de las celdas

Grid tiene dos niveles de alineación: el contenido dentro de las celdas (justify-items/align-items) y las celdas dentro del contenedor (justify-content/align-content). Funcionan igual que en Flexbox pero con un nivel más de control.

CSS/* ── En el CONTENEDOR ── */ /* Alinea el contenido DENTRO de cada celda */ .grid { justify-items: start | end | center | stretch; } /* horizontal */ .grid { align-items: start | end | center | stretch; } /* vertical */ /* Shorthand: place-items: align justify */ .grid { place-items: center; } /* = align-items:center + justify-items:center */ /* Distribuye las COLUMNAS/FILAS dentro del contenedor (si hay espacio sobrante) */ .grid { justify-content: start | end | center | space-between | space-around; } .grid { align-content: start | end | center | space-between; } /* Shorthand: place-content: align justify */ .grid { place-content: center; } /* ── En cada HIJO individualmente ── */ .item { justify-self: start | end | center | stretch; } /* horizontal */ .item { align-self: start | end | center | stretch; } /* vertical */ /* Shorthand: place-self: align justify */ .item { place-self: center end; }

Ejercicio integrador — Portal deportivo con Grid
Construye un layout de portal completo usando CSS Grid
✏️ Ejercicio — Portal de noticias deportivas con Grid completo Avanzado

Crea portal-deportivo.html usando CSS Grid para todo el layout. No uses Flexbox para el layout de página (solo para componentes internos pequeños como botones y navbars).

  1. Layout de página con grid-template-areas: Define las áreas header, nav, featured, sidebar, articles y footer. En desktop: featured y articles forman el contenido central con sidebar a la derecha. En móvil (media query): todo en una columna.
  2. Sección "Noticias destacadas" con colocación explícita: Un grid de 3 columnas y 2 filas donde la noticia principal ocupa grid-column: 1 / 3; grid-row: 1 / 3, y las noticias secundarias ocupan el resto con span.
  3. Galería de goles con grid-auto-flow: dense: Mínimo 8 fotos donde algunas son tall (span 2 filas) y otras wide (span 2 columnas). El dense rellena los huecos automáticamente.
  4. Dashboard de estadísticas: Un grid de 4 columnas con tarjetas de stats, un gráfico grande (span 2x2), un banner ancho (span 4) y tarjetas medianas (span 2).
  5. Grid de partidos responsive sin media queries: Usa repeat(auto-fit, minmax(220px, 1fr)) para una grilla de tarjetas de partidos que se adapta sola.
  6. Alineación con place-items: En el header usa place-items: center para centrar el contenido. En las tarjetas de stats usa align-items: end para que el texto quede abajo.

Bonus: Agrega una transición responsive usando @media (max-width: 768px) que cambie el grid-template-areas para que en móvil el sidebar quede debajo del contenido, y el orden cambie a: header → nav → featured → articles → sidebar → footer.

🟢 Resumen de lo que aprendiste en la Parte G

  • display: grid — activa Grid, los hijos son grid items
  • grid-template-columns/rows — define columnas y filas con cualquier unidad
  • fr — fracción del espacio disponible, descuenta gaps automáticamente
  • repeat(n, valor) — crea n columnas/filas iguales sin repetir código
  • minmax(min, max) — rango flexible para columnas y filas
  • auto-fill — llena el grid con columnas aunque queden vacías
  • auto-fit — solo crea columnas para los items existentes (los estira)
  • repeat(auto-fit, minmax(250px, 1fr)) — el one-liner responsive sin media queries
  • grid-auto-rows/columns — alto/ancho de filas/columnas implícitas
  • grid-auto-flow: dense — rellena huecos en galerías
  • grid-column/row: span N — el elemento ocupa N columnas/filas
  • grid-column: 1 / 3 — colocación explícita con líneas de grid
  • grid-column: 1 / -1 — hasta la última línea (ancho total)
  • grid-template-areas — dibuja el layout con texto, asigna con grid-area
  • justify/align-items — alineación dentro de las celdas
  • justify/align-content — distribución de las celdas en el contenedor
  • place-items / place-self / place-content — shorthands de alineación
