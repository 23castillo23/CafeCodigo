Position — Sacar elementos del flujo normal
El sistema de posicionamiento de CSS: static, relative, absolute, fixed, sticky

Por defecto, los elementos HTML se colocan uno debajo del otro en el orden en que aparecen en el código. Eso se llama flujo normal del documento. La propiedad position te permite romper ese flujo y colocar elementos exactamente donde quieras, incluso superponiéndolos.

🏗️ Analogía: los pisos de un edificio en construcción

En el flujo normal, los elementos son como habitaciones en un plano: cada una ocupa su espacio y la siguiente va debajo. Con position tienes cuatro herramientas distintas: una mesa que puedes mover un poco sin salir del plano (relative), un mueble que puedes anclar en cualquier rincón de la habitación (absolute), un letrero que se pega al techo y siempre está visible sin importar dónde estés (fixed), y un separador que te acompaña mientras haces scroll pero que tiene un límite (sticky).

Los 5 valores de position

Valor¿Está en el flujo?¿Qué referencia usa top/left/right/bottom?Cuándo usarlo
static ✅ Sí (por defecto) No aplica — top/left no funcionan Comportamiento normal. No necesitas escribirlo explícitamente.
relative ✅ Sí (reserva su espacio) Su propia posición original Mover el elemento ligeramente sin afectar a los demás. O crear un contexto para hijos absolute.
absolute ❌ No (sale del flujo) El ancestro posicionado más cercano (que tenga position ≠ static) Badges, tooltips, menús desplegables, íconos dentro de tarjetas.
fixed ❌ No (sale del flujo) La ventana del navegador (viewport) Navbar fija, botón "volver arriba", chat flotante, banners de cookies.
sticky ✅ Sí (hasta que llega al umbral) Su contenedor padre scroll Encabezados de sección que se quedan fijos al hacer scroll dentro de un contenedor.

💡 Las propiedades de desplazamiento: top, right, bottom, left

Estas cuatro propiedades solo funcionan cuando el elemento tiene un position diferente a static. Definen la distancia desde cada borde: top: 20px significa "20px desde el borde superior de la referencia". Puedes usar valores negativos para mover en dirección opuesta.

↕️
position: relative — El de doble personalidad
Se mueve visualmente pero su espacio original sigue reservado

position: relative mueve el elemento visualmente respecto a donde estaría normalmente, pero el espacio que ocupaba en el flujo queda reservado. Los demás elementos no se mueven para llenarlo. Es como si el elemento se volviera invisible en su lugar original y apareciera en otro lugar.

🪑 Analogía: levantarse de la silla pero dejar el abrigo

En un cine concurrido, si te levantas de tu asiento nadie se sienta ahí porque dejaste el abrigo. Te moviste tú, pero tu espacio sigue ocupado. Eso es position: relative: el elemento se mueve visualmente, pero su "abrigo" (el espacio original) sigue ahí.

CSS.caja-movida { position: relative; top: 20px; /* Baja 20px desde donde estaría normalmente */ left: 15px; /* Se mueve 15px a la derecha */ }
Observa el espacio reservado — la caja naranja se movió pero su hueco sigue ahí:

El uso más importante de relative: crear contexto para absolute

Este es el uso de position: relative que más vas a usar en el mundo real. Cuando un hijo tiene position: absolute, busca su referencia en el ancestro posicionado más cercano. Si no hay ninguno, usa el <body>. Al poner position: relative en el padre, le dices "usa a mí como referencia":

HTML + CSS/* ❌ Sin relative en el padre: el badge se posiciona respecto al body */ <div class="tarjeta"> <!-- sin position --> <span class="badge">Nuevo</span> <!-- position:absolute → se va al body --> </div> /* ✅ Con relative en el padre: el badge se posiciona respecto a la tarjeta */ .tarjeta { position: relative; } /* ← El padre es la referencia */ .badge { position: absolute; top: 10px; right: 10px; background: red; color: white; border-radius: 20px; padding: 2px 10px; }
📌
position: absolute — El elemento que se sale del mundo
Sale del flujo normal y se ancla a su ancestro posicionado

position: absolute saca el elemento completamente del flujo normal: los demás elementos actúan como si no existiera, y el espacio que ocupaba desaparece. El elemento flota encima del contenido y se posiciona respecto a su ancestro con position distinto de static.

🪧 Analogía: el letrero de "Se renta" en una vitrina

Un letrero de "Se renta" no ocupa espacio dentro del local — está pegado en el vidrio de la vitrina. Puedes verlo sobre el fondo, pero el espacio de la vitrina no se reduce por él. Eso es position: absolute: flota sobre el contenido en una posición exacta sin afectar a nadie.

El patrón padre relative + hijo absolute

HTML + CSS/* HTML */ <div class="producto-card"> <img src="zapato.jpg" alt="Zapato deportivo"> <span class="badge-oferta">-30%</span> <!-- Esquina superior derecha --> <button class="btn-favorito">♡</button> <!-- Esquina inferior derecha --> <h3>Air Max 2025</h3> </div> /* CSS */ .producto-card { position: relative; /* ← Contexto para los hijos absolute */ border-radius: 12px; overflow: hidden; } .badge-oferta { position: absolute; top: 12px; right: 12px; /* ← Respecto a .producto-card */ background: var(--primary); color: white; border-radius: 20px; padding: 4px 12px; font-weight: 700; } .btn-favorito { position: absolute; bottom: 12px; right: 12px; background: white; border: none; border-radius: 50%; width: 36px; height: 36px; cursor: pointer; font-size: 1.1rem; }

Centrado perfecto con absolute

CSS/* Técnica clásica para centrar un elemento absolutamente */ .centrado { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); /* top:50% + left:50% lleva la esquina superior izquierda al centro */ /* transform:-50%,-50% corrige moviendo el elemento hacia atrás */ }

position: fixed y sticky — Elementos que persisten al hacer scroll
Navbars fijas, sidebars que siguen al usuario, encabezados de tabla

position: fixed — Siempre visible, siempre en el mismo lugar

position: fixed ancla el elemento al viewport (la ventana del navegador), no al documento. No importa cuánto hagas scroll: el elemento siempre aparece exactamente en la misma posición de la pantalla.

🖼️ Analogía: el marco de una foto vs la foto

El documento HTML es como una foto muy larga que puedes enrollar hacia arriba. position: fixed es como el marco: siempre está en el mismo lugar en la pared (tu pantalla), sin importar cuánto subas o bajes el papel de la foto.

CSS/* Navbar fija en la parte superior */ .navbar { position: fixed; top: 0; left: 0; width: 100%; /* Ocupa todo el ancho */ z-index: 1000; /* Encima de todo lo demás */ background: white; box-shadow: 0 2px 10px rgba(0,0,0,.1); } /* ⚠️ Importante: añade padding-top al body para que el contenido no quede tapado por la navbar fija */ body { padding-top: 70px; } /* Mismo alto que la navbar */ /* Botón flotante de WhatsApp (abajo a la derecha) */ .whatsapp-btn { position: fixed; bottom: 24px; right: 24px; z-index: 999; background: #25D366; color: white; border-radius: 50%; width: 56px; height: 56px; font-size: 1.5rem; }

position: sticky — El punto medio entre relative y fixed

position: sticky es el más reciente e interesante. El elemento se comporta como relative hasta que el usuario hace scroll y llega al umbral definido (por ejemplo top: 0); en ese momento se "pega" como fixed y se queda ahí mientras sigue el scroll. Cuando el scroll llega al final del contenedor padre, el elemento vuelve a su lugar.

🧲 Analogía: el imán de un refrigerador con límite

Imagina un imán en una regleta vertical. Mientras está en la parte de la regleta que está visible, se mueve con ella (como relative). Pero cuando llega al borde superior del refrigerador, se queda pegado ahí (como fixed). Y cuando la regleta termina, el imán ya no puede seguir pegado y sube con el contenedor.

CSS/* Encabezado de sección sticky — se pega al top al hacer scroll */ .seccion-header { position: sticky; top: 0; /* Se pega cuando llega a 0px del tope */ background: white; /* Necesita fondo para no transparentarse */ z-index: 10; padding: .75rem 1rem; border-bottom: 1px solid #eee; } /* Sidebar sticky que acompaña el scroll del artículo */ .sidebar { position: sticky; top: 80px; /* Se pega 80px debajo del tope (debajo de la navbar) */ height: fit-content; /* Solo tan alto como su contenido */ }

⚠️ El error más común con sticky

Para que position: sticky funcione, el elemento debe tener espacio para hacer scroll dentro de su contenedor padre. Si el padre tiene overflow: hidden o overflow: auto, el sticky no funciona. Además, el padre debe ser más alto que el elemento sticky.

z-index — El orden de apilamiento
Quién queda encima de quién cuando los elementos se superponen

Cuando varios elementos posicionados (non-static) se superponen, el z-index determina cuál queda encima. Mayor número = más adelante. Solo funciona en elementos con position diferente a static.

🃏 Analogía: una baraja de cartas boca arriba

Cuando pones varias cartas encima de la mesa superpuestas, la que está más "arriba del montón" es la que más ves. En el mundo del DOM, el z-index es el orden del montón: la carta con z-index más alto está encima de las demás. Si no especificas z-index, el orden lo decide el HTML (la que aparece más abajo en el código queda encima).

CSS/* Sin z-index: el orden en el HTML decide (el último escrito queda encima) */ .caja-a { position: absolute; top: 0; left: 0; } /* queda debajo */ .caja-b { position: absolute; top: 0; left: 0; } /* queda encima */ /* Con z-index: los números mandan */ .caja-a { position: absolute; z-index: 3; } /* Encima de todo */ .caja-b { position: absolute; z-index: 1; } /* Debajo */ /* Convención de z-index en proyectos reales */ /* Usar saltos de 10 o 100 para dejar espacio entre capas */ /* --z-base: 1 → elementos posicionados normales */ /* --z-dropdown: 100 → menús desplegables */ /* --z-sticky: 200 → headers sticky */ /* --z-modal: 1000 → ventanas modales */ /* --z-toast: 1100 → notificaciones flotantes */
Las cajas con z-index más alto quedan encima:
z-index: 1
(gris)
z-index: 2
(naranja)
z-index: 3
(rojo)

El rojo (z-index:3) está encima del naranja (z-index:2), que está encima del gris (z-index:1).

💡 Contexto de apilamiento — el concepto avanzado

Cuando un elemento tiene position + z-index diferente de auto, crea su propio contexto de apilamiento. Dentro de ese contexto, los z-index de sus hijos son independientes del resto de la página. Por eso a veces un z-index: 9999 no soluciona nada: si el padre tiene un z-index bajo, sus hijos no pueden salir de él sin importar qué z-index tengan.

Ventanas Modales — La aplicación real de position: fixed
Cómo construir un popup/dialog profesional con HTML, CSS y JavaScript mínimo

Un modal es una ventana que aparece encima del contenido de la página y bloquea la interacción con el fondo hasta que el usuario la cierra. Los usas a diario: confirmaciones de pago, formularios de login, alertas de cookies, galerías de fotos. Se construyen combinando position: fixed, z-index alto y un overlay semitransparente.

🚪 Analogía: la puerta de seguridad del banco

Cuando entras a una zona restringida del banco, hay una puerta de seguridad que se cierra detrás de ti y no puedes salir ni entrar hasta que el guardia te atiende. El modal es exactamente eso: bloquea todo lo demás de la página hasta que el usuario toma una decisión.

Estructura del modal

HTML<!-- Botón que abre el modal --> <button id="abrir-modal">Confirmar pedido</button> <!-- El overlay cubre toda la pantalla --> <div class="modal-overlay" id="modal"> <!-- La caja del modal va centrada dentro del overlay --> <div class="modal-box"> <button class="modal-cerrar" id="cerrar-modal">✕</button> <h2 class="modal-title">¿Confirmar el pedido?</h2> <p class="modal-body">Tu pedido de 3 artículos por $450 MXN será procesado.</p> <div class="modal-footer"> <button class="btn-ghost" id="cerrar-modal-2">Cancelar</button> <button class="btn-primary">Sí, confirmar</button> </div> </div> </div>
CSS/* El overlay cubre toda la pantalla */ .modal-overlay { display: none; /* Oculto por defecto */ position: fixed; inset: 0; /* = top:0; right:0; bottom:0; left:0 */ background: rgba(0,0,0,.55); backdrop-filter: blur(3px); /* Desenfoca el fondo (moderno) */ z-index: 1000; display: none; align-items: center; justify-content: center; } .modal-overlay.abierto { display: flex; } /* JS agrega esta clase */ /* La caja del modal con animación de entrada */ .modal-box { background: white; border-radius: 16px; padding: 2rem; max-width: 420px; width: 90%; position: relative; animation: modalEntrar .25s ease; } @keyframes modalEntrar { from { opacity: 0; transform: scale(.9) translateY(10px); } to { opacity: 1; transform: scale(1) translateY(0); } }
JavaScript// Solo 3 líneas de JS — todo lo demás es HTML y CSS const modal = document.getElementById('modal'); const abrir = document.getElementById('abrir-modal'); const cerrar = document.querySelectorAll('#cerrar-modal, #cerrar-modal-2'); abrir.addEventListener('click', () => modal.classList.add('abierto')); cerrar.forEach(btn => btn.addEventListener('click', () => modal.classList.remove('abierto'))); // Cerrar al hacer clic en el overlay (fuera del modal-box) modal.addEventListener('click', (e) => { if (e.target === modal) modal.classList.remove('abierto'); });

🟢 La propiedad inset — el atajo moderno

inset: 0 es equivalente a escribir top: 0; right: 0; bottom: 0; left: 0 en una sola línea. Perfecto para overlays que cubren toda la pantalla. También puedes usarlo parcialmente: inset: 0 0 auto 0 significa top+right+left en 0, bottom en auto.

Transition — Suavizar los cambios de estado
Del estado A al estado B con una animación fluida y sin JavaScript

transition le dice al navegador que, cuando una propiedad CSS cambie (por :hover, :focus, agregar una clase con JS, etc.), en vez de cambiar bruscamente, lo haga de forma gradual y suave durante un tiempo definido. Es la diferencia entre un interruptor y un dimmer de luz.

🌅 Analogía: el atardecer vs el apagador

Un apagador cambia la luz de día a oscuridad en un instante. Un atardecer hace la misma transición, pero en 30 minutos, suavemente. Las transitions de CSS son ese atardecer: el cambio existe, pero se hace de forma gradual y agradable a la vista.

Anatomía de transition

transition: propiedad duración timing retardo;
¿qué cambia? ¿cuánto tarda? ¿cómo acelera? ¿espera inicial?

/* Ejemplo: */
transition: background .3s ease 0s;
/* Múltiples propiedades separadas por coma */
transition: background .3s, transform .2s, box-shadow .3s;
/* O transicionar todo (cuidado con el rendimiento) */
transition: all .3s ease;

El timing-function — cómo acelera y frena la animación

El timing-function controla la curva de velocidad de la transición: ¿empieza rápido y frena? ¿va constante? ¿tiene rebote?

🟢 Propiedades que puedes (y no debes) transicionar

  • ✅ Baratas (GPU): transform, opacity — usa estas siempre que puedas
  • ⚠️ Costosas (CPU): width, height, top, left, margin, padding — pueden causar janking (frames perdidos) en páginas complejas
  • ❌ No transicionan: display, visibility (en parte), font-family

Overflow — Qué pasa cuando el contenido no cabe
Controlar el desbordamiento con visible, hidden, scroll y auto

Cuando el contenido de un elemento es más grande que su contenedor, ocurre desbordamiento (overflow). La propiedad overflow controla qué pasa en ese caso: ¿se muestra todo aunque salga? ¿se corta? ¿aparece una barra de scroll?

🥤 Analogía: verter líquido en un vaso

Si viertes demasiado líquido en un vaso, tienes cuatro opciones: dejar que se derrame (visible), usar un vaso mágico que lo corta sin derramar (hidden), que el vaso crezca y aparezca una agarradera lateral para verter más (scroll), o que el vaso solo crezca si lo necesita (auto).

ValorQué haceCuándo usarlo
visibleEl contenido se desborda fuera del contenedor (por defecto)Tooltips y dropdowns que deben salir del contenedor
hiddenEl contenido que se sale queda cortado y ocultoImágenes con border-radius, ocultar content de tabs/acordeones
scrollSiempre muestra scrollbar, aunque no lo necesiteCuando quieres que el layout no salte al aparecer la scrollbar
autoScrollbar solo aparece si el contenido no cabeLa opción más natural para la mayoría de casos
clipComo hidden pero no crea contexto de apilamientoCasos específicos donde hidden interfiere con sticky
CSS/* Separar overflow horizontal y vertical */ .tabla-responsive { overflow-x: auto; /* Scroll horizontal para tablas anchas */ overflow-y: visible; /* Vertical sin restricción */ } /* Imagen con esquinas redondeadas — overflow:hidden necesario */ .avatar { width: 80px; height: 80px; border-radius: 50%; overflow: hidden; /* Sin esto, la imagen se sale del círculo */ } /* Zona de scroll independiente — como un chat */ .chat-mensajes { height: 400px; overflow-y: auto; scroll-behavior: smooth; /* Scroll suave */ }

Control de flujo de texto — Truncar, romper y alinear
white-space, text-overflow, word-break, hyphens, text-wrap y más

El texto tiene su propio sistema de flujo: se envuelve automáticamente cuando llega al borde del contenedor, pero a veces necesitas controlarlo. ¿Quieres que nunca salte de línea? ¿Que se corte con puntos suspensivos? ¿Que corte las palabras largas? ¿Que el título quede visualmente equilibrado? Todo eso se controla con estas propiedades.

Truncar texto en una línea — el truco de los tres

CSS/* Las tres propiedades van siempre juntas para truncar en una línea */ .truncar { white-space: nowrap; /* 1. Prohíbe el salto de línea */ overflow: hidden; /* 2. Corta el texto que se sale */ text-overflow: ellipsis; /* 3. Agrega "..." al final */ }

Ejercicio integrador — Landing page de un estadio
Posicionamiento, modal, transitions, overflow y texto en un proyecto real
✏️ Ejercicio — Página de venta de boletos de estadio Avanzado

Crea un archivo estadio.html con todo el CSS en el mismo archivo. El objetivo es construir una landing page de venta de boletos para un partido de fútbol, aplicando todo lo de esta parte.

  1. Navbar fija (position: fixed): Un <header> con logo y navegación que siempre se quede en la parte superior. Recuerda agregar padding-top al body para que el contenido no quede tapado. Agrega una transition en el box-shadow para que aparezca suavemente.
  2. Sección hero con badge absolute: Un <section class="hero"> con position: relative que contenga:
    • Un badge position: absolute en la esquina superior derecha que diga "¡HOY!"
    • Un ícono de estadio position: absolute en la esquina inferior izquierda
  3. Tabla de precios con overflow: Una tabla de precios de zonas (General, Preferente, VIP, Palco) con overflow-x: auto en un contenedor, para que en móvil sea scrolleable horizontalmente.
  4. Lista de partidos con sticky header: Una sección <div> con altura fija y overflow-y: auto que muestre 10+ partidos. El encabezado de la lista debe ser position: sticky; top: 0.
  5. Tarjetas de jugadores con transitions: Al menos 4 tarjetas de jugadores del equipo. Cada una debe tener en :hover:
    • transform: translateY(-6px)
    • box-shadow que aparezca suavemente
    • El nombre del jugador con text-wrap: balance
  6. Modal de compra: Un botón "Comprar boletos" que abra un modal con position: fixed, overlay semitransparente con backdrop-filter: blur(4px), animación de entrada con @keyframes, y botones de confirmar/cancelar.
  7. Footer con z-index correcto: Asegúrate de que el modal (z-index: 1000) quede siempre por encima de la navbar (z-index: 100) y del contenido normal (z-index: 1).

Bonus: Agrega un botón flotante position: fixed; bottom: 24px; right: 24px con el texto "↑ Subir" que lleve al inicio de la página con <a href="#"> y aparezca solo cuando el usuario haga scroll (usando una clase que JS agrega/quita).

🟢 Resumen de lo que aprendiste en la Parte D

  • position: static — flujo normal, sin top/left/z-index
  • position: relative — se mueve visualmente pero reserva su espacio; crea contexto para absolute
  • position: absolute — sale del flujo, se ancla al ancestro posicionado más cercano
  • position: fixed — se ancla al viewport, siempre visible al hacer scroll
  • position: sticky — relative hasta llegar al umbral, luego se queda pegado
  • z-index — orden de apilamiento; solo funciona con position ≠ static
  • Modal — overlay fixed + caja centrada + clase JS para abrir/cerrar
  • transition — propiedad duración timing delay; preferir transform y opacity
  • timing-function — linear, ease, ease-in, ease-out, ease-in-out, cubic-bezier()
  • overflow — visible, hidden, scroll, auto; overflow-x / overflow-y por separado
  • Flujo de texto — white-space, text-overflow: ellipsis, word-break, hyphens, text-wrap: balance
