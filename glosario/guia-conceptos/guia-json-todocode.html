¿Qué es JSON?
El formato de datos más usado en el desarrollo web · 0:00 – 1:30

Cuando dos sistemas necesitan intercambiar información — tu app y un servidor, un frontend y un backend, dos APIs distintas — necesitan ponerse de acuerdo en el formato que usarán para empacar esos datos. El formato ganador en el mundo moderno es JSON.

📛
JSON
JavaScript Object Notation. Notación de Objetos de JavaScript. A pesar del nombre, sirve con cualquier lenguaje.
📄
Formato de texto
JSON es simplemente texto plano con una estructura específica. Cualquier lenguaje puede leerlo y escribirlo.
⚖️
Ligero
Ocupa poco espacio comparado con alternativas como XML. Fácil de leer para humanos y para máquinas.
🌍
Universal
Python, Java, PHP, Go, Ruby, Kotlin, Swift… todos tienen soporte nativo para JSON. Es el estándar de facto.
🔗
Intercambio
Su función principal es viajar entre sistemas: del servidor al cliente, entre microservicios, entre APIs.
📁
Archivo .json
Los archivos JSON tienen extensión .json y tipo MIME application/json. Son texto plano.

📮 Analogía: el formulario aduanal internacional

Cuando mandas un paquete de México a Japón, la aduana requiere un formulario estándar con campos específicos: descripción, peso, valor, destinatario. Sin ese formulario estandarizado, cada país tendría su propio sistema y nada funcionaría. JSON es ese formulario universal para los datos digitales: un formato que todos los países (lenguajes de programación) entienden sin necesidad de traducción, y que permite que los datos "crucen fronteras" entre sistemas completamente distintos.

💡 ¿Por qué no usar JavaScript directamente?

JavaScript tiene cosas que JSON no puede representar: funciones, métodos, referencias circulares, undefined. JSON es solo datos puros. Al eliminar esas cosas, se vuelve un formato neutral que cualquier lenguaje puede procesar, no solo JavaScript.

La sintaxis de JSON: pares clave-valor
La estructura fundamental de todo JSON · 1:30 – 3:00

JSON se basa en un concepto muy simple: clave-valor. Para cada clave (nombre del dato) existe un valor asociado. Un conjunto de pares clave-valor forma un objeto, representado entre llaves {}.

JSON// Objeto JSON básico — como se ve en el video de TodoCode { "nombre" : "Cristiano", "apellido" : "Ronaldo", "edad" : 36 } // Estructura: "clave" : valor // ↑ string ↑ cualquier tipo válido

🏷 Analogía: la ficha del expediente

Imagina una ficha de paciente en un consultorio. Tiene campos con su etiqueta: Nombre: Juan, Edad: 30, Alergias: Ninguna. Cada campo tiene una clave (la etiqueta impresa) y un valor (lo que se rellena). JSON funciona exactamente igual, pero en formato digital. La clave siempre va entre comillas dobles y el valor puede ser de varios tipos.

Reglas básicas de la sintaxis clave-valor

  • 🔤La clave siempre va entre comillas dobles: "nombre". Nunca con comillas simples ni sin comillas.
  • ➡️Se separan con dos puntos: "clave" : valor
  • ➕Los pares se separan entre sí con comas: "a":1, "b":2
  • 🚫El último par NO lleva coma al final. {"a":1, "b":2} ✅ vs {"a":1, "b":2,} ❌
  • 🚫No se permiten comentarios en JSON (// ni /* */). JSON es solo datos.

Tipos de datos en JSON
Los 6 tipos que JSON acepta como valores · 3:00 – 4:30

El valor en un par clave-valor no puede ser cualquier cosa. JSON acepta exactamente 6 tipos de datos. Ni más ni menos.

string

Texto entre comillas dobles. "Hola mundo", "Veracruz"

number

Enteros o decimales, sin comillas. 36, 3.14, -5

boolean

Solo true o false. En minúsculas, sin comillas.

null

Representa ausencia de valor. null. Sin comillas, en minúsculas.

object

Otro objeto anidado entre {}. Permite estructuras jerárquicas.

array

Lista ordenada de valores entre []. Puede contener cualquier tipo.

JSON{ "nombre" : "Luisina", // string "edad" : 28, // number (entero) "promedio" : 9.75, // number (decimal) "activo" : true, // boolean "telefono" : null, // null (sin dato) "direccion" : { "ciudad": "BsAs" }, // object anidado "habilidades" : ["JS", "Python"] // array }

⚠️ Tipos que JSON NO acepta

JSON no puede representar: undefined, funciones/métodos, fechas como Date (solo strings de fecha), Infinity, NaN, ni referencias circulares. Si intentas convertir estos con JSON.stringify(), los omite o los convierte a null.

🎲 Analogía: el formulario oficial del gobierno

Un formulario oficial del gobierno solo acepta respuestas en campos muy específicos: nombre (texto), edad (número), casado (sí/no), dirección (otro formulario adjunto), hijos (lista). No puedes poner una foto, ni un archivo MP3, ni dejar campos con respuestas inventadas. JSON es igual de estricto: solo esos 6 tipos, nada más. Lo que no cabe en esos 6 tipos simplemente no se puede enviar en JSON tal cual.

Objetos JSON con {}
La estructura principal de JSON · 3:00 – 5:00

Un objeto JSON es una colección de pares clave-valor encerrados entre llaves {}. Es la unidad principal de JSON — todo JSON empieza siendo un objeto o un array.

JSON// Objeto completo de un jugador (ejemplo del video) { "nombre" : "Cristiano", "apellido" : "Ronaldo", "edad" : 36, "activo" : true, "equipo" : "Al Nassr", "goles" : 874 }

🪪 Analogía: la credencial de identificación

Un objeto JSON es como una credencial de identificación: tiene campos definidos (nombre, apellido, fecha de nacimiento, número de ID) y cada uno tiene su valor. El objeto agrupa todo lo que describe a una entidad — una persona, un producto, un pedido. Las llaves {} son el marco físico de la credencial: definen dónde empieza y termina la información de esa entidad.

🟢 JSON vs Objeto de JavaScript — las diferencias

  • En JS un objeto puede tener claves sin comillas: {nombre: "Juan"}. En JSON las claves siempre llevan comillas dobles.
  • En JS puedes poner funciones como valores. En JSON no — solo datos.
  • En JS puedes usar undefined. En JSON no existe.
  • JSON no admite comentarios. JS sí.

Arrays JSON con []
Listas ordenadas de valores · 5:00 – 6:30

Un array JSON es una lista ordenada de valores encerrada entre corchetes []. Puede contener cualquier tipo de dato válido en JSON: strings, números, booleans, objetos, o incluso otros arrays.

JSON// Array de strings ["JavaScript", "Python", "Java", "Go"] // Array de números [10, 20, 30, 40, 50] // Array de objetos — el patrón MÁS COMÚN en APIs [ { "id": 1, "nombre": "Cristiano", "goles": 874 }, { "id": 2, "nombre": "Messi", "goles": 838 }, { "id": 3, "nombre": "Mbappé", "goles": 302 } ]

📋 Analogía: la lista del super con compartimentos

Un array JSON es como una lista del supermercado: tiene posiciones ordenadas (0, 1, 2, 3…), puedes tener ítems repetidos, y cada ítem puede ser cualquier cosa. Un array de objetos es como una lista de la compra donde cada ítem trae su ficha completa: "Producto: manzana, Cantidad: 3, Precio: $25, Disponible: true". El orden importa — el primer ítem siempre será el índice 0.

💡 El patrón más común en APIs: array de objetos

Cuando una API te devuelve varios registros (lista de usuarios, lista de productos, lista de pedidos), casi siempre lo hace como un array de objetos. Cada objeto en el array representa un registro. Este es el patrón que más vas a ver consumiendo APIs reales.

JSON Anidado — objetos dentro de objetos
Representar estructuras jerárquicas · 6:30 – 7:45

Una de las capacidades más poderosas de JSON es el anidamiento: un valor puede ser otro objeto o array completo. Esto permite representar datos con jerarquías complejas de forma natural.

JSON — objeto anidado completo{ "id": 1, "nombre": "Cristiano Ronaldo", "activo": true, // Objeto anidado "equipo": { "nombre": "Al Nassr", "pais": "Arabia Saudita", "fundado": 1955 }, // Array de strings "posiciones": ["delantero", "extremo"], // Array de objetos "titulos": [ { "competicion": "Champions League", "cantidad": 5 }, { "competicion": "Balón de Oro", "cantidad": 5 }, { "competicion": "Liga española", "cantidad": 2 } ] }

🪆 Analogía: las muñecas matrushka rusas

El JSON anidado es como las famosas muñecas matrushka: una muñeca grande contiene una mediana, que contiene una pequeña, que contiene otra más pequeña. El objeto principal ("jugador") contiene un objeto "equipo", que a su vez podría contener otro objeto "ciudad", que podría contener coordenadas geográficas. Puedes anidar tan profundo como necesites, aunque en APIs reales conviene no pasarse de 3-4 niveles.

Las reglas estrictas de JSON
Los errores más comunes y cómo evitarlos · 7:45 – 9:00

JSON es mucho más estricto que JavaScript. Un solo carácter incorrecto hace que el JSON completo sea inválido y no pueda procesarse. Estas son las reglas que más se violan:

Regla❌ Incorrecto✅ Correcto
Claves entre comillas dobles {nombre: "Juan"} {"nombre": "Juan"}
Sin comillas simples {'nombre': 'Juan'} {"nombre": "Juan"}
Sin coma final {"a": 1, "b": 2,} {"a": 1, "b": 2}
Sin comentarios {"a": 1 // comentario} {"a": 1}
Boolean en minúsculas {"activo": True} {"activo": true}
null en minúsculas {"tel": Null} {"tel": null}
Sin undefined {"valor": undefined} {"valor": null}
Sin funciones {"fn": function(){}} No tiene equivalente — omitir

🧾 Analogía: el formulario de la DIAN / SAT

Un formulario fiscal es extremadamente estricto: si escribes el CURP en el campo del RFC, si dejas un campo en blanco que era obligatorio, si usas una abreviación no autorizada — el formulario es rechazado. No importa que "casi" esté correcto. JSON es igual: una coma de más, unas comillas simples en lugar de dobles, o un True en lugar de true — y todo el JSON es inválido. Los parsers no adivinan tu intención.

JSON.stringify() — de objeto a texto JSON
Serializar datos para enviarlos · 9:00 – 10:10

JSON.stringify() convierte un objeto JavaScript en una cadena de texto JSON. Se usa cuando necesitas enviar datos a un servidor o guardarlos como texto.

JavaScript// Objeto JavaScript normal const jugador = { nombre: "Cristiano", apellido: "Ronaldo", edad: 36, activo: true }; // Convertir a string JSON const jsonString = JSON.stringify(jugador); console.log(jsonString); // → '{"nombre":"Cristiano","apellido":"Ronaldo","edad":36,"activo":true}' // Con formato legible (indentación de 2 espacios) const jsonBonito = JSON.stringify(jugador, null, 2); console.log(jsonBonito); // → { // "nombre": "Cristiano", // ... // } // ⚠️ Las funciones y undefined se OMITEN const obj = { nombre: "Ana", saludar: function(){}, valor: undefined }; JSON.stringify(obj); // → '{"nombre":"Ana"}' ← función y undefined desaparecen

📦 Analogía: empacar una caja para enviar por correo

JSON.stringify() es como el proceso de empacar objetos en una caja para enviarlos por correo. Tienes tu objeto JavaScript (los artículos: libros, ropa, gadgets) y stringify los "aplana" en una representación de texto que puede viajar por internet (como una lista de contenidos en papel). Cuando lleguen al destino, el receptor los "desempacará" con JSON.parse() para usar los objetos de nuevo.

JSON.parse() — de texto JSON a objeto
Deserializar datos recibidos · 10:10 – 11:10

JSON.parse() hace lo contrario a stringify: convierte una cadena de texto JSON en un objeto JavaScript con el que puedes trabajar. Se usa cuando recibes datos de un servidor o lees un archivo .json.

JavaScript// Supón que recibiste este string desde una API const respuestaAPI = '{"nombre":"Cristiano","apellido":"Ronaldo","edad":36,"activo":true}'; // Convertir el string JSON a objeto JavaScript const jugador = JSON.parse(respuestaAPI); // Ahora puedes acceder a sus propiedades normalmente console.log(jugador.nombre); // → "Cristiano" console.log(jugador.edad); // → 36 console.log(jugador.activo); // → true // Parsear un array JSON const listaJSON = '[{"id":1,"nombre":"Messi"},{"id":2,"nombre":"CR7"}]'; const lista = JSON.parse(listaJSON); console.log(lista[0].nombre); // → "Messi" console.log(lista.length); // → 2 // ⚠️ Si el string no es JSON válido, lanza SyntaxError JSON.parse("esto no es json"); // SyntaxError!

📬 Analogía: abrir y desempacar la caja del correo

JSON.parse() es el proceso inverso al stringify: es como recibir la caja del correo y abrirla. El texto JSON (la caja sellada con la lista de contenidos) se "desempaca" y se convierte en los objetos reales con los que puedes interactuar: acceder a propiedades, modificarlas, recorrerlas. Sin parse, el texto JSON es solo un string — un papel con nombres escritos. Con parse, se convierte en los objetos reales.

🟢 El ciclo completo: stringify ↔ parse

  • Enviar datos al servidor → JSON.stringify(objeto) → viaja como texto
  • Recibir datos del servidor → JSON.parse(texto) → usas como objeto
  • En el medio → el texto JSON viaja por HTTP, se guarda en archivos .json, se almacena en bases de datos NoSQL

¿Dónde se usa JSON en el mundo real?
JSON en APIs, configuraciones y bases de datos · 11:10 – 11:58

JSON no es solo teoría. Está en absolutamente todo el desarrollo moderno. Aquí los casos más importantes que menciona el video:

UsoDescripciónEjemplo
🌐 APIs REST Las respuestas de todas las APIs modernas vienen en JSON. Es el estándar. Instagram, Twitter, Google Maps, Spotify
⚙️ Archivos de configuración Muchas herramientas usan JSON para configuración de proyectos. package.json, tsconfig.json, .eslintrc.json
💾 Bases de datos NoSQL MongoDB, CouchDB, Firebase almacenan los datos en formato JSON/BSON. Cada "documento" es un objeto JSON
📱 Apps móviles Apps iOS y Android consumen APIs que devuelven JSON. Toda app que tenga backend usa JSON
🔄 Comunicación entre microservicios Los microservicios se comunican entre sí usando JSON sobre HTTP. Netflix, Uber, Airbnb usan esta arquitectura
💻 localStorage del navegador Para guardar objetos en el navegador se usa JSON.stringify y JSON.parse. Guardar preferencias, carrito de compras

🗣 Analogía: el inglés como idioma universal de negocios

En el mundo de los negocios internacionales, el inglés es el idioma neutral que usan dos partes que hablan lenguas distintas: un coreano y un brasileño negocian en inglés porque los dos lo entienden. JSON es exactamente eso en el mundo del software: un servidor en Python y una app en Swift no "hablan el mismo idioma", pero los dos entienden JSON. Es el inglés del intercambio de datos.

Resumen — Todo lo que explica el video de TodoCode
JSON de la A a la Z en una sola vista

🟢 Los 6 conceptos clave del video

  • JSON = JavaScript Object Notation. Formato de texto para intercambiar datos entre sistemas.
  • Se basa en pares clave-valor. La clave siempre entre comillas dobles, el valor puede ser 1 de 6 tipos.
  • Objeto {} = colección de pares clave-valor. Array [] = lista ordenada de valores.
  • Permite anidamiento: objetos dentro de objetos, arrays dentro de objetos, etc.
  • JSON.stringify() → objeto JS a string. JSON.parse() → string a objeto JS.
  • Es el estándar universal de APIs REST, configuraciones, bases de datos NoSQL y más.
