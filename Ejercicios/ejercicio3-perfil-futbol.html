PropiedadExplicación

CONCEPTO 1 — Box Model: recordatorio rápido
Ya lo viste a fondo en el Ejercicio 2 — aquí solo refrescamos lo esencial antes de usarlo

💡 ¿Lo estudiaste en el Ejercicio 2?

En el Ejercicio 2 aprendiste las 4 capas completas (content · padding · border · margin), el shorthand de margin/padding, el truco margin: 0 auto y por qué box-sizing: border-box es esencial. Todo eso lo usamos aquí. Si tienes dudas, revisa el Ejercicio 2 antes de continuar.

En este ejercicio aplicarás el Box Model en 3 situaciones nuevas: la foto de perfil circular (border-radius: 50%), las tarjetas de estadísticas (padding + border decorativo) y las pastillas de habilidades (padding + border-radius grande). La diferencia con el Ejercicio 2 es que ahora vas a combinar el Box Model dentro de un contenedor Flexbox.

CSS/* Recordatorio rápido — las propiedades clave */ .elemento { padding: 24px 16px; /* espacio interior ↑↓ ←→ */ border: 1px solid #E5E7EB; /* borde visible */ border-radius: 12px; /* esquinas redondeadas */ margin: 0 auto; /* centra horizontalmente */ box-sizing: border-box; /* el padding NO agranda el elemento */ } /* NUEVO en este ejercicio: imagen perfectamente circular */ .foto-perfil { width: 150px; height: 150px; border-radius: 50%; /* ← 50% con mismo ancho y alto = círculo */ border: 4px solid #D97706; }

CONCEPTO 2 — Flexbox básico
El sistema más usado para organizar elementos en fila o columna

Con solo poner display: flex en el contenedor padre, los hijos se organizan solos en fila horizontal. Funciona en 2 pasos:

  1. Le dices al contenedor padre que use Flexbox.
  2. Los elementos hijos se organizan automáticamente según las reglas que definas.

🚂 Analogía: el tren

El contenedor flex es el tren. Los elementos hijos son los vagones. Con Flexbox decides: ¿los vagones van en fila (row) o en columna (column)? ¿Cómo se distribuye el espacio entre ellos? ¿Se alinean al centro, arriba o abajo? Tú controlas el tren — los vagones obedecen.

CSS/* El CONTENEDOR (el padre — el tren) */ .contenedor { display: flex; /* ACTIVA flexbox — lo único obligatorio */ flex-direction: row; /* los hijos van en fila (izq a der) — es el default */ justify-content: center; /* centra en el eje horizontal */ align-items: center; /* centra en el eje vertical */ gap: 16px; /* espacio entre hijos */ flex-wrap: wrap; /* si no caben, saltan a nueva línea */ } /* justify-content — más valores útiles: */ /* flex-start → al inicio (izquierda) */ /* flex-end → al final (derecha) */ /* space-between → espacio entre (sin bordes) */ /* space-evenly → espacio igual entre todos */
Propiedad Flexbox¿Dónde va?¿Qué controla?
display: flexPadreActiva Flexbox. Los hijos pasan a modo flex automáticamente.
flex-directionPadrerow = en fila (default) · column = en columna.
justify-contentPadreDistribución en el eje principal (horizontal si direction es row).
align-itemsPadreAlineación en el eje cruzado (vertical si direction es row).
gapPadreEspacio entre hijos. Reemplaza márgenes manuales.
flex-wrap: wrapPadrePermite que los hijos salten de línea si no caben.
flex: 1HijoEl hijo ocupa una fracción igual del espacio disponible.

CONCEPTO 3 — Variables CSS: de nuevo en acción
Ya las usaste en el Ejercicio 2 — aquí las aplicamos con más colores y un patrón más completo

💡 Las variables CSS no son nuevas para ti

En el Ejercicio 2 definiste :root con --azul-oscuro, --amarillo, etc. y las usaste con var(). Aquí el concepto es el mismo — la diferencia es que ahora tienes más variables (incluyendo una para el radio de borde) y las vas a usar en más lugares del CSS, especialmente dentro de reglas Flexbox.

CSS/* Mismo patrón que el Ejercicio 2, ahora con más variables */ :root { --color-primario: #1D4ED8; /* azul principal */ --color-secundario: #1E3A8A; /* azul oscuro */ --color-acento: #D97706; /* naranja */ --color-texto: #1F2937; --color-fondo: #F8FAFC; --color-borde: #E5E7EB; --radio-borde: 12px; /* NUEVO: guardar medidas también, no solo colores */ } /* Usarlas con var() — igual que antes */ .stat-card { border-radius: var(--radio-borde); /* reutiliza la medida */ border-top: 4px solid var(--color-primario); } .stat-card.destacada { background-color: var(--color-primario); /* mismo color, diferente propiedad */ }

💡 ¿Qué aporta --radio-borde?

Guardar medidas como variables (no solo colores) te da consistencia visual. Todas las tarjetas del portfolio tendrán exactamente el mismo radio de esquina porque todas usan var(--radio-borde). Si decides cambiar a 8px, cambias una línea y se actualiza todo.

BLOQUE 1 — Estructura base con Google Fonts

Esta vez importamos una fuente de Google directamente en el HTML usando <link>.

HTML<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Perfil de Jugador — Carlos Mendoza</title> <!-- Google Fonts: importamos la fuente 'Poppins' --> <link rel="preconnect" href="https://fonts.googleapis.com"> <link href="https://fonts.googleapis.com/css2?family=Poppins:wght@400;600;700&display=swap" rel="stylesheet"> <link rel="stylesheet" href="styles.css"> </head> <body> <!-- El contenido va aquí --> </body> </html>
Línea nueva¿Qué hace?
rel="preconnect"Le dice al navegador que establezca conexión con Google antes de necesitar la fuente. Hace que cargue más rápido.
family=Poppins:wght@400;600;700Pedimos 3 pesos de la fuente: 400 (normal), 600 (semi-bold) y 700 (bold). Así podemos usar los tres en el CSS.
&display=swapMientras carga la fuente, muestra una alternativa. El texto aparece de inmediato y luego "swap" cambia a Poppins.

💡 Las habilidades como pastillas con Flexbox

Los <li> normalmente se apilan en columna. Al ponerle display: flex al <ul> y flex-wrap: wrap, los elementos se acomodan en fila y saltan de línea automáticamente si no caben. Es el mismo patrón que usamos con las tarjetas, pero con pastillas de texto.

CSS BLOQUE 1 — Reset y Variables CSS

CSS/* ────────────────────────────────── RESET ────────────────────────────────── */ *, *::before, *::after { margin: 0; padding: 0; box-sizing: border-box; } /* ────────────────────────────────── VARIABLES CSS ────────────────────────────────── */ :root { --primario: #1D4ED8; /* azul principal */ --primario-d: #1E3A8A; /* azul oscuro */ --acento: #D97706; /* naranja */ --texto: #1F2937; --texto-suave:#6B7280; --fondo: #F8FAFC; --surface: #FFFFFF; --borde: #E5E7EB; --radio: 12px; } body { font-family: 'Poppins', Arial, sans-serif; background-color: var(--fondo); color: var(--texto); font-size: 16px; line-height: 1.6; }
CódigoExplicación
*, *::before, *::afterEl reset se aplica también a los pseudo-elementos ::before y ::after que CSS puede insertar antes/después del contenido.
:root { --primario: ... }El doble guión (--) indica que es una variable CSS. El nombre lo inventas tú. Se define en :root para que sea global.
var(--primario)Así se usa la variable. La función var() llama al valor definido en :root.

CSS BLOQUE 2 — El hero con Flexbox

Aquí usamos Flexbox por primera vez. .hero es el contenedor y .hero-foto + .hero-info son sus hijos en fila.

CSS.hero { display: flex; /* ← ACTIVA FLEXBOX. Los hijos van en fila */ align-items: center; /* centra verticalmente foto e info */ gap: 40px; /* 40px entre la foto y la info */ background-color: var(--primario); padding: 48px 60px; color: white; } .foto-perfil { width: 150px; height: 150px; border-radius: 50%; /* ← 50% = círculo perfecto */ border: 4px solid var(--acento); display: block; } .posicion { display: inline-block; /* necesario para que padding funcione */ background-color: var(--acento); color: white; font-size: 13px; font-weight: 600; padding: 4px 14px; /* 4px arriba/abajo, 14px izq/der = pastilla */ border-radius: 20px; /* muy redondeado = forma pastilla */ margin-bottom: 10px; } .hero h1 { font-size: 42px; font-weight: 700; line-height: 1.1; margin-bottom: 6px; } .equipo { font-size: 16px; font-weight: 600; color: rgba(255, 255, 255, 0.75); margin-bottom: 12px; } .descripcion { font-size: 15px; color: rgba(255, 255, 255, 0.85); max-width: 480px; }
PropiedadExplicación detallada
display: flexActiva Flexbox en .hero. Sus hijos directos (.hero-foto y .hero-info) se ponen automáticamente en fila horizontal.
align-items: centerAlinea los hijos verticalmente al centro. Sin esto, la foto quedaría alineada arriba y el texto también arriba — se vería descuadrado.
border-radius: 50%Con 50%, si el elemento tiene el mismo ancho y alto (150px × 150px), se convierte en un círculo perfecto.
display: inline-blockEl <span> es inline por defecto e ignora el padding vertical. Con inline-block respeta el padding y se ve como pastilla.
rgba(255, 255, 255, 0.75)Blanco con 75% de opacidad. Más suave que blanco puro — crea jerarquía visual entre el nombre (blanco puro) y el equipo (blanco translúcido).

CSS BLOQUE 3 — Secciones y tarjetas de estadísticas

CSS.seccion { max-width: 860px; margin: 0 auto; padding: 40px 20px; } .titulo-seccion { font-size: 24px; font-weight: 700; color: var(--primario); border-left: 5px solid var(--acento); /* barra naranja a la izquierda */ padding-left: 16px; margin-bottom: 24px; } .stats-grid { display: flex; /* las 4 tarjetas en FILA */ gap: 16px; flex-wrap: wrap; } .stat-card { flex: 1; /* ocupa igual espacio que sus hermanas */ min-width: 140px; background-color: var(--surface); border: 1px solid var(--borde); border-radius: var(--radio); padding: 24px 16px; text-align: center; box-shadow: 0 2px 8px rgba(0, 0, 0, 0.06); /* sombra suave */ } /* Selector de doble clase: AMBAS clases en el mismo elemento */ .stat-card.destacada { background-color: var(--primario); border-color: var(--primario); color: white; } .stat-numero { display: block; /* el span ocupa su propia línea */ font-size: 40px; font-weight: 700; color: var(--primario); line-height: 1; } /* Selector descendiente + doble clase */ .stat-card.destacada .stat-numero { color: white; /* solo el número DENTRO de la tarjeta destacada */ } .stat-label { display: block; font-size: 13px; font-weight: 600; color: var(--texto-suave); text-transform: uppercase; letter-spacing: 0.08em; margin-top: 6px; } .stat-card.destacada .stat-label { color: rgba(255, 255, 255, 0.8); }
Selector / PropiedadExplicación
.stat-card.destacadaSin espacio entre las dos clases = el mismo elemento debe tener ambas clases. Con espacio sería diferente: .stat-card .destacada = elemento hijo de stat-card.
.stat-card.destacada .stat-numeroSelector compuesto: el .stat-numero que esté dentro de un elemento que tenga ambas clases stat-card y destacada. Muy específico.
box-shadow: 0 2px 8px ...Sombra de caja. Formato: desplaz-x desplaz-y desenfoque color. El 0.06 en rgba es opacidad — muy suave para que no sea agresiva.
border-radius: var(--radio)Aquí usamos la variable --radio definida en :root. Si cambias el valor en :root, todas las tarjetas actualizan su radio de esquinas.

CSS BLOQUE 4 — La tabla de partidos

CSS.tabla-partidos { width: 100%; border-collapse: collapse; /* fusiona bordes — evita el doble borde */ font-size: 15px; } .tabla-partidos th { background-color: var(--primario); color: white; font-weight: 600; padding: 12px 16px; text-align: left; font-size: 13px; text-transform: uppercase; letter-spacing: 0.06em; } .tabla-partidos td { padding: 12px 16px; border-bottom: 1px solid var(--borde); } /* Filas alternas — efecto "zebra" */ .tabla-partidos tbody tr:nth-child(even) { background-color: #F1F5F9; } /* Al pasar el ratón por una fila */ .tabla-partidos tbody tr:hover { background-color: #DBEAFE; } .resultado { font-weight: 700; } .ganado { color: #065F46; } .empate { color: #92400E; } .perdido { color: #B91C1C; }
Selector / PropiedadExplicación
border-collapse: collapsePor defecto las tablas tienen espacio entre bordes (se ve como doble borde). Con collapse se fusionan en uno solo.
.tabla-partidos thSelector descendiente: solo los <th> dentro de .tabla-partidos. No afecta a otras tablas de la página.
tr:nth-child(even)Pseudo-clase estructural: aplica a las filas pares (2ª, 4ª, 6ª...). Crea el efecto "zebra". odd sería las impares.
tbody tr:hoverSelector descendiente + pseudo-clase: el <tr> que esté dentro del <tbody> y tenga el ratón encima.

CSS BLOQUE 5 — Habilidades y footer

CSS.lista-habilidades { display: flex; flex-wrap: wrap; /* las pastillas saltan de línea si no caben */ gap: 10px; list-style: none; /* quita los bullets de la lista */ padding: 0; } .habilidad { background-color: var(--surface); border: 1px solid var(--borde); border-radius: 20px; padding: 8px 18px; font-size: 14px; font-weight: 600; color: var(--texto); transition: all 0.2s; /* anima todos los cambios en 0.2s */ } .habilidad:hover { background-color: var(--primario); color: white; border-color: var(--primario); } .habilidad.top { background-color: var(--acento); color: white; border-color: var(--acento); } .pie { background-color: var(--primario-d); color: rgba(255, 255, 255, 0.8); text-align: center; padding: 32px 20px; margin-top: 60px; font-size: 14px; } .pie a { color: var(--acento); text-decoration: none; font-weight: 600; } .pie a:hover { text-decoration: underline; }
PropiedadExplicación
transition: all 0.2sAnima todos los cambios de estilo en 0.2 segundos. Sin esta línea el hover sería instantáneo y brusco. Con ella es suave y profesional.
.habilidad:hoverAl pasar el ratón, cambia fondo, color de texto y borde simultáneamente. El transition los anima todos juntos.
.habilidad.topSelector de doble clase: la habilidad especial (la que tiene class="habilidad top") tiene color naranja permanente, no solo en hover.
.pie aSelector descendiente: solo los <a> dentro del .pie reciben este color. No afecta a ningún otro enlace de la página.

🔍 DevTools — tu mejor herramienta de debugging

PARTE 5 — Retos para practicar más
Intenta cada reto sin ver soluciones. Los errores también enseñan
🟡 Fácil

Reto 1 — Cambia colores con las variables

En el :root, cambia --primario a #065F46 (verde) o #7C3AED (morado). Como todo el CSS usa var(--primario), la página entera cambiará de color modificando solo esa línea.

Lo que practicas: el poder de las variables CSS para cambios globales.

🟠 Medio

Reto 2 — 5ª tarjeta de estadística

Agrega un 5° <div class="stat-card"> con el número 12 y la etiqueta Tarjetas amarillas. Ponle la clase extra advertencia y en CSS dale background: #FEF3C7 y border-color: #D97706.

Lo que practicas: clases dobles y Flexbox con flex: 1 adaptándose automáticamente.

🟠 Medio

Reto 3 — Nueva fila en la tabla

Agrega un <tr> nuevo en el <tbody> con datos de otro partido inventado. Aplica la clase correcta al resultado: ganado, empate o perdido. Verifica que el efecto zebra se actualice solo.

Lo que practicas: estructura de tablas y :nth-child(even).

⚫ Avanzado

Reto 4 — Layout de dos columnas

Crea un <div class="dos-columnas"> que contenga la sección de habilidades y una sección nueva que inventes (por ejemplo: "🏆 Palmarés" con una lista de títulos). En CSS ponle display: flex y gap: 40px.

Lo que practicas: Flexbox para layouts de dos columnas reales.

ConceptoLo que aprendiste
Box Model4 capas: content, padding, border, margin. box-sizing: border-box hace que el padding no aumente el tamaño total.
Flexboxdisplay: flex en el padre pone los hijos en fila. justify-content eje horizontal, align-items eje vertical, gap espacio entre hijos.
Variables CSS:root { --nombre: valor; } para definir. var(--nombre) para usar. Un cambio en :root actualiza toda la página.
Imagen circularborder-radius: 50% con mismo ancho y alto = círculo perfecto.
display: block en spanLos <span> son inline por defecto. Con display: block ocupan su propia línea.
transitionAnima cambios de estilo suavemente. transition: all 0.2s hace que todos los cambios tarden 0.2 segundos.
.clase1.clase2Sin espacio = el mismo elemento tiene ambas clases. Con espacio = elemento hijo.
:nth-child(even)Pseudo-clase estructural. Aplica a hijos pares (2, 4, 6...). Crea el efecto zebra en tablas.
Tablas HTML<table> → <thead>/<tbody> → <tr> → <th>/<td>. Semántica correcta para datos tabulares.
box-shadowSombra de caja. Formato: x y desenfoque color. Crea profundidad visual.
Selector descendiente.padre .hijo aplica solo a los .hijo dentro de .padre.
