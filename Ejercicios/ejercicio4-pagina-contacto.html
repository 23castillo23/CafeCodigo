CSS — Propiedades en uso que no conocías aún

Usadas en los bloques de código de este ejercicio pero sin tabla de referencia.

PropiedadExplicación
cursor: pointer;Cambia el cursor del ratón a la manita que indica "esto es clickeable". Se usa en botones, labels clicables y cualquier elemento interactivo que no sea un enlace o input nativo.
outline: none;Quita el anillo de enfoque por defecto del navegador. Úsalo solo si vas a reemplazarlo con tu propio estilo en :focus — eliminarlo sin reemplazo daña la accesibilidad.
resize: vertical;Solo en <textarea>. Permite al usuario redimensionar el campo solo verticalmente (no horizontalmente, lo que rompería el layout). resize: none lo bloquea del todo.
top / left / right / bottom:Coordenadas de posicionamiento. Solo funcionan cuando el elemento tiene position diferente de static. Con position: absolute ubican el elemento relativo a su padre posicionado.
z-index:Controla qué elemento queda encima cuando dos elementos se superponen. Mayor número = más arriba. Solo funciona con position diferente de static.
min-height:Altura mínima. El elemento puede crecer más si el contenido lo requiere, pero nunca será menor que este valor. Muy usado en heroes y secciones que deben verse bien aunque estén vacías.
flex-direction: column;Cambia la dirección de Flexbox de horizontal (por defecto) a vertical. Los hijos se apilan uno debajo del otro. Con column-reverse el orden se invierte.

CONCEPTO 1 — Formularios HTML
La forma en que los usuarios envían información a través de una página web

Todo lo que conoces en internet — registro, login, búsqueda, compras, comentarios — funciona con formularios. Son la única manera que tiene el usuario de enviar datos sin JavaScript.

📝 Analogía: el formulario del banco

Cuando vas al banco a hacer una transferencia, te dan un formulario en papel: campos para el beneficiario, la CLABE, el monto, y al final lo firmas y lo entregas al cajero. Un formulario HTML funciona exactamente igual: campos para que el usuario llene (<input>), un área de texto (<textarea>), y un botón para enviarlo (<button type="submit">). El "cajero" que recibe los datos es el servidor web.

Estructura básica de un formulario

<form action="#" method="post"> contenedor del formulario
  <label for="nombre">Tu nombre:</label> etiqueta descriptiva
  <input type="text" id="nombre" name="nombre" required> campo de entrada
  <button type="submit">Enviar</button> botón de envío
</form>
Atributo / EtiquetaDefinición completa
<form>Contenedor de todo el formulario. Sin él, los campos no se agrupan y el botón no sabe qué datos enviar.
action="/url"A dónde se envían los datos al hacer clic en Enviar. En práctica puedes poner action="#".
method="post"post manda los datos de forma oculta (seguro para contraseñas). get los pone visibles en la URL.
novalidateDesactiva los globos de error feos del navegador. Así podemos usar solo nuestra validación visual con CSS.
<label for="id">Etiqueta descriptiva. Al hacer clic en el label, el cursor va al campo con ese id. Accesibilidad esencial.
name="campo"El nombre con que se envía el dato al servidor. Si el usuario escribe "Hugo", el servidor recibe nombre=Hugo.
requiredAtributo booleano. El formulario no se envía si este campo está vacío.
autocompleteLe dice al navegador que puede sugerir valores guardados. autocomplete="email" sugiere emails usados antes.

Todos los tipos de <input> que debes conocer

type=¿Qué muestra y cuándo usarlo?
textCampo de texto libre. Para nombre, apellido, dirección, cualquier texto sin formato especial.
emailValida automáticamente que tenga @ y un dominio. En móvil abre teclado con @ visible.
telCampo de teléfono. En móvil abre el teclado numérico. No valida el formato (varía por país).
passwordOculta los caracteres con puntos. Para contraseñas y PINs.
numberSolo acepta números. Muestra flechas para subir/bajar. Atributos: min, max, step.
dateAbre un selector de calendario. El valor se guarda en formato YYYY-MM-DD.
checkboxCasilla de verificación. Para opciones múltiples independientes (puede haber varias marcadas).
radioBotón de opción. Solo uno del grupo puede estar activo. Todos del mismo grupo deben tener el mismo name.
fileBotón para subir archivos. accept="image/*" limita a imágenes.
rangeDeslizador (slider). Con min, max y step. Para volumen, precio, valoraciones.
colorAbre un selector de color. Devuelve un valor hexadecimal (#FF0000).
hiddenCampo invisible al usuario. Para enviar datos que el usuario no debe ver ni cambiar.

CONCEPTO 2 — Pseudo-clases de estado
CSS detecta el estado del campo automáticamente — sin JavaScript

🚦 Analogía: los semáforos del formulario

Imagina que cada campo tiene su propio semáforo. Cuando el usuario hace clic, el semáforo se pone amarillo (:focus). Si escribe un email correcto, pasa a verde (:valid). Si escribe algo inválido, pasa a rojo (:invalid). Todo esto sin JavaScript — CSS lo detecta solo porque el navegador ya sabe qué es un email válido, qué es un número, etc.

Estado normal

Campo en reposo. Sin interacción. Borde gris por defecto.

:focus

El usuario hizo clic dentro. Borde morado + anillo de enfoque.

:valid

El contenido cumple las reglas (email correcto, no vacío). Borde verde.

:invalid

El contenido no cumple las reglas. Borde rojo + mensaje de error visible.

CSS/* :focus — el usuario hizo clic dentro del campo */ input:focus { border-color: var(--primario); outline: none; /* quita el borde azul feo del navegador */ box-shadow: 0 0 0 3px rgba(109, 40, 217, 0.15); /* anillo suave */ } /* :valid — campo con contenido correcto */ /* :not(:placeholder-shown) evita que verde aparezca en campos vacíos */ input:not(:placeholder-shown):valid { border-color: var(--exito); /* verde */ } /* :invalid — campo con contenido incorrecto (y ya tocado por el usuario) */ input:not(:placeholder-shown):invalid { border-color: var(--error); /* rojo */ } /* ~ (tilde) = selector hermano siguiente: .campo-msg que siga al input inválido */ input:not(:placeholder-shown):invalid ~ .campo-msg { display: block; /* muestra el mensaje de error */ } /* ::placeholder (pseudo-ELEMENTO, doble ::) — el texto gris dentro del campo */ input::placeholder { color: #9CA3AF; font-style: italic; } /* :has() — aplica al .radio-label si CONTIENE un input:checked */ .radio-label:has(input:checked) { background: var(--primario); color: white; } /* accent-color — cambia el color del checkbox/radio en 1 línea */ input[type='checkbox'] { accent-color: var(--primario); }
Pseudo-clase¿Cuándo se activa?
:focusCuando el usuario hace clic dentro del campo o navega con Tab. Se desactiva al hacer clic afuera.
:validCuando el valor cumple todas las restricciones: required + type correcto + pattern si lo hay.
:invalidCuando el valor no cumple las restricciones. Sin :not(:placeholder-shown) se activa desde que carga la página.
:placeholder-shownMientras el placeholder es visible, es decir, mientras el campo está vacío.
:checkedCuando un checkbox o radio está marcado.
:has(selector)Aplica al elemento si contiene algo que coincida. Pseudo-clase moderna (Chrome 105+).
:disabledCuando el campo tiene el atributo disabled. El usuario no puede interactuar.
:activeMientras el botón está siendo presionado. Permite el efecto de "hundirse".
~ (tilde)Selector de hermano siguiente. input:invalid ~ .msg selecciona .msg que venga después del input en el mismo padre.

⚠️ El problema de :valid e :invalid desde cero

Por defecto, un input required vacío ya es :invalid desde que carga la página — antes de que el usuario lo toque. Eso hace que todos los campos aparezcan en rojo al abrir el formulario. La solución: combinar con :not(:placeholder-shown). Así la pseudo-clase solo se activa cuando el campo tiene texto (el placeholder desapareció).

CONCEPTO 3 — Animaciones CSS con @keyframes
Movimiento y efectos visuales sin una sola línea de JavaScript

🎬 Analogía: el guion de una película

@keyframes es como escribir el guion de una animación. Defines qué pasa en cada escena: "al inicio (0%) el elemento está transparente y 30px abajo. Al final (100%) es opaco y está en su posición normal". Luego la propiedad animation en el elemento es como decirle al actor: "sigue este guion, tarda 0.6 segundos y repítelo 1 vez".

CSS/* PASO 1: Define la animación con @keyframes */ @keyframes aparecer { from { /* = 0% — estado INICIAL */ opacity: 0; /* transparente */ transform: translateY(30px); /* 30px hacia abajo */ } to { /* = 100% — estado FINAL */ opacity: 1; /* visible */ transform: translateY(0); /* en su posición normal */ } } /* PASO 2: Aplica la animación al elemento */ .hero { animation: aparecer 0.5s ease forwards; /* ↑nombre ↑dur ↑curva ↑qué-hacer-al-final */ } /* Con delay: la tarjeta aparece 0.2s después del hero */ .form-card { opacity: 0; /* empieza invisible para que la animación tome control */ animation: aparecer 0.6s ease 0.2s forwards; /* ↑delay */ } /* Animación de pulso continuo (el botón de enviar) */ @keyframes latir { 0%, 100% { box-shadow: 0 0 0 0px rgba(109,40,217,0.4); } 50% { box-shadow: 0 0 0 8px rgba(109,40,217,0); } } .btn-enviar { animation: latir 2s ease-in-out infinite; /* se repite para siempre */ } .btn-enviar:hover { animation: none; /* detiene el pulso al hacer hover */ }
ConceptoExplicación
@keyframes nombreDefine la animación. El nombre lo inventas tú. Dentro usas porcentajes 0% … 100% o from … to.
animation: nombre dur curvaAplica la animación. Mínimo necesitas nombre y duración. El 4° valor es el delay.
forwardsEl elemento mantiene el estado final de la animación. Sin forwards, regresa al estado inicial.
infiniteLa animación se repite para siempre. Para spinners, pulsaciones, indicadores.
opacity: 0El elemento empieza invisible antes de que inicie la animación. Sin esto aparecería un instante antes.
translateY(30px)Mueve el elemento 30px hacia abajo. En el to está en 0 = posición normal. Efecto: sube desde abajo.
ease / linear / ease-outCurva de velocidad. ease = natural (lento-rápido-lento). linear = constante. ease-out = frena al final.

CONCEPTO 4 — Pseudo-elementos ::before y ::after
Contenido decorativo desde CSS — sin tocar el HTML

🎭 Analogía: los extras de una película

::before y ::after son como contratar extras para una escena. Los actores principales ya están en el guion (HTML). Pero puedes añadir personajes de fondo desde la dirección (CSS) sin modificar el guion. Aparecen antes o después del actor principal y pueden ser decorativos, llevar un traje especial, o hacer un movimiento específico.

CSS/* ::before y ::after SIEMPRE necesitan la propiedad content (aunque esté vacía) */ /* Ejemplo 1: el asterisco * de los campos obligatorios */ .requerido::after { content: ' *'; /* el asterisco — CSS lo añade, no el HTML */ color: var(--error); font-weight: 700; } /* Ejemplo 2: la línea naranja decorativa bajo el título */ .titulo-deco::after { content: ''; /* vacío — solo queremos la forma visual */ display: block; width: 48px; height: 3px; background: var(--acento); margin-top: 8px; border-radius: 2px; } /* Ejemplo 3: la línea animada bajo los links del navbar */ .nav-links a::after { content: ''; position: absolute; bottom: -4px; left: 0; width: 0; /* empieza con ancho 0 — invisible */ height: 2px; background: var(--primario); transition: width 0.25s ease; /* anima el crecimiento */ } .nav-links a:hover::after { width: 100%; /* crece a 100% al hacer hover */ }
ConceptoExplicación
::beforeInserta contenido antes del contenido real del elemento. Siempre necesita content.
::afterInserta contenido después del contenido real. También necesita content.
content: ''Vacío = solo queremos el elemento visual (la línea, el punto), no texto. Obligatorio de todas formas.
content: ' *'Texto que CSS inserta. Se usa para asteriscos, numeración, iconos de texto, sin tocar el HTML.
display: blockLos pseudo-elementos son inline por defecto. Con block pueden tener width y height propios.
position: absoluteNecesario cuando el pseudo-elemento debe posicionarse relativo a su padre (como la línea del navbar).

💡 :: (doble dos puntos) vs : (un dos puntos)

En CSS moderno, los pseudo-elementos usan :: doble: ::before, ::after, ::placeholder. Las pseudo-clases usan : simple: :hover, :focus, :valid. Esta distinción es intencional — los elementos crean algo nuevo en el DOM, las clases aplican a elementos existentes.

CSS BLOQUE 4 — Campos, estados y validación

El bloque más importante: aquí entran en juego todas las pseudo-clases.

CSS/* ── FORMULARIO ── */ .formulario { display: flex; flex-direction: column; gap: 1.5rem; } /* 2 campos lado a lado con CSS Grid */ .form-fila { display: grid; grid-template-columns: 1fr 1fr; /* 2 columnas iguales */ gap: 1.25rem; } .campo-grupo { display: flex; flex-direction: column; gap: 0.4rem; } .campo-grupo label, .label-bloque { font-size: 0.875rem; font-weight: 600; } /* El asterisco rojo — CSS lo pone, el HTML no lo tiene */ .requerido::after { content: ' *'; color: var(--error); font-weight: 700; } /* Estilos BASE para todos los campos */ input[type='text'], input[type='email'], input[type='tel'], select, textarea { width: 100%; padding: 0.75rem 1rem; border: 2px solid var(--borde); border-radius: var(--radio); font-family: inherit; font-size: 0.95rem; background-color: var(--fondo); color: var(--texto); outline: none; transition: border-color 0.2s, box-shadow 0.2s, background-color 0.2s; } /* :FOCUS — campo activo */ input[type='text']:focus, input[type='email']:focus, input[type='tel']:focus, select:focus, textarea:focus { border-color: var(--primario); background-color: var(--surface); box-shadow: 0 0 0 3px rgba(109, 40, 217, 0.15); /* anillo */ } /* :VALID — campo con contenido correcto (y no vacío) */ input[type='text']:not(:placeholder-shown):valid, input[type='email']:not(:placeholder-shown):valid { border-color: var(--exito); /* verde */ } /* :INVALID — campo con error (ya fue tocado) */ input[type='text']:not(:placeholder-shown):invalid, input[type='email']:not(:placeholder-shown):invalid { border-color: var(--error); /* rojo */ } /* Mensaje de error — oculto por defecto */ .campo-msg { font-size: 0.8rem; color: var(--error); display: none; } /* ~ = selector hermano siguiente: muestra .campo-msg si el input es :invalid */ input:not(:placeholder-shown):invalid ~ .campo-msg { display: block; } /* Placeholder */ input::placeholder, textarea::placeholder { color: #9CA3AF; font-style: italic; } /* Textarea: solo puede redimensionarse verticalmente */ textarea { resize: vertical; min-height: 120px; }
Selector / PropiedadExplicación
grid-template-columns: 1fr 1frCSS Grid: 2 columnas iguales. Los 2 campos dentro de .form-fila quedan lado a lado automáticamente.
input[type='email']Selector de atributo: aplica solo a inputs con type='email'. Más preciso que solo input que afectaría también checkboxes y radios.
transition: border-color, box-shadow, background-colorAnima 3 propiedades a la vez separadas por coma. Todas tardan 0.2s.
box-shadow: 0 0 0 3px rgba()El "anillo de enfoque" accesible. 0 0 sin desplazamiento, 0 sin desenfoque, 3px es el spread (expansión del anillo).
:not(:placeholder-shown):validEl campo es válido y el placeholder ya no es visible (hay texto). Sin :not(), todos los campos aparecerían verdes vacíos desde el inicio.
input:invalid ~ .campo-msgEl selector ~ (tilde) es "hermano siguiente". Selecciona .campo-msg que venga después del input inválido en el mismo contenedor padre.
resize: verticalEl textarea solo puede redimensionarse verticalmente. Sin esto el usuario puede estirar el formulario horizontalmente y romper el layout.

CSS BLOQUE 5 — Radio buttons, Checkbox y Botón de envío

CSS/* ── RADIO BUTTONS ── */ .radio-grupo { display: flex; flex-wrap: wrap; gap: 0.75rem; margin-top: 0.5rem; } .radio-label { display: flex; align-items: center; gap: 0.5rem; cursor: pointer; font-size: 0.9rem; padding: 0.5rem 1rem; border: 2px solid var(--borde); border-radius: 20px; transition: all 0.2s; } .radio-label:hover { border-color: var(--primario); color: var(--primario); } /* :has() — el label cambia color si CONTIENE un radio:checked */ .radio-label:has(input:checked) { background: var(--primario); border-color: var(--primario); color: white; font-weight: 600; } /* ── CHECKBOX ── */ .campo-check { display: flex; align-items: flex-start; gap: 0.75rem; font-size: 0.9rem; } .campo-check input[type='checkbox'] { width: 18px; height: 18px; margin-top: 2px; accent-color: var(--primario); /* color del check al marcarse */ cursor: pointer; flex-shrink: 0; } .campo-check a { color: var(--primario); font-weight: 600; } /* ── BOTÓN DE ENVÍO ── */ .btn-enviar { width: 100%; padding: 1rem; background: var(--primario); color: white; border: none; border-radius: var(--radio); font-family: inherit; font-size: 1rem; font-weight: 700; cursor: pointer; letter-spacing: 0.02em; transition: background-color 0.2s, transform 0.15s; animation: latir 2s ease-in-out infinite; /* pulso constante */ } .btn-enviar:hover { background-color: var(--primario-d); transform: translateY(-2px); /* sube 2px */ animation: none; /* detiene el pulso */ } .btn-enviar:active { transform: translateY(0); /* vuelve abajo al presionar */ } /* ── FOOTER ── */ .pie { background-color: var(--texto); color: rgba(255,255,255,0.7); text-align: center; padding: 3rem 2rem; margin-top: 4rem; } .pie-contenido { display: flex; flex-direction: column; gap: 0.5rem; align-items: center; } .pie-logo { font-size: 1.2rem; font-weight: 700; color: white; } .pie-copy { font-size: 0.8rem; opacity: 0.5; margin-top: 0.5rem; }
Selector / PropiedadExplicación
.radio-label:has(input:checked):has() es una pseudo-clase moderna: aplica al elemento si contiene lo que indica el selector. Aquí: colorea el label entero si tiene un radio marcado.
accent-color: var(--primario)Cambia el color del checkbox/radio con una sola línea. Antes requería técnicas complejas con SVG o posicionamiento absoluto.
animation: latir 2s infiniteEl botón pulsa suavemente para siempre. infinite repite la animación.
animation: noneEn :hover desactivamos el pulso. El hover tiene su propio efecto (translateY(-2px)) más dramático y claro.
:active { transform: translateY(0) }Mientras el botón está presionado, baja de vuelta. Combinado con el -2px del hover, simula un botón físico que se hunde.

Del más sencillo al más desafiante
🟡 Fácil

Reto 1 — Modo oscuro con variables

En el :root, cambia: --fondo a #111827, --surface a #1F2937, --texto a #F9FAFB, --borde a #374151.

Toda la página cambia a modo oscuro con solo 4 líneas. Observa el poder de las variables CSS.

🟠 Medio

Reto 2 — Nueva animación

Crea un nuevo @keyframes llamado entrar-izquierda que mueva el elemento de translateX(-40px) a translateX(0) con opacity: 0 → 1. Aplícalo al .form-card con un delay de 0.4s.

🟠 Medio

Reto 3 — Campo de archivos

Agrega un .campo-grupo con <input type="file" accept="image/*,.pdf"> y el label "Adjuntar archivos (opcional)". En CSS dale estilos al input[type='file']: borde punteado, fondo gris claro, padding generoso y cursor: pointer.

⚫ Avanzado

Reto 4 — Contador de caracteres

Al textarea agrégale maxlength="500". Debajo pon un <span class="contador">0 / 500</span>. Con CSS puedes cambiar el color del contador cuando el textarea es :invalid. Para que cuente dinámicamente necesitarías JavaScript — ¡el siguiente paso en tu camino!
