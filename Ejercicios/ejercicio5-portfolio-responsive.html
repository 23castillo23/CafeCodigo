Propiedades que aparecen en el código de este ejercicio sin tabla de referencia.

PropiedadExplicación
position: relative;El elemento mantiene su lugar en el flujo normal pero se convierte en referencia para hijos con position: absolute. También permite usar top/left para moverlo ligeramente sin sacarlo del flujo.

CONCEPTO 1 — Diseño Mobile-First
La filosofía que usan todos los equipos profesionales

Mobile-First es una filosofía de diseño: escribes los estilos base para la pantalla más pequeña (celular) y luego usas Media Queries para añadir estilos conforme la pantalla crece. Nunca quitas estilos, solo agregas capas.

🧥 Analogía: vestirse por capas

Imagina que vas a salir y no sabes qué tanto frío hará. Primero te pones la ropa base (playera y pantalón) — eso es el diseño para celular. Si hace un poco de frío, añades un suéter — eso es el diseño para tablet. Si hace mucho frío, añades también un abrigo — eso es el diseño para escritorio. Nunca quitas ropa, solo añades capas. Mobile-First funciona igual.

🖥 Escritorio — 3 columnas

¿Por qué Mobile-First y no al revés?

  • Más del 60% del tráfico web viene de celulares. El celular es la prioridad.
  • Es más fácil expandir un diseño simple que comprimir uno complejo.
  • Los navegadores móviles cargan menos CSS innecesario — páginas más rápidas.
  • Google usa el Mobile-First Indexing: indexa la versión móvil para el posicionamiento en buscadores.
CSS/* ✅ MOBILE-FIRST — así se hace */ /* Base: estilos para CELULAR (sin @media) */ .galeria { display: grid; grid-template-columns: 1fr; /* 1 columna en celular */ gap: 1rem; } /* Tablet — añadimos estilos cuando la pantalla crece */ @media (min-width: 640px) { .galeria { grid-template-columns: 1fr 1fr; } /* 2 columnas */ } /* Escritorio — añadimos más */ @media (min-width: 1024px) { .galeria { grid-template-columns: repeat(3, 1fr); } /* 3 columnas */ } /* ❌ DESKTOP-FIRST — lo que NO harías */ .galeria { grid-template-columns: repeat(3, 1fr); } /* empieza con 3 */ @media (max-width: 1024px) { /* ... tiene que reducir a 2 */ } @media (max-width: 640px) { /* ... tiene que reducir a 1 */ }

CONCEPTO 2 — Media Queries
La herramienta que hace posible el diseño responsive

Una Media Query es una condición CSS: "aplica estos estilos solo si la pantalla cumple esta condición". Es como un if en programación pero para estilos.

🔭 Analogía: el telescopio adaptable

Un telescopio moderno detecta la distancia del objeto y ajusta automáticamente el zoom, el filtro y el enfoque. Tu página web hace lo mismo con las Media Queries: detecta el tamaño de la pantalla y ajusta columnas, tamaños de fuente, márgenes y todo lo necesario para que el contenido se vea perfecto en esa pantalla específica.

📱 Celular 320–639px
📲 Celular grande 640–767px
📟 Tablet 768–1023px
🖥 Escritorio 1024px+
CSS/* Anatomía de una Media Query */ @media (condición) { /* Estilos que SOLO aplican si se cumple la condición */ } /* ── Los 4 breakpoints más usados ── */ @media (min-width: 640px) { } /* celulares grandes / iPhone 15 */ @media (min-width: 768px) { } /* tablets verticales / iPad */ @media (min-width: 1024px) { } /* laptops y tablets horizontales */ @media (min-width: 1280px) { } /* escritorios y monitores Full HD */ /* ── Condiciones combinadas ── */ @media (min-width: 640px) and (max-width: 1023px) { } /* SOLO tablet */ /* ── Orientación ── */ @media (orientation: landscape) { } /* pantalla horizontal */ @media (orientation: portrait) { } /* pantalla vertical */ /* ── Preferencias del sistema operativo ── */ @media (prefers-color-scheme: dark) { :root { --fondo: #111827; --surface: #1F2937; } /* modo oscuro */ } @media (prefers-reduced-motion: reduce) { * { animation: none !important; transition: none !important; } }
min-widthDispositivos que cubre
Sin @media (base)Celulares pequeños (320–639px). iPhone SE, Android gama baja. Es el punto de partida Mobile-First.
640pxCelulares grandes. iPhone 15, Samsung Galaxy S.
768pxTablets verticales. iPad en modo retrato.
1024pxTablets horizontales y laptops pequeñas. iPad Pro horizontal, MacBook Air 11".
1280pxLaptops y escritorios estándar. MacBook Pro 13", monitores Full HD.

CONCEPTO 3 — CSS Grid avanzado
grid-template-areas, minmax(), auto-fit y grid-column: span

grid-template-areas — El mapa del layout

🗺️ Analogía: el plano de un departamento

grid-template-areas es como hacer el plano de un departamento en papel cuadriculado. Dibujas las habitaciones por nombre: "sala sala / cocina baño / cuarto cuarto". Luego le dices a cada elemento dónde vive: "tú eres la sala", "tú eres la cocina". El navegador los coloca automáticamente en su lugar. En celular puedes reorganizar el plano completamente con solo una Media Query.

/* Celular — todo en 1 columna */
grid-template-areas:
  "nav" fila 1
  "hero" fila 2
  "main" fila 3
  "footer";

/* Escritorio — sidebar a la derecha */
grid-template-areas:
  "nav nav" navbar ocupa las 2 columnas
  "hero hero"
  "main sidebar" main + sidebar en fila
  "footer footer";
CSS/* El contenedor padre define las áreas */ .pagina { display: grid; grid-template-columns: 1fr; /* celular: 1 columna */ grid-template-areas: "nav" "hero" "main" "footer"; } @media (min-width: 1024px) { .pagina { grid-template-columns: 1fr 280px; /* escritorio: main + sidebar */ grid-template-areas: "nav nav" "hero hero" "main sidebar" "footer footer"; } } /* Cada elemento va a su área */ .navbar { grid-area: nav; } .hero { grid-area: hero; } .main { grid-area: main; } .sidebar { grid-area: sidebar; } .pie { grid-area: footer; }

minmax(), auto-fit y auto-fill

CSS/* auto-fit + minmax: grid que se adapta SOLO — sin Media Queries */ .galeria { display: grid; grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)); /* ↑ ↑ ↑ */ /* auto-fit mínimo máximo */ gap: 1.5rem; } /* Resultado automático según el ancho disponible: */ /* En celular (350px): 1 columna de 350px */ /* En tablet (750px): 2 columnas de ~360px */ /* En escritorio (1200px): 3 columnas de ~373px */ /* ¡Sin una sola Media Query! */ /* grid-column: span — un elemento ocupa varias columnas */ .proyecto-destacado { grid-column: 1 / -1; /* ocupa TODAS las columnas (-1 = última línea) */ } /* repeat() simplifica columnas repetitivas */ grid-template-columns: 1fr 1fr 1fr; /* ← largo */ grid-template-columns: repeat(3, 1fr); /* ← igual, más limpio */
Función / ValorExplicación
repeat(3, 1fr)Repite 3 veces 1fr. Equivale a escribir 1fr 1fr 1fr. Más limpio y fácil de cambiar.
repeat(auto-fit, ...)El número de columnas se calcula automáticamente según el ancho disponible.
minmax(280px, 1fr)La columna es mínimo 280px y máximo 1 fracción del espacio. Nunca se hace más pequeña de 280px.
grid-column: 1 / -1El elemento empieza en la línea 1 y termina en la última (-1). Ocupa todas las columnas del grid.
grid-area: nombreUbica el elemento en la zona nombrada del grid-template-areas.

CONCEPTO 4 — Unidades y funciones avanzadas
clamp(), min(), vw, vh, aspect-ratio y object-fit
CSS/* clamp(mínimo, ideal, máximo) — tipografía fluida */ h1 { font-size: clamp(1.8rem, 4vw, 3.5rem); /* En celular (360px): 4vw = 14.4px → usa el mínimo → 1.8rem */ /* En tablet (768px): 4vw = 30.7px → usa ese valor */ /* En escritorio (1400px): 4vw = 56px → usa el máximo → 3.5rem */ } /* min(a, b) — usa el valor MÁS PEQUEÑO */ .foto { width: min(280px, 80vw); /* En pantalla de 400px: 80vw = 320px → usa el mínimo → 280px */ /* En pantalla de 300px: 80vw = 240px → usa 240px (cede a 80vw) */ } /* aspect-ratio — proporciones fijas sin calcular alturas */ .thumbnail { width: 100%; aspect-ratio: 16 / 9; /* siempre formato 16:9 como un video */ overflow: hidden; } .thumbnail img { width: 100%; height: 100%; object-fit: cover; /* llena el espacio, recorta bordes si es necesario */ } /* vw y vh — relativos al viewport */ .hero { min-height: 90vh; } /* 90% de la altura de la pantalla */ .titulo { font-size: 5vw; } /* 5% del ancho de la pantalla */
Función / UnidadExplicación
clamp(min, ideal, max)El valor ideal (en vw) escala con el viewport, pero nunca baja del mínimo ni sube del máximo. Tipografía fluida sin Media Queries.
vwViewport Width. 1vw = 1% del ancho de la ventana. Con 4vw, el texto crece conforme crece la pantalla.
vhViewport Height. 1vh = 1% del alto de la ventana. 100vh = pantalla completa de alto.
min(a, b)Devuelve el valor más pequeño entre los dos. width: min(100%, 1200px) es el reemplazo moderno de max-width + width: 100%.
aspect-ratio: 16/9Mantiene proporciones ancho/alto fijas. Sin esto tendrías que calcular padding-top en % (la técnica antigua).
object-fit: coverPara imágenes: rellena el espacio definido y recorta los bordes si la proporción no coincide. Como background-size: cover pero para <img>.

💡 style="width: 85%" — por qué está en el HTML y no en el CSS

El ancho de cada barra es un dato específico de esa habilidad. El CSS da el estilo (altura, color de fondo, animación). El HTML da el valor. La animación @keyframes llenar empieza desde width: 0 y el navegador interpola hasta el ancho definido en el style inline.

CSS BLOQUE 1 — Reset, Variables y Base

🔀 La técnica display:none ↔ display:flex

En móvil: .nav-toggle { display: block } (visible) y .nav-menu { display: none } (oculto). En tablet+: se intercambian con @media. El menú hamburguesa desaparece y el menú horizontal aparece. Es la técnica más usada en diseño responsive.

PARTE 5 — Retos para practicar
Del más sencillo al más desafiante
🟡 Fácil

Reto 1 — Modo oscuro automático

Ya está en el CSS el bloque @media (prefers-color-scheme: dark). Activa el modo oscuro en tu sistema operativo (Configuración → Apariencia → Oscuro) y observa cómo toda la página cambia automáticamente.

Lo que practicas: Media Queries de preferencias del usuario.

🟠 Medio

Reto 2 — 4° proyecto

Agrega un 4° <article class="proyecto-card"> en el HTML. Observa cómo el Grid lo acomoda automáticamente: en celular nueva fila, en tablet segunda fila, en escritorio llena el hueco de la última fila.

Lo que practicas: cómo Grid responsive distribuye elementos automáticamente.

🟠 Medio

Reto 3 — Animación de entrada escalonada

Añade estas reglas CSS para que las tarjetas aparezcan en secuencia:

.proyecto-card { animation: aparecer 0.5s ease forwards; opacity: 0; }

.proyecto-card:nth-child(2) { animation-delay: 0.15s; }

.proyecto-card:nth-child(3) { animation-delay: 0.30s; }

Y crea @keyframes aparecer con opacity: 0 → 1 y translateY(20px → 0).

⚫ Avanzado

Reto 4 — Breakpoint extra pequeño

Agrega una Media Query de max-width: 360px para celulares muy pequeños (iPhone SE primera generación). Reduce el padding del hero a 2rem 1rem y el font-size del .hero-nombre al mínimo del clamp. Pruébalo seleccionando "iPhone SE" en las DevTools.

Ejercicio 5 = Ejercicios 1+2+3+4 + todo esto
ConceptoLo que aprendiste
Mobile-FirstEscribir estilos base para celular y añadir con min-width. La filosofía profesional estándar. Nunca quitas estilos, solo añades capas.
@media (min-width:)La herramienta del responsive. Aplica estilos solo cuando la pantalla tiene al menos X píxeles. Los 4 breakpoints estándar: 640, 768, 1024, 1280.
grid-template-areasNombrar zonas del layout como un mapa. Cada elemento se ubica con grid-area: nombre. Reorganizas todo el layout en una Media Query.
repeat(auto-fit, minmax())Grid que calcula automáticamente cuántas columnas caben. Sin Media Queries para galerías de herramientas.
grid-column: 1 / -1Un elemento ocupa todas las columnas. -1 = la última línea del grid.
clamp(min, ideal, max)Tipografía fluida: escala con vw pero respeta límites. Sin Media Queries para el tamaño del texto.
min(a, b)Usa el valor más pequeño. width: min(280px, 80vw) reemplaza la combinación width + max-width.
aspect-ratioMantiene proporciones fijas. Esencial para thumbnails de proyectos. Sin calcular alturas en %.
object-fit: coverImágenes que llenan su espacio sin deformarse. Recorta bordes si la proporción no coincide.
img { max-width: 100% }La regla fundamental de imágenes responsive. Siempre en el reset.
scroll-behavior: smoothLos links de ancla hacen scroll animado suave. Una línea en html { }.
display:none ↔ display:flexTécnica hamburguesa: ocultar/mostrar el menú y el botón según el breakpoint.
prefers-color-schemeMedia Query que detecta el modo oscuro del sistema operativo. Cambia toda la página con Variables CSS.
prefers-reduced-motionAccesibilidad: desactiva animaciones para usuarios que lo prefieren por razones de salud.
