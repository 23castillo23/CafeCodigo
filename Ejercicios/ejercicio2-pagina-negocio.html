El Box Model — El concepto más importante de CSS
Todo elemento HTML es una caja. Aquí aprendes cómo funciona esa caja

En CSS, cada elemento es una caja rectangular. No importa si es un párrafo, un título, una imagen o un div: todos son cajas. El Box Model define cómo se calcula el tamaño y el espacio de esa caja.

📦 Analogía: enviar un producto por paquetería

Imagina que vendes una taza de cerámica por internet. La taza en sí es el contenido. Para protegerla, la rodeas de plástico de burbujas: eso es el padding (espacio interior). Luego la metes en una caja de cartón con sus paredes: eso es el border (el borde). Finalmente, en la bodega dejas espacio entre tu caja y las cajas de otros vendedores: eso es el margin (espacio exterior).

Capa¿Qué es?Color en el diagrama

margin y padding: shorthand (forma corta)

En lugar de escribir cuatro propiedades, puedes escribirlas todas en una sola línea. A esto se le llama shorthand.

CSS/* ── PADDING shorthand ── */ /* 1 valor: aplica a los 4 lados */ .caja { padding: 20px; } /* arriba derecha abajo izquierda = todos 20px */ /* 2 valores: arriba/abajo izquierda/derecha */ .caja { padding: 20px 40px; } /* ↑↓ 20px ←→ 40px */ /* 4 valores: sentido del reloj ↑ → ↓ ← */ .caja { padding: 10px 20px 30px 40px; } /* arriba derecha abajo izquierda */ /* ── MARGIN funciona exactamente igual ── */ .caja { margin: 0 auto; } /* centra horizontalmente */ .caja { margin-top: 24px; } /* solo arriba */

🕐 Memoriza el orden: sentido del reloj

Cuando escribes 4 valores, siempre van en orden de las manecillas del reloj: arriba → derecha → abajo → izquierda. Una forma de recordarlo: "TRouBLe" (Top, Right, Bottom, Left).

El problema de box-sizing

Por defecto, cuando le das width: 200px a un elemento, ese es el ancho del contenido. Si luego le agregas padding y border, el elemento crece más de 200px — y eso rompe layouts.

CSS/* ❌ Comportamiento por defecto (box-sizing: content-box) */ .caja { width: 200px; padding: 20px; border: 5px solid black; } /* Ancho real = 200 + 20 + 20 + 5 + 5 = 250px ← SORPRESA 😱 */ /* ✅ Con box-sizing: border-box (lo que ya usas en el reset) */ * { box-sizing: border-box; } .caja { width: 200px; padding: 20px; border: 5px solid black; } /* Ancho real = 200px exactos. El padding y border se restan por dentro. ✅ */

💡 Por eso el reset siempre empieza con box-sizing: border-box

En el Ejercicio 1 pusiste * { box-sizing: border-box; } en el reset y no supiste por qué. Ahora ya lo sabes: sin esa línea, el padding y el border agrandan el elemento más allá del width que tú definiste. Con ella, el width siempre es el ancho final — exacto y predecible.

margin: 0 auto — el truco de centrado

CSS/* Para centrar un bloque horizontalmente necesitas 2 cosas: */ /* 1. Que el elemento sea de bloque (div, p, section...) */ /* 2. Que tenga un width definido (no 100%) */ .contenedor { width: 800px; /* o max-width */ margin: 0 auto; /* 0 = arriba/abajo, auto = izquierda/derecha se reparten igual */ } /* max-width es mejor que width fijo porque en pantallas pequeñas */ /* el elemento puede achicarse en lugar de salirse de la pantalla */ .contenedor { max-width: 800px; margin: 0 auto; }

⚠️ margin: 0 auto NO centra texto

Este truco centra el bloque completo dentro de su contenedor padre. Para centrar el texto dentro del bloque, sigues usando text-align: center. Son dos cosas diferentes: centrar el contenedor vs centrar el contenido.

💡 ¿Por qué un <a> en lugar de <button>?

Porque este "botón" nos lleva a otra sección de la página (lleva un href). Los <button> son para acciones de JavaScript (enviar formularios, abrir menús). Si el elemento navega: usa <a>. Si ejecuta una acción: usa <button>.

⚠️ ¿Por qué <span> y no <p> para el ícono?

Es tentador usar <p class="tarjeta-icono"> porque parece un simple párrafo más. El problema aparece más adelante en el CSS: la regla .tarjeta p (que le da estilo al texto de descripción) selecciona cualquier <p> dentro de .tarjeta — incluido el del ícono. Y como .tarjeta p tiene más especificidad (clase + etiqueta) que .tarjeta-icono (solo clase), termina ganando y aplastando el font-size: 36px del ícono con el font-size: 14px pensado para los párrafos normales.

Por eso el ícono va en un <span>: como no es un <p>, la regla .tarjeta p nunca lo toca, y .tarjeta-icono manda sin competencia. Es un ejemplo real de cómo la especificidad CSS puede arruinar un estilo aunque el selector "correcto" también exista en tu hoja — conviene revisarlo cuando un elemento no toma el tamaño o color que le pusiste.

💡 ¿Qué es <span> exactamente?

<span> es una etiqueta genérica, igual que <div>: no tiene significado semántico, solo sirve para poder aplicarle una clase de CSS. La diferencia es el tipo de elemento:

  • <div> es de bloque: siempre ocupa todo el ancho disponible y empieza en línea nueva. Se usa para agrupar secciones grandes.
  • <span> es en línea: solo ocupa el espacio de su contenido y se acomoda dentro del flujo del texto, sin romper la línea. Se usa para estilizar una parte pequeña de un párrafo o título, como El precio es <span class="oferta">$99</span>.

En nuestro caso, el ícono va en <span> por dos razones: es un elemento pequeño que no necesita comportarse como bloque, y al no ser un <p> evita el choque de especificidad con .tarjeta p que vimos arriba.

🔍 <section> y <div> en el mismo bloque — ¿por qué van juntos?

Mira el código de arriba: hay un <section> por fuera y varios <div> por dentro. No es un error ni algo intercambiable — casi siempre se anidan porque cada uno hace un trabajo distinto:

  • <section> = bloque con significado. Le dice al navegador, a Google y a los lectores de pantalla "aquí empieza un tema completo" — por eso trae su propio <h2>Nuestros servicios</h2>.
  • <div> = una caja sin significado, solo para agrupar visualmente y poder aplicarle CSS. El <div class="tarjetas"> no le dice nada a nadie sobre el contenido — solo existe para poder ponerle display: flex y acomodar las tarjetas en fila.

Pregunta clave para decidir cuál usar: ¿este bloque tendría sentido como su propio tema si lo sacaras de la página? Si sí → section. Si solo lo necesitas para acomodar visualmente algo que ya está dentro de una section (como .tarjetas o .contenedor) → div.

Por eso ves id y class juntos en la misma etiqueta, como <section id="servicios" class="seccion-servicios">: el id es único en toda la página y sirve para que el link <a href="#servicios"> salte exactamente ahí; la class se puede repetir en varios elementos y sirve para compartir estilos de CSS. No compiten entre sí — cada uno responde una pregunta distinta, y muchas veces necesitas los dos a la vez.

🟢 El patrón .contenedor que usamos en todo el HTML

Notaste que casi todas las secciones tienen un <div class="contenedor"> adentro. Ese div limita el ancho del contenido a 900px y lo centra. PARTE 3 — Escribe el CSS
Aquí es donde el Box Model entra en acción en cada sección

CSS — PARTE 1: Reset y variables de color

CSS/* ── RESET ── */ * { margin: 0; padding: 0; box-sizing: border-box; /* ← el héroe del Box Model */ } /* ── VARIABLES DE COLOR ── */ :root { --azul-oscuro: #1e3a5f; --azul-medio: #2d5986; --amarillo: #f59e0b; --gris-claro: #f0f4f8; --texto: #1a1814; --texto-suave: #64748b; --borde: #dde4ee; } /* ── BODY ── */ body { font-family: Arial, Helvetica, sans-serif; font-size: 16px; color: var(--texto); line-height: 1.6; } /* ── PATRÓN CONTENEDOR (se usa en toda la página) ── */ .contenedor { max-width: 900px; /* ancho máximo */ margin: 0 auto; /* centra horizontalmente */ padding: 0 24px; /* respiro a los lados en móvil */ }

💡 Variables CSS con :root

Las variables CSS (--nombre: valor) te permiten definir un color una sola vez y reutilizarlo en todo el archivo. Si mañana quieres cambiar el azul, lo cambias en un solo lugar y se actualiza en todo el sitio. Se usan así: color: var(--azul-oscuro).

En el Ejercicio 3 ampliarás este patrón guardando también medidas (no solo colores) con variables como --radio-borde: 12px, y las usarás dentro de reglas Flexbox.

CSS — PARTE 2: Navbar

Aquí practicamos padding para dar espacio interior a la barra y border-bottom para la línea decorativa.

CSS/* ── NAVBAR ── */ #navbar { background-color: var(--azul-oscuro); border-bottom: 4px solid var(--amarillo); /* línea amarilla decorativa */ } .nav-contenedor { max-width: 900px; margin: 0 auto; padding: 16px 24px; /* 16px arriba/abajo, 24px izquierda/derecha */ display: flex; /* ← alinea logo y links en la misma fila */ justify-content: space-between; align-items: center; } .logo { color: white; font-size: 22px; font-weight: 800; } .nav-links { list-style: none; /* quita los bullets de la lista */ display: flex; gap: 24px; /* espacio entre los enlaces */ } .nav-links a { color: rgba(255,255,255,0.85); text-decoration: none; font-size: 15px; padding: 4px 0; /* área clicable más grande */ border-bottom: 2px solid transparent; transition: border-color 0.2s; } .nav-links a:hover { border-bottom-color: var(--amarillo); }
Propiedad nuevaExplicación
list-style: none;Quita los bullets (•) de los <ul>. Necesario para convertir una lista en menú de navegación.
gap: 24px;En un contenedor flex o grid, gap pone espacio entre los hijos sin afectar los extremos. Es más limpio que usar margin en cada elemento.
border-bottom: 2px solid transparent;Crea un borde de 2px pero invisible. Al hacer hover lo hacemos amarillo. Así el texto no "salta" al aparecer el borde.
transition: border-color 0.2s;Anima el cambio de color del borde en 0.2 segundos. Sin esto el cambio sería instantáneo y brusco.

CSS — PARTE 3: Hero

Mucho padding para crear impacto visual y max-width + margin: 0 auto para limitar el texto.

CSS/* ── HERO ── */ .hero { background-color: var(--azul-oscuro); color: white; text-align: center; padding: 80px 24px; /* mucho padding arriba/abajo = sección grande */ } .hero-contenido { max-width: 600px; /* el texto no se estira más de 600px */ margin: 0 auto; /* centra ese bloque de 600px dentro del hero */ } .hero h1 { font-size: 42px; font-weight: 800; margin-bottom: 16px; /* espacio entre el h1 y el párrafo */ } .hero p { font-size: 18px; opacity: 0.85; margin-bottom: 32px; /* espacio entre el párrafo y el botón */ } .btn-principal { display: inline-block; /* necesario para que padding funcione en <a> */ background-color: var(--amarillo); color: #1a1814; font-weight: 700; text-decoration: none; padding: 14px 32px; /* el padding crea el "cuerpo" del botón */ border-radius: 8px; border: 2px solid transparent; transition: background-color 0.2s, border-color 0.2s; } .btn-principal:hover { background-color: transparent; color: var(--amarillo); border-color: var(--amarillo); }

🔍 ¿Qué significa .hero p? — selector descendiente

El espacio entre .hero y p no es un error ni un adorno: es un tipo de selector llamado selector descendiente. Significa "cualquier <p> que esté adentro de un elemento con clase hero" — el <p> no necesita tener su propia clase, porque el CSS también puede buscar elementos por dónde están ubicados en el HTML, no solo por su clase o id.

EscribesSignifica
.heroEl elemento que tiene class="hero" (la caja completa)
.hero pCon espacio → cualquier <p> que esté adentro de .hero, tenga clase o no
p.heroSin espacio → un <p> que él mismo tenga class="hero" (muy distinto al anterior)

Este mismo patrón se repite todo el ejercicio: .nav-links a, .seccion-servicios h2. Siempre dice lo mismo: "el elemento X que vive dentro de la caja con esta clase".

💡 display: inline-block en el botón — ¿por qué?

Los elementos <a> son inline por defecto. Los elementos inline ignoran el padding vertical y el width. Al ponerle display: inline-block le damos lo mejor de los dos mundos: se queda en la misma línea (inline) pero acepta padding, width y height (block). Sin esta línea, el botón se vería aplastado.

CSS — PARTE 4: Tarjetas de servicios

Aquí practicamos el Box Model completo: cada tarjeta tiene padding interno, border visible y margin entre tarjetas (usando gap).

CSS/* ── SECCIÓN SERVICIOS ── */ .seccion-servicios { background-color: var(--gris-claro); padding: 64px 0; /* espacio arriba/abajo, el .contenedor maneja los lados */ } .seccion-servicios h2 { font-size: 30px; color: var(--azul-oscuro); border-bottom: 3px solid var(--amarillo); padding-bottom: 8px; /* espacio entre el texto y la línea */ margin-bottom: 8px; /* espacio entre el h2 y el párrafo */ } .seccion-descripcion { color: var(--texto-suave); margin-bottom: 40px; /* espacio entre el párrafo y las tarjetas */ } .tarjetas { display: flex; gap: 20px; /* espacio entre tarjetas (reemplaza margin lateral) */ } .tarjeta { background-color: white; border: 1px solid var(--borde); /* borde sutil */ border-radius: 12px; padding: 24px; /* espacio interior generoso */ flex: 1; /* cada tarjeta ocupa el mismo espacio */ border-top: 4px solid var(--azul-oscuro); /* acento decorativo arriba */ } .tarjeta-icono { /* es un <span>, no un <p>, para que .tarjeta p no lo pise */ font-size: 36px; margin-bottom: 12px; } .tarjeta h3 { font-size: 18px; color: var(--azul-oscuro); margin-bottom: 8px; } .tarjeta p { font-size: 14px; color: var(--texto-suave); line-height: 1.7; }
PropiedadExplicación
flex: 1;Le dice a cada tarjeta que ocupe una fracción igual del espacio disponible. Si hay 3 tarjetas con flex: 1, cada una ocupa exactamente un tercio.
border-top: 4px solid ...Solo el borde superior, más grueso y de color diferente. Técnica de acento visual muy usada en diseño de tarjetas.

Etiquetas semánticas: cuándo y por qué usarlas
La diferencia entre <section>, <div>, <address>, <main> y <nav>

Una etiqueta semántica le dice al navegador (y a Google) qué tipo de contenido hay dentro, no solo cómo se ve. Esto ayuda a la accesibilidad, al SEO y a que tu código sea más fácil de leer.

🏠 Analogía: las habitaciones de una casa

Imagina que todas las habitaciones de tu casa tuvieran la misma puerta genérica sin letrero. Podrías entrar a cualquiera, pero no sabrías si es la cocina, el baño o la recámara hasta ver adentro. Las etiquetas semánticas son como poner letreros en las puertas: <nav> = pasillo de entrada, <main> = sala principal, <section> = habitación con tema propio, <address> = directorio de contactos. El <div> sería la caja sin letrero — útil, pero no dice nada.

Las 5 etiquetas que necesitas dominar

Etiqueta¿Para qué sirve?¿Cuándo usarla?
<nav> Menú de navegación principal del sitio. Cuando tienes un grupo de enlaces que llevan a otras secciones o páginas: el menú de arriba, un menú lateral. No para cualquier grupo de enlaces.
<main> El contenido principal de la página. Solo existe uno por página. Contiene todo lo importante: las secciones, artículos, etc. El <header>, <nav> y <footer> quedan fuera.
<section> Un bloque de contenido con tema propio. Cuando el contenido tiene su propio título (h2 o h3). Regla fácil: si le puedes poner un título, usa <section>. Ejemplos: "Nuestros Servicios", "Sobre Nosotros", "Contacto".
<div> Una caja genérica sin significado especial. Solo para agrupar cosas con fines de diseño/CSS, cuando ninguna etiqueta semántica encaja. Las tarjetas .demo-card del ejercicio son <div> porque son cajas visuales, no secciones con significado propio.
<address> Información de contacto. Para teléfono, email, dirección física, redes sociales del negocio o autor. En este ejercicio la usarás en la sección de contacto del Café Roble. No para cualquier dirección postal genérica.

💡 La diferencia clave: <section> vs <div>

<section> tiene significado semántico: le dice al navegador "esto es una parte importante del contenido con tema propio". <div> no dice nada, solo agrupa visualmente. Como regla práctica: usa siempre la etiqueta más específica que encaje. Solo llega al <div> cuando ninguna otra funciona.

Cómo se estructuran en una página real

Así quedaría la estructura del Café Roble usando estas etiquetas correctamente:

HTML<header> <!-- Logo y título del sitio --> </header> <nav> <!-- Menú: Inicio | Servicios | Contacto --> <a href="#servicios">Servicios</a> <a href="#contacto">Contacto</a> </nav> <main> <!-- Solo uno por página --> <section id="servicios"> <!-- Tiene su propio título --> <h2>Nuestros servicios</h2> <div class="tarjetas"> <!-- Solo agrupa visualmente --> <div class="tarjeta">☕ Café de especialidad</div> <div class="tarjeta">🥐 Repostería artesanal</div> <div class="tarjeta">💻 Espacio de trabajo</div> </div> </section> <section id="contacto"> <h2>Encuéntranos</h2> <address> <!-- Solo para datos de contacto --> <p>📍 Calle Roble 42, Centro Histórico</p> <p>📞 <a href="tel:2291234567">229 123 4567</a></p> <p>✉️ <a href="mailto:hola@caferoble.com">hola@caferoble.com</a></p> </address> </section> </main> <footer> <!-- Pie de página --> </footer>

✅ Regla práctica para elegir la etiqueta correcta

  • ¿Es la barra de menú del sitio? → <nav>
  • ¿Es todo el contenido importante de la página? → <main> (solo uno)
  • ¿Tiene su propio título (h2/h3) y tema definido? → <section>
  • ¿Es solo una caja para aplicar CSS o agrupar visualmente? → <div>
  • ¿Son datos de contacto (tel, email, dirección)? → <address>
  • ¿Es contenido que tendría sentido fuera de esta página? → <article>

⚠️ Error común: usar <div> para todo

Al principio es tentador usar <div> para todo porque "funciona igual visualmente". El problema es que pierdes información importante para el navegador, para los lectores de pantalla (accesibilidad) y para Google (SEO). Un sitio bien estructurado semánticamente se posiciona mejor y es más accesible para personas con discapacidades visuales.
