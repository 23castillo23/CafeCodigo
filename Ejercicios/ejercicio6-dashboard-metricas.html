CSS — Propiedades en uso que no conocías aún

Propiedades presentes en el código de este ejercicio que aún no tenían referencia.

PropiedadExplicación
white-space: nowrap;Impide que el texto se rompa en varias líneas. El contenido se desborda horizontalmente en lugar de saltar de línea. Útil en celdas de tabla, labels de sidebar y elementos que no deben partirse.
overflow-x / overflow-y:Controla el desbordamiento solo en el eje horizontal o vertical. overflow-x: auto agrega scroll horizontal solo cuando el contenido es más ancho — ideal para tablas en móvil.
pointer-events: none;Hace que el elemento ignore todos los eventos del ratón (clics, hover, etc.) como si no existiera. Los clics "pasan a través" hacia los elementos de abajo. Muy usado en tooltips y overlays decorativos.
border-top-color:Cambia solo el color del borde superior, sin redefinir el grosor ni estilo. Permite sobrescribir un solo aspecto del borde en variantes o estados. Funciona igual para border-right-color, etc.
max-height:Altura máxima. El elemento puede ser más bajo, pero nunca más alto que este valor. Combinado con overflow-y: auto crea áreas de scroll limitadas (como el feed de actividad del dashboard).
scroll-behavior: smooth;Hace que los saltos de ancla (href="#seccion") se animen suavemente en lugar de saltar de golpe. Va en el selector html o body. Un detalle de pulido que mejora mucho la experiencia.
border-right:Solo el borde derecho. Mismo patrón que border-left y border-bottom. En el dashboard se usa en el sidebar para separarlo visualmente del contenido principal.

CONCEPTO 1 — La propiedad position
Los 5 valores que todo desarrollador web debe conocer de memoria

Cada elemento HTML tiene un valor de position. Por defecto es static. Cambiarlo modifica completamente cómo el navegador lo coloca en la página.

🏗️ Analogía: los tipos de trabajadores en una obra

static es el trabajador normal: ocupa su lugar en la fila, no se mueve. relative es el trabajador que puede desplazarse un poco de su posición original sin afectar a los demás. absolute es el trabajador que vuela — sale del flujo, se posiciona exactamente donde tú le dices relativo a su jefe. fixed es el capataz que está pegado a la cámara — no importa cuánto te desplaces, siempre está visible. sticky es el trabajador que está en su lugar pero cuando llegas a él, se queda pegado al borde de la pantalla mientras sigues haciendo scroll.

Valor¿Qué hace?¿Usa top/left/right/bottom?¿Sale del flujo?
staticComportamiento por defecto. El elemento va donde le toca según el HTML.No — las ignoraNo
relativeSe desplaza desde su posición original. El hueco que ocupaba sigue reservado.Sí — desplazamiento desde origenNo
absoluteSale del flujo. Se posiciona relativo al ancestro posicionado más cercano (cualquiera que no sea static).Sí — coordenadas exactasSí
fixedSale del flujo. Se posiciona relativo a la ventana del navegador. No se mueve con el scroll.Sí — coordenadas exactasSí
stickyFlujo normal hasta que llega al borde definido — entonces se comporta como fixed. Vuelve al flujo cuando su contenedor desaparece.Sí — define el punto de "pegado"No

La regla más importante de position: absolute

⚠️ La regla que nadie te había dicho explícitamente

Un elemento con position: absolute busca su ancestro posicionado más cercano para tomar sus coordenadas. "Posicionado" significa cualquier ancestro cuyo position no sea static. Si no encuentra ninguno, se posiciona relativo al <body>. Por eso casi siempre vas a ver position: relative en el padre de un elemento absolute — el padre no se mueve, solo actúa como referencia.

CSS/* El patrón más usado en el mundo real: */ /* 1. El PADRE se convierte en referencia */ .tarjeta { position: relative; /* no se mueve, solo actúa como ancla */ } /* 2. El HIJO se posiciona dentro del padre */ .badge-notificacion { position: absolute; top: -8px; /* 8px arriba del borde superior del padre */ right: -8px; /* 8px fuera del borde derecho del padre */ } /* Para top/left/right/bottom: */ /* Valores POSITIVOS alejan del borde. Negativos van más allá del borde. */ top: 0; /* pegado al borde superior del padre */ bottom: 0; /* pegado al borde inferior del padre */ left: 50%; /* en el centro horizontal del padre */ right: 0; /* pegado al borde derecho del padre */ /* Centrar con absolute */ .centrado { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); /* ajusta el offset del propio elemento */ }

position: sticky — cómo funciona de verdad

Lo usaste en la navbar de todos los ejercicios. Aquí está la explicación completa.

CSS.navbar { position: sticky; top: 0; /* ← define el punto de "pegado". Cuando llegue a 0px del tope, se fija */ z-index: 100; /* ← para que quede por encima del contenido al hacer scroll */ } /* sticky tiene 2 limitaciones importantes: */ /* 1. Solo funciona si el contenedor padre tiene altura suficiente */ /* 2. Si el padre tiene overflow: hidden, el sticky no funciona */ /* position: fixed — diferencia con sticky */ .chat-boton { position: fixed; bottom: 24px; right: 24px; /* Se posiciona relativo a la VENTANA, no al documento */ /* Siempre visible sin importar cuánto se haga scroll */ }

CONCEPTO 2 — z-index: el orden de apilamiento
¿Quién va encima de quién cuando los elementos se superponen?

📚 Analogía: la pila de papeles en un escritorio

Imagina que tienes papeles apilados en un escritorio. El que está más arriba es el que ves. El z-index es el número de orden: papel con z-index 3 va encima del 2, que va encima del 1. Si quieres que el menú desplegable aparezca por encima de todo el contenido, le das un z-index alto. El topbar de esta guía tiene z-index: 200, la navbar de guía tiene z-index: 100, y el contenido normal tiene z-index implícito de 0.

CSS/* z-index solo funciona en elementos posicionados (no static) */ .contenido { position: relative; z-index: 1; } .sidebar { position: relative; z-index: 10; } .dropdown { position: absolute; z-index: 50; } .modal { position: fixed; z-index: 100; } .toast { position: fixed; z-index: 200; } .tooltip { position: absolute; z-index: 300; } /* Buena práctica: definir z-indexes como variables CSS */ :root { --z-base: 1; --z-sidebar: 10; --z-navbar: 100; --z-modal: 200; --z-tooltip: 300; } /* Ventaja: sabes exactamente qué valor le corresponde a cada "capa" */ .navbar { z-index: var(--z-navbar); }

💡 La regla de z-index que confunde a todos

z-index solo funciona en elementos cuyo position no sea static. Si le pones z-index: 999 a un elemento con position: static (el valor por defecto), no pasará absolutamente nada. Primero ponle position: relative, absolute, fixed o sticky — luego el z-index funcionará.

CONCEPTO 3 — overflow: qué pasa cuando el contenido no cabe
Controlar qué hace el navegador cuando el contenido desborda su contenedor

🛁 Analogía: la bañera y el agua

Tu contenedor es una bañera con un tamaño fijo. El contenido es el agua. overflow: visible (default) = el agua se derrama y moja el suelo — se ve fuera de la bañera. overflow: hidden = hay una tapa — el agua que sobra desaparece. overflow: scroll = hay un desagüe visible con una palanca siempre presente. overflow: auto = el desagüe aparece solo si hay demasiada agua.

CSS/* Los 4 valores principales */ .caja { overflow: visible; } /* default — el contenido se derrama fuera */ .caja { overflow: hidden; } /* recorta — el contenido extra desaparece */ .caja { overflow: scroll; } /* barras de scroll siempre visibles */ .caja { overflow: auto; } /* barras solo cuando hacen falta ← el más usado */ /* overflow-x y overflow-y por separado */ .tabla-wrap { overflow-x: auto; /* scroll horizontal si la tabla es ancha */ overflow-y: visible; /* vertical sin cambios */ } /* Scroll personalizado (Webkit) */ .panel-scroll::-webkit-scrollbar { width: 4px; } .panel-scroll::-webkit-scrollbar-track { background: transparent; } .panel-scroll::-webkit-scrollbar-thumb { background: #334155; border-radius: 4px; } /* overflow: hidden tiene 2 usos importantes: */ /* 1. Recortar imágenes al hacer hover (zoom que no sale de la tarjeta) */ .proyecto-img { overflow: hidden; } .proyecto-img img:hover { transform: scale(1.05); } /* no se sale ✅ */ /* 2. Crear la máscara de las barras de progreso animadas (Ejercicio 3) */ .barra-contenedor { overflow: hidden; border-radius: 20px; } .barra-progreso { animation: llenar 1.5s ease; } /* empieza en 0 sin salirse */
ValorQué haceCuándo usarlo
visibleEl contenido se muestra fuera del contenedor. Sin barras.Default. Casi nunca se escribe explícitamente.
hiddenRecorta el contenido que sobra. No muestra scroll.Zoom de imágenes, barras de progreso, elementos decorativos que no deben salirse.
scrollMuestra barras de scroll siempre, aunque no hagan falta.Raro. Generalmente es mejor auto.
autoBarras de scroll solo cuando el contenido desborda.Tablas anchas, listas largas, sidebars con muchos items, cualquier área scrollable.

CONCEPTO 4 — calc(): mezclar unidades en CSS
El problema que CSS no podía resolver antes de esta función

🧮 Analogía: la receta con medidas mixtas

Imagina que tienes que llenar un molde y sabes que el molde mide "toda la encimera menos 30cm para el fregadero". No puedes calcular eso sin saber el ancho exacto de la encimera. calc() resuelve exactamente ese problema: width: calc(100% - 30px) significa "todo el ancho disponible menos 30px fijos" — CSS lo calcula en tiempo real según el tamaño de la pantalla.

CSS/* EL PROBLEMA: estas dos líneas son impossibles sin calc() */ /* No puedes restar píxeles de porcentajes directamente */ .main { width: 100% - 240px; } /* ❌ CSS incorrecto */ /* LA SOLUCIÓN: calc() */ .main { width: calc(100% - 240px); } /* ✅ sidebar de 240px, main ocupa el resto */ /* ── Operadores disponibles ── */ width: calc(100% - 2rem); /* resta — espacio para padding lateral */ height: calc(100vh - 60px); /* resta — pantalla completa menos navbar */ font-size: calc(1rem + 0.5vw); /* suma — tamaño base más valor fluido */ padding: calc(var(--espacio) * 2); /* multiplicar con variable */ top: calc(50% - 10px); /* centrar verticalmente con absolute */ /* IMPORTANTE: los operadores + y - necesitan espacios alrededor */ width: calc(100%-240px); /* ❌ sin espacios — no funciona */ width: calc(100% - 240px); /* ✅ con espacios — funciona */ width: calc(100%/3); /* ✅ división no necesita espacios */ /* Caso real: sidebar fijo + contenido que ocupa el resto */ :root { --sidebar-w: 240px; } .sidebar { width: var(--sidebar-w); position: fixed; } .main { margin-left: var(--sidebar-w); /* deja espacio al sidebar */ width: calc(100% - var(--sidebar-w)); }
Expresión¿Qué calcula?
calc(100% - 240px)Todo el ancho disponible menos 240px. Perfecto para el contenido principal cuando hay un sidebar fijo.
calc(100vh - 60px)La altura de la pantalla menos 60px (la navbar). Hace que el panel ocupe exactamente el espacio visible.
calc(50% - 10px)Centra un elemento absolute desplazando la mitad de su propio tamaño. Con transform: translate(-50%,-50%) es más elegante, pero esta forma también funciona.
calc(var(--base) * 1.5)Multiplica una variable CSS. Permite crear escalas tipográficas proporcionales.

CSS BLOQUE 2 — Layout: sidebar fixed + main con calc()

CSS/* ── LAYOUT PRINCIPAL ── */ .app-layout { display: flex; min-height: 100vh; } /* ① SIDEBAR: position fixed — siempre visible al hacer scroll */ .sidebar { position: fixed; /* sale del flujo, se fija a la ventana */ top: 0; /* pegado al borde superior de la pantalla */ left: 0; /* pegado al borde izquierdo */ width: var(--sidebar-w); height: 100vh; /* toda la altura de la pantalla */ background: var(--panel); border-right: 1px solid var(--borde); display: flex; flex-direction: column; overflow-y: auto; /* scroll vertical si hay muchos items de nav */ z-index: var(--z-sidebar); } /* ② MAIN-AREA: compensar el sidebar con margin-left y calc() */ .main-area { margin-left: var(--sidebar-w); /* ← deja espacio al sidebar fixed */ width: calc(100% - var(--sidebar-w)); /* ← ocupa el resto */ min-height: 100vh; display: flex; flex-direction: column; } /* Estilos del sidebar */ .sidebar-logo { padding: 20px 18px; border-bottom: 1px solid var(--borde); } .logo-texto { font-weight: 700; font-size: 1.1rem; color: white; } .sidebar-logo small { font-size: .7rem; color: var(--texto-s); display: block; margin-top: 2px; } .nav-titulo { padding: 14px 18px 4px; font-size: .65rem; font-weight: 700; text-transform: uppercase; letter-spacing: .1em; color: #475569; } .sidebar-nav a { display: flex; align-items: center; gap: 10px; padding: 9px 18px; color: var(--texto-s); text-decoration: none; font-size: .85rem; transition: all .15s; } .sidebar-nav a:hover { background: #334155; color: white; } .sidebar-nav a.activo { background: rgba(59,130,246,.15); color: #60A5FA; border-right: 3px solid var(--azul); }
Propiedad / patrónExplicación
position: fixed + top:0 left:0El sidebar se fija a la esquina superior izquierda de la ventana. No se mueve con el scroll. Es exactamente lo que hace la barra de YouTube, Gmail o VS Code.
height: 100vhEl sidebar ocupa toda la altura de la pantalla visible. 100vh = 100% del viewport height.
margin-left: var(--sidebar-w)Como el sidebar tiene position: fixed y sale del flujo, el main-area no lo "ve". Hay que empujarlo manualmente con margin-left del mismo ancho que el sidebar.

Reto 1 — Botón flotante de acción

Agrega un botón de + Nueva campaña que esté siempre visible en la esquina inferior derecha de la pantalla, sin importar el scroll.

CSS: position: fixed; bottom: 24px; right: 24px; + color de fondo azul y borde redondeado.

🟠 Medio

Reto 2 — Badge en el link de Pedidos

Agrega un span con el número 5 dentro del <a> de "Pedidos" en el sidebar. Hazlo aparecer a la derecha del texto como un chip de contador.

CSS: dale al <a> justify-content: space-between. Al span: background, color, border-radius: 20px, font-size: .65rem, padding: 1px 8px.

Lo que practicas: elementos inline dentro de Flexbox sin necesitar position.

🟠 Medio

Reto 3 — Tooltip en los KPIs

Agrega un .tooltip-wrap con ícono ? y .tooltip-caja en una de las tarjetas KPI. El tooltip debe decir qué período representa el número (ej. "Últimos 30 días").

Atención: el KPI tiene overflow: hidden — el tooltip quedará recortado. Tendrás que quitarle el overflow al KPI y buscar otra solución para el círculo decorativo. ¿Puedes resolverlo?

⚫ Avanzado

Reto 4 — Sidebar colapsable

Añade un botón ☰ en el topbar. Con :has() (que aprendiste en el Ejercicio 4) crea una clase .colapsado para el body.

Cuando el sidebar tenga .colapsado: width: 60px (solo íconos), el main-area cambia su margin-left y width con calc(). Usa transition: width 0.3s para animarlo suavemente.

Lo que practicas: calc() + variables CSS + transition + position interactuando.

ConceptoLo que aprendiste
position: staticEl valor por defecto. El elemento va donde le toca en el flujo del HTML.
position: relativeSe desplaza desde su posición original. El hueco original sigue reservado. Principalmente se usa como ancla para hijos absolute.
position: absoluteSale del flujo. Se posiciona relativo al ancestro posicionado más cercano. Si no hay ninguno, va al body. Necesita un padre con position ≠ static.
position: fixedSale del flujo. Se posiciona relativo a la ventana del navegador. No se mueve con el scroll. Para sidebars, navbars y botones flotantes.
position: stickyFlujo normal hasta que llega al valor definido (top: 0) — entonces se queda fijo. Vuelve al flujo cuando su contenedor sale de la vista. Limitado por overflow: hidden del padre.
top / right / bottom / leftCoordenadas que solo funcionan en elementos posicionados (≠ static). Definen distancia al borde del ancestro posicionado (o de la ventana en fixed).
z-indexOrden de apilamiento. Solo funciona en elementos posicionados. Mayor número = encima. Mejor organizarlo como variables CSS: --z-navbar: 100.
overflow: hiddenRecorta el contenido que desborda. Esencial para: zoom de imágenes, barras de progreso animadas, bordes redondeados que recortan hijos. Cuidado: rompe position: sticky de hijos.
overflow-x / overflow-yControl independiente por eje. overflow-x: auto para tablas anchas. overflow-y: auto para listas largas con altura máxima.
::-webkit-scrollbarPseudo-elemento para personalizar la barra de scroll en Chrome/Safari. Tres partes: scrollbar (ancho), track (fondo), thumb (la barra deslizable).
calc()Mezcla unidades distintas en un cálculo. Obligatorio para: main = 100% - sidebar, altura = 100vh - navbar. Los operadores + y - necesitan espacios.
Badge de notificaciónPatrón: padre con position: relative, badge con position: absolute + top: -5px + right: -5px + border del color del fondo.
Tooltip con CSS puroPatrón: padre con position: relative, tooltip con position: absolute + opacity: 0, aparece con :hover. Flecha con ::after border trick.
white-space: nowrapImpide que el texto salte de línea. Necesario en tablas con overflow horizontal para que las celdas no se compriman.
pointer-events: noneEl elemento no recibe eventos del ratón. Usado en tooltips invisibles para que no "intercepten" clics cuando están ocultos.
