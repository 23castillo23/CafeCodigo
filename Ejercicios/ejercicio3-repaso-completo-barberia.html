Repaso visual: el Box Model
Las 4 capas de todo elemento HTML, de afuera hacia adentro
margin
border
padding
contenido
CapaQué controla
contentEl texto o imagen real. Su tamaño lo define width/height, o el contenido mismo.
paddingEspacio dentro del borde, entre el contenido y el borde. Con border-box, se resta del width.
borderLa línea visible alrededor del padding.
marginEspacio fuera del borde, hacia los elementos vecinos. Nunca tiene fondo ni borde propio — es aire.

🖼️ Analogía

Piensa en un cuadro colgado en la pared: la pintura es el contenido, el marco es el border, el passe-partout (el cartón blanco entre pintura y marco) es el padding, y el espacio en la pared antes del siguiente cuadro es el margin.

PropiedadQué hace
justify-content: space-between;En un flex container, empuja el primer hijo al inicio y el último al final, repartiendo el espacio sobrante entre ellos.
align-items: center;Centra los hijos en el eje vertical (cuando la fila es horizontal) — soluciona el clásico problema de "centrar verticalmente".
list-style: none;Quita los puntos (•) que el navegador pone por defecto a los <li>.
opacity: 0.8;Transparencia de 0 (invisible) a 1 (sólido). Aquí suaviza el texto sin necesidad de un gris distinto.
display: inline-block;Un <a> normal es "en línea" e ignora padding vertical y width/height. inline-block le permite comportarse como bloque (aceptar padding completo) sin dejar de fluir en línea con el texto.

Glosario final combinado
Las 33 etiquetas y propiedades reales de los Ejercicios 1 y 2 — qué son y para qué sirven

📗 Del Ejercicio 1 (fundamentos)

<header>

Encabezado de la página o de una sección: título y datos generales. Va una vez, arriba de todo.

<header><h1>Tacos</h1></header>
<section id="...">

Agrupa contenido relacionado con su propio tema, como un capítulo de un libro. El id sirve para enlazarlo con #id.

<section id="ingredientes">...</section>
<footer>

Pie de página — información de cierre. Va una vez, al final.

<h1> / <h2>

<h1>: el título más importante, solo uno por página. <h2>: subtítulos de cada sección.

<p>

Párrafo de texto. Crea espacio arriba/abajo automáticamente — es un elemento de bloque.

<ul> / <ol> / <li>

<ul>: lista SIN orden importante (viñetas •). <ol>: lista CON orden (1, 2, 3...). <li>: cada elemento de cualquiera de las dos.

<ol><li>Paso 1</li></ol>
<li class="a b">

Un <li> puede llevar una sola clase o dos a la vez, separadas por espacio — cada una aportando su propio estilo en el CSS.

<li class="servicio destacado">...</li>
<img>

Muestra una imagen. src = la ruta/URL. alt = texto alternativo si no carga (nunca lo omitas). width/height = tamaño, evita que la página "salte" al cargar.

<img src="foto.jpg" alt="Tacos" width="600" height="350">
<a href="...">

Enlace. Puede apuntar a otra página, a un sitio externo, o (como verás abajo) a un teléfono, correo o sección de la misma página.

target="_blank" + rel="noopener"

target="_blank" abre el enlace en una pestaña nueva. rel="noopener" es una medida de seguridad que siempre debe acompañarlo.

<a href="..." target="_blank" rel="noopener">
<span>

Contenedor en línea (no hace salto de línea) sin apariencia propia. Se usa para rodear una parte de un texto y darle estilo distinto sin afectar el resto del párrafo.

<p><span class="tiempo">⏱ 30 min</span></p>
<strong> / <em>

<strong>: marca importancia (negrita). <em>: marca énfasis (cursiva). No son solo estilo visual, también tienen significado.

selector de ID (#nombre)

Apunta a un único elemento con ese id. Tiene más especificidad (más "fuerza") que un selector de clase.

#encabezado { color: red; }
selector de clase (.nombre)

Apunta a TODOS los elementos que tengan esa clase, sin importar cuántos haya en la página.

.tarjeta { padding: 16px; }
selector descendiente (#id p)

Aplica solo a los <p> que estén DENTRO del elemento con ese id — más específico que una clase suelta.

#contacto p { margin-bottom: 8px; }
selector múltiple (a, b, c { })

Aplica la misma regla a varios selectores distintos a la vez, separados por comas, para no repetir código.

h1, h2, .titulo { font-weight: 700; }
font-family / font-size / color / background-color

Las 4 propiedades básicas de texto y color: tipografía, tamaño del texto, color del texto y color de fondo.

text-align / text-transform / letter-spacing / text-decoration

Alineación (left/center/right), forzar mayúsculas/minúsculas, espacio entre letras, y subrayado o quitarlo — muy usado en enlaces.

line-height / border / border-radius / border-bottom

Espacio vertical entre líneas, borde completo alrededor, esquinas redondeadas, o borde solo en la parte de abajo.

margin: 0 auto / max-width

Centra un bloque horizontalmente, siempre que tenga un max-width definido — sin eso, margin: 0 auto no tiene efecto.

a:hover

Estilo que se aplica SOLO mientras el mouse está encima del enlace — nunca se ve en una captura de pantalla, solo al interactuar.

📘 Del Ejercicio 2 (semántica y Box Model)

<nav>

Agrupa los enlaces de navegación principal del sitio — no cualquier lista de links, sino específicamente el menú.

<main>

El contenido único e importante de la página. Solo puede haber uno por página — tener dos es un error semántico.

<div>

Caja genérica, sin significado propio. Úsala solo cuando necesitas agrupar visualmente algo que no encaja como sección, artículo o dato de contacto.

<article>

Contenido que tendría sentido por sí solo, fuera de esta página (ej. una tarjeta de servicio, una noticia).

<address>

Datos de contacto de un negocio o persona. El navegador la muestra en cursiva por defecto, sin que tú lo pidas.

href="tel:..."

En celular, ofrece llamar directamente al número al tocar el enlace.

<a href="tel:2291234567">229...</a>
href="mailto:..."

Abre el programa de correo del usuario con esa dirección ya escrita.

<a href="mailto:hola@negocio.com">
href="#id"

Enlace de ancla: hace que la página salte hasta el elemento que tenga ese mismo id.

<a href="#contacto">Contacto</a>
box-sizing: border-box

Hace que padding y border se cuenten DENTRO del width, no que se sumen. Nunca "se ve" directamente, pero si falta, todos tus anchos se descuadran.

margin / padding shorthand

Varios valores en una línea: 4 valores = arriba derecha abajo izquierda; 2 valores = vertical horizontal.

padding: 24px 32px;
margin: 0 auto

Centra un bloque horizontalmente, siempre que tenga max-width definido.

max-width

Límite de ancho para que el contenido no se estire demasiado en pantallas grandes.

:root + variables

Declara valores reutilizables (colores, medidas). Por sí sola, no pinta nada — solo guarda el valor.

var(--nombre)

Así USAS una variable ya declarada en :root. Sin esto, la variable existe pero no hace nada.

selector descendiente (.clase p)

Aplica solo a los <p> que estén DENTRO de .clase. Más específico que una clase sola.

list-style: none

Quita las viñetas (•) de una lista <ul>.

display: flex

Convierte al elemento en un contenedor que acomoda a sus hijos en fila. Por sí solo NO reparte espacio ni separa.

gap

Espacio ENTRE elementos flex, sin usar margin.

flex: 1

Le dice a un hijo flex "reparte el espacio sobrante en partes iguales" — por eso varias tarjetas terminan midiendo lo mismo.

justify-content / align-items

justify-content alinea los hijos flex en el eje principal (horizontal, si es fila). align-items los alinea en el eje cruzado (vertical, si es fila).

width: 100% / opacity

width: 100% ocupa todo el ancho disponible del contenedor. opacity controla la transparencia — de 0 (invisible) a 1 (sólido).

border-top / border-left

Acento de color en un solo lado — muy usado en tarjetas.

border-radius parcial

0 8px 8px 0 redondea solo un lado — combina bien con un border-left de acento en el lado opuesto.

display: inline-block

Hace que un elemento se comporte como texto en línea pero acepte width, height, padding y margin como una caja de verdad.

transition

Anima el cambio entre dos estados (ej. antes/después de :hover). Es invisible hasta que interactúas — nunca se ve en una captura de pantalla.

:last-child

Selecciona solo el último hijo de su tipo — útil para quitar el margin-bottom extra del último elemento de una lista.

Especificidad CSS (span vs p)

Cuando dos reglas compiten por el mismo elemento, gana la más específica. Un selector de clase en un <span> le gana a un selector de etiqueta como p, aunque el p esté escrito después en el archivo.
