Estructuras Condicionales — if / else if / else
Tomar decisiones en el código · 1:10:45 – 1:20:43

Las estructuras condicionales le permiten al programa tomar decisiones: si se cumple una condición, hace una cosa; si no, hace otra. Es la base de cualquier lógica en programación.

🚦 Analogía: el semáforo

Cuando llegas a un semáforo, tu cerebro evalúa una condición: ¿está en rojo? Si sí → para. Si no, evalúa: ¿está en amarillo? Si sí → prepárate. Si no → avanza. Tu cerebro ejecuta un if / else if / else en milisegundos sin que te des cuenta. JavaScript hace exactamente lo mismo, pero con condiciones que tú defines.

Sintaxis de if / else if / else

JavaScriptconst hora = 14; if (hora < 12) { console.log("Buenos días 🌅"); } else if (hora < 20) { console.log("Buenas tardes ☀️"); // ← Este se ejecuta } else { console.log("Buenas noches 🌙"); }

Condiciones compuestas

JavaScriptconst edad = 22; const tieneEntrada = true; // Ambas condiciones deben cumplirse (AND) if (edad >= 18 && tieneEntrada) { console.log("Puedes entrar al evento ✅"); } else { console.log("No puedes entrar ❌"); } // Al menos una condición (OR) const esAdmin = false; const esModerador = true; if (esAdmin || esModerador) { console.log("Tienes acceso al panel"); } // Negación (NOT) const estaLogueado = false; if (!estaLogueado) { console.log("Por favor, inicia sesión"); }

Truthy y Falsy — qué considera JS verdadero/falso

JavaScript// Valores FALSY (se comportan como false en un if): if (0) { } // no entra if ("") { } // no entra if (null) { } // no entra if (undefined) { } // no entra if (NaN) { } // no entra if (false) { } // no entra // Todo lo demás es TRUTHY (se comporta como true): if (1) { console.log("entra"); } if ("hola") { console.log("entra"); } if ([]) { console.log("entra"); } // ¡array vacío es truthy! if ({}) { console.log("entra"); } // ¡objeto vacío es truthy!

Condicional Switch — Múltiples casos exactos
La alternativa a muchos if/else if · 1:20:43 – 1:28:44

switch es ideal cuando tienes una variable que puede tener varios valores exactos y quieres hacer algo diferente en cada caso. Es más legible que una cadena larga de else if cuando comparas el mismo valor.

🎰 Analogía: la máquina de bebidas

Presionas el botón 1 → sale Coca-Cola. Botón 2 → agua. Botón 3 → jugo. Botón desconocido → "selección no válida". La máquina no evalúa rangos ni condiciones complejas — solo compara el número que presionaste con cada opción disponible. Eso es exactamente lo que hace switch: compara un valor contra casos exactos.

Sintaxis del switch

JavaScriptconst dia = 3; switch (dia) { case 1: console.log("Lunes"); break; // ← IMPORTANTE: sin break, sigue al siguiente case case 2: console.log("Martes"); break; case 3: console.log("Miércoles"); // ← Este se ejecuta break; case 4: console.log("Jueves"); break; case 5: console.log("Viernes"); break; default: // Se ejecuta si ningún case coincide console.log("Fin de semana"); }

⚠️ El break es obligatorio (casi siempre)

Sin break, switch sigue ejecutando los casos siguientes aunque ya encontró el correcto — esto se llama fall-through. A veces es útil (para agrupar casos), pero generalmente es un bug. Acostúmbrate a poner break al final de cada caso o a usar return si estás dentro de una función.

Fall-through intencional — agrupar casos

JavaScriptconst mes = 4; // Abril switch (mes) { case 12: case 1: case 2: console.log("Invierno ❄️"); break; case 3: case 4: // ← Sin break en 3, cae al 4 case 5: console.log("Primavera 🌸"); // ← Ejecuta para mes 3, 4 o 5 break; case 6: case 7: case 8: console.log("Verano ☀️"); break; default: console.log("Otoño 🍂"); }

Estructura Repetitiva while
Repetir mientras se cumpla una condición · 1:42:30 – 1:52:20

while repite un bloque de código mientras la condición sea verdadera. Se usa cuando no sabes de antemano cuántas veces vas a repetir — solo sabes cuándo parar.

🏃 Analogía: correr hasta quedar sin energía

Imagina que corres en una cinta y la condición de parada es: "¿me queda energía?". Mientras la respuesta sea sí → sigues corriendo. En el momento en que la respuesta sea no → paras. No sabes de antemano cuántas vueltas darás — depende de cuánta energía tengas. Eso es exactamente el while: repite mientras la condición sea verdadera, sin importar cuántas veces.

Sintaxis del while

JavaScriptlet contador = 1; while (contador <= 5) { console.log(`Vuelta número ${contador}`); contador++; // ← CRÍTICO: sin esto, loop infinito 💀 } // Salida: // Vuelta número 1 // Vuelta número 2 // Vuelta número 3 // Vuelta número 4 // Vuelta número 5

⚠️ Loop infinito — el error más temido

Si la condición del while nunca se vuelve falsa, el programa entra en un bucle infinito y se congela. Siempre asegúrate de que algo dentro del while modifique la condición para que eventualmente sea falsa. Si el navegador se congela, cierra la pestaña.

while con break — salida anticipada

JavaScriptlet numero = 1; while (true) { // Loop "infinito" controlado if (numero > 10) { break; // Sale del while cuando número > 10 } console.log(numero); numero++; } console.log("Loop terminado");

Estructura Repetitiva for
Cuando sabes cuántas veces repetir · 1:52:20 – 2:00:29

for es el ciclo más usado en JavaScript. Se usa cuando sabes exactamente cuántas veces quieres repetir algo, o cuando quieres recorrer una colección de datos.

📋 Analogía: la lista de asistencia

Un profesor tiene 30 alumnos en la lista. Llama al alumno 1, anota si vino. Llama al alumno 2, anota. Así hasta el alumno 30. Sabe exactamente cuántas veces va a repetir la acción. El for funciona igual: tú defines el inicio (alumno 1), la condición de parada (hasta el 30), y cómo avanzar (siguiente alumno).

Anatomía del for

JavaScript// inicio condición incremento for (let i = 0; i < 5; i++ ) { console.log(`Iteración ${i}`); } // i = 0, 1, 2, 3, 4 (empieza en 0, termina ANTES del 5)

Variantes del for

JavaScript// Hacia atrás for (let i = 10; i >= 1; i--) { console.log(i); // 10, 9, 8...1 } // De 2 en 2 (saltos) for (let i = 0; i <= 20; i += 2) { console.log(i); // 0, 2, 4, 6...20 } // Recorrer un string carácter por carácter const palabra = "Hola"; for (let i = 0; i < palabra.length; i++) { console.log(palabra[i]); // H, o, l, a } // for...of — la forma moderna de recorrer arrays const frutas = ["🍎", "🍌", "🍇"]; for (const fruta of frutas) { console.log(fruta); // 🍎, 🍌, 🍇 } // continue — salta a la siguiente iteración for (let i = 1; i <= 10; i++) { if (i % 2 === 0) continue; // Salta los pares console.log(i); // 1, 3, 5, 7, 9 }

Estructura Repetitiva do...while
Ejecutar al menos una vez · 2:00:29 – 2:09:19

La diferencia clave del do...while: el bloque de código se ejecuta al menos una vez, y después evalúa la condición. En while, si la condición es falsa desde el principio, nunca entra. En do...while, siempre entra al menos una vez.

🍕 Analogía: probar antes de decidir

En un restaurante, el mesero te trae un plato de prueba y te pregunta si quieres más. Primero te lo dan (se ejecuta al menos una vez), y después preguntan. Si dijeras "while tengas hambre, come" pero no tienes hambre desde el principio, nunca comerías nada. El do...while te asegura que siempre comas al menos ese primer plato de prueba.

JavaScript// WHILE: si condición empieza en false, NUNCA entra let x = 10; while (x < 5) { console.log("while ejecutado"); // Nunca se imprime } // DO...WHILE: entra siempre la primera vez let y = 10; do { console.log("do...while ejecutado"); // Se imprime UNA vez y++; } while (y < 5); // Condición falsa → para después de la primera vuelta // Caso típico: menú que se muestra siempre la primera vez let intentos = 0; do { intentos++; console.log(`Intento número ${intentos}`); } while (intentos < 3); // Ejecuta 3 veces: intento 1, 2 y 3

prompt() y confirm() — Interacción con el usuario
Pedir y confirmar datos por ventana emergente · 2:31:26 – 2:43:10

prompt() y confirm() son funciones del navegador que abren ventanas emergentes para pedir información al usuario. Son simples y útiles para prototipos y ejercicios — aunque en producción se usan formularios HTML.

📞 Analogía: la recepcionista

prompt() es como una recepcionista que te llama por teléfono y te pregunta algo esperando tu respuesta: "Buenos días, ¿cuál es su nombre?" — y espera a que respondas antes de continuar. confirm() es como cuando te pregunta algo de sí o no: "¿Confirma su cita para mañana?" — solo puede decir Sí o Cancelar.

prompt() — pedir texto al usuario

JavaScript// prompt() siempre devuelve un STRING o null si cancela const nombre = prompt("¿Cuál es tu nombre?"); if (nombre === null) { console.log("El usuario canceló"); } else if (nombre === "") { console.log("El usuario no escribió nada"); } else { console.log(`Hola, ${nombre}!`); } // ⚠️ prompt() devuelve STRING — convierte si necesitas número const edadStr = prompt("¿Cuántos años tienes?"); const edad = Number(edadStr); // ← Convertir a número if (isNaN(edad)) { console.log("Eso no es un número válido"); } else { console.log(`Tienes ${edad} años`); }

confirm() — preguntas de sí/no

JavaScript// confirm() devuelve true (Aceptar) o false (Cancelar) const confirmado = confirm("¿Deseas eliminar este archivo?"); if (confirmado) { console.log("Archivo eliminado"); } else { console.log("Operación cancelada"); }

💡 Por qué no usamos prompt aquí en los ejercicios

Las funciones prompt(), confirm() y alert() bloquean el navegador hasta que el usuario responde — esto impide que el editor de esta guía funcione correctamente. En los ejercicios de abajo simulamos las respuestas del usuario con variables. Pero puedes probarlas directamente en la consola de tu navegador (F12) escribiendo prompt("hola").

Manejo de Fechas — El objeto Date
Trabajar con fechas y horas en JavaScript · 2:43:10 – 2:58:53

JavaScript tiene el objeto Date para trabajar con fechas y horas. Puede obtener la fecha actual, crear fechas específicas, calcular diferencias y formatear cómo se muestra la información.

🕐 Analogía: el reloj con memoria

Imagina un reloj que no solo te dice la hora, sino que también puede calcular cuántos días faltan para tu cumpleaños, qué día de la semana fue el 15 de julio de 1969, o cuántas horas llevas trabajando. El objeto Date de JavaScript es ese reloj inteligente: te da la fecha y hora actuales, pero también puede hacer cálculos complejos con cualquier momento en el tiempo.

Crear y leer fechas

JavaScript// Fecha y hora actuales const ahora = new Date(); console.log(ahora); // Ej: 2025-06-10T14:30:00.000Z // Crear una fecha específica const navidad = new Date(2025, 11, 25); // ojo: mes 11 = diciembre (0-indexado) const desde_string = new Date("2025-12-25"); // Métodos GET — leer partes de la fecha const hoy = new Date(); console.log(hoy.getFullYear()); // 2025 → Año completo console.log(hoy.getMonth()); // 0-11 → OJO: 0=enero, 11=diciembre console.log(hoy.getDate()); // 1-31 → Día del mes console.log(hoy.getDay()); // 0-6 → Día semana (0=domingo) console.log(hoy.getHours()); // 0-23 → Hora console.log(hoy.getMinutes()); // 0-59 → Minutos console.log(hoy.getSeconds()); // 0-59 → Segundos console.log(hoy.getTime()); // Milisegundos desde 01/01/1970 (timestamp)

Calcular diferencias entre fechas

JavaScript// Los Date se pueden restar → resultado en milisegundos const inicio = new Date("2025-01-01"); const fin = new Date("2025-12-31"); const diferencia = fin - inicio; // en milisegundos const diasEnMili = 1000 * 60 * 60 * 24; // ms en un día const dias = Math.round(diferencia / diasEnMili); console.log(`Días entre fechas: ${dias}`); // → 364
