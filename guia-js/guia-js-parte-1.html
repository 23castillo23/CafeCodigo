Introducción a JavaScript
El lenguaje que da vida a las páginas web · 0:00 – 10:55

Si HTML es el esqueleto de una página web y CSS es su ropa, JavaScript es el cerebro: el que hace que las cosas sucedan cuando el usuario interactúa.

🏠 Analogía: la casa inteligente

HTML construye la casa (paredes, puertas, ventanas). CSS la decora (colores, muebles, iluminación). JavaScript la hace inteligente: cuando llegas a casa, las luces se encienden solas. Cuando detecta que hay alguien en la puerta, suena un timbre. Cuando presionas un botón, el aire acondicionado se ajusta. Esa inteligencia, esa reacción a eventos, es lo que hace JavaScript en una página web.

¿Qué puede hacer JavaScript?

🖱️
Interactividad
Responder a clics, teclas, movimientos del mouse, formularios enviados.
🔄
Cambiar el HTML
Agregar, quitar o modificar elementos de la página sin recargarla.
🌐
Comunicación con servidores
Pedir y enviar datos a APIs sin recargar la página (AJAX/Fetch).
✅
Validación de formularios
Verificar que el email tenga @, que la contraseña tenga 8 caracteres, etc.
🎮
Animaciones y juegos
Mover elementos, crear efectos visuales, construir juegos en el navegador.
🖥️
Backend (Node.js)
Con Node.js, JavaScript también corre en servidores, no solo en el navegador.

¿Cómo se vincula JavaScript a HTML?

Igual que CSS, JS se puede incluir de dos formas: inline (dentro del HTML) o en un archivo externo (recomendado).

HTML<!-- Forma 1: inline (no recomendada para proyectos reales) --> <script> console.log("Hola desde JavaScript"); </script> <!-- Forma 2: archivo externo (recomendada) --> <!-- Siempre al FINAL del body, antes de </body> --> <script src="js/main.js"></script>

💡 ¿Por qué al final del body?

El navegador lee el HTML de arriba hacia abajo. Si pones el script al principio, JS intenta modificar elementos HTML que todavía no existen — y falla. Al ponerlo al final, el HTML ya está completamente cargado cuando JS empieza a ejecutarse.

La consola del navegador — tu mejor herramienta

Presiona F12 o clic derecho → Inspeccionar → pestaña "Console". Aquí verás los mensajes de console.log() y los errores de tu código.

Variables — Las cajas del programa
Qué son, para qué sirven y cómo crearlas · 10:55 – 21:33

Una variable es un espacio en la memoria del programa donde guardas un valor para poder usarlo y modificarlo más adelante. Sin variables, cada dato que el usuario ingresa desaparecería inmediatamente.

📦 Analogía: las cajas etiquetadas

Imagina que te mudas de casa. Tienes cajas para guardar tus cosas. Cada caja tiene una etiqueta: "libros", "ropa", "cocina". En programación, las variables son esas cajas. La etiqueta es el nombre de la variable, y lo que metes dentro es el valor. Cuando necesitas algo, no buscas en todas las cajas — solo buscas la caja por su etiqueta.

Declarar una variable

JavaScript// Declarar = crear la caja let nombre; // Asignar = meter algo dentro de la caja nombre = "Dalto"; // Declarar Y asignar en una sola línea (lo más común) let edad = 28; let ciudad = "Buenos Aires"; let esEstudiante = true; // Mostrar en consola console.log(nombre); // → "Dalto" console.log(edad); // → 28 console.log(esEstudiante); // → true

💡 camelCase — la convención de JavaScript

Cuando un nombre de variable tiene varias palabras, en JavaScript se usa camelCase: la primera palabra en minúscula y cada palabra siguiente empieza con mayúscula. Ej: nombreCompleto, edadDelUsuario, precioTotalConDescuento. No es una regla del lenguaje, pero es la convención universal de la comunidad JS.

Tipos de Datos en JavaScript
Los 7 tipos primitivos + Objetos · 10:55 – 21:33

Cada valor en JavaScript pertenece a un tipo de dato. Saber el tipo importa porque determina qué operaciones puedes hacer con él: no puedes sumar textos como si fueran números (bueno, sí puedes, pero no de la forma que esperas).

🧪 Analogía: los recipientes del laboratorio

En un laboratorio hay diferentes recipientes según lo que vas a guardar: matraces para líquidos, cajas de Petri para muestras sólidas, jeringas para inyectables. No puedes guardar un gas en una caja de Petri. En JavaScript, los tipos de datos son como esos recipientes: cada tipo está diseñado para guardar un tipo específico de información.

Los tipos primitivos

JavaScript// 1. NUMBER — números (enteros y decimales, JS no los distingue) let entero = 42; let decimal = 3.14; let negativo = -100; let infinito = Infinity; // resultado de 1/0 let noEsNum = NaN; // Not a Number: resultado de "texto" * 2 // 2. STRING — texto, siempre entre comillas let simple = 'Comillas simples'; let doble = "Comillas dobles"; let template = `Template literal: puedo usar ${entero} variables aquí`; // 3. BOOLEAN — solo dos valores posibles: verdadero o falso let esMayor = true; let estaVacio = false; // 4. UNDEFINED — variable declarada pero sin valor asignado let sinValor; console.log(sinValor); // → undefined // 5. NULL — ausencia intencional de valor (tú lo asignas a propósito) let usuarioActual = null; // "no hay usuario, a propósito" // 6. BIGINT — números MUY grandes (más de 15 dígitos) let numeroGrande = 9007199254740991n; // nota la n al final // 7. SYMBOL — identificadores únicos (uso avanzado) let id = Symbol("id");

typeof — saber el tipo de un valor

JavaScriptconsole.log(typeof 42); // → "number" console.log(typeof "hola"); // → "string" console.log(typeof true); // → "boolean" console.log(typeof undefined); // → "undefined" console.log(typeof null); // → "object" ⚠️ BUG histórico de JS console.log(typeof []); // → "object" console.log(typeof {}); // → "object" console.log(typeof function(){}); // → "function"

⚠️ El bug de typeof null

typeof null devuelve "object" — esto es un error histórico de JavaScript que nunca se corrigió por compatibilidad con código antiguo. null NO es un objeto. Para verificar si algo es null, usa comparación directa: valor === null.

Conversión de tipos (Type Coercion)

JavaScript// JS convierte tipos automáticamente — a veces de formas sorprendentes console.log("5" + 3); // → "53" (concatena, no suma) console.log("5" - 3); // → 2 (resta, convierte "5" a número) console.log("5" * 2); // → 10 (multiplica, convierte) console.log("abc" * 2); // → NaN (no puede convertir "abc") // Conversión explícita (mejor práctica) Number("42") // → 42 String(42) // → "42" Boolean(0) // → false Boolean("hola") // → true parseInt("42px") // → 42 (extrae el número del inicio del string)

Asignación de Variables
Operadores de asignación y cómo actualizar valores · 21:33 – 35:01

Asignar una variable es meterle un valor. Puedes hacerlo cuando la creas o en cualquier momento después. JavaScript tiene varios operadores de asignación que te permiten actualizar un valor de forma compacta.

🌡️ Analogía: el marcador de temperatura

Un termostato tiene una temperatura guardada. Puedes decirle "ponlo en 22°" (asignación directa), o "súbelo 2 grados" (asignación con operación). No necesitas saber la temperatura actual para subirla — solo le dices cuánto más. Los operadores de asignación compuesta hacen exactamente eso: modifican el valor actual sin tener que repetirlo.

%=Módulo y asignax %= 3x = x % 3

var, let y const — Las tres formas de declarar variables
Sus diferencias, el scope y cuándo usar cada una · 35:01 – 44:38

En JavaScript hay tres palabras para declarar variables: var (la antigua), let (la moderna) y const (para valores que no cambian). Elegir mal genera errores difíciles de detectar.

🚗 Analogía: tres tipos de contratos de arrendamiento

var es como un contrato verbal sin papeles — flexible pero impredecible, puede generar problemas legales (bugs) que no esperabas. let es un contrato estándar — claro, seguro, con reglas bien definidas. const es un contrato de propiedad — una vez firmado, el bien no puede reasignarse a otra persona (aunque su contenido interno sí puede cambiar).

var — la forma antigua (evitar)

JavaScript// VAR tiene dos problemas graves: // 1. Se puede redeclarar sin error (genera bugs silenciosos) var nombre = "Dalto"; var nombre = "Otro"; // ¡Sin error! Pero perdiste el valor original // 2. Tiene scope de función, NO de bloque if (true) { var mensaje = "Hola"; // Definida dentro del if... } console.log(mensaje); // → "Hola" (¡escapó del bloque! Bug potencial) // 3. Hoisting — se "sube" al tope, puede dar undefined console.log(x); // → undefined (no error, pero raro) var x = 5;

let — la forma moderna

JavaScript// LET: scope de bloque, no se puede redeclarar, sí reasignar let contador = 0; contador = 1; // ✅ Se puede cambiar el valor // let contador = 2; // ❌ Error: no se puede redeclarar if (true) { let local = "Solo existo aquí dentro"; console.log(local); // ✅ } // console.log(local); // ❌ Error: local no está definida aquí fuera // Tampoco se puede usar antes de declarar: // console.log(y); // ❌ ReferenceError let y = 10;

const — para valores que no cambian

JavaScript// CONST: no se puede reasignar, pero los objetos/arrays SÍ se pueden modificar const PI = 3.14159; // PI = 3; // ❌ Error: no se puede reasignar una constante const COLORES = ["rojo", "azul"]; COLORES.push("verde"); // ✅ El array se puede modificar // COLORES = []; // ❌ Pero no se puede reasignar la constante completa const persona = { nombre: "Dalto" }; persona.nombre = "Lucas"; // ✅ Podemos modificar propiedades // persona = {}; // ❌ Pero no reasignar

🟢 Regla de oro de Dalto: const primero, let si necesitas cambiar

Cuando declares una variable, empieza siempre con const. Si en algún momento necesitas cambiar su valor, cámbiala a let. Nunca uses var en código moderno. Esta regla reduce bugs y hace el código más predecible.

Operadores en JavaScript
Aritméticos, de comparación, lógicos y más · 44:38 – 1:01:38

Los operadores son los símbolos que le dicen a JavaScript qué hacer con los valores. Sin operadores no hay cálculos, no hay comparaciones, no hay lógica.

1. Operadores Aritméticos

JavaScriptlet a = 10, b = 3; console.log(a + b); // 13 → Suma console.log(a - b); // 7 → Resta console.log(a * b); // 30 → Multiplicación console.log(a / b); // 3.33... → División console.log(a % b); // 1 → Módulo (resto de la división) console.log(a ** b); // 1000 → Potencia (10³) // El módulo es muy útil para saber si un número es par o impar: console.log(10 % 2); // 0 → par (resto 0) console.log(7 % 2); // 1 → impar (resto 1)

2. Operadores de Comparación

Siempre devuelven true o false.

JavaScript// == vs === : la diferencia más importante de JS console.log(5 == "5"); // true → == compara valor, ignora tipo console.log(5 === "5"); // false → === compara valor Y tipo (recomendado) console.log(5 != "5"); // false → != ignora tipo console.log(5 !== "5"); // true → !== compara valor Y tipo console.log(10 > 5); // true console.log(10 < 5); // false console.log(10 >= 10); // true console.log(10 <= 9); // false

💡 Siempre usa === en vez de ==

El operador == hace conversión de tipos automáticamente, lo que produce resultados confusos: 0 == false es true, "" == false es true, null == undefined es true. El operador === (triple igual) compara valor Y tipo — sin conversiones, sin sorpresas. Úsalo siempre.

3. Operadores Lógicos

JavaScript// AND (&&) — true solo si AMBOS son true console.log(true && true); // true console.log(true && false); // false console.log(5 > 3 && 10 > 8); // true (ambas condiciones se cumplen) // OR (||) — true si AL MENOS UNO es true console.log(true || false); // true console.log(false || false); // false console.log(5 > 10 || 3 > 1); // true (la segunda se cumple) // NOT (!) — invierte el valor console.log(!true); // false console.log(!false); // true console.log(!(5 > 3)); // false (5 > 3 es true, !true = false) // Nullish Coalescing (??) — usa el derecho si el izquierdo es null/undefined let usuario = null; let nombre = usuario ?? "Invitado"; console.log(nombre); // → "Invitado" // Optional Chaining (?.) — accede a propiedad sin error si el objeto es null let perfil = null; console.log(perfil?.nombre); // → undefined (sin error)

4. Operador Ternario

Un if/else en una sola línea.

JavaScript// condición ? valorSiTrue : valorSiFalse let edad = 20; let acceso = edad >= 18 ? "Permitido" : "Denegado"; console.log(acceso); // → "Permitido" // Equivalente al if/else: let acceso2; if (edad >= 18) { acceso2 = "Permitido"; } else { acceso2 = "Denegado"; }
