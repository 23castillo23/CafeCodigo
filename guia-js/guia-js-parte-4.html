¿Qué es el DOM?
Document Object Model — el puente entre JS y HTML · 5:29:23 – 5:40:39

El DOM (Document Object Model) es la representación que hace el navegador de tu página HTML convertida en objetos JavaScript. Es el puente que permite a JavaScript leer y modificar el HTML y CSS en tiempo real, sin recargar la página.

🗺️ Analogía: el plano editable del edificio

Imagina que tienes un edificio (tu página HTML). El DOM es el plano digital de ese edificio — pero no un plano estático, sino uno que puedes editar en vivo. Si modificas una pared en el plano, la pared real del edificio cambia al instante. JavaScript trabaja con ese plano: puede mover paredes, pintar habitaciones, agregar pisos nuevos o demoler secciones, y el edificio (la página que ve el usuario) se actualiza inmediatamente.

El DOM como árbol de nodos

El navegador convierte cada etiqueta HTML en un nodo del árbol del DOM. Cada nodo tiene un padre, puede tener hijos y hermanos.

document └── html ├── head │ ├── title │ │ └── "Mi Página" ← nodo de texto │ └── link rel="stylesheet" └── body ├── header │ └── h1 │ └── "Bienvenido" ├── main │ ├── p id="intro" │ │ └── "Texto del párrafo" │ └── ul │ ├── li class="item" │ └── li class="item" └── footer

💡 document — la puerta de entrada al DOM

document es el objeto raíz del DOM en JavaScript. Desde él accedes a todo: document.body, document.head, document.title. Todos los métodos para seleccionar elementos (querySelector, getElementById...) se llaman sobre document. Cada vez que quieras hacer algo con el HTML, empiezas por document.

🎯
Seleccionar elementos del DOM
querySelector, getElementById y más · 5:29:23 – 5:40:39

Antes de modificar un elemento HTML con JavaScript, tienes que seleccionarlo: decirle a JS cuál es el elemento que quieres manipular. Hay varias formas de hacerlo.

JavaScript// ── Los dos métodos modernos (los más usados) ────────── // querySelector — devuelve el PRIMER elemento que coincide con el selector CSS const titulo = document.querySelector("h1"); // por etiqueta const intro = document.querySelector("#intro"); // por id const boton = document.querySelector(".btn-primario"); // por clase const link = document.querySelector("nav a"); // selector compuesto // querySelectorAll — devuelve TODOS los elementos que coinciden (NodeList) const todosLosItems = document.querySelectorAll(".item"); const todosLosParraf = document.querySelectorAll("p"); // Recorrer NodeList con forEach todosLosItems.forEach(item => { item.style.color = "red"; }); // ── Métodos clásicos (siguen funcionando) ────────────── document.getElementById("intro"); // por id, sin "#" document.getElementsByClassName("item"); // por clase, sin "." document.getElementsByTagName("p"); // por etiqueta // ── Navegar por el árbol ─────────────────────────────── const lista = document.querySelector("ul"); console.log(lista.children); // hijos directos console.log(lista.parentElement); // elemento padre console.log(lista.nextElementSibling); // hermano siguiente console.log(lista.firstElementChild); // primer hijo

🟢 querySelector vs getElementById — ¿cuál usar?

querySelector es más flexible porque acepta cualquier selector CSS — lo usarás el 90% del tiempo. getElementById es marginalmente más rápido (directo al id sin parsear CSS) — úsalo si tienes miles de elementos y necesitas máxima performance. En proyectos normales, querySelector para todo.

Manipular el DOM — Leer y modificar la página
Cambiar texto, HTML, estilos, clases y atributos · 5:29:23 – 5:40:39

Leer y cambiar contenido

JavaScriptconst parrafo = document.querySelector("#mi-parrafo"); // textContent — solo texto plano (más seguro, más rápido) console.log(parrafo.textContent); // leer parrafo.textContent = "Nuevo texto"; // escribir // innerHTML — texto con HTML (puede insertar etiquetas) parrafo.innerHTML = "<strong>Texto en negrita</strong>"; // ⚠️ NUNCA pongas datos del usuario en innerHTML → XSS // parrafo.innerHTML = usuarioInput; // ← PELIGROSO // parrafo.textContent = usuarioInput; // ← SEGURO

Cambiar estilos CSS

JavaScriptconst caja = document.querySelector(".caja"); // Estilo inline — cambia una propiedad CSS // (usa camelCase: background-color → backgroundColor) caja.style.backgroundColor = "#c44a2a"; caja.style.color = "white"; caja.style.fontSize = "20px"; caja.style.display = "none"; // ocultar caja.style.display = "block"; // mostrar

Manipular clases CSS (la forma recomendada)

JavaScriptconst btn = document.querySelector(".mi-boton"); // classList — el objeto para gestionar clases btn.classList.add("activo"); // agregar clase btn.classList.remove("activo"); // quitar clase btn.classList.toggle("activo"); // agregar si no está, quitar si está btn.classList.contains("activo"); // ¿tiene la clase? → boolean btn.classList.replace("activo", "inactivo"); // reemplazar // En CSS tienes: .activo { background: green; } // JS solo agrega/quita la clase — CSS define el aspecto

Leer y cambiar atributos

JavaScriptconst img = document.querySelector("img"); // getAttribute / setAttribute console.log(img.getAttribute("src")); // leer src img.setAttribute("src", "nueva.jpg"); // cambiar src img.setAttribute("alt", "Nueva imagen"); // cambiar alt img.removeAttribute("hidden"); // quitar atributo // Propiedades directas para atributos comunes const input = document.querySelector("input"); console.log(input.value); // valor del input console.log(input.disabled); // ¿está deshabilitado? input.placeholder = "Escribe aquí...";

Crear e insertar elementos

JavaScript// createElement — crear un elemento nuevo const nuevoParrafo = document.createElement("p"); nuevoParrafo.textContent = "Soy un párrafo nuevo"; nuevoParrafo.classList.add("destacado"); // Insertarlo en el DOM const contenedor = document.querySelector("#contenedor"); contenedor.appendChild(nuevoParrafo); // al final contenedor.prepend(nuevoParrafo); // al inicio contenedor.insertAdjacentElement("afterend", nuevoParrafo); // después // Eliminar un elemento nuevoParrafo.remove(); // innerHTML para insertar HTML complejo (con cuidado) contenedor.innerHTML += ` <div class="tarjeta"> <h3>Título</h3> <p>Descripción</p> </div> `;

Eventos — Reaccionar a las acciones del usuario
addEventListener, el objeto event y delegación · 5:29:23 – 5:40:39

Los eventos son lo que hace que JavaScript sea interactivo. Un evento es algo que sucede en la página — un clic, una tecla presionada, el mouse moviéndose, un formulario enviado — y JavaScript puede "escuchar" esos eventos y ejecutar código cuando ocurren.

🔔 Analogía: las alarmas del hogar inteligente

En una casa inteligente instalas sensores: uno en la puerta (detecta cuando alguien entra), uno en la cocina (detecta humo), uno en la ventana (detecta que se abre). Cada sensor está "escuchando" un evento específico. Cuando el evento ocurre, dispara una acción: enciende luces, llama a los bomberos, suena una alarma. addEventListener es exactamente eso: le dices a un elemento "escucha este evento, y cuando ocurra, ejecuta esta función".

addEventListener — la forma correcta de escuchar eventos

JavaScriptconst boton = document.querySelector("#mi-boton"); // Sintaxis: elemento.addEventListener("evento", función) boton.addEventListener("click", () => { console.log("¡El botón fue clickeado!"); }); // El objeto 'event' — información sobre lo que ocurrió boton.addEventListener("click", (event) => { console.log(event.target); // el elemento que recibió el clic console.log(event.type); // "click" console.log(event.clientX, event.clientY); // coordenadas del mouse event.preventDefault(); // evita el comportamiento por defecto event.stopPropagation(); // evita que el evento suba al padre });

Delegación de eventos — el truco profesional

JavaScript// PROBLEMA: Si agregas elementos dinámicamente, el listener no les aplica // SOLUCIÓN: Poner el listener en el padre y filtrar por el hijo const lista = document.querySelector("#mi-lista"); // Un solo listener en el padre para todos los hijos (actuales y futuros) lista.addEventListener("click", (e) => { if (e.target.tagName === "LI") { // filtrar solo los LI e.target.classList.toggle("completado"); // toggle la clase } }); // Ahora si agregas nuevos LI dinámicamente, también responden al clic

    Proyecto Final — Lista de Tareas (To-Do App)
    DOM + eventos + arrays + localStorage · 5:40:39 – fin

    El proyecto final integra todo el curso: arrays para guardar tareas, funciones para la lógica, manipulación del DOM para mostrar y actualizar la lista, eventos para la interactividad, y localStorage para que las tareas persistan al recargar la página.

    🗂️ To-Do App — Proyecto Final Interactivo

      Código comentado del proyecto

      JavaScript — To-Do App completa// ══════════════════════════════════════════ // LISTA DE TAREAS — Código explicado // ══════════════════════════════════════════ // ── Estado de la aplicación ─────────────── let tareas = JSON.parse(localStorage.getItem("tareas")) || []; // Si hay tareas guardadas en localStorage, las carga. // Si no, empieza con un array vacío. let filtroActual = "todas"; let contadorId = tareas.length > 0 ? Math.max(...tareas.map(t => t.id)) + 1 : 1; // ── Guardar en localStorage ─────────────── function guardar() { localStorage.setItem("tareas", JSON.stringify(tareas)); } // ── Agregar tarea ───────────────────────── function agregarTarea() { const texto = document.querySelector("#todo-input").value.trim(); if (!texto) return; // no agregar vacías tareas.push({ id: contadorId++, texto, completada: false }); document.querySelector("#todo-input").value = ""; guardar(); renderizar(); } // ── Completar / descompletar ────────────── function toggleTarea(id) { tareas = tareas.map(t => t.id === id ? { ...t, completada: !t.completada } : t ); guardar(); renderizar(); } // ── Eliminar tarea ──────────────────────── function eliminarTarea(id) { tareas = tareas.filter(t => t.id !== id); guardar(); renderizar(); } // ── Renderizar la lista ─────────────────── function renderizar() { const lista = document.querySelector("#todo-list"); // Filtrar según selección const tareasFiltradas = tareas.filter(t => { if (filtroActual === "pendientes") return !t.completada; if (filtroActual === "completadas") return t.completada; return true; }); // Construir HTML lista.innerHTML = tareasFiltradas.length === 0 ? `<li class="empty">No hay tareas ${filtroActual}</li>` : tareasFiltradas.map(t => ` <li class="todo-item ${t.completada ? 'completada' : ''}"> <input class="todo-check" type="checkbox" ${t.completada ? 'checked' : ''} onchange="toggleTarea(${t.id})"> <span class="todo-texto">${t.texto}</span> <button class="todo-del" onclick="eliminarTarea(${t.id})">✕</button> </li> `).join(""); // Actualizar estadísticas const completadas = tareas.filter(t => t.completada).length; document.querySelector("#todo-stats").innerHTML = tareas.length > 0 ? `<span>Total: ${tareas.length}</span> <span>✅ ${completadas} completadas</span> <span>⏳ ${tareas.length - completadas} pendientes</span>` : ""; }
