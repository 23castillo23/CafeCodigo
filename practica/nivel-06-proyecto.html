La estructura HTML

Primero el esqueleto — qué elementos necesita la app

📌 Antes de escribir JavaScript, necesitas el HTML. Una lista de tareas necesita:

• Un input para escribir la tarea
• Un botón para agregarla
• Una lista donde aparecen las tareas

Observa la estructura y responde: ¿qué id tiene el input? ¿Y la lista?

HTML de la app:

<!-- La app completa en HTML --> <div class="app"> <h1>Mis Tareas</h1> <<!-- Formulario para agregar --> <div class="form-agregar"> <input type="text" id="inputTarea" placeholder="Nueva tarea..."> <button id="btnAgregar">Agregar</button> </div> <<!-- Lista donde aparecen las tareas --> <ul id="listaTareas"></ul> </div>
✅ Regla fundamental antes de escribir JavaScript:
Siempre escribe primero el HTML y asígnale ids a los elementos que vas a necesitar desde JavaScript. Sin esos ids no puedes seleccionarlos con querySelector.

✏️ ¿Qué id tiene el input? ¿Qué id tiene la lista? Escríbelo:

💡 Mira el HTML de arriba. Cada elemento que necesitas desde JS tiene un id.
El input: id="inputTarea" → seleccionas con querySelector('#inputTarea')
La lista: id="listaTareas" → seleccionas con querySelector('#listaTareas')

Agregar tareas

Leer el input, crear el elemento y agregarlo a la lista

📌 Cuando el usuario hace clic en "Agregar":
1. Leer el texto del input
2. Si está vacío, no hacer nada
3. Crear un <li> con ese texto
4. Agregarlo a la lista
5. Limpiar el input

Escribe la función agregarTarea() y conéctala al botón.

Demo interactiva — prueba tu código aquí:

    La lógica paso a paso:

    function agregarTarea() { // 1. Leer el input const texto = document.querySelector('#inputTarea').value.trim() // 2. Validar que no esté vacío if (!texto) return // 3. Crear el elemento de lista const li = document.createElement('li') li.textContent = texto // 4. Agregarlo a la lista document.querySelector('#listaTareas').appendChild(li) // 5. Limpiar el input document.querySelector('#inputTarea').value = '' } // Conectar al botón document.querySelector('#btnAgregar').addEventListener('click', agregarTarea)

      Guardar con localStorage

      Las tareas sobreviven aunque cierres el navegador

      📌 El problema ahora: si recargas la página, las tareas desaparecen.

      La solución: guardar las tareas en localStorage cada vez que cambien.

      El patrón es siempre el mismo:
      1. Tener un arreglo tareas en JavaScript
      2. Cada cambio → actualizar el arreglo → guardar con JSON.stringify
      3. Al cargar la página → leer con JSON.parse → dibujar todo

      El patrón de localStorage con arreglos:

      // El arreglo de tareas — la fuente de verdad let tareas = [] // Al cargar la página — recuperar lo guardado const guardadas = localStorage.getItem('mis-tareas') if (guardadas) tareas = JSON.parse(guardadas) // Guardar después de cada cambio function guardar() { localStorage.setItem('mis-tareas', JSON.stringify(tareas)) } // Agregar una tarea function agregar(texto) { tareas.push({ id: Date.now(), texto, hecha: false }) guardar() dibujar() ← redibuja toda la lista } // Dibujar la lista desde el arreglo function dibujar() { const lista = document.querySelector('#listaTareas') lista.innerHTML = '' ← limpiar primero tareas.forEach(tarea => { lista.appendChild(crearElemento(tarea)) }) }

      ✏️ Escribe en la consola las respuestas a estas preguntas:

      💡 Date.now() devuelve los milisegundos desde 1970 — siempre es único.
      Sirve como id porque dos tareas nunca se crean en el mismo milisegundo exacto.
      Vaciar con innerHTML = '' antes de dibujar evita que los elementos se dupliquen.
