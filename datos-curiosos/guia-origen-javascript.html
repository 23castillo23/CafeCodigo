Introducción — un nombre que confunde desde 1995
El malentendido más viejo de la programación web

Casi todo el que empieza a programar se hace la misma pregunta al ver el nombre: "¿JavaScript es una versión chiquita de Java?" La respuesta corta es no. La respuesta larga es una historia de una empresa apurada, un lenguaje escrito en 10 días y una decisión de marketing que quedó pegada para siempre.

💡 El dato en una frase

JavaScript se llama así por una alianza comercial entre Netscape y Sun Microsystems en 1995 — no porque sea una variación de Java. Técnicamente, se parecen tanto como un carro y una alfombra: ambas palabras en inglés empiezan distinto, pero eso es todo.

📅
10 días que crearon un lenguaje
Brendan Eich y la carrera contra el calendario

En mayo de 1995, un ingeniero de Netscape llamado Brendan Eich recibió el encargo de crear un lenguaje de programación para que las páginas web dejaran de ser documentos estáticos y pudieran reaccionar a lo que hacía el usuario. Tenía diez días.

🏃 Analogía: la receta de emergencia

Imagina que te avisan una hora antes de que llegan invitados a cenar y tienes que inventar un platillo con lo que hay en la alacena. Eso hizo Eich, pero con un lenguaje de programación: tomó ideas de varios lenguajes que ya conocía (Scheme, Self, Java) y las mezcló rápido para cumplir con el plazo de Netscape.

El resultado fue un lenguaje con funciones como "ciudadanos de primera clase" (tomado de Scheme), prototipos en lugar de clases (tomado de Self), y una sintaxis de llaves y punto y coma parecida a la de Java — para que se sintiera familiar a los programadores de la época.

🔄
Tres nombres en menos de un año
Mocha → LiveScript → JavaScript

El lenguaje no nació llamándose JavaScript. Pasó por tres nombres distintos en cuestión de meses:

NombreMomentoQué pasó
MochaMayo 1995Nombre interno de desarrollo, usado por Eich y el equipo de Netscape.
LiveScriptSeptiembre 1995Nombre público al lanzarlo en el navegador Netscape Navigator 2.0 beta.
JavaScriptDiciembre 1995Renombrado como parte de un acuerdo comercial con Sun Microsystems, dueña de Java.

El cambio final, de LiveScript a JavaScript, no tuvo nada que ver con el código — fue una decisión de negocio.

📈
La jugada de marketing detrás del nombre
1995 era el año en que "Java" vendía solo

En 1995, Java —el lenguaje de Sun Microsystems— era la palabra de moda en la industria del software. Netscape tenía un acuerdo de colaboración con Sun, y renombrar su nuevo lenguaje para que sonara relacionado con Java era, sobre todo, una estrategia de mercadotecnia para subirse a esa ola de popularidad.

🟢 No fue una decisión técnica

  • El lenguaje no se rediseñó para parecerse más a Java al cambiar de nombre.
  • La sintaxis con llaves ya se parecía a Java desde antes, como a muchos lenguajes de la época.
  • El nombre "JavaScript" es una marca registrada de Oracle (heredada de Sun) hasta hoy.
☕
¿Entonces qué relación real tienen?
Casi ninguna — y las diferencias lo demuestran
🏗️
Tipado
Java es de tipado estático (defines el tipo de cada variable). JavaScript es de tipado dinámico (el tipo se decide en tiempo de ejecución).
🧬
Modelo de objetos
Java usa clases desde el inicio. JavaScript nació basado en prototipos — las clases llegaron muchos años después, en ES6, como una capa de azúcar sintáctica.
⚙️
Cómo se ejecutan
Java se compila a bytecode y corre en la JVM. JavaScript corre interpretado (y hoy compilado JIT) directamente en el motor del navegador.
🎯
Para qué se creó
Java se creó para aplicaciones independientes de la plataforma. JavaScript se creó específicamente para reaccionar a eventos dentro de una página web.

⚠️ El error que persigue al lenguaje

Confundir Java con JavaScript sigue siendo, treinta años después, uno de los malentendidos más comunes fuera del mundo de la programación — un recordatorio de que un nombre elegido por marketing puede pesar más que la lógica técnica, para siempre.

💡 Resumen del dato curioso

JavaScript nació como Mocha, se lanzó como LiveScript, y terminó llamándose JavaScript por un acuerdo comercial con Sun Microsystems en 1995 — no por parecido técnico con Java. Ambos lenguajes comparten una sintaxis de llaves heredada de la misma época, pero su tipado, su modelo de objetos y su forma de ejecutarse son completamente distintos.
