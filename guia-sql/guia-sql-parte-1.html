¿Qué es SQL y por qué aprenderlo?
El lenguaje más usado para bases de datos relacionales · Introducción

SQL (Structured Query Language) es el lenguaje estándar para comunicarse con bases de datos relacionales. Con SQL puedes crear estructuras de datos, insertar información, consultarla, actualizarla y eliminarla. Es el lenguaje más solicitado en ofertas de trabajo de tecnología.

📚 Analogía: la biblioteca organizada

Imagina una biblioteca gigante con millones de libros. Sin un sistema de organización, encontrar un libro sería imposible. La biblioteca tiene ficheros, categorías, estantes numerados — una estructura que permite encontrar cualquier libro en segundos. Una base de datos es esa biblioteca digital, y SQL es el lenguaje con el que le hablas al bibliotecario: "tráeme todos los libros de ciencia ficción publicados después del 2010, ordenados por autor". El bibliotecario entiende ese lenguaje y te devuelve exactamente lo que pediste.

¿Qué es una base de datos relacional?

Una base de datos relacional organiza la información en tablas (como hojas de Excel) que pueden estar relacionadas entre sí. Cada tabla tiene filas (registros) y columnas (campos). La "relación" viene de poder conectar tablas: la tabla de clientes se relaciona con la tabla de pedidos, que se relaciona con la tabla de productos.

SQL es el lenguaje. MySQL es el programa que lo interpreta y gestiona las bases de datos. Es el más popular del mundo, gratuito, open source, y el que usa el curso de Dalto.

🟢 Instalación de MySQL + Workbench paso a paso

  • Ve a dev.mysql.com/downloads/ y descarga MySQL Installer (Windows) o el paquete de tu sistema operativo
  • Durante la instalación, selecciona "Developer Default" — incluye MySQL Server + Workbench
  • Elige una contraseña para el usuario root — no la olvides, la necesitarás siempre
  • Al terminar, abre MySQL Workbench, conéctate a localhost con el usuario root
  • Verás el editor SQL donde ejecutarás todas las consultas del curso

💡 Cómo ejecutar SQL en Workbench

Escribe tu consulta SQL en el área de texto central. Para ejecutarla: Ctrl+Enter (ejecuta la línea donde está el cursor) o Ctrl+Shift+Enter (ejecuta todo). El resultado aparece en el panel inferior. El rayo ⚡ en la barra también ejecuta.

Crear, mostrar y eliminar Bases de Datos
CREATE DATABASE · SHOW DATABASES · DROP DATABASE · USE

Una base de datos es el contenedor principal que agrupa todas las tablas relacionadas de un proyecto. Es lo primero que debes crear antes de hacer cualquier otra cosa.

🏢 Analogía: el edificio de oficinas

Una base de datos es como un edificio. Dentro del edificio hay oficinas (tablas). Dentro de cada oficina hay archiveros (registros). Puedes tener varios edificios (varias bases de datos): uno para tu tienda online, otro para tu sistema de recursos humanos, otro para tu blog. MySQL puede gestionar todos esos edificios a la vez, pero en cada momento solo trabajas "dentro de" uno.

Comandos para gestionar bases de datos

SQL-- Ver todas las bases de datos existentes SHOW DATABASES; -- Crear una nueva base de datos CREATE DATABASE tienda_online; -- Crear solo si NO existe (evita error si ya existe) CREATE DATABASE IF NOT EXISTS tienda_online; -- Seleccionar la base de datos con la que vamos a trabajar USE tienda_online; -- Ver en qué base de datos estamos SELECT DATABASE(); -- Eliminar una base de datos (¡CUIDADO! No hay vuelta atrás) DROP DATABASE tienda_online; -- Eliminar solo si existe (evita error si no existe) DROP DATABASE IF EXISTS tienda_online;

⚠️ DROP DATABASE es irreversible

El comando DROP DATABASE elimina la base de datos y todos sus datos para siempre. No hay papelera de reciclaje ni Ctrl+Z. Antes de ejecutarlo, asegúrate de estar completamente seguro. En producción, siempre haz un backup antes.

Crear, mostrar y eliminar Tablas
CREATE TABLE · SHOW TABLES · DESCRIBE · DROP TABLE

Las tablas son el corazón de una base de datos relacional. Cada tabla representa una entidad de tu sistema: clientes, productos, pedidos, empleados. Al crear una tabla defines su estructura: qué columnas tiene y qué tipo de dato guarda cada una.

📊 Analogía: la hoja de cálculo estructurada

Una tabla SQL es como una hoja de Excel muy estricta. Primero defines las columnas (nombre, tipo de dato, restricciones) — como poner encabezados y configurar validaciones. Después agregas filas de datos. La diferencia clave: Excel te deja poner cualquier cosa en cualquier celda. SQL te obliga a respetar la estructura que definiste — eso garantiza consistencia y permite hacer consultas eficientes.

CREATE TABLE — crear la estructura

SQLUSE cafe_curso; -- Crear una tabla de productos CREATE TABLE productos ( id INT AUTO_INCREMENT PRIMARY KEY, -- ID único, se incrementa solo nombre VARCHAR(100) NOT NULL, -- Texto, máx 100 chars, obligatorio precio DECIMAL(10,2) NOT NULL, -- Número decimal (ej: 1999.99) stock INT DEFAULT 0, -- Entero, por defecto 0 categoria VARCHAR(50), -- Texto opcional (puede ser NULL) activo BOOLEAN DEFAULT TRUE, -- Booleano, por defecto activo fecha_alta DATETIME DEFAULT NOW() -- Fecha y hora de creación automática ); -- Ver todas las tablas de la base de datos actual SHOW TABLES; -- Ver la estructura de una tabla (columnas, tipos, restricciones) DESCRIBE productos; -- También funciona: DESC productos; -- Ver el SQL que generó la tabla SHOW CREATE TABLE productos; -- Eliminar una tabla DROP TABLE IF EXISTS productos;

Insertar, actualizar y eliminar Registros
INSERT INTO · UPDATE · DELETE · TRUNCATE

Una vez que tienes la estructura (tablas), es hora de llenarla con datos. Los registros son las filas de datos dentro de una tabla.

INSERT INTO — agregar registros

SQLUSE cafe_curso; -- Insertar un registro (especificando columnas) INSERT INTO clientes (nombre, email, telefono, ciudad) VALUES ('Ana García', 'ana@gmail.com', '5551234567', 'México DF'); -- Insertar múltiples registros en una sola consulta (más eficiente) INSERT INTO clientes (nombre, email, ciudad) VALUES ('Luis Pérez', 'luis@hotmail.com', 'Guadalajara'), ('María López', 'maria@gmail.com', 'Monterrey'), ('Carlos Ruiz', 'carlos@yahoo.com', 'Guadalajara'), ('Sofia Méndez', 'sofia@outlook.com', 'México DF'); -- Insertar productos en el menú INSERT INTO menu (nombre, precio, categoria) VALUES ('Café Americano', 45.00, 'bebida'), ('Cappuccino', 65.00, 'bebida'), ('Croissant', 55.00, 'comida'), ('Cheesecake', 80.00, 'postre'), ('Latte', 70.00, 'bebida'), ('Sandwich Club', 95.00, 'comida');

UPDATE — modificar registros existentes

SQL-- Actualizar UN registro (siempre usa WHERE para no actualizar todos) UPDATE clientes SET telefono = '5559876543', ciudad = 'Puebla' WHERE id = 1; -- Actualizar VARIOS registros con una condición UPDATE menu SET precio = precio * 1.10 -- aumentar precios 10% WHERE categoria = 'bebida'; -- Desactivar un producto UPDATE menu SET disponible = FALSE WHERE nombre = 'Croissant';

⚠️ UPDATE sin WHERE — el error más peligroso en SQL

Si ejecutas UPDATE clientes SET ciudad = 'Prueba' sin el WHERE, actualizarás la ciudad de TODOS los clientes. Antes de ejecutar un UPDATE, siempre escribe primero el WHERE. MySQL Workbench tiene un modo "Safe Updates" que bloquea esto — mantenlo activado mientras aprendes.

DELETE y TRUNCATE — eliminar registros

SQL-- Eliminar UN registro específico DELETE FROM clientes WHERE id = 3; -- Eliminar VARIOS registros con condición DELETE FROM menu WHERE disponible = FALSE; -- TRUNCATE: vacía la tabla ENTERA y resetea el auto_increment -- Más rápido que DELETE sin WHERE, pero no se puede deshacer TRUNCATE TABLE menu;

Tipos de Datos en MySQL
Numéricos, texto, fechas y especiales

Elegir el tipo de dato correcto para cada columna es fundamental: afecta el espacio en disco, la velocidad de las consultas y qué operaciones puedes hacer con el dato.

📦 Analogía: los moldes de fábrica

En una fábrica de chocolates, tienes moldes de diferentes formas y tamaños. No puedes poner un chocolate de 500g en un molde de 50g, ni tiene sentido usar un molde enorme para uno pequeño. Los tipos de datos son esos moldes: cada columna tiene exactamente el molde adecuado para lo que va a guardar. Usar el molde equivocado desperdicia espacio, ralentiza la base de datos o directamente no permite guardar el dato.

Tipos numéricos

Entero pequeño
TINYINT
-128 a 127 (o 0-255 UNSIGNED). Para estados, booleanos, edades pequeñas. 1 byte.
Entero estándar
INT
-2,147,483,648 a 2,147,483,647. El más usado para IDs y contadores. 4 bytes.
Entero grande
BIGINT
Hasta ±9.2 quintillones. Para IDs de alta escala (Twitter usa BIGINT para IDs de tweets). 8 bytes.
Decimal exacto
DECIMAL(m,d)
Para dinero y precios. DECIMAL(10,2) → hasta 99,999,999.99. Exacto, sin redondeo.
Decimal flotante
FLOAT / DOUBLE
Para cálculos científicos. Puede tener pequeños errores de precisión — NO usar para dinero.
Booleano
BOOLEAN / BOOL
TRUE o FALSE. Internamente es TINYINT(1). 0=FALSE, cualquier otro=TRUE.

Tipos de texto

Texto variable
VARCHAR(n)
El más usado. Hasta n caracteres. Solo usa el espacio que necesita. Ideal para nombres, emails, títulos. Máx: 65,535.
Texto fijo
CHAR(n)
Siempre ocupa n caracteres (rellena con espacios). Para datos de longitud fija: códigos postales, códigos de país.
Texto largo
TEXT
Para textos largos: descripciones, artículos, comentarios. Hasta 65,535 caracteres. No se puede indexar completo.
Lista fija
ENUM('a','b','c')
Solo acepta los valores listados. Para categorías fijas: estados, roles, tipos. Eficiente en almacenamiento.

SELECT — La consulta más importante de SQL
Leer y filtrar datos de las tablas

SELECT es el comando que usarás el 80% del tiempo en SQL. Sirve para leer datos de una o más tablas, con filtros, ordenamiento, agrupaciones y más.

🔎 Analogía: el buscador inteligente

SELECT es como un buscador de archivos muy avanzado. No solo te muestra archivos — te deja filtrar por fecha, tipo, tamaño; ordenarlos por nombre o fecha; mostrar solo 10 resultados; combinar archivos de diferentes carpetas. La diferencia con buscadores web: tú defines exactamente qué quieres ver y cómo lo quieres ver, con una precisión quirúrgica.

Anatomía de un SELECT

SQL-- Estructura completa de un SELECT (no todas las partes son obligatorias) SELECT columnas -- ¿Qué columnas quiero ver? FROM tabla -- ¿De qué tabla? WHERE condición -- ¿Con qué filtro? (opcional) ORDER BY columna ASC/DESC -- ¿En qué orden? (opcional) LIMIT número; -- ¿Cuántos resultados máximo? (opcional)
SQLUSE cafe_curso; -- Seleccionar TODAS las columnas SELECT * FROM clientes; -- Seleccionar columnas específicas SELECT nombre, email, ciudad FROM clientes; -- Alias — cambiar el nombre de la columna en el resultado SELECT nombre AS 'Cliente', email AS 'Correo electrónico', ciudad AS 'Ubicación' FROM clientes; -- Columnas calculadas SELECT nombre, precio, precio * 1.16 AS 'precio_con_iva' FROM menu; -- SELECT sin FROM — útil para calcular o ver funciones SELECT 2 + 2; SELECT NOW(); SELECT VERSION();

WHERE — Filtrar resultados
Condiciones, operadores y patrones de búsqueda

WHERE filtra las filas que cumplan una condición. Sin él, SELECT devuelve todo. Con él, devuelves exactamente lo que necesitas.

SQL-- WHERE con comparadores simples SELECT * FROM clientes WHERE ciudad = 'México DF'; SELECT * FROM menu WHERE precio > 70; SELECT * FROM menu WHERE precio BETWEEN 50 AND 80; -- Condiciones múltiples SELECT * FROM menu WHERE categoria = 'bebida' AND precio < 70; SELECT * FROM clientes WHERE ciudad = 'México DF' OR ciudad = 'Guadalajara'; -- IN: equivalente a múltiples OR SELECT * FROM clientes WHERE ciudad IN ('México DF', 'Guadalajara', 'Monterrey'); -- NOT IN: todo lo que NO esté en la lista SELECT * FROM menu WHERE categoria NOT IN ('postre'); -- LIKE: búsqueda de patrones en texto -- % = cualquier cantidad de caracteres, _ = exactamente 1 carácter SELECT * FROM clientes WHERE email LIKE '%@gmail.com'; -- termina en @gmail.com SELECT * FROM menu WHERE nombre LIKE '%Latte%'; -- contiene "Latte" SELECT * FROM clientes WHERE nombre LIKE 'A%'; -- empieza con A -- IS NULL / IS NOT NULL: verificar valores vacíos SELECT * FROM clientes WHERE telefono IS NULL; SELECT * FROM clientes WHERE telefono IS NOT NULL;

ORDER BY · LIMIT · DISTINCT
Ordenar, limitar y eliminar duplicados en resultados
SQL-- ORDER BY: ordenar resultados -- ASC = ascendente (A-Z, 1-9) — es el predeterminado -- DESC = descendente (Z-A, 9-1) SELECT nombre, precio FROM menu ORDER BY precio ASC; -- del más barato al más caro SELECT nombre, precio FROM menu ORDER BY precio DESC; -- del más caro al más barato -- Ordenar por múltiples columnas SELECT nombre, categoria, precio FROM menu ORDER BY categoria ASC, precio DESC; -- Ordena primero por categoría A-Z, -- y dentro de cada categoría, por precio de mayor a menor -- LIMIT: traer solo N resultados SELECT nombre, precio FROM menu ORDER BY precio DESC LIMIT 3; -- los 3 más caros -- LIMIT con OFFSET: paginación -- LIMIT cantidad OFFSET desde_dónde SELECT * FROM clientes LIMIT 3 OFFSET 0; -- página 1 (registros 1-3) SELECT * FROM clientes LIMIT 3 OFFSET 3; -- página 2 (registros 4-6) -- DISTINCT: eliminar duplicados en el resultado SELECT DISTINCT ciudad FROM clientes; -- Sin DISTINCT: 7 filas (con ciudades repetidas) -- Con DISTINCT: 4 filas (cada ciudad una sola vez) SELECT DISTINCT categoria FROM menu;
