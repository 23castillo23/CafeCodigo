Cómo usar esta guía
La misma tarea, resuelta de dos formas

Esta guía junta cada acción cotidiana de terminal en una tabla de tres columnas: la acción que quieres lograr, cómo se hace en Linux (Bash), y cómo se hace en CMD (Windows). Cuando una acción tiene varias formas válidas de lograrse, se listan todas como filas separadas para que veas el abanico completo de opciones.

Verde = sintaxis de Linux / Bash Terracota = sintaxis de CMD / Windows

💡 La diferencia de fondo entre ambas terminales

CMD nació para DOS en los años 80 y arrastra esa herencia: comandos cortos, flags con /, y muchas tareas que requieren herramientas externas o PowerShell. Bash nació en un mundo Unix diseñado desde el inicio para encadenar comandos pequeños con pipes (|) y trabajar con texto plano. Por eso Linux suele tener más de una forma "elegante" de resolver algo, mientras que CMD suele depender de flags específicas de un comando monolítico.

💡 Tres reglas para no perderte

  • Linux distingue mayúsculas de minúsculas en todo — archivos, carpetas y comandos. CMD no.
  • Linux usa / para rutas, CMD usa \ (aunque CMD moderno también acepta / en muchos casos).
  • En Linux casi todo se resuelve con un puñado de comandos + flags. En CMD, para lo más avanzado (JSON, objetos, bucles complejos) normalmente ya te mueves a PowerShell.

💡 CMD se queda corto aquí — y está bien

Notarás que en esta sección varias filas dicen "no existe en CMD". No es un error tuyo: CMD fue diseñado en los 80 para tareas simples. Cuando Windows necesita algo más potente, la solución moderna es PowerShell, que si tiene equivalentes directos a head, tail y compañía.

Permisos y administrador
Un concepto que Linux expone y CMD casi esconde

🔑 Analogía: el mismo edificio, dos formas de gestionar llaves

Linux te da un panel de control detallado: cada archivo dice exactamente quién puede leer, escribir o ejecutar. Windows, en cambio, maneja permisos por listas de control de acceso (ACL) mucho más elaboradas mediante icacls, pero en el uso diario del CMD casi nunca las tocas — en vez de eso, simplemente abres CMD "como Administrador" para tener paso libre.

💡 Aquí es donde más se parecen

Los operadores de redirección (|, >, >>, 2>) son casi idénticos en ambas terminales porque los dos heredaron esta idea del mismo lugar: los sistemas Unix de los años 70. Si dominas pipes en una, ya sabes el 90% en la otra.
