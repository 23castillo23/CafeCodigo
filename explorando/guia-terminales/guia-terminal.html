Terminal: moverte sin mouse.

La terminal es la herramienta que más vas a usar como programador después del propio editor de código. Aquí ves qué es, por qué importa, y los comandos que vas a repetir todos los días hasta que se vuelvan automáticos.

01 · Fundamentos

¿Qué es una terminal?

Una terminal es una ventana donde le escribes instrucciones a tu computadora en forma de texto, en vez de hacer clic en íconos. Escribes un comando, presionas Enter, y la computadora lo ejecuta y te devuelve el resultado — también en texto.

Terminal

La ventana o programa donde escribes. Es la "cara" visible: la app que abres.

Shell

El intérprete que entiende tus comandos y los ejecuta. Bash, Zsh y PowerShell son shells distintos.

Consola

Se usa casi como sinónimo de terminal. Históricamente era la pantalla física conectada a una computadora central.

En la práctica, cuando alguien dice "abre la terminal" o "abre la consola", casi siempre se refiere a lo mismo: esa ventana negra (o de colores) donde vas a escribir comandos.

02 · Motivación

¿Por qué usarla en vez del mouse?

No es que el mouse esté mal — es que hay tareas que con la terminal son instantáneas y con clics se vuelven eternas.

  • Velocidad: renombrar 200 archivos con un patrón toma un comando, no 200 clics.
  • Repetibilidad: un comando se puede guardar, compartir y volver a ejecutar exactamente igual.
  • Es donde vive el resto de tus herramientas: Git, servidores locales, gestores de paquetes (npm, pip) — casi todo el ecosistema de programación se controla desde la terminal.
  • Acceso remoto: cuando te conectas a un servidor que no tiene pantalla ni mouse, la terminal es la única forma de trabajar en él.
💡

No necesitas memorizar todo de un jazo. Con 10-15 comandos básicos ya puedes moverte, crear, editar y borrar sin abrir el explorador de archivos. El resto se va pegando con la práctica.

03 · Lo esencial

Navegación básica

Todo en tu computadora vive dentro de carpetas (también llamadas directorios), una dentro de otra. Navegar es simplemente moverte entre esas carpetas usando comandos en lugar de doble clic.

terminal
$ pwd
/home/hugo/proyectos
// pwd = "print working directory": te dice dónde estás parado
$ ls
capitulos.html index.html img/ css/
// ls (Linux/Mac) o dir (Windows): lista lo que hay en la carpeta actual
$ cd img
// cd = "change directory": entras a la carpeta "img"
$ cd ..
// ".." significa "un nivel arriba" — regresas a la carpeta anterior
ComandoQué hace
pwdMuestra la ruta completa de la carpeta donde estás.
ls / dirLista archivos y carpetas del directorio actual. ls -la muestra también los ocultos y con detalle.
cd <carpeta>Entra a esa carpeta. cd .. sube un nivel, cd ~ te lleva a tu carpeta de usuario.
cd /Te lleva a la raíz del sistema de archivos.
04 · Lo esencial

Crear, mover y borrar archivos

Una vez que sabes moverte, el siguiente paso es manipular lo que hay dentro de cada carpeta.

ComandoQué hace
mkdir nombreCrea una carpeta nueva llamada "nombre".
touch archivo.txtCrea un archivo vacío (en Windows, type nul > archivo.txt).
cp origen destinoCopia un archivo o carpeta (con -r para carpetas completas).
mv origen destinoMueve o renombra un archivo — no deja copia en el lugar original.
rm archivoBorra un archivo. Con -r borra una carpeta completa.
⚠️

rm no manda nada a la papelera. Cuando borras algo desde la terminal, se borra directo — no hay "deshacer". Antes de usar rm -r en una carpeta, revisa dos veces qué estás borrando y desde dónde.

05 · Lo esencial

Ver el contenido de un archivo

No siempre necesitas abrir un editor completo solo para revisar qué hay dentro de un archivo.

terminal
$ cat notas.txt
Aquí aparece todo el contenido del archivo de una sola vez
$ head -n 5 notas.txt
Muestra solo las primeras 5 líneas

cat (de "concatenate") imprime todo el archivo en la terminal. Es perfecto para archivos cortos como configuraciones o notas rápidas — para archivos largos, se usa más un editor.

06 · Un poco más allá

Permisos, sin miedo

En sistemas tipo Linux/Mac, cada archivo tiene permisos que definen quién puede leerlo, escribirlo o ejecutarlo. No necesitas dominarlos desde el día uno, pero sí reconocerlos.

🔐

rwx

Read (leer), write (escribir), execute (ejecutar). Al escribir ls -l vas a ver algo como -rwxr--r--: son los permisos del dueño, del grupo y de todos los demás, en ese orden.

🧑‍💻

sudo

"Super user do" — ejecuta un comando con permisos de administrador. Se usa cuando el sistema te bloquea una acción por seguridad. Úsalo solo cuando sepas exactamente qué comando estás corriendo.

08 · Un poco más allá

Tuberías y redirección

Uno de los grandes poderes de la terminal es poder conectar comandos entre sí, para que la salida de uno sea la entrada del siguiente.

terminal
$ ls | grep .html
// el símbolo | ("pipe") pasa el resultado de ls a grep, que filtra solo lo que contiene ".html"
$ ls > lista.txt
// el símbolo > guarda el resultado del comando en un archivo, en vez de mostrarlo en pantalla

No necesitas usarlas todos los días, pero reconocerlas te va a ahorrar mucha confusión cuando las veas en guías o en código de otras personas.

09 · Referencia rápida

Glosario

Terminal
La ventana donde escribes comandos de texto para controlar la computadora.
Shell
El programa que interpreta y ejecuta los comandos que escribes (Bash, Zsh, PowerShell).
Comando
Una instrucción de texto que le das a la terminal, como ls o cd.
Argumento
Información extra que le das a un comando, como el nombre de la carpeta en cd carpeta.
Flag / bandera
Un modificador que cambia el comportamiento de un comando, como el -r en rm -r.
Ruta (path)
La dirección de un archivo o carpeta dentro del sistema. Puede ser absoluta o relativa.
Ruta absoluta
Empieza desde la raíz del sistema, ej. /home/hugo/proyectos.
Ruta relativa
Parte de donde ya estás parado, ej. ../imagenes.
