Arrow functions

La forma moderna y corta de escribir funciones

✂️ Como escribir un mensaje de texto en lugar de una carta formal — dices lo mismo pero de forma más corta y directa. Las arrow functions son funciones normales escritas con menos código. Son la forma que más verás en código moderno, tutoriales y librerías.

Una arrow function usa una flecha => en lugar de la palabra function. Hace exactamente lo mismo pero con una sintaxis más limpia. Además tiene atajos cuando el cuerpo es muy sencillo.

JS
// Función normal
function sumar(a, b) {
  return a + b
}

// Arrow function — misma función, menos código
const sumar = (a, b) => {
  return a + b
}

// Atajo 1: si el cuerpo es solo un return, quita las llaves y el return
const sumar = (a, b) => a + b
// hace exactamente lo mismo

// Atajo 2: si hay un solo parámetro, quita los paréntesis
const doblar = numero => numero * 2
// doblar(5) → 10

// Sin parámetros: los paréntesis vacíos son obligatorios
const saludar = () => "Hola mundo"

Las arrow functions brillan dentro de métodos como forEach, map y filter porque hacen el código mucho más legible:

JS
const numeros = [1, 2, 3, 4, 5]

// Con función normal (más largo)
const dobles = numeros.map(function(n) { return n * 2 })

// Con arrow function (limpio)
const dobles = numeros.map(n => n * 2)

// Más ejemplos
const pares  = numeros.filter(n => n % 2 === 0)   ← [2, 4]
const total  = numeros.reduce((acc, n) => acc + n, 0) ← 15

Callbacks

Una función que le pasas a otra para que la llame después

📞 Como dejar tu número de teléfono — "cuando termines, llámame a este número". El callback es ese número: una función que le das a otra función para que la ejecute cuando la necesite.

En JavaScript las funciones son valores — igual que un número o un texto, puedes guardar una función en una variable y pasarla como argumento a otra función. Esa función que pasas se llama callback.

JS
// Una función que recibe otra función como argumento
function ejecutarDespues(tarea, callback) {
  console.log("Ejecutando: " + tarea)
  callback()   ← llama a la función que recibió
}

function avisar() {
  console.log("¡Terminé!")
}

ejecutarDespues("limpiar", avisar)
// "Ejecutando: limpiar"
// "¡Terminé!"

Ya usabas callbacks sin saberlo — addEventListener y forEach los usan siempre:

JS
// La función que pasas al addEventListener es un callback
boton.addEventListener('click', () => {   ← callback
  console.log("clic!")
})

// La función que pasas al forEach es un callback
[1, 2, 3].forEach(numero => {            ← callback
  console.log(numero)
})

// setTimeout — ejecuta el callback después de N milisegundos
setTimeout(() => {                        ← callback
  console.log("Pasaron 2 segundos")
}, 2000)
¿Por qué existen los callbacks?

Porque JavaScript necesita manejar cosas que no ocurren al instante — un clic del usuario, una respuesta de un servidor, un temporizador. No puedes simplemente "esperar" porque bloquearías toda la página. En cambio, dices "cuando esto ocurra, llama a esta función". Eso es exactamente un callback.

Scope — el alcance de las variables

Desde dónde puedes ver y usar cada variable

🏠 Como las habitaciones de una casa — lo que está en el salón lo puede ver toda la familia (scope global), pero lo que está en tu cuarto solo lo ves tú (scope local). Si alguien intenta usar algo de tu cuarto desde afuera, no puede — no existe para ellos.

El scope define desde qué partes del código puedes acceder a una variable. Hay dos tipos principales: scope global (la variable existe en todo el programa) y scope local (la variable solo existe dentro del bloque donde fue creada).

JS
// SCOPE GLOBAL — visible en todo el programa
const idioma = "español"

function saludar() {
  console.log(idioma)   ← ✅ puede verlo
}

// SCOPE LOCAL — solo existe dentro de la función
function calcular() {
  const resultado = 42
  console.log(resultado)  ← ✅ funciona
}

console.log(resultado)   ← ❌ ERROR — no existe aquí afuera

// Scope de bloque — if, for, while
if (true) {
  const mensaje = "hola"
  console.log(mensaje)   ← ✅ funciona
}
console.log(mensaje)     ← ❌ ERROR — no existe fuera del if
var vs let vs const — y el scope:

var (la forma antigua) tiene scope de función — ignora los bloques if y for.
let y const (la forma moderna) tienen scope de bloque — respetan todo { }.

Por eso en código moderno siempre se usa const y let, nunca var. El scope de bloque hace el código más predecible y evita errores difíciles de encontrar.

Closure

Una función que recuerda el entorno donde nació

🎒 Como una mochila que llevas al salir de casa — aunque salgas de la casa (la función termine), sigues cargando todo lo que metiste en la mochila (las variables del scope donde naciste). Un closure es una función que lleva consigo el contexto donde fue creada.

Un closure ocurre cuando una función interna recuerda las variables de la función externa, incluso después de que esa función externa ya terminó de ejecutarse. Esto suena complicado, pero ocurre constantemente en código real.

JS
// Ejemplo clásico — un contador con memoria
function crearContador() {
  let cuenta = 0      ← vive en el scope de crearContador

  return () => {      ← devuelve una función
    cuenta++
    return cuenta
  }
}

const contador = crearContador()
// crearContador ya terminó — pero "cuenta" sigue viva en la mochila

contador()   ← 1
contador()   ← 2
contador()   ← 3
// "cuenta" no se puede tocar directamente — está protegida
JS
// Uso práctico — funciones configurables
function crearSaludo(idioma) {
  return (nombre) => {
    if (idioma === "español") return "Hola, " + nombre
    if (idioma === "inglés")  return "Hello, " + nombre
  }
}

const saludarEs = crearSaludo("español")
const saludarEn = crearSaludo("inglés")

saludarEs("Ana")   ← "Hola, Ana"
saludarEn("Ana")   ← "Hello, Ana"
// Cada función recuerda el idioma con el que fue creada

Funciones de orden superior

Funciones que reciben o devuelven otras funciones

🏭 Como una fábrica que produce herramientas — en lugar de producir un producto directamente, produce máquinas que luego producen el producto. Una función de orden superior trabaja con otras funciones como si fueran datos normales.

Una función de orden superior es cualquier función que recibe una función como argumento O devuelve una función. Ya conoces varias: forEach, map, filter, reduce y addEventListener son todas funciones de orden superior.

JS
const precios = [100, 250, 80, 400, 150]

// filter — conserva los que cumplen la condición
const caros = precios.filter(p => p > 200)
// [250, 400]

// map — transforma cada elemento
const conDescuento = precios.map(p => p * 0.9)
// [90, 225, 72, 360, 135]

// reduce — acumula en un solo valor
const total = precios.reduce((acum, p) => acum + p, 0)
// 980 — suma todos los precios

// Combinando — filtrar y calcular total
const totalCaros = precios
  .filter(p => p > 200)
  .reduce((acum, p) => acum + p, 0)
// 650
💡

Ejemplo en la práctica: Cualquier tienda en línea usa esta combinación: filter para mostrar solo los productos de una categoría, map para crear la tarjeta HTML de cada uno, reduce para calcular el total del carrito. Son las herramientas más poderosas para trabajar con listas de datos.

🔗 Todo junto en código real

Arrow functions + callbacks + closure + funciones de orden superior trabajando juntos:

JS
// Un buscador con memoria — closure + arrow fn + filter
function crearBuscador(lista) {
  return (termino) =>              ← arrow fn devuelta (closure)
    lista.filter(item =>           ← recuerda "lista" del scope externo
      item.toLowerCase().includes(termino.toLowerCase())
    )
}

const frutas = ["manzana", "mango", "naranja", "mandarina"]
const buscar = crearBuscador(frutas)

buscar("man")   ← ["manzana", "mango", "mandarina"]
buscar("an")    ← ["manzana", "naranja", "mandarina"]
