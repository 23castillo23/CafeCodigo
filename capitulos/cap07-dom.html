Crear elementos desde JavaScript

Agregar cosas a la página sin tocar el HTML

🏗️ Como construir con LEGO — en lugar de tener las piezas ya puestas en el plano, las vas creando y colocando en tiempo real según lo que necesitas. JavaScript puede fabricar cualquier elemento HTML y agregarlo donde quieras.

En el capítulo 02 aprendiste a encontrar elementos que ya existían en el HTML. Ahora aprenderás a crear elementos nuevos desde cero con JavaScript y agregarlos a la página. Esto es lo que hacen las listas dinámicas, los resultados de búsqueda y cualquier contenido que se genera automáticamente.

JS
// Paso 1: crear el elemento (aún no está en la página)
const tarjeta = document.createElement("div")

// Paso 2: configurarlo — texto, clases, atributos
tarjeta.textContent = "Mi nueva tarjeta"
tarjeta.className   = "cap-card"
tarjeta.id          = "tarjeta-1"

// Paso 3: agregarlo a la página
const contenedor = document.getElementById("miContenedor")
contenedor.appendChild(tarjeta)   ← se agrega al FINAL del contenedor

// También puedes usar innerHTML para crear HTML complejo de golpe
tarjeta.innerHTML = \`
  <h3>Título</h3>
  <p>Descripción aquí</p>
  <button>Ver más</button>
\`

// Insertar antes de un elemento específico
contenedor.insertBefore(tarjeta, contenedor.firstChild)  ← al principio

// Insertar en posiciones específicas
contenedor.insertAdjacentHTML("beforeend", "<p>Al final</p>")
contenedor.insertAdjacentHTML("afterbegin", "<p>Al principio</p>")
textContent vs innerHTML — cuál usar:

textContent inserta texto plano — lo que escribas aparece tal cual como texto, sin interpretar HTML. Es más seguro.
innerHTML interpreta HTML — puedes escribir etiquetas y las convierte en elementos reales. Es más poderoso pero tiene que usarse con cuidado cuando el texto viene del usuario (puede ser un riesgo de seguridad llamado XSS).

Modificar elementos existentes

Cambiar texto, estilos y atributos en tiempo real

🖊️ Como editar un documento en tiempo real — puedes cambiar el texto, el color, el tamaño o cualquier propiedad de un elemento mientras el usuario está viendo la página, sin tener que recargarla.

Una vez que tienes una referencia a un elemento del DOM, puedes cambiar prácticamente cualquier cosa de él: su texto, sus estilos visuales, sus atributos HTML y más.

JS
const titulo = document.getElementById("miTitulo")

// Cambiar el TEXTO
titulo.textContent = "Nuevo título"

// Cambiar ESTILOS directamente
titulo.style.color       = "red"
titulo.style.fontSize    = "24px"
titulo.style.fontWeight  = "bold"
titulo.style.display     = "none"     ← esconde el elemento
titulo.style.display     = "block"    ← lo muestra de nuevo

// Cambiar ATRIBUTOS
const enlace = document.getElementById("miEnlace")
enlace.setAttribute("href", "https://google.com")
enlace.getAttribute("href")       ← lee el valor actual
enlace.removeAttribute("disabled") ← elimina el atributo

// Cambiar el HTML interno completo
const caja = document.getElementById("miCaja")
caja.innerHTML = "<strong>Nuevo contenido</strong>"

Eliminar elementos

Quitar cosas de la página desde JavaScript

🗑️ Como borrar un párrafo en un documento — el elemento desaparece de la pantalla completamente, no solo se esconde. Si quieres que desaparezca pero pueda volver, usa display: none. Si quieres que se vaya para siempre, elimínalo del DOM.
JS
// Forma moderna — el elemento se elimina a sí mismo
const elemento = document.getElementById("miElemento")
elemento.remove()

// Forma clásica — el padre elimina al hijo
const padre = document.getElementById("contenedor")
const hijo  = document.getElementById("miElemento")
padre.removeChild(hijo)

// Vaciar un contenedor completo (borrar todos sus hijos)
const lista = document.getElementById("miLista")
lista.innerHTML = ""   ← deja el contenedor vacío

// Ocultar vs eliminar
elemento.style.display = "none"   ← invisible pero sigue en el DOM
elemento.remove()                   ← desaparece del DOM completamente
¿Cuándo ocultar y cuándo eliminar?

Usa display: none cuando el elemento va a volver a aparecer — un modal, un menú desplegable, una sección que se muestra al hacer clic.
Usa remove() cuando el elemento ya no se necesita — una notificación que el usuario cerró, un elemento de una lista que se borró, un item completado.

Buscar elementos — métodos avanzados

Más formas de encontrar exactamente lo que necesitas

🔎 Como una búsqueda avanzada en Google — puedes buscar por muchos criterios diferentes, no solo por el nombre exacto. El DOM tiene múltiples métodos para encontrar elementos según lo que necesites.
JS
// getElementById — busca por id (ya lo conoces)
document.getElementById("miId")             ← uno exacto

// querySelector — busca por selector CSS (el más flexible)
document.querySelector(".mi-clase")          ← primero que coincida
document.querySelector("#miId")             ← por id
document.querySelector("button")            ← primer botón
document.querySelector(".card .titulo")     ← .titulo dentro de .card
document.querySelector("input[type='text']") ← input de tipo texto

// querySelectorAll — busca TODOS los que coincidan
document.querySelectorAll(".leccion")        ← todos con esa clase
document.querySelectorAll("button")          ← todos los botones

// querySelectorAll devuelve una NodeList — se recorre con forEach
const botones = document.querySelectorAll("button")
botones.forEach(btn => {
  btn.addEventListener("click", () => console.log("clic!"))
})

// Buscar DENTRO de un elemento (no en toda la página)
const formulario = document.getElementById("miForm")
formulario.querySelector("input")            ← input dentro del form
formulario.querySelectorAll("input")          ← todos los inputs del form

// Navegar entre elementos relacionados
elemento.parentElement        ← el elemento padre
elemento.children             ← todos los hijos directos
elemento.firstElementChild    ← el primer hijo
elemento.lastElementChild     ← el último hijo
elemento.nextElementSibling   ← el hermano siguiente
elemento.previousElementSibling ← el hermano anterior

Manipular clases CSS desde JavaScript

Cambiar el diseño de un elemento al vuelo

👔 Como cambiarle la ropa a alguien — en lugar de modificar el estilo directamente, le cambias la clase CSS y CSS hace el resto. Es más limpio porque mantiene el diseño en CSS y la lógica en JavaScript, cada uno en su lugar.

En lugar de modificar los estilos uno por uno con elemento.style.propiedad, la forma profesional es agregar o quitar clases CSS. Así el diseño queda definido en CSS y JavaScript solo decide qué clase aplicar.

JS
const boton = document.getElementById("miBoton")

// classList.add — agrega una clase
boton.classList.add("activo")
boton.classList.add("grande", "resaltado")   ← varias a la vez

// classList.remove — quita una clase
boton.classList.remove("activo")

// classList.toggle — si la tiene la quita, si no la tiene la pone
boton.classList.toggle("activo")   ← perfecto para botones on/off

// classList.contains — comprueba si tiene la clase
boton.classList.contains("activo")   ← true o false

// Ejemplo práctico — menú que se abre y cierra
const menu = document.getElementById("menu")
const btnMenu = document.getElementById("btnMenu")

btnMenu.addEventListener("click", () => {
  menu.classList.toggle("abierto")   ← abre o cierra según el estado
})

// En CSS tienes definido:
// .menu { display: none }
// .menu.abierto { display: block }

Eventos avanzados

El objeto evento, delegación y más

📋 Como un reporte de lo que pasó — cuando ocurre un evento, JavaScript no solo te avisa "hubo un clic", sino que te da un reporte completo: dónde fue el clic, qué tecla se presionó, qué elemento lo disparó. Ese reporte es el objeto evento.

Cada vez que ocurre un evento, el callback recibe automáticamente un objeto que contiene toda la información sobre lo que pasó. Ese objeto suele llamarse e o event.

JS
// El parámetro "e" es el objeto evento — contiene toda la info
boton.addEventListener("click", (e) => {
  e.target           ← el elemento que recibió el clic
  e.target.id        ← el id de ese elemento
  e.target.textContent ← el texto de ese elemento
  e.preventDefault() ← evita el comportamiento por defecto
  e.stopPropagation()← evita que el evento suba al padre
})

// preventDefault — uso más común: evitar que un form recargue la página
formulario.addEventListener("submit", (e) => {
  e.preventDefault()  ← sin esto la página se recargaría
  // ahora puedes procesar los datos tú mismo
})

// Delegación de eventos — escuchar en el padre para los hijos
// Útil cuando los hijos se crean dinámicamente
const lista = document.getElementById("miLista")

lista.addEventListener("click", (e) => {
  if (e.target.classList.contains("item")) {
    console.log("Clic en item:", e.target.textContent)
  }
})
// Aunque agregues nuevos items dinámicamente, el listener funciona
¿Qué es la delegación de eventos?

Cuando creas elementos dinámicamente con JavaScript (por ejemplo, una tarjeta nueva al hacer clic en "agregar"), esos elementos nuevos no existían cuando pusiste el addEventListener, así que no lo tienen.

La solución es poner el listener en el padre (que sí existe desde el principio) y dentro verificar qué hijo disparó el evento con e.target. Esto se llama delegación de eventos y es una técnica muy usada en aplicaciones reales.

🔗 Construir una lista dinámica completa

Crear + modificar + eliminar + eventos — todo junto:

JS
const datos = ["Matemáticas", "Historia", "Física"]
const contenedor = document.getElementById("lista")

function renderLista(items) {
  contenedor.innerHTML = ""   ← vacía primero

  items.forEach((item, i) => {
    const div = document.createElement("div")
    div.className = "item"
    div.innerHTML = \`
      <span>\${item}</span>
      <button data-index="\${i}">Eliminar</button>
    \`
    contenedor.appendChild(div)
  })
}

// Delegación — el contenedor escucha los clics de todos los botones
contenedor.addEventListener("click", (e) => {
  if (e.target.tagName === "BUTTON") {
    const i = parseInt(e.target.dataset.index)
    datos.splice(i, 1)    ← elimina del arreglo
    renderLista(datos)    ← vuelve a dibujar
  }
})

renderLista(datos)   ← dibuja la primera vez
