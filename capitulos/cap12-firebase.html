Base de Datos en la Nube

Firebase y Firestore - almacenamiento permanente, autenticacion y sincronizacion en tiempo real.

🔥

Que es Firebase

Backend completo sin escribir codigo de servidor

🏢 Como rentar un apartamento totalmente amueblado - en lugar de construir tu propia casa (servidor, base de datos, autenticacion), Google ya tiene todo listo y solo pagas por lo que usas. Firebase es el Backend as a Service mas popular del mundo.

Firebase es una plataforma de Google que provee servicios de backend listos para usar. El mas importante es Firestore - una base de datos NoSQL en la nube. Tambien tiene autenticacion, almacenamiento de archivos, hosting y mucho mas. Todo se controla desde JavaScript, sin necesidad de escribir codigo de servidor.

Firebase provee todo esto:
────────────────────────────────────────
Firestore    → Base de datos en la nube
Auth         → Login con email, Google, etc.
Storage      → Guardar archivos e imagenes
Hosting      → Publicar tu app en internet
Functions    → Codigo que corre en el servidor
                                               
Todo accesible desde JavaScript en el navegador

Configurar Firebase en tu proyecto

Los pasos para conectar tu app a Firebase

JS 
// Firebase se importa desde su CDN (igual que Google Fonts pero para JS)
// En tu HTML antes del cierre del body:
<script type="module">
  import { initializeApp } from "https://www.gstatic.com/firebasejs/11.0.1/firebase-app.js"
  import { getFirestore, collection, addDoc, getDocs }
    from "https://www.gstatic.com/firebasejs/11.0.1/firebase-firestore.js"

  // Tu configuracion — la obtienes al crear un proyecto en console.firebase.google.com
  const firebaseConfig = {
    apiKey:        "AIzaSy...",
    authDomain:    "mi-proyecto.firebaseapp.com",
    projectId:     "mi-proyecto",
    storageBucket: "mi-proyecto.appspot.com",
    messagingSenderId: "123456789",
    appId:         "1:123456789:web:abc123"
  }

  // Inicializar Firebase
  const app = initializeApp(firebaseConfig)
  const db  = getFirestore(app)     // la base de datos
</script>

Firestore - como organiza los datos

Colecciones y documentos en lugar de tablas y filas

📁 Como un archivero de folders - cada folder es una coleccion (tipo de datos), y dentro de cada folder hay fichas individuales (documentos). Cada ficha tiene su propio contenido (propiedades).
SQL (tabla)                  Firestore (coleccion)
────────────────             ────────────────────────
tabla: usuarios              coleccion: usuarios
┌─────────────────────┐        documento: {
│ id | nombre | edad  │          id: "abc123",
│ 1  | Ana   | 25     │          nombre: "Ana",
│ 2  | Luis  | 17     │          edad: 25
└─────────────────────┘        }
                               documento: {
                                 id: "def456",
                                 nombre: "Luis",
                                 edad: 17
                               }

En Firestore no hay tablas ni filas. Hay colecciones (como carpetas) que contienen documentos (como fichas). Cada documento es un objeto con sus propiedades. Los documentos pueden tener subcolecciones dentro de ellos, lo que permite estructuras muy flexibles.

📖

Leer datos de Firestore

getDocs y getDoc - obtener uno o varios documentos

JS 
// Leer TODOS los documentos de una coleccion
async function leerUsuarios() {
  const referencia    = collection(db, "usuarios")
  const instantanea   = await getDocs(referencia)

  // instantanea.docs es un arreglo de documentos
  instantanea.docs.forEach(doc => {
    console.log(doc.id)        // el id del documento
    console.log(doc.data())    // los datos como objeto
  })
}

// Convertir a arreglo de objetos normal
const usuarios = instantanea.docs.map(doc => ({
  id: doc.id,
  ...doc.data()     // spread — expande todas las propiedades
}))
// [{ id: "abc123", nombre: "Ana", edad: 25 }, ...]

// Leer UN documento especifico por id
const docRef     = doc(db, "usuarios", "abc123")
const docSnap    = await getDoc(docRef)

if (docSnap.exists()) {
  console.log(docSnap.data())   // el objeto con los datos
} else {
  console.log("No existe ese documento")
}

Escribir, actualizar y eliminar

Las operaciones de modificacion en Firestore

JS 
// AGREGAR un documento nuevo (Firestore genera el id)
const docCreado = await addDoc(collection(db, "usuarios"), {
  nombre: "Ana",
  edad:   25,
  activo: true
})
console.log("ID creado:", docCreado.id)

// ESTABLECER un documento con id especifico
await setDoc(doc(db, "usuarios", "mi-id-personalizado"), {
  nombre: "Luis",
  edad:   17
})

// ACTUALIZAR solo algunas propiedades (sin borrar las demas)
await updateDoc(doc(db, "usuarios", "abc123"), {
  edad: 26    // solo actualiza edad, nombre y activo quedan igual
})

// ELIMINAR un documento
await deleteDoc(doc(db, "usuarios", "abc123"))

Tiempo real con onSnapshot

Recibir actualizaciones automaticas cuando los datos cambian

📱 Como las notificaciones del telefono - no tienes que preguntar cada segundo "hay algo nuevo?", sino que te avisan automaticamente cuando algo cambia. onSnapshot pone a escuchar tu app para actualizaciones en tiempo real.
JS 
// onSnapshot — escucha cambios en tiempo real
const cancelarEscucha = onSnapshot(
  collection(db, "mensajes"),
  (snapshot) => {
    // esto se ejecuta cada vez que cambia algo en la coleccion
    snapshot.docs.forEach(doc => {
      console.log(doc.data())
    })
  }
)

// cancelarEscucha() — cuando ya no necesitas escuchar mas
// importante llamarlo para liberar memoria

// Esto es lo que hace WhatsApp, Slack, chats en vivo:
// cuando alguien manda un mensaje a Firestore,
// todos los que tienen onSnapshot activo reciben la actualizacion
// sin necesidad de recargar la pagina

Autenticacion con Firebase Auth

Login de usuarios sin construir tu propio sistema

Firebase Auth maneja el registro, login, logout y sesiones de usuarios de forma segura. Soporta email/contrasena, Google, Facebook, GitHub y mas. Es uno de los sistemas de autenticacion mas seguros y faciles de implementar disponibles.

JS 
import { getAuth, createUserWithEmailAndPassword,
         signInWithEmailAndPassword, signOut, onAuthStateChanged }
  from "https://www.gstatic.com/firebasejs/11.0.1/firebase-auth.js"

const auth = getAuth(app)

// REGISTRAR un usuario nuevo
async function registrar(correo, contrasena) {
  try {
    const credencial = await createUserWithEmailAndPassword(auth, correo, contrasena)
    console.log("Usuario creado:", credencial.user.uid)
  } catch (e) {
    console.log("Error:", e.message)
  }
}

// INICIAR SESION
async function login(correo, contrasena) {
  const cred = await signInWithEmailAndPassword(auth, correo, contrasena)
  console.log("Bienvenido:", cred.user.email)
}

// CERRAR SESION
await signOut(auth)

// DETECTAR si hay sesion activa
onAuthStateChanged(auth, (usuario) => {
  if (usuario) {
    console.log("Sesion activa:", usuario.email)
  } else {
    console.log("Sin sesion")
  }
})
