Herramientas del Desarrollador

VS Code, extensiones, Live Server y DevTools — las herramientas que todo desarrollador usa todos los días y que nadie suele explicar bien.

✏️

VS Code — el editor de código

El taller donde construyes todo

🔨 Como el taller de un carpintero — puedes clavar clavos con cualquier cosa, pero con las herramientas correctas el trabajo es más rápido, más preciso y mucho menos frustrante. VS Code es ese taller: el editor de código más usado del mundo, gratuito y con miles de extensiones.

Visual Studio Code (VS Code) es un editor de código desarrollado por Microsoft. Es gratuito, de código abierto, y está disponible para Windows, Mac y Linux. Más del 70% de los desarrolladores web lo usan como su herramienta principal.

No es un procesador de texto — está diseñado específicamente para escribir código. Colorea la sintaxis, detecta errores mientras escribes, autocompleta, tiene integración con Git, y se puede extender con miles de plugins.

Las partes más importantes de VS Code:

Explorer (el árbol de archivos) — ve y abre cualquier archivo del proyecto
Editor — donde escribes el código, con colores y autocompletado
Terminal integrada — puedes usar Git y comandos sin salir del editor
Paleta de comandos — presiona Ctrl+Shift+P para buscar cualquier acción
Barra de estado — abajo muestra el idioma del archivo, errores y más

Extensiones esenciales

Los plugins que hacen tu vida mucho más fácil

🧩 Como las aplicaciones de tu celular — el teléfono ya funciona sin apps, pero con las correctas se convierte en algo completamente diferente. Las extensiones de VS Code son así: opcionales pero transformadoras.

Las extensiones se instalan desde el ícono de bloques en la barra lateral (o con Ctrl+Shift+X). Busca el nombre y haz clic en Install. Aquí están las más importantes para desarrollo web: 

EXTENSIÓN                    QUÉ HACE
─────────────────────────────────────────────────────────

Live Server                  Servidor local con recarga automática
(Ritwick Dey)                Indispensable — la primera que debes instalar

Prettier                     Formatea tu código automáticamente
(Prettier)                   Alinea, indentra y hace todo consistente

ESLint                       Detecta errores en JavaScript antes de ejecutar
(Microsoft)                  Como el corrector ortográfico pero para código

Auto Rename Tag              Renombra automáticamente la etiqueta de cierre
(Jun Han)                    <div> → cambias a <section> y cierre se actualiza solo

HTML CSS Support             Autocompletado de clases CSS dentro del HTML
(ecmel)                      Escribe cla... y sugiere tus clases

GitLens                      Muestra quién cambió cada línea y cuándo
(GitKraken)                  Historial de Git inline en el editor

CSS Peek                     Haz Ctrl+Click en una clase CSS y abre el estilo
(Pranay Prakash)             Navega entre HTML y CSS sin perder el hilo

Error Lens                   Muestra los errores inline en la misma línea
(Alexander)                  Sin tener que buscarlos en otro panel

Indent Rainbow               Colorea los niveles de indentación
(oderwat)                    Muy útil para ver el anidamiento en HTML
Cómo configurar Prettier para que formatee al guardar:

1. Instala la extensión Prettier
2. Abre Configuración: Ctrl+,
3. Busca "format on save" y activa la casilla
4. Busca "default formatter" y selecciona Prettier

A partir de ahí, cada vez que guardes con Ctrl+S, Prettier alinea y formatea tu código automáticamente. Tu código siempre se verá limpio y consistente.

Live Server

Ver los cambios en el navegador al instante

📺 Como una pantalla que se actualiza sola — cada vez que guardas un archivo, el navegador recarga automáticamente y muestra los cambios al instante. Sin Live Server tendrías que presionar F5 manualmente después de cada cambio.

Live Server crea un servidor local en tu computadora y abre tu proyecto en el navegador. Cada vez que guardas cualquier archivo del proyecto, el navegador se recarga solo. También resuelve el problema de abrir archivos con file://, que limita algunas funciones como los Service Workers y las peticiones a APIs.

CÓMO USARLO 
// Formas de iniciar Live Server:

// 1. Clic derecho en index.html → "Open with Live Server"

// 2. Botón "Go Live" en la esquina inferior derecha de VS Code

// 3. Paleta de comandos: Ctrl+Shift+P → "Live Server: Open with Live Server"

// El navegador abre en:
http://127.0.0.1:5500/index.html

// 127.0.0.1 = "esta misma computadora" = localhost
// 5500 = el puerto donde Live Server escucha

// Para detenerlo:
// Clic en "Port: 5500" en la barra de estado inferior → Stop
¿Por qué no abrir el HTML directamente con doble clic?

Cuando abres un archivo con doble clic, el navegador usa el protocolo file:// en lugar de http://. Con file:// no funcionan los Service Workers, las peticiones fetch() a APIs locales, ni algunos módulos de JavaScript. Live Server usa http:// igual que un servidor real, así que todo funciona correctamente.

DevTools del navegador

La radiografía de cualquier página web

🔬 Como un microscopio — puedes ver exactamente qué hay dentro de cualquier página web, cómo funciona, qué errores tiene y cómo está construida. Los DevTools son la herramienta más poderosa para aprender y también para depurar cuando algo falla.

Las DevTools (Herramientas de Desarrollador) vienen incluidas en todos los navegadores modernos. Para abrirlas:

CÓMO ABRIR 
// Formas de abrir DevTools en cualquier navegador:

F12                    ← la forma más rápida
Ctrl + Shift + I       ← atajo de teclado (Windows/Linux)
Cmd + Option + I       ← atajo de teclado (Mac)
Clic derecho → "Inspeccionar elemento"   ← para inspeccionar algo específico

// Las pestañas más importantes:
Elements / Inspector   ← ver y editar el HTML y CSS en tiempo real
Console                ← ejecutar JavaScript y ver errores
Network                ← ver todas las peticiones HTTP
Sources                ← ver los archivos JS y poner breakpoints
Application            ← localStorage, cookies, Service Worker
Lighthouse             ← auditar rendimiento, accesibilidad y PWA

La Consola

Tu ventana directa a JavaScript

🗣️ Como hablarle directamente a la página — puedes escribir cualquier instrucción JavaScript y se ejecuta al instante en la página que tienes abierta. También es donde aparecen todos los errores y los mensajes de console.log().

La Consola es la herramienta que más vas a usar al principio. Sirve para dos cosas: ver los mensajes y errores de tu código, y ejecutar JavaScript directamente para probar cosas.

CONSOLA 
// console.log() — la instrucción más usada para depurar
console.log("Hola desde JavaScript")
console.log(miVariable)
console.log("El valor es:", miVariable)

// Otras variantes útiles
console.error("Esto es un error")    ← aparece en rojo
console.warn("Esto es una advertencia") ← aparece en amarillo
console.table(miArreglo)              ← muestra un arreglo como tabla
console.clear()                       ← limpia la consola

// En la consola puedes escribir JS directamente:
document.body.style.background = "red"    ← cambia el fondo
document.querySelectorAll("p").length    ← cuántos párrafos hay
localStorage.clear()                      ← borra el localStorage

// Los errores se ven así en la consola:
// ❌ Uncaught ReferenceError: miVariable is not defined
//    at script.js:42   ← en qué línea está el error
Cómo leer un error de la consola:

Un error típico tiene tres partes:
❌ TypeError: Cannot read properties of null — el tipo de error
at script.js:87 — el archivo y la línea donde ocurrió
reading 'textContent' — el detalle específico

Siempre empieza por la línea que dice el número. Abre ese archivo en VS Code, ve a esa línea y ahí está el problema. Ignorar los errores de consola es el error más común al aprender.
💡

Truco profesional: Cuando algo no funciona, lo primero que hace cualquier desarrollador es abrir la consola. Si hay un error rojo ahí, ese es el problema. Leer el mensaje del error (aunque parezca raro) casi siempre te dice exactamente qué está mal y en qué línea.

El Inspector de Elementos

Ver y editar HTML y CSS en tiempo real

🏗️ Como los planos de una construcción que puedes editar mientras el edificio está de pie — puedes cambiar colores, textos, clases y estructura directamente en el navegador y ver los resultados al instante. Los cambios son temporales — al recargar vuelve a la versión original.

El Inspector (pestaña Elements en Chrome, Inspector en Firefox) muestra el árbol HTML de la página y los estilos CSS que aplican a cada elemento. Es la herramienta perfecta para experimentar antes de escribir código.

CÓMO USARLO 
// Para inspeccionar un elemento específico:
// Clic derecho en cualquier cosa → "Inspeccionar"
// El elemento se resalta en el árbol HTML

// Panel izquierdo — el árbol HTML:
// - Haz doble clic en un texto para editarlo
// - Haz doble clic en un atributo para cambiarlo
// - Clic derecho → Delete element para eliminarlo temporalmente
// - Arrastra elementos para moverlos de lugar

// Panel derecho — los estilos CSS:
// - Haz clic en un valor para editarlo (color, tamaño, etc.)
// - Activa/desactiva una propiedad con la casilla de la izquierda
// - Agrega propiedades nuevas al final de cualquier selector
// - Estilos tachados = sobreescritos por otra regla más específica

// Selector de elementos (ícono de cursor arriba a la izquierda):
// Actívalo y luego haz clic en cualquier elemento de la página
// para ir directo a ese elemento en el árbol HTML
Flujo recomendado para ajustar estilos:

1. Haz clic derecho en el elemento → Inspeccionar
2. En el panel CSS de la derecha, edita los valores directamente
3. Cuando encuentres los valores que te gustan, anótalos
4. Cópialos a tu archivo style.css real
5. Guarda y verifica con Live Server

Esto evita el ciclo de editar el archivo → guardar → ver → repetir. Pruebas en el inspector y copias cuando estás seguro.

La pestaña Network (Red)

Ver todas las peticiones que hace tu página

📦 Como el historial de pedidos de una tienda online — puedes ver exactamente qué pidió tu página, cuándo, cuánto tardó en llegar y qué respondió el servidor. Es indispensable cuando trabajas con APIs y fetch().

La pestaña Network registra todas las peticiones HTTP que hace la página: los archivos CSS y JS que carga, las imágenes, y especialmente las llamadas fetch() a APIs. Para que funcione, debe estar abierta antes de cargar la página — si la abres después, no verás las peticiones anteriores.

QUÉ VER EN NETWORK 
// Cada fila es una petición. Las columnas más importantes:

Name      ← nombre del archivo o URL de la API
Status    ← código de respuesta (200=ok, 404=no existe, 500=error servidor)
Type      ← tipo: document, script, stylesheet, fetch, xhr
Size      ← tamaño de la respuesta
Time      ← cuánto tardó en responder

// Al hacer clic en una petición fetch verás:
Headers   ← info de la petición y la respuesta
Preview   ← el JSON formateado
Response  ← el texto crudo de la respuesta
Timing    ← desglose de cuánto tardó cada fase

// Filtros útiles (barra superior):
All       ← todo
Fetch/XHR ← solo las llamadas a APIs
JS        ← solo archivos JavaScript
CSS       ← solo archivos de estilos
Img       ← solo imágenes
💡

Ejemplo en la práctica: Si tu fetch() no trae datos, abre Network, haz la petición y busca la fila correspondiente. Si el Status es rojo (400, 404, 500), haz clic en esa fila y ve a la pestaña Response — ahí el servidor casi siempre explica qué salió mal.

Atajos de teclado esenciales

Los que usan todos los desarrolladores todos los días

⚡ Como los comandos de un videojuego — al principio miras el mando y piensas en cada botón, pero con práctica los dedos van solos y puedes concentrarte en el juego. Los atajos de teclado son exactamente así — al principio los buscas, con el tiempo los usas sin pensar.
VS CODE — ATAJOS ESENCIALES 
// ARCHIVOS Y NAVEGACIÓN
Ctrl + S              ← guardar archivo actual
Ctrl + P              ← buscar y abrir cualquier archivo del proyecto
Ctrl + Shift + P      ← paleta de comandos (buscar cualquier acción)
Ctrl + `              ← abrir/cerrar la terminal integrada
Ctrl + B              ← mostrar/ocultar la barra lateral
Ctrl + \              ← dividir el editor en dos paneles

// EDICIÓN DE CÓDIGO
Ctrl + D              ← seleccionar la siguiente coincidencia del texto
Ctrl + /              ← comentar/descomentar línea seleccionada
Alt + ↑ / ↓           ← mover una línea hacia arriba o abajo
Shift + Alt + ↓       ← duplicar línea actual hacia abajo
Ctrl + H              ← buscar y reemplazar texto
Ctrl + Z              ← deshacer
Ctrl + Shift + Z      ← rehacer
Tab                   ← indentar selección
Shift + Tab           ← des-indentar selección

// EMMET — atajos para HTML (solo en VS Code)
div.clase             ← escribe esto y presiona Tab
                         → <div class="clase"></div>
ul>li*3               ← escribe esto y presiona Tab
                         → <ul><li></li><li></li><li></li></ul>

Flujo de trabajo diario

Cómo se ve un día real de desarrollo

🏃 Como la rutina de un deportista — los movimientos individuales no son complicados, pero saber el orden correcto y hacerlos de forma consistente es lo que marca la diferencia. Este es el flujo que sigue casi todo desarrollador web.

Ahora que conoces las herramientas, aquí está cómo encajan en un día típico de trabajo. No tienes que seguirlo al pie de la letra, pero tener un flujo claro evita mucho tiempo perdido.

FLUJO DIARIO 
// 1. ABRIR EL PROYECTO
// Abrir VS Code → File → Open Folder → tu proyecto
// O desde la terminal: cd mi-proyecto && code .

// 2. SINCRONIZAR CON GITHUB (si trabajaste en otra computadora)
git pull   ← trae los cambios más recientes

// 3. INICIAR LIVE SERVER
// Clic derecho en index.html → Open with Live Server
// El navegador abre en http://127.0.0.1:5500

// 4. ABRIR DEVTOOLS
// F12 → pestaña Console para ver errores mientras trabajas

// 5. ESCRIBIR CÓDIGO
// VS Code + Ctrl+S para guardar → Live Server recarga automático
// Ver resultado en el navegador
// Si algo no funciona → revisar la Consola primero

// 6. GUARDAR PROGRESO EN GITHUB
git add .
git commit -m "Descripción clara de qué cambiaste"
git push

// 7. REPETIR
// El desarrollo es este ciclo: escribir → ver → corregir → guardar
EL CICLO DEL DESARROLLO WEB
────────────────────────────────────────────

  VS Code          Live Server          DevTools
  (escribes)  →   (recarga solo)  →   (revisas errores)
      ↑                                      │
      └──────────── corriges ←───────────────┘

  Cuando funciona:
  git add . → git commit -m "..." → git push → GitHub
Los errores más comunes al principio:

❌ Cambias el código pero no ves el cambio → ¿guardaste con Ctrl+S? ¿Live Server está corriendo?

❌ Algo no funciona y no sabes por qué → Abre la Consola. Casi siempre hay un error rojo ahí.

❌ El CSS no aplica → Abre el Inspector, encuentra el elemento, mira si la propiedad está tachada (sobreescrita por otra regla).

❌ fetch() no trae datos → Abre Network, encuentra la petición, revisa el Status y la pestaña Response.

❌ Git push rechaza → Primero haz git pull para sincronizar, luego vuelve a intentar el push.
