LocalStorage - que es y para que sirve

Un minialmacen en el navegador del usuario

💵 Como una libretita pegada al interior del navegador - puedes anotar cosas ahi y las encuentras exactamente igual la proxima vez que abras la pagina, incluso despues de cerrar el navegador y la computadora. Los datos persisten indefinidamente hasta que los borras.

LocalStorage es un mecanismo de almacenamiento que el navegador provee a cada sitio web. Permite guardar datos en el dispositivo del usuario de forma permanente. Es ideal para preferencias, configuracion, carritos de compra, datos de sesion y cualquier cosa que quieras recordar entre visitas.

JS 
// LocalStorage solo guarda TEXTO
// Todo lo que guardes se convierte a string automaticamente
// Si quieres guardar objetos o arreglos, necesitas JSON (lo vemos despues)

// Caracteristicas importantes:
// - Persiste aunque cierres el navegador
// - Cada dominio tiene su propio espacio separado
// - Capacidad de ~5MB por dominio
// - Solo accesible desde el mismo dominio (no entre sitios)
// - No se envia al servidor automaticamente (diferente a las cookies)
📥

Guardar datos

setItem - la forma de escribir en el almacenamiento

JS 
// setItem(clave, valor) — guarda un dato
localStorage.setItem("nombre", "Carlos")
localStorage.setItem("tema",   "oscuro")
localStorage.setItem("idioma", "es")

// Si ya existe la clave, sobreescribe el valor anterior
localStorage.setItem("nombre", "Ana")
// ahora "nombre" es "Ana", no "Carlos"

// Los numeros se guardan como texto automaticamente
localStorage.setItem("puntos", 100)
localStorage.getItem("puntos")   // "100" — es un string, no un numero
📤

Leer datos

getItem - recuperar lo que guardaste

JS 
// getItem(clave) — lee un dato guardado
const nombre = localStorage.getItem("nombre")   // "Ana"
const tema   = localStorage.getItem("tema")     // "oscuro"

// Si la clave no existe, devuelve null
localStorage.getItem("algo-que-no-existe")   // null

// Patron comun: usar un valor por defecto si no existe
const idioma = localStorage.getItem("idioma") || "es"
// si no hay idioma guardado, usa "es" como default

// Convertir numeros de vuelta
const puntos = parseInt(localStorage.getItem("puntos"))
// puntos es el numero 100, no el string "100"
🗑

Eliminar datos

removeItem y clear

JS 
// removeItem — elimina una clave especifica
localStorage.removeItem("nombre")

// clear — borra TODO el localStorage del sitio
localStorage.clear()
// usar con cuidado — borra todos los datos guardados

// Verificar si existe algo antes de usarlo
if (localStorage.getItem("usuario")) {
  console.log("Hay un usuario guardado")
} else {
  console.log("No hay usuario")
}
📋

Guardar objetos y arreglos con JSON

Como almacenar datos complejos

💾 Como guardar una foto en un formato especifico - antes de guardarla la conviertes a bytes (JSON.stringify), y cuando la recuperas la vuelves a convertir a imagen (JSON.parse). Es el mismo proceso para objetos y arreglos.
JS 
// JSON.stringify — convierte objeto/arreglo a texto
const usuario = { nombre: "Ana", edad: 25, activo: true }
const texto = JSON.stringify(usuario)
// '{"nombre":"Ana","edad":25,"activo":true}'  — texto

localStorage.setItem("usuario", texto)

// JSON.parse — convierte texto de vuelta a objeto
const textoGuardado = localStorage.getItem("usuario")
const usuarioRecuperado = JSON.parse(textoGuardado)
// { nombre: "Ana", edad: 25, activo: true }  — objeto de nuevo

usuarioRecuperado.nombre   // "Ana"
usuarioRecuperado.edad     // 25

// Lo mismo con arreglos
const favoritos = ["Matematicas", "Historia", "Fisica"]
localStorage.setItem("favoritos", JSON.stringify(favoritos))

const recuperados = JSON.parse(localStorage.getItem("favoritos"))
// ["Matematicas", "Historia", "Fisica"]
El patron mas comun con localStorage:

1. Al cargar la pagina: leer con getItem y JSON.parse
2. Al hacer cambios: actualizar el estado en JavaScript
3. Guardar inmediatamente: JSON.stringify y setItem

Asi siempre tienes los datos actualizados guardados y los recuperas correctamente al recargar.
🅏

SessionStorage - almacenamiento temporal

Como localStorage pero se borra al cerrar la pestana

📋 Como las notas adhesivas del escritorio - las ves mientras trabajas pero al irse a casa (cerrar la pestana) desaparecen. Son perfectas para datos que solo necesitas durante la sesion actual.
JS 
// SessionStorage tiene exactamente la misma API que localStorage
sessionStorage.setItem("paso", "2")
sessionStorage.getItem("paso")
sessionStorage.removeItem("paso")
sessionStorage.clear()

// La unica diferencia: se borra automaticamente cuando
// el usuario cierra la pestana o el navegador

// Cuando usar cada uno:
// localStorage  → preferencias, tema, idioma, carrito guardado
// sessionStorage → datos de un proceso de varios pasos, formulario de varios pasos

Preferencias de usuario con localStorage

Un sistema completo de tema oscuro/claro que persiste entre visitas:

JS 
// Al cargar la pagina — recuperar preferencia guardada
const temaGuardado = localStorage.getItem("tema") || "claro"
document.body.classList.add(temaGuardado)

// Boton para cambiar el tema
const btnTema = document.getElementById("btnTema")
btnTema.addEventListener("click", () => {
  const esOscuro = document.body.classList.contains("oscuro")
  const nuevoTema = esOscuro ? "claro" : "oscuro"

  document.body.classList.remove("claro", "oscuro")
  document.body.classList.add(nuevoTema)
  localStorage.setItem("tema", nuevoTema)   // guardar preferencia
})
