HTML

Los bloques de construcción

Etiquetas, elementos y atributos — el vocabulario base de toda página web.

🏷️

Etiqueta

El ladrillo básico de HTML

🧱 Como un ladrillo de construcción — define qué tipo de cosa es ese elemento. No dice cómo se ve ni qué hace, solo qué es.

Es la instrucción básica de HTML. Le dice al navegador qué tipo de elemento existe en la página: un botón, un título, una imagen, una caja. Siempre tiene una etiqueta de apertura y una de cierre, y el contenido va en medio.

HTML 
<etiqueta> contenido </etiqueta>
↑ apertura             ↑ cierre — lleva una diagonal /

Las etiquetas más importantes en cualquier página web:

HTML 
<h1> Título principal </h1>     ← el más grande, solo uno por página
<h2> Subtítulo </h2>              ← título secundario
<h3> Título pequeño </h3>         ← aún más pequeño
<p> Un párrafo de texto </p>     ← texto normal
<button> Haz clic </button>      ← botón clickeable
<input>                          ← campo donde el usuario escribe
<img>                            ← imagen
<div> cosas adentro </div>       ← caja invisible que agrupa cosas
La etiqueta más usada de todas: <div>

No tiene forma, color ni significado propio. Solo sirve para agrupar elementos. Es como una caja de cartón — el cartón no importa, importa lo que está adentro. En tu proyecto FotoApuntes hay decenas de divs organizando todo el contenido.
💡

En tu proyecto FotoApuntes: button, h1, h2, p, div, input, section, form — todas son etiquetas de tu index.html. Ábrelo en VS Code y verás cómo cada línea de contenido empieza y termina con una etiqueta.

🪆

Elemento

La cosa completa — no solo el nombre

🪆 Como una muñeca rusa completa — no es solo la cáscara de afuera, es la etiqueta de apertura + todo el contenido + la etiqueta de cierre juntos. La etiqueta es solo el nombre. El elemento es la cosa entera.

Esta diferencia confunde a todos al principio. Aquí está la explicación clara:

HTML 
<!-- La ETIQUETA es solo el nombre: -->
<button>

<!-- El ELEMENTO es todo esto junto: -->
<button class="btn-new-group" id="btnNewGroup">
  Nuevo Grupo
</button>

↑ etiqueta apertura + atributos
  ↑ contenido
    ↑ etiqueta de cierre

Un elemento también puede tener otros elementos adentro como contenido. Aquí el div es un elemento que contiene otros dos elementos:

HTML 
<div class="header-inner">       ← elemento padre
  <h1>FotoApuntes</h1>           ← elemento hijo 1
  <p>Álbum Escolar</p>            ← elemento hijo 2
</div>
Regla para no confundirlos:
Etiqueta → ¿Cómo se llama el tipo? (button, div, h1)
Elemento → ¿Qué es la cosa completa, incluyendo atributos y contenido?
📋

Atributo

Datos extra que describen el elemento

🏷️ Como las características de un producto en su etiqueta — color: rojo, talla: M, precio: $50. Los atributos son información adicional que va dentro de la etiqueta de apertura y describe o modifica el elemento.

Siempre tienen la misma forma: nombre="valor". Un elemento puede tener muchos atributos al mismo tiempo — todos van dentro de la etiqueta de apertura, separados por espacios.

HTML 
<button class="btn-new-group" id="btnNewGroup">
         ↑ atributo 1          ↑ atributo 2
         nombre="valor"        nombre="valor"

<!-- Más atributos de tu proyecto -->
<input type="text" id="galleryName" placeholder="Ej: Matemáticas">
<img   src="foto.jpg" alt="Foto de clase">

Dos atributos son especiales porque los usan CSS y JavaScript para encontrar y modificar elementos. Son tan importantes que merecen su propia explicación:

CSS class — para dar estilo

CSS lo usa para aplicar el mismo diseño a varios elementos a la vez. Puedes poner el mismo class en muchos elementos y todos recibirán el mismo estilo. Como un uniforme escolar — todos los que lo usan reciben el mismo trato.

En CSS se escribe con punto antes del nombre:
.btn-new-group { color: orange; }

JS id — para que JS lo encuentre

JavaScript lo usa para encontrar ese elemento exacto en toda la página. Solo puede haber un elemento con ese id — es único. Es como tu CURP — nadie más tiene el mismo.

En JS se usa así:
document.getElementById('btnNewGroup')

Nota importante: class e id son atributos como cualquier otro. Lo que los hace especiales es para qué los usa CSS y JavaScript — pero siguen siendo solo datos que van dentro de la etiqueta.

¿Por qué los nombres se escriben diferente?

CSS usa guiones: btn-new-group — se llama kebab-case
JS usa mayúsculas: btnNewGroup — se llama camelCase

Son el mismo botón, nombrado diferente según quién lo va a usar. Como decirle "Profe Martínez" en la escuela pero "papá" en casa — misma persona, diferente contexto.
💡

En tu proyecto: class, id, type, placeholder, href, src, accept, maxlength — todos son atributos de tu index.html. Abre el archivo, busca cualquier etiqueta y lo que va adentro antes del > son sus atributos.

Anidamiento

Elementos dentro de elementos

🏠 Como una casa — la casa contiene cuartos, los cuartos contienen muebles, los muebles contienen cajones. Cada cosa vive dentro de otra más grande. El elemento de afuera es el padre, el de adentro es el hijo.

El anidamiento describe dónde vive cada elemento. Esta es la diferencia que confunde al principio:

La diferencia clave:

Elemento responde: ¿QUÉ es esa cosa?
Anidamiento responde: ¿DÓNDE está esa cosa?

El h1 es un elemento. Que ese h1 esté dentro de un div, que está dentro de un header — eso es el anidamiento.

Así se ve el anidamiento real en tu proyecto:

HTML 
<header class="header">                      ← abuelo
  <div class="header-inner">               ← padre
    <div class="header-icon">📸</div>    ← hijo 1
    <h1 class="logo">FotoApuntes</h1>   ← hijo 2
    <p class="tagline">Álbum Escolar</p> ← hijo 3
  </div>
</header>

El mismo código como árbol familiar:

header.header                   ← abuelo
  └── div.header-inner          ← padre
        ├── div.header-icon     ← hijo 1  →  📸
        ├── h1.logo             ← hijo 2  →  "FotoApuntes"
        └── p.tagline           ← hijo 3  →  "Álbum Escolar"

La regla más importante del anidamiento es que todo elemento que abres debe cerrarse antes de cerrar su padre:

HTML 
<!-- ✅ CORRECTO — el hijo cierra antes que el padre -->
<div>
  <h1>FotoApuntes</h1>   ← h1 cierra primero
</div>                   ← luego div

<!-- ❌ INCORRECTO — el h1 no cerró en el lugar correcto -->
<div>
  <h1>FotoApuntes
</div>
</h1>   ← demasiado tarde, el div ya cerró
💡

En tu proyecto: cuando algo se ve raro o roto en la página, la causa más común es exactamente esta — una etiqueta que no se cerró en el lugar correcto. El navegador intenta arreglarlo solo pero no siempre lo hace bien. Por eso VS Code colorea el código — para que puedas ver claramente qué abre y qué cierra.
