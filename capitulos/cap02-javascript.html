📦

Variable

Una caja con nombre donde guardas algo

📦 Como cajas etiquetadas en tu cuarto — guardas algo adentro y lo recuperas usando el nombre de la caja cuando lo necesites. Sin variables no podrías recordar nada entre una línea de código y la siguiente.

Una variable guarda un valor para usarlo después. Tiene tres partes: la palabra que la crea, el nombre que tú inventas, y el valor que guardas adentro.

JS 
// Las tres partes de una variable
const  nombre  =  "Carlos"
  ↑       ↑      ↑      ↑
  │       │      │      └── el valor que guardas
  │       │      └───────── el signo igual significa "guarda esto aquí"
  │       └────────────── el nombre de la caja (tú lo inventas)
  └───────────────────── la palabra que crea la caja

Existen dos palabras para crear variables y cada una tiene un propósito distinto:

const — caja que no cambia 🔒

Una vez que guardas algo, ya no puedes cambiar lo que hay adentro. Úsala cuando el valor es fijo y no va a variar nunca. Si intentas cambiarla, JavaScript te dará un error.

let — caja que sí cambia 🔄

Puedes cambiar lo que hay adentro las veces que quieras. Úsala cuando el valor va a variar con el tiempo, como un contador o el estado actual de algo.

JS 
// const — no cambia
const nombre = "Carlos"
nombre = "Juan"      ← ❌ ERROR — const no se puede cambiar

// let — sí cambia
let puntos = 0
puntos = 10         ← ✅ bien
puntos = 25         ← ✅ bien de nuevo

// Tipos de cosas que puedes guardar
const texto   = "Hola mundo"   ← texto, siempre entre comillas
const numero  = 17              ← número, sin comillas
const verdad  = true            ← verdadero o falso (boolean)
let   actual  = null            ← vacío a propósito, se llenará después

A veces creas una caja vacía porque todavía no sabes qué vas a guardar, y la llenas después cuando el usuario hace algo:

JS 
let temporizador      ← caja vacía — se llenará cuando el usuario empiece algo
let contador = 0      ← empieza en cero — cambia cada vez que ocurre algo
let seleccionado      ← vacía — se llenará cuando el usuario elija una opción
💡

Ejemplo en la práctica: let contador = 0 guarda cuántas veces ha pasado algo — const TITULO = 'Mi App' guarda un texto fijo que no cambia — let seleccionado empieza vacía y se llena cuando el usuario elige algo. Si abres el archivo JavaScript de cualquier proyecto verás estas variables declaradas al principio, antes de cualquier función.

Función

Una receta que solo se cocina cuando alguien la llama

📋 Como una receta de cocina — existe escrita en el papel pero no pasa nada hasta que alguien la cocina. Puedes usarla mil veces sin tener que reescribirla cada vez. Eso es exactamente lo que hace una función: guarda instrucciones para ejecutarlas cuando las necesites.

Una función es un bloque de instrucciones con nombre. No se ejecuta sola — alguien tiene que llamarla. Y puedes llamarla cuantas veces quieras.

JS 
// DEFINIR la función — solo la escribe, no la ejecuta todavía
function saludar() {
  console.log("Hola!")
}

// LLAMAR la función — aquí sí pasa algo
saludar()   ← Hola!
saludar()   ← Hola!
saludar()   ← Hola!

// Sin función tendrías que repetir el código tres veces
// Con función lo escribes una vez y lo usas mil veces

Los paréntesis () sirven para pasarle información a la función. Esa información se llama parámetros. Mientras más parámetros tiene una función, más información necesita para trabajar.

JS 
// Sin parámetros — no necesita información extra
function cerrarModal() {
  // cierra el modal — no necesita saber nada más
}

// Con un parámetro — necesita saber de qué elemento
function mostrarDetalle(elementoId) {
  // muestra el detalle de ESE elemento específico
  // elementoId es como una variable temporal que solo existe aquí adentro
}

// Con dos parámetros — necesita la categoría Y su nombre para mostrarla
function abrirSeccion(categoriaId, titulo) {
  // abre la sección con ese título
  // necesita saber QUÉ abrir y CÓMO se llama
}

// Así se llaman con sus datos:
cerrarModal()
mostrarDetalle('elemento_123')
abrirSeccion('noticias', 'Últimas noticias')
Cómo leer el nombre de una función:

El nombre de una función casi siempre describe exactamente qué hace. cerrarModal → cierra el modal. mostrarDetalle → muestra un detalle. abrirSeccion → abre una sección.

Cuando no entiendes qué hace una función, lee su nombre despacio — normalmente te lo dice todo.
💡

Ejemplo en la práctica: si abres el archivo JavaScript de cualquier proyecto y buscas la palabra function encontrarás docenas de funciones, cada una con una tarea específica. Cada función tiene un nombre que describe exactamente qué hace.

🔔

Evento

Alguien hace algo — tú reaccionas

🔔 Como un timbre — alguien toca el timbre (hace algo) y tú reaccionas (abres la puerta). Sin eventos la página sería una hoja de papel estática. Los eventos son los que hacen que la página reaccione al usuario.

Es algo que ocurre en la página — un clic, escribir texto, enviar un formulario. JavaScript puede "escuchar" esos eventos con addEventListener y ejecutar código cuando ocurran.

JS 
boton.addEventListener('click', function() {
  abrirModal()
})

↑              ↑              ↑
│              │              └── qué hacer cuando ocurra el evento
│              └──────────────── qué evento escuchar
└────────────────────────────── a quién escuchar

// Traducido al español:
// "Oye boton, quédate escuchando un clic, y cuando ocurra, ejecuta abrirModal()"

Los eventos más comunes que encontrarás en cualquier proyecto:

JS 
'click'      → el usuario hace clic con el mouse o toca en móvil
'input'      → el usuario escribe en un campo, letra por letra
'submit'     → el usuario envía un formulario
'mouseover'  → el usuario pasa el mouse encima de algo
'keydown'    → el usuario presiona una tecla del teclado

Así se ven los eventos en código real:

JS 
// Clic en el botón cancelar → cierra el modal
btnCancelar.addEventListener('click', cerrarModal)

// Enviar el formulario → procesa y guarda los datos
formulario.addEventListener('submit', guardarDatos)

// Escribir en el campo → actualiza algo mientras el usuario escribe
campoBusqueda.addEventListener('input', filtrarResultados)
// ↑ nota que este usa 'input' y no 'click'
// porque reacciona letra por letra mientras escribe, no solo al hacer clic
💡

Ejemplo en la práctica: si abres el archivo JavaScript de cualquier proyecto y buscas addEventListener cada botón, cada campo de texto y cada formulario tiene al menos uno. Son los que hacen que la app haga cosas cuando el usuario interactúa.

DOM

El mapa vivo de tu página

🗺️ Como el plano de una casa — el DOM es el mapa de todos los elementos de tu página que JavaScript puede ver, leer y modificar en tiempo real. Sin el DOM, JavaScript no podría tocar nada de lo que hay en la pantalla.

Cuando el navegador lee tu index.html lo convierte en un árbol de elementos que JavaScript puede usar. A ese árbol se le llama DOM (Document Object Model). La palabra document en JavaScript representa toda tu página completa.

JS 
// document es toda tu página
// .getElementById() le dice "entra y tráeme el que tenga este id"

// Encontrar un elemento por id — regresa uno exacto
document.getElementById('btnGuardar')

// Encontrar un elemento por clase — regresa el primero que encuentre
document.querySelector('.btn-guardar')

// Guardar el elemento en una variable para usarlo después
const boton = document.getElementById('btnGuardar')
// ahora "boton" ES ese botón — puedes hacer cosas con él

Una vez que tienes el elemento, puedes leerlo, modificarlo, mostrarlo u ocultarlo:

JS 
const titulo = document.getElementById('tituloPrincipal')

// Leer el texto que tiene ahora
console.log(titulo.textContent)         ← muestra el texto en consola

// Cambiar el texto
titulo.textContent = "Hola, bienvenido" ← la pantalla se actualiza al instante

// Esconder un elemento (desaparece de la pantalla)
titulo.style.display = 'none'

// Mostrarlo de nuevo
titulo.style.display = 'block'
Pruébalo ahora mismo:

Abre cualquier página web en el navegador, presiona F12 y ve a la pestaña Console. Escribe esto y presiona Enter:

document.getElementById('miElemento')

Vas a ver aparecer ese elemento. Eso es JavaScript encontrando un elemento de la página en vivo — eso es el DOM.
💡

Ejemplo en la práctica: cuando una app muestra u oculta una sección sin recargar la página, JavaScript usa el DOM para lograrlo. Por ejemplo, esconder una sección con style.display = 'none' y mostrar otra con style.display = 'block'. Eso es el DOM en acción.

🔗 Cómo se conectan todos los conceptos

Este patrón se repite constantemente en cualquier proyecto JavaScript. Una vez que lo reconoces, puedes leer casi cualquier archivo:

JS 
// 1. VARIABLE — usa el DOM para encontrar el elemento en la página
const boton = document.getElementById('btnGuardar')

// 2. EVENTO — escucha lo que hace el usuario con ese elemento
boton.addEventListener('click', function() {

  // 3. FUNCIÓN — ejecuta la tarea cuando el evento ocurre
  abrirSeccion('noticias', 'Últimas noticias')

})

// Variable encuentra · Evento escucha · Función actúa
// Así funciona cada botón, cada campo, cada acción en cualquier app
