El cerebro de la página

Variables, funciones, eventos y el DOM — cómo JavaScript hace que todo funcione.

📦

Variable

Una caja con nombre donde guardas algo

📦 Como cajas etiquetadas — guardas algo adentro y lo recuperas usando el nombre cuando lo necesites.

const crea una caja que no cambia. let crea una caja que sí puede cambiar.

JS 
// Las tres partes de una variable
const  nombre  =  "Carlos"
  ↑       ↑      ↑      ↑
  │       │      │      └── el valor que guardas
  │       │      └───────── "guarda esto aquí"
  │       └────────────── el nombre (tú lo inventas)
  └───────────────────── la palabra que crea la caja

const texto  = "Hola"   ← no cambia 🔒
let   puntos = 0        ← sí cambia 🔄

const — caja fija 🔒

Una vez que guardas algo ya no cambia. Para valores fijos.

let — caja flexible 🔄

Puedes cambiar lo que hay adentro. Para valores que varían.

💡

En tu proyecto: let subidas = 0 cuenta fotos · const PIN = '1234' guarda el pin fijo · let pressTimer guarda un temporizador.

📋

Función

Una receta que solo se cocina cuando alguien la llama

📋 Como una receta — existe escrita pero no pasa nada hasta que alguien la cocina. Puedes cocinarla mil veces sin reescribirla.

Los paréntesis pueden recibir información extra llamada parámetros. Mientras más parámetros, más información necesita para trabajar.

JS 
// Sin parámetros — no necesita información extra
function closeCommentsModal() {
  // cierra el modal de comentarios
}

// Con un parámetro — necesita saber qué foto
function listenComments(photoId) {
  // escucha comentarios de ESA foto específica
}

// Con dos parámetros — necesita la materia Y su nombre
function openSubjectComments(subjectId, title) {
  // abre el modal con ese título
}
Regla fácil: El nombre de una función casi siempre describe exactamente qué hace. closeCommentsModal → cierra el modal de comentarios.
🔔

Evento

Alguien hace algo — tú reaccionas

🔔 Como un timbre — alguien toca el timbre y tú abres la puerta. Sin eventos la página sería una hoja estática.
JS 
boton.addEventListener('click', function() {
  abrirModal()
})
↑              ↑              ↑
│              │              └── qué hacer cuando ocurra
│              └──────────────── qué evento escuchar
└────────────────────────────── a quién escuchar

// Eventos de tu proyecto real
newGroupCancel.addEventListener('click', closeNewGroupModal)
commentsForm.addEventListener('submit', guardarComentario)
galleryName.addEventListener('input', generarTagAutomatico)
💡

Nota: galleryName usa 'input' y no 'click' porque reacciona letra por letra mientras escribes — eso es lo que genera el tag automáticamente en tu app.

🗺️

DOM

El mapa vivo de tu página

🗺️ Como el plano de una casa — el mapa de todos los elementos que JavaScript puede leer y modificar en tiempo real.

document es toda tu página. A través de él JavaScript puede encontrar y cambiar cualquier elemento.

JS 
// Encontrar elementos
document.getElementById('btnNewGroup')    ← por id, uno exacto
document.querySelector('.btn-new-group')  ← por clase

// Cambiar contenido
const titulo = document.getElementById('galleryTitle')
titulo.textContent = "Matemáticas"   ← cambia el texto

// Mostrar / esconder
seccion.style.display = 'none'   ← esconde
seccion.style.display = 'block'  ← muestra
💡

En tu proyecto: cuando abres una materia, JS esconde albumsSection y muestra gallerySection usando el DOM. Sin recargar la página.

🔗 Cómo se conectan todos los conceptos

Esto se repite en el 70% de tu escolar.js:

JS 
// 1. VARIABLE — usa el DOM para encontrar el elemento
const boton = document.getElementById('btnNewGroup')

// 2. EVENTO — escucha lo que hace el usuario
boton.addEventListener('click', function() {

  // 3. FUNCIÓN — ejecuta la tarea
  openSubjectComments('matematicas', 'Matemáticas')

})
