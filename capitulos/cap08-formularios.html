Formularios y Validaciones

Cómo recolectar, validar y procesar datos — el puente entre el usuario y los datos.

📝

Tipos de input — lo que el usuario puede ingresar

HTML tiene muchos tipos de campo, cada uno para un dato diferente

📋 Como un formulario médico en papel — hay un campo para el nombre, otro para la fecha de nacimiento, otro para marcar casillas, otro para elegir de una lista. HTML tiene un tipo de input específico para cada caso.

Un formulario es la forma principal de recolectar información del usuario. HTML tiene muchos tipos de campos con comportamientos específicos que el navegador ya maneja automáticamente.

HTML
<!-- El contenedor del formulario -->
<form id="miForm">

  <!-- Texto libre -->
  <input type="text"     id="nombre"    placeholder="Tu nombre">

  <!-- Correo electrónico — valida el formato automáticamente -->
  <input type="email"    id="correo"    placeholder="tu@correo.com">

  <!-- Contraseña — oculta lo que se escribe -->
  <input type="password" id="clave">

  <!-- Número -->
  <input type="number"   id="edad"     min="1" max="120">

  <!-- Fecha -->
  <input type="date"     id="nacimiento">

  <!-- Casilla de verificación -->
  <input type="checkbox" id="acepto">

  <!-- Selector de opción única -->
  <select id="pais">
    <option value="mx">México</option>
    <option value="ar">Argentina</option>
  </select>

  <!-- Área de texto multilínea -->
  <textarea id="mensaje" rows="4"></textarea>

  <!-- Botón de envío -->
  <button type="submit">Enviar</button>

</form>
📖

Leer los valores del formulario

Cómo obtener lo que el usuario escribió

Cada tipo de input guarda su valor de una forma diferente en JavaScript. Aquí está cómo leer cada uno:

JS
// Texto, email, password, number — se usa .value
const nombre = document.getElementById("nombre").value
const correo = document.getElementById("correo").value
const edad   = document.getElementById("edad").value   ← viene como texto
const edadNum = Number(edad)                               ← convertir a número

// Checkbox — se usa .checked (true o false)
const acepta = document.getElementById("acepto").checked   ← true/false

// Select — se usa .value (el value de la opción seleccionada)
const pais = document.getElementById("pais").value   ← "mx" o "ar"

// Textarea — se usa .value igual que el input de texto
const mensaje = document.getElementById("mensaje").value

// .value siempre devuelve texto (string)
// Si esperas un número, convierte con Number() o parseInt()
const precio = parseInt(document.getElementById("precio").value)
✅

Validar los datos

Verificar que lo que el usuario escribió es correcto antes de usarlo

🛂 Como el control de pasaportes — antes de dejarte pasar, verifican que tu documento es válido, que no está vencido, que la foto coincide. La validación verifica que los datos tienen el formato correcto antes de procesarlos.

Siempre hay que validar los datos del usuario antes de usarlos. El usuario puede equivocarse, dejar campos vacíos o escribir algo en el formato incorrecto. La validación protege tu app de datos incorrectos.

JS
// Validaciones básicas
const nombre = document.getElementById("nombre").value.trim()
// .trim() quita espacios al principio y al final

if (!nombre) {
  console.log("El nombre es obligatorio")
}

if (nombre.length < 2) {
  console.log("El nombre es muy corto")
}

// Validar email con una expresión regular
const correo = document.getElementById("correo").value
const emailValido = /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(correo)
if (!emailValido) {
  console.log("El correo no tiene formato válido")
}

// Validar número en rango
const edad = parseInt(document.getElementById("edad").value)
if (isNaN(edad) || edad < 1 || edad > 120) {
  console.log("La edad no es válida")
}

// Función de validación completa
function validarFormulario() {
  const errores = []

  if (!nombre) errores.push("El nombre es obligatorio")
  if (!emailValido) errores.push("El correo no es válido")
  if (!acepta) errores.push("Debes aceptar los términos")

  return errores   ← arreglo vacío = todo válido
}

Mostrar errores al usuario

Comunicar qué está mal de forma clara y útil

🗺️ Como las señales de tránsito — no solo te dicen que no puedes pasar, sino por qué y qué debes hacer. Los buenos mensajes de error explican exactamente qué está mal y cómo corregirlo, no solo "hay un error".
JS
// Forma simple — mostrar mensaje bajo el campo
function mostrarError(campoId, mensaje) {
  const campo   = document.getElementById(campoId)
  const errorEl = document.getElementById(campoId + "-error")

  campo.classList.add("campo-error")          ← borde rojo
  errorEl.textContent = mensaje                  ← mensaje visible
  errorEl.style.display = "block"
}

function limpiarError(campoId) {
  const campo   = document.getElementById(campoId)
  const errorEl = document.getElementById(campoId + "-error")

  campo.classList.remove("campo-error")
  errorEl.style.display = "none"
}

// Limpiar errores cuando el usuario empieza a corregir
document.getElementById("nombre").addEventListener("input", () => {
  limpiarError("nombre")
})

Procesar el envío del formulario

Qué hacer cuando el usuario hace clic en Enviar

JS
const formulario = document.getElementById("miForm")

formulario.addEventListener("submit", (e) => {
  e.preventDefault()   ← SIEMPRE primero — evita que la página recargue

  // 1. Recolectar los datos
  const nombre = document.getElementById("nombre").value.trim()
  const correo = document.getElementById("correo").value.trim()

  // 2. Validar
  const errores = validarFormulario()

  if (errores.length > 0) {
    // Hay errores — mostrarlos y no continuar
    errores.forEach(err => console.log(err))
    return   ← detiene la ejecución aquí
  }

  // 3. Todo válido — procesar los datos
  const datos = { nombre, correo }
  console.log("Datos listos:", datos)

  // Aquí normalmente los enviarías al servidor con fetch()
  // (lo veremos en el capítulo de APIs)

  // 4. Limpiar el formulario al terminar
  formulario.reset()   ← borra todos los campos
})
La regla de oro del submit:

Siempre empieza con e.preventDefault(). Sin eso, al hacer clic en Enviar la página recarga y pierdes todos los datos y el estado de la app. Es el error más común al trabajar con formularios por primera vez.

🔗 Formulario completo con validación

Inputs + lectura + validación + errores + submit — todo el flujo:

JS
const form = document.getElementById("registro")

form.addEventListener("submit", (e) => {
  e.preventDefault()

  const nombre = document.getElementById("nombre").value.trim()
  const correo = document.getElementById("correo").value.trim()
  const acepta = document.getElementById("acepto").checked

  let hayErrores = false

  if (!nombre) {
    mostrarError("nombre", "El nombre es obligatorio")
    hayErrores = true
  }

  if (!correo || !correo.includes("@")) {
    mostrarError("correo", "Escribe un correo válido")
    hayErrores = true
  }

  if (!acepta) {
    mostrarError("acepto", "Debes aceptar los términos")
    hayErrores = true
  }

  if (hayErrores) return

  console.log("Registro exitoso:", { nombre, correo })
  form.reset()
})
