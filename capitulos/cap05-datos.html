Arreglos y Objetos

Cómo guardar, organizar y manipular colecciones de datos — la base de cualquier aplicación real.

📚

Arreglos

Una lista ordenada de cosas

📋 Como una lista de compras — una sola variable que guarda muchos valores en orden, uno tras otro. En lugar de crear una variable para cada elemento, los pones todos dentro de una sola lista.

Hasta ahora las variables guardaban un solo valor: un texto, un número, un elemento. Los arreglos guardan muchos valores a la vez, ordenados en posiciones numeradas. Se escriben con corchetes [ ] y los valores se separan con comas.

JS 
// Sin arreglo — una variable por cada cosa (muy repetitivo)
const materia1 = "Matemáticas"
const materia2 = "Historia"
const materia3 = "Física"

// Con arreglo — todo en una sola variable
const materias = ["Matemáticas", "Historia", "Física"]

// Un arreglo puede guardar cualquier tipo de dato
const numeros   = [1, 2, 3, 4, 5]
const mixto     = ["texto", 42, true, null]
const vacio     = []    ← arreglo vacío, se llenará después

// Saber cuántos elementos tiene
materias.length   ← 3
numeros.length    ← 5
vacio.length      ← 0
¿Por qué usar arreglos?

Imagina que tienes 100 materias. Sin arreglos necesitarías 100 variables distintas. Con un arreglo es una sola variable con 100 elementos adentro. Además puedes recorrerlos todos con un bucle forEach — algo imposible con 100 variables separadas.
🔑

Acceder a elementos

Cómo leer y modificar elementos individuales

🏢 Como los pisos de un edificio — cada elemento tiene un número de piso, pero la cuenta empieza desde el piso 0, no desde el 1. El primer elemento está en la posición 0, el segundo en la 1, y así. Ese número se llama índice.

Para acceder a un elemento específico usas su índice entre corchetes [número] junto al nombre del arreglo. El índice siempre empieza en 0 — esto confunde al principio pero es universal en programación.

JS 
const frutas = ["manzana", "naranja", "uva", "mango"]
//               índice 0    índice 1    índice 2   índice 3

// Leer un elemento por su índice
frutas[0]    ← "manzana"  (el primero)
frutas[1]    ← "naranja"
frutas[2]    ← "uva"
frutas[3]    ← "mango"   (el último)
frutas[4]    ← undefined  (no existe ese índice)

// Truco para el último elemento sin saber cuántos hay
frutas[frutas.length - 1]   ← "mango" (siempre el último)

// Modificar un elemento
frutas[1] = "limón"
// ahora frutas = ["manzana", "limón", "uva", "mango"]

// Guardar un elemento en una variable
const primera = frutas[0]
console.log(primera)   ← "manzana"
¿Por qué empieza en 0 y no en 1?

Es una convención que viene de los primeros lenguajes de programación y que todos los lenguajes modernos adoptaron. El índice 0 representa "ningún desplazamiento desde el inicio" — es decir, el primer elemento sin moverse nada. Con el tiempo se vuelve completamente natural, pero los primeros días es normal olvidarse y poner 1 donde debería ir 0.
🛠️

Métodos de arreglo

Las herramientas para manipular listas

🧰 Como una caja de herramientas que viene incluida con cada arreglo — funciones listas para usar que hacen cosas comunes: agregar, eliminar, buscar, filtrar, transformar. No tienes que crearlas tú, JavaScript ya las trae.

Los métodos son funciones que pertenecen a un arreglo y se llaman con un punto después del nombre: arreglo.metodo(). Son las herramientas más usadas del día a día en JavaScript.

JS 
const colores = ["rojo", "verde", "azul"]

// ---- AGREGAR Y QUITAR ----

// push — agrega al FINAL
colores.push("amarillo")
// colores = ["rojo", "verde", "azul", "amarillo"]

// pop — quita el ÚLTIMO y lo devuelve
const eliminado = colores.pop()
// eliminado = "amarillo"
// colores = ["rojo", "verde", "azul"]

// unshift — agrega al PRINCIPIO
colores.unshift("negro")
// colores = ["negro", "rojo", "verde", "azul"]

// shift — quita el PRIMERO y lo devuelve
const primero = colores.shift()
// primero = "negro"

// splice — quita elementos en una posición específica
colores.splice(1, 1)   ← desde índice 1, quita 1 elemento
// colores = ["rojo", "azul"]  (quitó "verde")
JS 
const numeros = [3, 1, 4, 1, 5, 9, 2, 6]

// ---- BUSCAR ----

// includes — ¿existe este valor? Devuelve true o false
numeros.includes(5)    ← true
numeros.includes(7)    ← false

// indexOf — ¿en qué posición está? Devuelve el índice o -1
numeros.indexOf(4)     ← 2 (está en la posición 2)
numeros.indexOf(7)     ← -1 (no existe)

// find — encuentra el primer elemento que cumple una condición
const mayorQueCinco = numeros.find(n => n > 5)
// mayorQueCinco = 9 (el primero que sea > 5)

// ---- TRANSFORMAR ----

// filter — crea un NUEVO arreglo con los que cumplen la condición
const pares = numeros.filter(n => n % 2 === 0)
// pares = [4, 2, 6]

// map — crea un NUEVO arreglo transformando cada elemento
const dobles = numeros.map(n => n * 2)
// dobles = [6, 2, 8, 2, 10, 18, 4, 12]

// sort — ordena el arreglo (modifica el original)
numeros.sort()
// [1, 1, 2, 3, 4, 5, 6, 9]

// join — convierte el arreglo en un texto
["Hola", "mundo"].join(" ")   ← "Hola mundo"
["a", "b", "c"].join(",")      ← "a,b,c"
filter y map — los más usados en desarrollo real:

filter es perfecto para buscadores y filtros — por ejemplo, mostrar solo los elementos que coinciden con lo que el usuario escribió.

map es perfecto para transformar datos — por ejemplo, convertir una lista de objetos en una lista de elementos HTML.

Ambos crean un nuevo arreglo sin modificar el original. Eso es importante — los datos originales quedan intactos.
💡

Ejemplo en la práctica: En cualquier app con buscador verás filter en acción: const resultados = elementos.filter(e => e.nombre.includes(busqueda)) — de toda la lista, devuelve solo los que contienen el texto buscado.

🗃️

Objetos

Agrupar datos relacionados bajo un solo nombre

🪪 Como una ficha de contacto — en lugar de guardar nombre, teléfono y correo en tres variables separadas, las agrupas todas dentro de una sola bajo el nombre de la persona. Un objeto es una colección de datos que describen una sola cosa.

Mientras los arreglos guardan listas de cosas similares, los objetos agrupan propiedades de una sola cosa. Se escriben con llaves { } y cada propiedad tiene un nombre (clave) y un valor separados por dos puntos.

JS 
// Sin objeto — variables sueltas sin relación clara
const nombreUsuario = "Carlos"
const edadUsuario   = 17
const ciudadUsuario = "Córdoba"

// Con objeto — todo agrupado y relacionado
const usuario = {
  nombre: "Carlos",
  edad:   17,
  ciudad: "Córdoba"
}

// Estructura de una propiedad:
// clave: valor
// nombre: "Carlos"
//   ↑         ↑
//   │         └── el dato que guardas
//   └── cómo se llama ese dato dentro del objeto

Los objetos pueden guardar cualquier tipo de dato como valor, incluyendo arreglos y hasta otros objetos:

JS 
const capitulo = {
  numero:    "01",
  titulo:    "Los bloques de construcción",
  lang:      "html",
  bloqueado: false,
  lecciones: ["Etiqueta", "Elemento", "Atributo"]
  //            ↑ un arreglo como valor de una propiedad
}
📌

Acceder a propiedades

Leer y modificar los datos de un objeto

📖 Como abrir una ficha y buscar un campo específico — usas el nombre del campo para llegar directamente al dato que buscas, sin tener que leer todo el objeto de principio a fin.

Hay dos formas de acceder a una propiedad de un objeto. La más común usa un punto: objeto.propiedad. La otra usa corchetes como los arreglos: objeto["propiedad"]. Ambas hacen lo mismo — la diferencia es cuándo usarlas.

JS 
const usuario = {
  nombre: "Carlos",
  edad:   17,
  ciudad: "Córdoba"
}

// Notación de PUNTO — la más común
usuario.nombre    ← "Carlos"
usuario.edad      ← 17
usuario.ciudad    ← "Córdoba"

// Notación de CORCHETES — cuando el nombre viene en una variable
const campo = "nombre"
usuario[campo]    ← "Carlos" (mismo resultado)

// Modificar una propiedad existente
usuario.edad = 18
// ahora usuario.edad es 18

// Agregar una propiedad nueva
usuario.email = "[email protected]"
// ahora el objeto tiene 4 propiedades

// Eliminar una propiedad
delete usuario.ciudad
// ahora el objeto ya no tiene "ciudad"

// Verificar si una propiedad existe
"nombre" in usuario    ← true
"ciudad" in usuario    ← false (la eliminamos)
¿Cuándo usar corchetes en lugar de punto?

Usa la notación de punto (objeto.propiedad) siempre que puedas — es más limpia y legible.

Usa corchetes (objeto[variable]) cuando el nombre de la propiedad está guardado en una variable y no lo sabes de antemano. Por ejemplo, si el usuario elige qué campo quiere ver, guardas su elección en una variable y accedes con corchetes.
🔧

Métodos de Object

Herramientas para trabajar con objetos

🔬 Como desmontar un reloj — a veces necesitas ver por separado las piezas (claves), los valores, o ambos a la vez. JavaScript tiene métodos para sacar esa información de cualquier objeto de forma ordenada.

A diferencia de los arreglos, los objetos tienen sus herramientas en Object (con O mayúscula) y el objeto va adentro como parámetro: Object.metodo(miObjeto).

JS 
const persona = {
  nombre: "Ana",
  edad:   25,
  ciudad: "Madrid"
}

// Object.keys — devuelve un arreglo con las CLAVES
Object.keys(persona)
// ["nombre", "edad", "ciudad"]

// Object.values — devuelve un arreglo con los VALORES
Object.values(persona)
// ["Ana", 25, "Madrid"]

// Object.entries — devuelve pares [clave, valor]
Object.entries(persona)
// [["nombre", "Ana"], ["edad", 25], ["ciudad", "Madrid"]]

// Muy útil para recorrer un objeto con forEach
Object.entries(persona).forEach(([clave, valor]) => {
  console.log(clave + ": " + valor)
})
// nombre: Ana
// edad: 25
// ciudad: Madrid

// Object.assign — copia propiedades de un objeto a otro
const copia = Object.assign({}, persona)
// copia es una copia independiente de persona

// Spread operator — otra forma de copiar (más moderna)
const copia2 = { ...persona }
// mismo resultado, sintaxis más limpia
💡

Ejemplo en la práctica: Object.keys() se usa cuando necesitas saber qué propiedades tiene un objeto sin conocerlas de antemano — por ejemplo, al recibir datos de una base de datos y querer mostrar todos los campos disponibles.

Arreglos de objetos — la combinación más poderosa

Listas donde cada elemento tiene múltiples propiedades

📇 Como una agenda de contactos — una lista (arreglo) donde cada contacto (objeto) tiene nombre, teléfono y correo. Cada elemento de la lista es un objeto completo. Esta combinación es la base de casi toda app del mundo real.

La mayoría de los datos del mundo real son arreglos de objetos. Una lista de usuarios, una lista de productos, una lista de mensajes — todos son arreglos donde cada elemento es un objeto con propiedades. Esta estructura aparece en bases de datos, APIs y en cualquier app medianamente compleja.

JS 
// Arreglo de objetos — cada elemento es un objeto completo
const usuarios = [
  { nombre: "Carlos", edad: 17, activo: true  },
  { nombre: "Ana",    edad: 22, activo: true  },
  { nombre: "Luis",   edad: 15, activo: false }
]

// Acceder a un usuario específico
usuarios[0]           ← { nombre: "Carlos", edad: 17, activo: true }
usuarios[0].nombre    ← "Carlos"
usuarios[1].edad      ← 22

// Recorrer todos con forEach
usuarios.forEach(usuario => {
  console.log(usuario.nombre + " tiene " + usuario.edad + " años")
})
// Carlos tiene 17 años
// Ana tiene 22 años
// Luis tiene 15 años

// Filtrar solo los activos
const activos = usuarios.filter(u => u.activo === true)
// activos = [Carlos, Ana]  (Luis no está activo)

// Obtener solo los nombres
const nombres = usuarios.map(u => u.nombre)
// nombres = ["Carlos", "Ana", "Luis"]

// Encontrar un usuario específico
const carlos = usuarios.find(u => u.nombre === "Carlos")
// carlos = { nombre: "Carlos", edad: 17, activo: true }
JSON — el formato de datos más usado en internet:

Cuando una app pide datos a un servidor, los recibe en formato JSON (JavaScript Object Notation). JSON es exactamente esto: arreglos y objetos escritos de la misma forma que acabas de aprender.

{"nombre": "Carlos", "edad": 17} — eso es JSON. Una base de datos te puede devolver miles de estos objetos en un arreglo. Por eso entender arreglos y objetos es tan fundamental.
💡

Ejemplo en la práctica: En cualquier app con lista de elementos — una tienda, una red social, un gestor de tareas — los datos llegan del servidor como un arreglo de objetos. Tu código recibe ese arreglo, lo recorre con forEach o map, y crea el HTML de cada tarjeta que el usuario ve en pantalla.

🔗 Cómo se conectan arreglos, objetos y todo lo anterior

Este patrón — arreglo de objetos + forEach + if + DOM — es el corazón de cualquier app que muestra listas de cosas en pantalla:

JS 
// Los datos — arreglo de objetos
const capitulos = [
  { titulo: "HTML",        bloqueado: false, lang: "html" },
  { titulo: "JavaScript",  bloqueado: false, lang: "js"   },
  { titulo: "CSS avanzado", bloqueado: true,  lang: "css"  }
]

// La función que construye la pantalla
function mostrarCapitulos() {
  const contenedor = document.getElementById("lista")  ← DOM

  capitulos.forEach(cap => {                               ← forEach

    const tarjeta = document.createElement("div")          ← DOM
    tarjeta.textContent = cap.titulo                      ← objeto

    if (cap.bloqueado) {                                   ← if/else
      tarjeta.classList.add("bloqueada")
    } else {
      tarjeta.classList.add("disponible")
      tarjeta.classList.add(cap.lang)                     ← objeto
    }

    contenedor.appendChild(tarjeta)                       ← DOM
  })
}

// Variables + DOM + forEach + objetos + if/else
// Todo junto construye lo que el usuario ve en pantalla
