El estilo de la página

Selectores, propiedades, Box Model, Flexbox y más — cómo CSS pinta y organiza todo lo que ves.

🎨

¿Qué es CSS?

El lenguaje que le da vida visual a HTML

👕 Si HTML es el esqueleto de una persona, CSS es la ropa — define cómo se ve cada cosa: colores, tamaños, fuentes, espacios y posición. Sin CSS, una página web es solo texto negro sobre fondo blanco.

CSS significa Cascading Style Sheets — Hojas de Estilo en Cascada. Es un lenguaje separado de HTML que vive en su propio archivo .css. Cualquier proyecto web tiene un archivo .css con cientos de líneas que definen exactamente cómo se ve cada parte de la página.

La conexión entre HTML y CSS es simple: HTML crea los elementos y CSS los encuentra por su class o id para pintarlos. Por eso los atributos class e id que aprendiste en el capítulo anterior son tan importantes.

CSS 
/* Así se conectan HTML y CSS */

/* En index.html */
<button class="btn-guardar">Guardar cambios</button>

/* En style.css */
.btn-guardar {
  background: #c44a2a;   ← color de fondo naranja
  color: white;           ← texto blanco
  border-radius: 8px;    ← esquinas redondeadas
  padding: 10px 16px;   ← espacio interno
}
¿Cómo se incluye CSS en la página?

Con una etiqueta <link> dentro del <head> de tu HTML. Se incluye en el HTML con una línea como esta:

<link rel="stylesheet" href="css/style.css">

Esa línea le dice al navegador: "antes de mostrar la página, ve a buscar ese archivo CSS y aplica todos los estilos que encuentres ahí".
💡

Ejemplo en la práctica: Abre el archivo style.css de cualquier proyecto en un editor de código. Todo lo que ves ahí — colores, esquinas redondeadas, el diseño completo — está escrito en ese archivo. Ahora que sabes CSS puedes leerlo y modificarlo.

🎯

Selectores

Cómo CSS apunta a los elementos que quiere pintar

🎯 Como apuntar con una linterna en un cuarto oscuro — el selector es el rayo de luz que decides dónde apuntar. Solo los elementos que iluminas reciben el estilo. Si apuntas mal, el estilo no llega donde quieres.

Un selector le dice a CSS cuáles elementos quiere estilizar. Hay tres formas principales de seleccionar elementos y cada una tiene un símbolo diferente:

CSS 
/* 1. Por ETIQUETA — aplica a TODOS los elementos de ese tipo */
button {
  cursor: pointer;   ← todos los botones de la página tendrán cursor de mano
}

/* 2. Por CLASS — se escribe con un PUNTO antes del nombre */
.btn-guardar {
  background: orange;  ← solo los elementos con class="btn-guardar"
}

/* 3. Por ID — se escribe con un NUMERAL # antes del nombre */
#btnGuardar {
  font-size: 14px;   ← solo el elemento con id="btnGuardar"
}

También puedes combinar selectores para apuntar con más precisión:

CSS 
/* Selector de HIJO — botones que estén dentro de .header */
.header button {
  color: white;
}

/* Selector de ESTADO — cuando el mouse está encima */
.btn-guardar:hover {
  background: darkred;   ← cambia de color al pasar el mouse
}

/* Selector de ESTADO — cuando el elemento está activo/presionado */
.btn-guardar:active {
  transform: scale(0.98);  ← se encoge ligeramente al hacer clic
}
La regla más importante de los selectores:

Usa class para casi todo — es lo más común y flexible.
Usa id solo cuando necesitas apuntar a un único elemento muy específico.
Usa etiqueta solo para estilos generales que apliquen a todos (como quitar márgenes de todos los párrafos).

En cualquier proyecto profesional casi todo usa clases con punto — ese es el estándar de la industria.
💡

Ejemplo en la práctica: Abre el archivo CSS de cualquier proyecto y busca un selector de botón. Cambia el valor de background por cualquier color y guarda — verás el cambio al instante en el navegador.

Propiedades y valores

Las instrucciones exactas de cómo verse

🖌️ Como las instrucciones para pintar una pared — la propiedad es qué cambias (el color), el valor es cómo quieres que quede (rojo). Siempre van juntos, separados por dos puntos, y terminan con punto y coma.
CSS 
propiedad: valor;
    ↑         ↑
    │         └── cómo quieres que quede
    └── qué aspecto vas a cambiar

/* Ejemplo real */
.logo-title {
  color: #1a1814;        ← propiedad: color | valor: café oscuro
  font-size: 18px;      ← propiedad: tamaño | valor: 18 pixeles
  font-weight: 700;     ← propiedad: grosor | valor: negrita
  line-height: 1.1;     ← propiedad: altura de línea | valor: 1.1x
}

Las propiedades más usadas organizadas por categoría:

CSS 
/* ---- TEXTO ---- */
color: red;                  ← color del texto
font-size: 16px;             ← tamaño del texto
font-weight: 700;            ← grosor: 400=normal, 700=negrita
font-family: 'DM Sans';      ← tipo de letra
text-align: center;          ← alineación: left, center, right
line-height: 1.7;            ← espacio entre líneas
text-decoration: none;       ← quita el subrayado de los links

/* ---- FONDO Y BORDE ---- */
background: #f7f4ef;         ← color de fondo (hex)
background: transparent;      ← fondo transparente
border: 1px solid #e8e4de;  ← borde: grosor tipo color
border-radius: 14px;         ← esquinas redondeadas
border-radius: 999px;        ← esquinas completamente redondas (píldora)

/* ---- TAMAÑO ---- */
width: 240px;               ← ancho fijo en pixeles
width: 100%;               ← ancho del 100% de su contenedor
height: 64px;              ← alto fijo
max-width: 1100px;         ← ancho máximo (no crece más)
min-height: 100vh;         ← alto mínimo de toda la pantalla

/* ---- VISIBILIDAD ---- */
display: none;              ← esconde el elemento completamente
display: block;             ← lo muestra como bloque
opacity: 0.5;              ← transparencia: 0=invisible, 1=visible
cursor: pointer;           ← cursor de mano al pasar encima

Los colores en CSS se pueden escribir de varias formas:

CSS 
/* Por nombre — solo para colores básicos */
color: red;
color: white;
color: black;

/* HEX — el más común en proyectos reales */
color: #c44a2a;    ← color naranja
color: #1a1814;    ← color café oscuro
color: #f7f4ef;    ← color beige

/* RGB — red, green, blue de 0 a 255 */
color: rgb(196, 74, 42);

/* RGBA — igual que RGB pero con transparencia */
color: rgba(196, 74, 42, 0.5);   ← 0.5 = 50% transparente
💡

Ejemplo en la práctica: En cualquier proyecto CSS moderno encontrarás un bloque :root al principio del archivo con todas las variables definidas. Por ejemplo --color-primario: #c44a2a define el color principal. Más adelante en este capítulo aprendemos exactamente cómo funcionan.

Box Model

Todo elemento es una caja con cuatro capas

📦 Como un regalo empaquetado — el contenido está adentro, el padding es el papel de relleno, el border es la caja, y el margin es el espacio entre esa caja y las demás. Todo elemento HTML, sin excepción, funciona así.

El Box Model es uno de los conceptos más importantes de CSS. Explica por qué los elementos ocupan el espacio que ocupan y cómo controlar ese espacio. Tiene cuatro capas, de adentro hacia afuera: 

┌─────────────────────────────────────────┐
│               MARGIN                    │  ← espacio exterior
│   ┌─────────────────────────────────┐   │     (separa del resto)
│   │            BORDER               │   │  ← el borde visible
│   │   ┌─────────────────────────┐   │   │
│   │   │         PADDING         │   │   │  ← relleno interior
│   │   │   ┌─────────────────┐   │   │   │     (espacio entre borde
│   │   │   │    CONTENIDO    │   │   │   │      y contenido)
│   │   │   │  (texto, imagen)│   │   │   │
│   │   │   └─────────────────┘   │   │   │
│   │   └─────────────────────────┘   │   │
│   └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
CSS 
.leccion {
  /* PADDING — espacio entre el borde y el contenido */
  padding: 24px 28px;     ← 24px arriba/abajo, 28px izq/der
  padding: 20px;          ← igual en los 4 lados
  padding-top: 10px;     ← solo arriba
  padding-left: 16px;    ← solo izquierda

  /* BORDER — el borde visible */
  border: 1px solid #e8e4de;         ← grosor tipo color
  border-left: 4px solid #c44a2a;    ← solo el borde izquierdo
  border-radius: 14px;               ← esquinas redondeadas

  /* MARGIN — espacio exterior entre elementos */
  margin: 0 auto;         ← 0 arriba/abajo, auto izq/der = centrar
  margin-bottom: 16px;   ← separación hacia abajo
  margin-top: 40px;      ← separación hacia arriba
}

La diferencia entre padding y margin es la que más confunde al principio:

padding — espacio interior 🏠

Es el espacio entre el borde del elemento y su contenido. Si el elemento tiene color de fondo, el padding también lo tiene. Úsalo para dar "aire" adentro del elemento.

margin — espacio exterior 🌳

Es el espacio entre el borde del elemento y los elementos de afuera. Es transparente — no tiene color de fondo. Úsalo para separar elementos entre sí.

El truco de margin: 0 auto:

Cuando le das margin: 0 auto a un elemento con un ancho fijo, CSS reparte el espacio sobrante igual en izquierda y derecha — lo que centra el elemento horizontalmente. Así funciona el centrado de en cualquier proyecto con elementos centrados:

max-width: 1100px; margin: 0 auto;
💡

Ejemplo en la práctica: Busca cualquier clase de tarjeta o contenedor en un archivo CSS. Verás algo como padding: 24px 28px — ese es el espacio interno. Si lo cambias a padding: 40px el elemento se verá más amplio. Pruébalo.

Flexbox

La forma moderna de acomodar elementos

🚂 Como un tren — los vagones (elementos hijos) se acomodan automáticamente dentro de la locomotora (elemento padre). Tú decides la dirección del tren, cómo se distribuyen los vagones y cómo se alinean. El padre controla a todos los hijos a la vez.

Antes de Flexbox, acomodar elementos en fila o centrarlos era complicado y lleno de trucos. Flexbox lo hace simple. Para activarlo solo necesitas una línea en el elemento padre: display: flex.

CSS 
/* El PADRE activa Flexbox */
.header-inner {
  display: flex;              ← activa Flexbox en este contenedor
  align-items: center;        ← alinea hijos verticalmente al centro
  justify-content: space-between; ← separa los hijos en los extremos
  gap: 12px;                 ← espacio entre cada hijo
}

/* Los HIJOS se acomodan solos */
/* .logo-group y .btn-header quedan uno a cada extremo */

Las dos propiedades más importantes de Flexbox son justify-content y align-items. Su dirección depende de si el flex va en fila o columna:

CSS 
/* flex-direction: row (por defecto) — los hijos van en FILA horizontal */
.contenedor {
  display: flex;
  flex-direction: row;            ← izquierda a derecha (es el default)

  justify-content: flex-start;   ← todos al inicio (izquierda)
  justify-content: center;       ← todos al centro
  justify-content: flex-end;     ← todos al final (derecha)
  justify-content: space-between; ← primero y último en extremos
  justify-content: space-around;  ← espacio igual alrededor de cada uno

  align-items: flex-start;      ← alineados arriba
  align-items: center;          ← alineados al centro vertical
  align-items: flex-end;        ← alineados abajo
}

/* flex-direction: column — los hijos van en COLUMNA vertical */
.contenedor-vertical {
  display: flex;
  flex-direction: column;
  /* ahora justify-content controla vertical y align-items horizontal */
}

/* flex-wrap — qué pasa cuando no caben en una fila */
.contenedor-wrap {
  display: flex;
  flex-wrap: wrap;   ← si no caben, se bajan a la siguiente línea
  gap: 16px;        ← espacio entre elementos en cualquier dirección
}

También puedes controlar cómo se comporta cada hijo individualmente:

CSS 
/* flex: 1 — el hijo ocupa todo el espacio disponible */
.install-banner-inner p {
  flex: 1;    ← este párrafo crece para llenar el espacio sobrante
}

/* flex-shrink: 0 — el hijo NO se encoge aunque no haya espacio */
.logo-icon {
  flex-shrink: 0;   ← el ícono siempre mantiene su tamaño
}
💡

Ejemplo en la práctica: Busca display: flex en cualquier archivo CSS — aparecerá decenas de veces. El encabezado de casi cualquier página usa Flexbox para poner el logo a la izquierda y los botones de navegación a la derecha. Es uno de los patrones más comunes del diseño web.

Variables CSS

Guarda valores para reutilizarlos en todo el archivo

🎨 Como una paleta de colores con nombre — defines el color una sola vez y lo usas en mil lugares. Si decides cambiar el naranja por azul, lo cambias en un solo lugar y se actualiza en toda la página automáticamente.

Las variables CSS siempre empiezan con dos guiones --. Se definen dentro de :root — que representa el elemento más alto de la página — para que estén disponibles en todo el archivo. Para usarlas se escribe var(--nombre).

CSS 
/* DEFINIR variables en :root (al principio del archivo) */
:root {
  /* Colores */
  --bg:       #f7f4ef;   ← fondo beige de toda la página
  --card:     #ffffff;   ← blanco para las tarjetas
  --ink:      #1a1814;   ← texto café oscuro
  --html:     #c44a2a;   ← naranja de HTML
  --css:      #1a7a5a;   ← verde de CSS
  --js:       #b07010;   ← dorado de JavaScript

  /* Tipografía */
  --font-body:  'DM Sans', sans-serif;
  --font-title: 'Fraunces', serif;
  --font-code:  'DM Mono', monospace;

  /* Tamaños */
  --sidebar-w:  220px;
  --header-h:   64px;
  --radius:     14px;
}

/* USAR las variables con var() */
body {
  background: var(--bg);        ← usa el beige definido arriba
  font-family: var(--font-body); ← usa la fuente definida arriba
  color: var(--ink);            ← usa el café oscuro definido arriba
}

.logo-code {
  color: var(--html);  ← el  naranja del logo usa esta variable
}

.leccion {
  border-radius: var(--radius);  ← esquinas redondeadas consistentes
}
¿Por qué usar variables en lugar de escribir el color directo?

Imagina que decides cambiar el color principal de un sitio por azul. Sin variables tendrías que buscar #c44a2a en todo el archivo y cambiarlo en cada lugar donde aparece — podría ser 50 veces.

Con variables solo cambias --html: #c44a2a por --html: #2a5cc4 en :root — una sola línea — y toda la guía cambia de color automáticamente.
💡

Ejemplo en la práctica: Abre cualquier archivo CSS y ve a las primeras líneas — ahí estará el bloque :root con las variables. Prueba cambiar un color y guarda. Verás cómo todos los elementos que usan esa variable cambian al instante en toda la página.

Responsive Design

Que la página se vea bien en cualquier pantalla

💧 Como el agua — toma la forma del recipiente donde está. Una página responsive se adapta sola a cualquier tamaño de pantalla: celular, tablet, computadora. No hay una versión para cada dispositivo, hay una sola página que se adapta.

El responsive se logra con media queries — condiciones que le dicen a CSS "si la pantalla es menor a cierto tamaño, aplica estos estilos diferentes". Se escriben con @media y contienen reglas que sobreescriben las normales cuando la condición se cumple.

CSS 
/* Estilos normales — aplican a TODAS las pantallas */
.hero-inner {
  display: grid;
  grid-template-columns: 1fr 1fr;   ← dos columnas en escritorio
  gap: 48px;
}

/* @media — aplica SOLO cuando la pantalla sea menor a 900px (tablet) */
@media (max-width: 900px) {
  .hero-inner {
    grid-template-columns: 1fr;   ← una sola columna en tablet
    gap: 32px;
  }
}

/* @media — aplica SOLO cuando la pantalla sea menor a 640px (móvil) */
@media (max-width: 640px) {
  .hero-inner {
    gap: 20px;
  }

  .hero-deco {
    display: none;   ← esconde la decoración en pantallas pequeñas
  }

  .leccion {
    padding: 18px 16px;   ← menos padding en móvil
  }
}

Los breakpoints más usados en la industria — los tamaños donde normalmente "rompes" el diseño para adaptarlo:

CSS 
/* Breakpoints comunes */
@media (max-width: 1200px) { /* pantalla grande → desktop normal */ }
@media (max-width: 900px)  { /* tablet apaisada */ }
@media (max-width: 640px)  { /* móvil — el más importante */ }
@media (max-width: 400px)  { /* móvil pequeño */ }
Unidades relativas — clave para el responsive:

px — pixeles fijos, no se adaptan
% — porcentaje del contenedor padre (ej: width: 100%)
vw — porcentaje del ancho de la pantalla (ej: 100vw = pantalla completa)
vh — porcentaje del alto de la pantalla (ej: 100vh = pantalla completa)
rem — relativo al tamaño de fuente base (ej: 1rem = 16px normalmente)
clamp(min, ideal, max) — se adapta entre un mínimo y un máximo automáticamente
💡

Ejemplo en la práctica: Ve al final de cualquier archivo CSS de un sitio moderno y encontrarás los bloques @media. Por ejemplo, en móvil los menús laterales se esconden usando display: none dentro de un @media — y en su lugar aparece un botón flotante ☰.

La Cascada y Especificidad

Por qué CSS hace lo que hace cuando hay conflictos

🌊 Como el agua que cae desde arriba — los estilos "fluyen" de reglas generales a específicas, y cuando dos reglas dicen cosas distintas sobre el mismo elemento, gana la más específica o la que viene después. Por eso se llama "en Cascada".

Cuando dos reglas CSS apuntan al mismo elemento y dicen cosas distintas, CSS tiene reglas para decidir cuál gana. Entender esto te evita mucha frustración cuando cambias algo y no pasa nada — o pasa lo que no querías.

CSS 
/* REGLA 1: El que viene DESPUÉS gana si tienen la misma especificidad */
.btn {
  color: red;     ← esta pierde
}
.btn {
  color: blue;    ← esta gana — viene después
}

/* REGLA 2: El más ESPECÍFICO gana */
.header .btn {   ← más específico — apunta a .btn dentro de .header
  color: green;  ← esta gana aunque venga antes
}
.btn {
  color: blue;
}

/* Orden de especificidad de menor a mayor: */
/* etiqueta → clase → id → estilos inline → !important */

button    { color: red; }     ← especificidad baja
.btn      { color: blue; }    ← más específico
#miBtn    { color: green; }   ← más específico aún
¿Cuándo usar !important?

color: red !important fuerza que ese estilo gane sin importar nada. Es tentador usarlo cuando algo no funciona, pero es mala práctica — hace el código difícil de mantener porque rompe el flujo natural de la cascada.

Si necesitas usar !important frecuentemente, normalmente significa que la estructura de tus selectores necesita revisión. Úsalo solo en casos muy específicos y excepcionales.
La herencia — estilos que se pasan de padre a hijo:

Algunas propiedades se heredan automáticamente de padre a hijo. Por ejemplo si defines font-family en el body, todos los elementos adentro usarán esa fuente sin que tengas que repetirlo en cada uno. Las propiedades de texto casi siempre se heredan. Las de caja (margin, padding, border) no se heredan.
💡

Ejemplo en la práctica: Al inicio de cualquier archivo CSS bien escrito hay estas líneas en el body: font-family y color. Esas propiedades se heredan a todos los elementos de la página — por eso no necesitas repetir la fuente en cada clase. Eso es la herencia en acción.

🎨 Cómo se conectan todos los conceptos de CSS

Así se ve una regla CSS completa usando todo lo que aprendiste:

CSS 
/* Las variables están definidas en :root */
:root {
  --html: #c44a2a;
  --radius: 14px;
}

/* El selector apunta a los elementos con esa clase */
.cap-card:hover {

  /* Propiedades que cambian el aspecto visual */
  border-color: var(--html);         ← variable → naranja
  transform: translateY(-2px);       ← sube 2px al hacer hover
  box-shadow: 0 8px 24px rgba(0,0,0,0.08); ← sombra suave

  /* Box model */
  padding: 22px;                    ← espacio interno
  border-radius: var(--radius);      ← esquinas redondeadas
}

/* Responsive — en móvil se ve diferente */
@media (max-width: 640px) {
  .cap-card { padding: 16px; }
}
