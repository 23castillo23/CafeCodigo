fetch - hacer peticiones al servidor

La funcion nativa de JavaScript para comunicarse con APIs

📞 Como hacer una llamada telefonica - marcas el numero (URL), esperas que contesten (la Promise), y cuando contestan recibes la informacion que pediste. fetch es la funcion que hace esa llamada.

fetch es una funcion global de JavaScript que hace peticiones HTTP. Devuelve una Promise, por lo que se usa con async/await. Toma la URL de la API como primer argumento y configuracion opcional como segundo.

JS 
// La peticion mas simple posible — GET a una URL
async function obtenerDatos() {
  const respuesta = await fetch("https://jsonplaceholder.typicode.com/users/1")
  const datos     = await respuesta.json()
  console.log(datos)
}

// fetch devuelve una Promise con el objeto Response
// Response no es directamente los datos — primero hay que convertirlos
// .json()  → convierte si la respuesta es JSON (lo mas comun)
// .text()  → convierte si la respuesta es texto plano
// .blob()  → convierte si la respuesta es un archivo (imagen, pdf)

// jsonplaceholder.typicode.com es una API de prueba gratuita
// perfecta para practicar sin necesitar crear tu propio servidor
📋

Procesar la respuesta JSON

Convertir los datos recibidos en objetos JavaScript

La respuesta de casi todas las APIs del mundo llega en formato JSON. Una vez convertida con .json(), tienes un objeto o arreglo JavaScript normal que puedes usar con todo lo que aprendiste en capitulos anteriores.

JS 
async function listarUsuarios() {
  const res     = await fetch("https://jsonplaceholder.typicode.com/users")
  const usuarios = await res.json()
  // usuarios es un ARREGLO de objetos — todo lo del cap05 aplica aqui

  console.log(usuarios.length)         // cuantos hay
  console.log(usuarios[0].name)        // nombre del primero

  // Mostrar en pantalla con forEach + DOM
  const contenedor = document.getElementById("lista")

  usuarios.forEach(user => {
    const div = document.createElement("div")
    div.innerHTML = `<strong>${user.name}</strong> - ${user.email}`
    contenedor.appendChild(div)
  })
}

GET y POST - los metodos HTTP

Tipos de peticion segun lo que quieres hacer

🖉 Como los tipos de mensaje que puedes enviar - GET es preguntar algo ("dame la lista de usuarios"), POST es enviar algo nuevo ("aqui tienes un usuario nuevo para guardar"). Cada operacion tiene su metodo apropiado.
JS 
// GET — leer datos (no modifica nada)
// Es el metodo por defecto de fetch
const res = await fetch("https://api.ejemplo.com/usuarios")

// POST — enviar datos nuevos al servidor
const nuevoUsuario = { nombre: "Ana", correo: "ana@mail.com" }

const res = await fetch("https://api.ejemplo.com/usuarios", {
  method:  "POST",
  headers: { "Content-Type": "application/json" },
  body:    JSON.stringify(nuevoUsuario)
})
const resultado = await res.json()

// PUT — actualizar un recurso existente completo
const res = await fetch("https://api.ejemplo.com/usuarios/1", {
  method:  "PUT",
  headers: { "Content-Type": "application/json" },
  body:    JSON.stringify({ nombre: "Ana García", correo: "ana@mail.com" })
})

// DELETE — eliminar un recurso
const res = await fetch("https://api.ejemplo.com/usuarios/1", {
  method: "DELETE"
})

Manejo de errores HTTP

Que hacer cuando el servidor no responde o responde mal

Algo importante sobre fetch: si el servidor responde con un error (404, 500, etc.), fetch NO lanza una excepcion automaticamente. Tienes que revisar manualmente si la respuesta fue exitosa con respuesta.ok.

JS 
async function obtenerUsuario(id) {
  try {
    const res = await fetch("https://api.ejemplo.com/usuarios/" + id)

    // respuesta.ok es true cuando el status es 200-299
    // Si es 404, 500, etc., ok es false pero fetch NO lanza error
    if (!res.ok) {
      throw new Error("Error del servidor: " + res.status)
    }

    const usuario = await res.json()
    return usuario

  } catch (error) {
    if (error.name === "TypeError") {
      // Sin conexion a internet
      console.log("Sin conexion a internet")
    } else {
      // Error del servidor
      console.log(error.message)
    }
  }
}

// Codigos de status mas comunes:
// 200 → OK, exito
// 201 → Created, recurso creado exitosamente
// 400 → Bad Request, datos enviados incorrectos
// 401 → Unauthorized, no estas autenticado
// 403 → Forbidden, no tienes permiso
// 404 → Not Found, el recurso no existe
// 500 → Internal Server Error, error del servidor

CORS - por que algunas APIs no funcionan

La politica de seguridad que bloquea peticiones entre dominios

🏢 Como las reglas de un edificio de oficinas - puedes entrar a tu oficina libremente, pero para entrar a la oficina de otra empresa del mismo edificio necesitas que ellos te den permiso explicito. CORS es ese sistema de permisos entre dominios.

CORS (Cross-Origin Resource Sharing) es una politica de seguridad del navegador. Por defecto, una pagina web no puede hacer peticiones a un dominio diferente al suyo. La API tiene que indicar explicitamente que permite peticiones desde otros dominios.

JS 
// Error tipico de CORS en la consola:
// "Access to fetch at 'https://api-sin-cors.com' from origin
// 'https://tudominio.com' has been blocked by CORS policy"

// Que puedes hacer cuando hay error de CORS:

// 1. Usar una API que SI tiene CORS habilitado
// (jsonplaceholder, openweathermap, etc. lo tienen activado)

// 2. Crear un proxy en tu backend que haga la peticion por ti
// Tu app -> Tu servidor (mismo dominio) -> API externa

// 3. Si controlas la API, agregar los headers de CORS
// Access-Control-Allow-Origin: * (en el servidor)

// Para practicar sin problemas de CORS, usa estas APIs gratuitas:
// https://jsonplaceholder.typicode.com  — datos de prueba
// https://api.publicapis.org            — lista de APIs publicas
// https://open-meteo.com               — clima gratuito

App completa que consume una API real

fetch + async/await + DOM + manejo de errores — el patron de cualquier app que consume datos:

JS 
const lista = document.getElementById("lista")
const buscar = document.getElementById("buscar")

async function cargarPosts() {
  lista.innerHTML = "Cargando..."

  try {
    const res   = await fetch("https://jsonplaceholder.typicode.com/posts")
    const posts = await res.json()
    mostrar(posts)
  } catch (e) {
    lista.textContent = "Error al cargar datos"
  }
}

function mostrar(posts) {
  lista.innerHTML = ""
  posts.slice(0, 10).forEach(post => {
    const div = document.createElement("div")
    div.innerHTML = `<h4>${post.title}</h4><p>${post.body}</p>`
    lista.appendChild(div)
  })
}

buscar.addEventListener("input", (e) => {
  // filtrar sin volver a pedir a la API
})

cargarPosts()
