Sincrono vs Asincrono

La diferencia mas importante que existe en JavaScript

🆘 Como pedir comida en un restaurante — en modo sincrono atiendes a un cliente, lo sirves completo, y solo entonces atiendes al siguiente. En modo asincrono tomas el pedido de todos, mandas todo a la cocina, y sirves a cada uno cuando su comida esta lista. JavaScript es asincrono por naturaleza.

El codigo sincrono se ejecuta linea por linea, esperando que cada linea termine antes de pasar a la siguiente. El codigo asincrono puede iniciar una tarea, pasar a la siguiente sin esperar, y retomar cuando la primera termine. Esto es fundamental para trabajar con operaciones lentas como peticiones a internet.

JS
// SINCRONO — espera linea por linea
console.log("1")
console.log("2")
console.log("3")
// resultado: 1, 2, 3 (en ese orden, siempre)

// ASINCRONO — no espera
console.log("1")
setTimeout(() => console.log("2"), 1000)   // espera 1 segundo
console.log("3")
// resultado: 1, 3, 2  (3 no espera a 2)

// El problema asincrono clasico
let datos = null

// imagina que aqui pides datos a internet (tarda tiempo)
pedirDatosAlServidor()   // asincrono

console.log(datos)   // null — los datos no llegaron todavia!
Por que JavaScript es asincrono:

JavaScript corre en el navegador en un solo hilo (thread) — solo puede hacer una cosa a la vez. Si tuviera que esperar cada peticion a internet antes de continuar, el navegador se congelaria completamente mientras espera la respuesta del servidor. El modelo asincrono permite que la pagina siga respondiendo al usuario mientras espera en segundo plano.

Callbacks asincronos

La forma original de manejar operaciones que toman tiempo

Antes de que existieran las Promises, la unica forma de saber cuando termino una operacion asincrona era pasarle un callback que se ejecutara al terminar. Hoy en dia casi nadie usa callbacks para esto porque las Promises son mucho mas limpias, pero es importante entenderlos porque apareceran en codigo antiguo.

JS
// Patron de callback asincrono
function cargarDatos(url, callback) {
  // simula una peticion que tarda 2 segundos
  setTimeout(() => {
    const datos = { nombre: "Ana", edad: 25 }
    callback(datos)   // llama al callback cuando termina
  }, 2000)
}

cargarDatos("https://api.ejemplo.com", (datos) => {
  console.log("Datos recibidos:", datos)
})

// El problema: "callback hell" — callbacks dentro de callbacks
cargarUsuario(id, (usuario) => {
  cargarPosts(usuario.id, (posts) => {
    cargarComentarios(posts[0].id, (comentarios) => {
      // ya muy dificil de leer...
    })
  })
})
// Por eso se inventaron las Promises
🤝

Promises

Un objeto que representa una operacion que terminara en el futuro

🈚 Como el numero de turno en un banco — te dan un papel que dice "vas a ser atendido". No sabes exactamente cuando, pero el papel promete que cuando llegue tu turno te avisaran. Mientras tanto puedes hacer otras cosas.

Una Promise es un objeto que representa el resultado eventual de una operacion asincrona. Puede estar en tres estados: pending (esperando), fulfilled (exitosa) o rejected (fallida). Cuando termina, llamas a .then() para el exito o .catch() para el error.

JS
// Crear una Promise
const miPromise = new Promise((resolve, reject) => {
  // operacion asincrona simulada
  setTimeout(() => {
    const exito = true

    if (exito) {
      resolve("Operacion exitosa")   // cumplida
    } else {
      reject("Algo salio mal")      // rechazada
    }
  }, 1000)
})

// Usar la Promise
miPromise
  .then(resultado => {
    console.log("Exito:", resultado)
  })
  .catch(error => {
    console.log("Error:", error)
  })
  .finally(() => {
    console.log("Esto siempre se ejecuta, haya exito o error")
  })

// Encadenar Promises — evita el callback hell
cargarUsuario(id)
  .then(usuario => cargarPosts(usuario.id))
  .then(posts   => cargarComentarios(posts[0].id))
  .then(comentarios => console.log(comentarios))
  .catch(error  => console.log("Error en algun paso:", error))

async / await

La forma moderna y legible de trabajar con Promises

☯ Como leer un libro en lugar de descifrar un mapa del tesoro — hace exactamente lo mismo que las Promises pero escrito de forma que parece codigo sincrono normal. Mucho mas facil de leer y entender.

async/await es azucar sintactica sobre las Promises — es la misma tecnologia por debajo, solo escrita diferente. Una funcion marcada con async siempre devuelve una Promise. Dentro de una funcion async, puedes usar await para esperar a que una Promise se resuelva antes de continuar.

JS
// Con Promises (correcto pero verboso)
function cargarDatos() {
  return fetch("https://api.ejemplo.com/datos")
    .then(respuesta => respuesta.json())
    .then(datos => console.log(datos))
    .catch(error => console.log(error))
}

// Con async/await (mas limpio, parece codigo normal)
async function cargarDatos() {
  const respuesta = await fetch("https://api.ejemplo.com/datos")
  const datos     = await respuesta.json()
  console.log(datos)
}

// await le dice a JavaScript: "espera aqui hasta que esta Promise se resuelva"
// y solo entonces pasa a la siguiente linea

// Arrow function asincrona
const cargarDatos = async () => {
  const respuesta = await fetch("https://api.ejemplo.com/datos")
  return await respuesta.json()
}
Regla importante:

await solo puede usarse dentro de una funcion marcada con async. Si intentas usarlo fuera, JavaScript da un error. Esta es la razon por la que veras async antes de la funcion cuando hay await adentro.

Manejo de errores asincronos

Que hacer cuando algo sale mal al pedir datos

Las operaciones asincronas pueden fallar — el servidor puede no responder, la conexion puede cortarse, los datos pueden venir en formato incorrecto. Siempre hay que anticipar estos casos y manejarlos de forma elegante.

JS
// try/catch con async/await
async function cargarDatos() {
  try {
    // intenta esto
    const respuesta = await fetch("https://api.ejemplo.com")

    if (!respuesta.ok) {
      throw new Error("Error del servidor: " + respuesta.status)
    }

    const datos = await respuesta.json()
    console.log("Datos:", datos)

  } catch (error) {
    // si algo falla, viene aqui
    console.log("Algo salio mal:", error.message)
    // mostrar mensaje al usuario, reintentar, etc.

  } finally {
    // esto siempre se ejecuta — util para ocultar un spinner
    console.log("Operacion terminada")
  }
}

Asincrono completo en un ejemplo real

async/await + try/catch + DOM — el patron que usaras en todo proyecto:

JS
async function mostrarUsuario(id) {
  const contenedor = document.getElementById("usuario")
  contenedor.textContent = "Cargando..."

  try {
    const res   = await fetch("https://api.ejemplo.com/usuarios/" + id)
    const user  = await res.json()
    contenedor.innerHTML = "Hola, " + user.nombre
  } catch (e) {
    contenedor.textContent = "No se pudo cargar el usuario"
  }
}

mostrarUsuario(42)
