Booleanos — verdadero o falso

El tipo de dato más simple y más poderoso

💡 Como un interruptor de luz — solo tiene dos posiciones: encendido o apagado, verdadero o falso, sí o no. No hay término medio. Todo el código que toma decisiones se basa en este concepto.

Un booleano es un tipo de dato que solo puede tener dos valores: true (verdadero) o false (falso). Ya los viste en el capítulo de variables, pero aquí son el centro de todo. Cada vez que el código necesita decidir qué hacer, evalúa algo que resulta en true o false.

JS 
// Un booleano es simplemente esto:
const encendido  = true
const apagado    = false
let   yaUsado    = false  ← ¿el usuario ya interactuó con este elemento?
const esAdmin    = false  ← ¿es administrador?

// También pueden ser el resultado de una comparación:
const mayorDeEdad  = 18 >= 18   ← true
const tieneElementos = 0 > 0   ← false

Hay valores que JavaScript trata como false automáticamente aunque no sean exactamente false. Se llaman valores falsy y son importantes conocerlos:

JS 
// Valores que JavaScript considera FALSE (falsy):
false        ← el booleano false
0            ← el número cero
""           ← texto vacío
null         ← vacío intencional
undefined    ← variable que existe pero no tiene valor

// Todo lo demás es TRUE (truthy):
true         ← el booleano true
1            ← cualquier número distinto de cero
"hola"       ← cualquier texto no vacío
[]           ← un arreglo vacío (sorpresa: es truthy)
{}           ← un objeto vacío (también es truthy)
¿Para qué sirve saber esto?

Cuando JavaScript evalúa una condición, convierte lo que recibe a true o false automáticamente. Por eso puedes escribir if (texto) en lugar de if (texto !== "") — si el texto está vacío es falsy y la condición no se cumple. Esto aparece constantemente en código real de cualquier proyecto JavaScript.
⚖️

Operadores de comparación

Herramientas para comparar valores y obtener true o false

⚖️ Como una balanza de justicia — pones dos valores en los platillos y el resultado siempre es verdadero o falso. ¿Son iguales? ¿Uno es mayor? ¿Son distintos? La balanza siempre da un veredicto.

Los operadores de comparación comparan dos valores y devuelven un booleano — true o false. Son la base de todas las decisiones en el código.

JS 
// === igual estricto — mismo valor Y mismo tipo
5 === 5        ← true
5 === "5"      ← false — uno es número, el otro texto
"hola" === "hola"  ← true

// !== diferente estricto — distinto valor O distinto tipo
5 !== 3        ← true — sí son diferentes
5 !== 5        ← false — son iguales, no son diferentes

// > mayor que
10 > 5         ← true
3  > 7         ← false

// < menor que
3  < 7         ← true
10 < 5         ← false

// >= mayor o igual que
5 >= 5         ← true — son iguales, cumple la condición
4 >= 5         ← false

// <= menor o igual que
5 <= 5         ← true
6 <= 5         ← false
¿Por qué === y no ==?

JavaScript tiene dos formas de comparar igualdad: == (igual flexible) y === (igual estricto).

5 == "5" da true porque == convierte los tipos antes de comparar — lo que puede causar errores inesperados.

5 === "5" da false porque === compara valor Y tipo sin conversiones.

Regla profesional: siempre usa ===. Evita sorpresas y hace el código más predecible. En cualquier archivo JavaScript verás === constantemente.
💡

Ejemplo en la práctica: Abre el archivo JavaScript de cualquier proyecto y busca ===. Lo verás en comparaciones como if (respuesta === 'ok') para comprobar si una operación tuvo éxito o no.

Operadores lógicos

Combinar varias condiciones en una sola

🚦 Como las condiciones de la vida real — para entrar al cine necesitas tener dinero Y ser mayor de edad (ambas a la vez). O para conseguir descuento necesitas ser estudiante O ser adulto mayor (cualquiera de las dos). Los operadores lógicos funcionan exactamente así.

Los operadores lógicos combinan dos o más comparaciones en una sola condición. Hay tres: && (y), || (o) y ! (no).

JS 
// && (AND) — ambas condiciones deben ser TRUE
// "esto Y aquello"
const tieneDinero  = true
const esMayorEdad  = true
const puedeEntrar  = tieneDinero && esMayorEdad  ← true

const tienePIN    = true
const pinCorrecto = false
const accesoAdmin = tienePIN && pinCorrecto      ← false (una falla = todo false)

// || (OR) — al menos UNA condición debe ser TRUE
// "esto O aquello"
const esEstudiante = false
const esAdultoMayor = true
const tieneDescuento = esEstudiante || esAdultoMayor  ← true (una basta)

// ! (NOT) — invierte el booleano
// "NO esto"
const estaAbierto  = false
const estaCerrado  = !estaAbierto   ← true (invierte el false)

const hayFotos     = true
const noHayFotos   = !hayFotos      ← false (invierte el true)

Una tabla para tenerlo claro siempre: 

&&  (Y)          — ambas deben ser true
true  && true  → true   ✅
true  && false → false  ❌
false && true  → false  ❌
false && false → false  ❌

||  (O)          — al menos una debe ser true
true  || true  → true   ✅
true  || false → true   ✅
false || true  → true   ✅
false || false → false  ❌

!   (NO)         — invierte el valor
!true  → false
!false → true
💡

Ejemplo en la práctica: Los operadores lógicos aparecen constantemente en buscadores. Una línea típica sería: const coincide = !busqueda || nombre.includes(busqueda)

Traducida: "coincide es verdadero si NO hay texto buscado (!busqueda) O si el nombre incluye ese texto". Así funcionan los filtros de búsqueda en cualquier app.

if / else

La estructura más importante del código — tomar decisiones

🌧️ Como decidir qué ponerse al salir de casa — "SI está lloviendo, lleva paraguas. SI NO, lleva lentes de sol". El código hace exactamente lo mismo: evalúa una condición y elige qué instrucciones ejecutar según el resultado.

if es la estructura de decisión más fundamental de la programación. Existe en todos los lenguajes. Funciona así: evalúa una condición entre paréntesis, si es true ejecuta el bloque de adentro, si es false lo salta. El else es opcional — es lo que pasa cuando la condición no se cumple.

JS 
// Estructura básica
if (condición) {
  // esto se ejecuta si la condición es TRUE
} else {
  // esto se ejecuta si la condición es FALSE
}

// Ejemplo simple
const llueve = true

if (llueve) {
  console.log("Lleva paraguas")
} else {
  console.log("Lleva lentes de sol")
}
// resultado: "Lleva paraguas"

Usando comparadores reales en las condiciones:

JS 
const fotos = 0

if (fotos > 0) {
  console.log("Hay fotos para mostrar")
} else {
  console.log("No hay fotos todavía")
}
// resultado: "No hay fotos todavía"

// También puedes usarlo sin else si no necesitas el caso contrario
const pin = '1234'

if (pin === '1234') {
  console.log("Acceso concedido")
}
// si el pin no es correcto, simplemente no pasa nada

El if también puede evaluar variables directamente aprovechando los valores truthy y falsy que vimos antes:

JS 
const nombre = ""   ← texto vacío = falsy

if (nombre) {
  console.log("El nombre es: " + nombre)
} else {
  console.log("No se escribió ningún nombre")
}
// resultado: "No se escribió ningún nombre"
// porque "" es falsy — equivale a if (false)

// Otro ejemplo con null
let currentGallery = null   ← null = falsy

if (currentGallery) {
  console.log("Hay una materia abierta")
} else {
  console.log("No hay ninguna materia abierta")
}
// resultado: "No hay ninguna materia abierta"
💡

Ejemplo en la práctica: Cualquier función que hace una petición a internet usa if/else para decidir qué mostrar. Si la respuesta es exitosa muestra el resultado, si no muestra un mensaje de error. Ese patrón aparece en absolutamente todos los proyectos web.

🪜

else if

Cuando hay más de dos posibilidades

🪜 Como una escalera de decisiones — subes peldaño por peldaño hasta encontrar la condición que se cumple, y en ese peldaño te quedas. Si ninguno se cumple, llegas al else que está abajo de todo.

Cuando hay más de dos posibilidades, encadenas varios else if entre el if inicial y el else final. JavaScript los evalúa de arriba hacia abajo y ejecuta el primero que sea true. Los demás los ignora completamente.

JS 
const nota = 75

if (nota >= 90) {
  console.log("Excelente")       ← ¿75 >= 90? No → sigue
} else if (nota >= 80) {
  console.log("Muy bien")        ← ¿75 >= 80? No → sigue
} else if (nota >= 70) {
  console.log("Bien")            ← ¿75 >= 70? Sí ✅ → ejecuta esto
} else if (nota >= 60) {
  console.log("Suficiente")      ← ya no llega aquí
} else {
  console.log("Reprobado")       ← tampoco aquí
}
// resultado: "Bien"
Orden importa — mucho:

JavaScript evalúa las condiciones de arriba hacia abajo y se detiene en la primera que sea true. Si pones la condición más general primero, nunca llegará a las específicas.

Por ejemplo, si pusieras if (nota >= 60) primero, cualquier nota de 60 en adelante entraría ahí y nunca llegaría a "Muy bien" o "Excelente", aunque la nota fuera 95. Siempre pon las condiciones más específicas primero.
💡

Ejemplo en la práctica: Esta estructura aparece cuando una app necesita reaccionar diferente según el tipo de error, el estado de un elemento, o el tipo de dato recibido. Busca else if en cualquier archivo JavaScript grande y encontrarás decenas.

switch

Elegir entre muchas opciones exactas

🎛️ Como un selector de canales de televisión — tienes muchas opciones numeradas y solo una se activa según el canal que elijas. Es más limpio que escribir diez else if seguidos cuando todas las condiciones comparan el mismo valor.

switch evalúa una expresión y la compara contra múltiples casos con case. Cuando encuentra el que coincide, ejecuta ese bloque. El break es importante — le dice al switch que pare ahí y no siga ejecutando los casos siguientes. El default es el equivalente al else — se ejecuta si ningún caso coincide.

JS 
const dia = "lunes"

switch (dia) {
  case "lunes":
    console.log("Inicio de semana")
    break   ← para aquí, no ejecuta los otros

  case "viernes":
    console.log("¡Por fin viernes!")
    break

  case "sábado":
  case "domingo":
    console.log("Fin de semana")  ← dos casos, mismo resultado
    break

  default:
    console.log("Día normal entre semana")
    ← no necesita break porque es el último
}
// resultado: "Inicio de semana"
¿Cuándo usar switch en lugar de else if?

Usa switch cuando comparas la misma variable contra muchos valores exactos. Usa else if cuando las condiciones son rangos o comparaciones diferentes.

switch es más legible cuando tienes 4 o más casos del mismo valor. Con 2 o 3 casos, if/else if suele ser más claro.

Bucle for

Repetir algo un número exacto de veces

🔢 Como contar en voz alta — "uno, dos, tres... hasta diez". Sabes cuántas veces vas a repetir desde el principio. El bucle for es perfecto cuando conoces exactamente cuántas veces necesitas hacer algo.

Un bucle ejecuta un bloque de código repetidamente. El bucle for tiene tres partes separadas por punto y coma dentro del paréntesis: dónde empieza, cuándo para, y cómo avanza.

JS 
for (inicio; condición; avance) {
  // código que se repite
}

// Las tres partes explicadas:
for (let i = 0; i < 5; i++) {
       ↑           ↑       ↑
       │           │       └── i++ significa "suma 1 a i cada vuelta"
       │           └────────── repite MIENTRAS i sea menor que 5
       └────────────────────── empieza con i en 0

  console.log(i)
}
// imprime: 0, 1, 2, 3, 4
// cuando i llega a 5, la condición (i < 5) es false y se detiene

Un uso muy común del bucle for es recorrer una lista de elementos para hacer algo con cada uno:

JS 
const materias = ["Matemáticas", "Historia", "Física"]

for (let i = 0; i < materias.length; i++) {
  console.log(materias[i])
}
// imprime:
// "Matemáticas"  (i=0)
// "Historia"     (i=1)
// "Física"       (i=2)
// cuando i=3, ya no hay más elementos y se detiene

// materias.length → da el número total de elementos (3)
// materias[0]     → accede al primer elemento ("Matemáticas")
// materias[i]     → accede al elemento en la posición i
¿Qué es i?

i es simplemente el nombre de la variable del bucle. Por convención se usa i (de "índice" o "iteración"), pero podría llamarse cualquier cosa: contador, numero, paso.

Empieza en 0 porque en programación casi todo empieza a contar desde 0, no desde 1. El primer elemento de una lista está en la posición [0], el segundo en [1], y así sucesivamente.
💡

Ejemplo en la práctica: Cualquier página que muestra una lista de elementos — productos, artículos, tarjetas — usa un bucle para recorrer los datos y crear el HTML de cada elemento automáticamente. Sin bucles tendrías que escribir el código de cada tarjeta a mano.

Bucle while

Repetir mientras algo sea verdadero

⏳ Como esperar en la fila del banco — sigues esperando MIENTRAS haya gente delante de ti. No sabes exactamente cuántas personas hay, solo sabes que paras cuando no quede ninguna. El bucle while repite sin saber de antemano cuántas veces.

A diferencia del for donde sabes exactamente cuántas veces repetir, el while repite mientras una condición sea true. Se detiene cuando esa condición se vuelve false. Es ideal cuando no sabes de antemano cuántas repeticiones necesitas.

JS 
// Estructura básica
while (condición) {
  // código que se repite mientras la condición sea true
  // ⚠️ algo dentro debe cambiar la condición, o será infinito
}

// Ejemplo: contar hasta 5
let contador = 1

while (contador <= 5) {
  console.log(contador)    ← imprime: 1, 2, 3, 4, 5
  contador++               ← suma 1 cada vuelta (¡importante!)
}
// cuando contador llega a 6, (6 <= 5) es false → para

// Ejemplo más real: intentar hasta que funcione
let intentos = 0
let exito = false

while (!exito && intentos < 3) {
  console.log("Intentando conectar... intento " + intentos)
  intentos++
  // imagina que aquí intentas conectarte a internet
  // si funciona, exito = true y el while para
}
⚠️ El bucle infinito — el error más común con while:

Si la condición del while nunca se vuelve false, el bucle corre para siempre y congela el navegador o programa.

while (true) { console.log("infinito") } — esto nunca para.

Siempre asegúrate de que algo dentro del bucle eventualmente haga que la condición sea false. En el ejemplo de arriba, contador++ hace que eventualmente contador <= 5 sea falso.
📋

forEach

La forma más limpia de recorrer una lista

📬 Como un cartero que reparte cartas — va casa por casa (elemento por elemento) y en cada una hace exactamente lo mismo: entregar la carta. No necesita saber cuántas casas hay — simplemente recorre todas las que existan hasta terminar.

forEach es un método que tienen los arreglos (listas) en JavaScript. Recorre cada elemento del arreglo y ejecuta una función con cada uno. Es más limpio y legible que un for tradicional cuando solo quieres hacer algo con cada elemento sin necesitar el índice.

JS 
// Sintaxis básica
arreglo.forEach(function(elemento) {
  // aquí haces algo con cada elemento
})

// Ejemplo con materias
const materias = ["Matemáticas", "Historia", "Física"]

materias.forEach(function(materia) {
  console.log("Materia: " + materia)
})
// imprime:
// "Materia: Matemáticas"
// "Materia: Historia"
// "Materia: Física"

// También funciona con funciones flecha (arrow function)
// Es la misma cosa escrita más corto — la verás mucho en código moderno
materias.forEach(materia => {
  console.log("Materia: " + materia)
})

Un ejemplo más práctico — crear elementos HTML para cada item de una lista:

JS 
const capitulos = [
  { titulo: "HTML",       numero: "01" },
  { titulo: "JavaScript",  numero: "02" },
  { titulo: "CSS",        numero: "03" }
]

capitulos.forEach(function(cap) {
  // para cada capítulo creamos una tarjeta en pantalla
  const tarjeta = document.createElement('div')
  tarjeta.textContent = cap.numero + " - " + cap.titulo
  document.body.appendChild(tarjeta)
})
// resultado: tres tarjetas aparecen en la pantalla
forEach vs for — ¿cuándo usar cada uno?

Usa forEach cuando quieras recorrer todos los elementos de una lista y hacer algo con cada uno. Es más limpio y expresivo.

Usa for cuando necesites el índice (i), cuando quieras poder saltar elementos con continue, o cuando quieras parar el bucle antes con break (forEach no admite break).

En código moderno verás forEach mucho más seguido que el for clásico para recorrer listas.
💡

Ejemplo en la práctica: Cualquier función que construye una lista de elementos en pantalla usa forEach. Por ejemplo: capitulos.forEach(cap => { ... }) recorre todos los capítulos y crea una tarjeta HTML por cada uno. Sin forEach tendrías que escribir el código de cada tarjeta manualmente.

🔗 Cómo se conectan todos los conceptos

Así se ven varios de estos conceptos trabajando juntos en código real. Este patrón aparece constantemente en cualquier archivo JavaScript:

JS 
function construirBiblioteca() {
  // forEach — recorre cada tema de la lista
  TEMAS.forEach(tema => {

    // if — solo construye los temas no bloqueados
    if (!tema.bloqueado) {

      // forEach anidado — recorre los capítulos de ese tema
      tema.capitulos.forEach(cap => {

        // if/else — decide qué clase poner según el lenguaje
        if (cap.lang === 'html') {
          tarjeta.classList.add('borde-html')
        } else if (cap.lang === 'css') {
          tarjeta.classList.add('borde-css')
        } else {
          tarjeta.classList.add('borde-js')
        }

      })
    }
  })
}
