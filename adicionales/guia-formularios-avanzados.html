Introducción — más allá del <input> básico
Por qué un formulario "bien hecho" no necesita casi JavaScript

Ya sabes crear un <form> con <input> y un botón de enviar. Esta guía va un paso más allá: cómo hacer que el propio navegador valide los datos por ti, sin escribir una sola línea de JavaScript para los casos más comunes — y cómo tomar el control con JS solo cuando de verdad lo necesitas.

💡 Lo que vas a aprender

La validación nativa de HTML5 (required, pattern, tipos de input), atributos que mejoran mucho la experiencia y casi nadie conoce, cómo diseñar los estados de error con CSS puro, y cómo personalizar los mensajes de error con la Constraint Validation API de JavaScript.

✅
Validación nativa de HTML5
El navegador ya sabe validar — solo hay que decirle las reglas

Antes de escribir JavaScript para "revisar" que un campo esté bien lleno, recuerda que HTML ya trae un sistema de validación integrado. Se activa solo con atributos.

HTML<form> <label>Correo: </label> <input type="email" required placeholder="tucorreo@ejemplo.com"> <label>Edad: </label> <input type="number" min="18" max="99" required> <button>Enviar</button> </form>

Con solo type="email" y required, el navegador ya rechaza correos mal escritos y campos vacíos — sin una línea de JavaScript.

AtributoQué hace
requiredEl campo no puede quedar vacío.
min / maxRango permitido para números y fechas.
minlength / maxlengthRango permitido de caracteres en texto.
type="email"Exige el formato de un correo válido.
patternExige que el valor cumpla una expresión regular.

🚪 Analogía: el portero del club

La validación nativa es como un portero en la puerta: revisa que cumplas las reglas básicas (identificación, código de vestimenta) antes de dejarte pasar. No necesitas contratar a alguien más adentro (JavaScript) para revisar lo mismo otra vez — el portero ya filtró lo obvio.

🧩
Atributos poco usados (pero muy útiles)
Pequeños detalles que mejoran mucho la experiencia
🔤
pattern
Valida el valor contra una expresión regular. Ej: pattern="[0-9]{5}" exige exactamente 5 dígitos.
📋
list + datalist
Convierte un input de texto en un autocompletado con sugerencias, sin JavaScript ni librerías externas.
⌨️
inputmode
Le dice al teclado del celular qué mostrar: inputmode="numeric" abre el teclado numérico aunque el input siga siendo de texto.
🚫
novalidate
Atributo del <form> que desactiva la validación nativa por completo, para cuando quieres controlarla 100% con JavaScript.
🔁
autocomplete
Le dice al navegador qué tipo de dato es (autocomplete="email", "tel", "cc-number") para autocompletar mejor.

Ejemplo: datalist para autocompletado

HTML<input list="lenguajes" name="lenguaje"> <datalist id="lenguajes"> <option value="HTML"> <option value="CSS"> <option value="JavaScript"> </datalist>
🎨
Estilizar la validación con CSS
El navegador ya sabe si el campo es válido — solo hay que pintarlo

Cuando un campo tiene reglas de validación, el navegador le agrega automáticamente un estado que puedes usar en CSS, sin ninguna clase manual.

CSS/* Campo válido */ input:valid { border-color: green; } /* Campo inválido, pero solo cuando el usuario ya interactuó */ input:invalid:not(:placeholder-shown) { border-color: crimson; } /* Foco visible por teclado, no por mouse */ input:focus-visible { outline: 2px solid dodgerblue; }

🟢 El truco de :not(:placeholder-shown)

Usar solo :invalid pinta de rojo el campo desde que carga la página, antes de que el usuario escriba nada — se ve agresivo. Combinarlo con :not(:placeholder-shown) hace que el rojo aparezca solo después de que el usuario empezó a escribir y lo dejó incompleto o mal.

⚙️
Mensajes personalizados con JavaScript
Constraint Validation API — cuando el mensaje del navegador no alcanza

El mensaje de error que muestra el navegador por defecto ("Rellena este campo") no se puede traducir ni personalizar solo con HTML. Para eso existe la Constraint Validation API.

JavaScriptconst input = document.querySelector('#usuario') input.addEventListener('invalid', () => { input.setCustomValidity('Escribe un nombre de usuario válido') }) input.addEventListener('input', () => { // Limpiar el mensaje personalizado en cuanto el usuario corrige input.setCustomValidity('') })

⚠️ No olvides limpiar el mensaje

Si llamas a setCustomValidity() con un texto y nunca lo vuelves a limpiar con setCustomValidity(''), el campo queda inválido para siempre, aunque el usuario lo corrija. Siempre limpia el mensaje en el evento input.

🟢 Resumen de la guía

  • Validación nativa: required, min/max, pattern, tipos de input.
  • Atributos útiles: datalist, inputmode, autocomplete, novalidate.
  • CSS: :valid, :invalid, :focus-visible, :not(:placeholder-shown).
  • JavaScript: Constraint Validation API para mensajes personalizados.
